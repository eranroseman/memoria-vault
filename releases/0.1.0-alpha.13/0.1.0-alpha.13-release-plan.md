# Release 0.1.0-alpha.13

Draft release parent issue body. Live release state belongs in the GitHub milestone,
the "Release 0.1.0-alpha.13" parent issue, and its gate/stage sub-issues. Delete
this scratch file before the release/checkpoint closes, after durable decisions move
to ADRs, docs, issues, or release notes.

## Status

Draft sandbox release plan for the alpha.13 catalog enrichment checkpoint. The main
blocker is the alpha.12 DB catalog prerequisite: source lookup, digest compilation,
`references.bib`, projections, attention, and rollback must resolve catalog sources
by SQLite `source_id` before alpha.13 removes source Markdown from the DOI path.

## Scope

Alpha.13 delivers the DOI-only multi-source enrichment path on top of the alpha.12
SQLite catalog engine in `Memoria-test`. `capture-source` stages a durable unchecked
source; `enrich-source` fetches Crossref, OpenAlex, and Unpaywall through a declared
network allowlist, caches raw provider payloads under `.memoria/blobs/`, merges
canonical CSL with per-field provenance, blocks required-provider failures and
retracted/contested records from consumable `checked`, materializes `references.bib`,
and keeps the current tracked `capabilities/ai-catalog.json` projection.

Later work: ISBN and URL-primary depth, Semantic Scholar, PubMed, arXiv, full citation
graph, embeddings, full-text acquisition, PI catalog edit UI, knowledge folder
migration, directory-per-capability manifests, `_generated/` projection paths, and
universal Concept frontmatter enforcement.

## Implementation Slices

1. Close the alpha.12 DB catalog prerequisite for all required source consumers.
2. Add the enrichment DB/schema migration and provider payload blob storage.
3. Ship `vault-template/.memoria/enrichment/providers.yaml` and enforce it through
   `enrich-source.md` `allowed_network`.
4. Split DOI capture from checked promotion: capture stages unchecked; enrichment promotes.
5. Implement Crossref, OpenAlex, and Unpaywall adapters with cached raw payloads.
6. Implement deterministic merge, field provenance, conflict handling, and retraction gate.
7. Materialize `references.bib` from durable replay data before operation completion.
8. Add attention items for missing keys, provider outages, conflicts, and retractions.
9. Keep `capabilities/ai-catalog.json` in tracked projections for alpha.13.

## Definition Of Done

This release is done when every gate/stage sub-issue below is closed.

| Gate | Proves | Evidence home |
|---|---|---|
| Source | Repo contracts, DB/source consumers, provider policy, schema migrations, Python tests, and static checks are coherent. | CI and `scripts/verify pr` summary |
| Package | Disposable vault assembly includes provider policy, blob storage, operation manifests, projections, and model-free replay. | `scripts/verify package` summary |
| Runtime | Worker queue, SQLite catalog, provider allowlist, materialization recovery, Obsidian bridge, and policy boundaries work live or have explicit skip reasons. | `scripts/verify runtime` or skip evidence |
| Product | DOI capture/enrich produces a checked usable source, portable `references.bib`, provider evidence, conflict/retraction attention, and no source Markdown dependency. | Product/manual sub-issue |
| Release | Blockers, docs, versioning, release notes, scratch cleanup, and rollout/rollback decisions are ready. | Release parent/sub-issues |

## Gate Sub-Issues To Create

- G1 Source gate: alpha.12 DB source prerequisite plus alpha.13 failing tests pass.
- G2 Package gate: disposable vault ships provider config, blob directories, operation
  manifest, `references.bib`, and `capabilities/ai-catalog.json`.
- G3 Runtime gate: attended or mocked live enrichment run proves allowlist, retry,
  materialization recovery, and checked/read-barrier behavior.
- G4 Product gate: PI-reviewable DOI workflow evidence, including conflict/retraction
  attention and Citation survival payloads.
- G5 Release gate: docs/ADR routing, release-please readiness, milestone cleanup, and
  deletion of `scratch/releases/0.1.0-alpha.13/`.

## Known Limitations

- Limitation: sandbox-only; no installed-vault migration target. Impact: production
  vaults must not be upgraded through this checkpoint. Workaround: test only in
  `~/Memoria-test`. Tracking: alpha.13 parent issue.
- Limitation: DOI path only. Impact: ISBN books, URL-primary sources, Semantic Scholar,
  PubMed, arXiv, citation graph, embeddings, and full text remain deferred. Workaround:
  use existing capture paths or schedule follow-up issues. Tracking: alpha.13 follow-ups.
- Limitation: no PI catalog edit UI. Impact: CLI/engine remains operator/test tooling;
  PI-facing edits and human resolutions wait for Obsidian queued controls. Workaround:
  leave blocked records unchecked with attention evidence. Tracking: ADR-106/ADR-121 UI
  follow-up.
- Limitation: no structural layout migration. Impact: `knowledge/views` and
  `capabilities/ai-catalog.json` remain current alpha.13 paths. Workaround: defer
  `_views/` and `_generated/` to a separate structural PR. Tracking: layout migration
  follow-up.
