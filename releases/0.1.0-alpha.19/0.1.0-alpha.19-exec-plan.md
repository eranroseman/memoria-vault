# ExecPlan - Implement 0.1.0-alpha.19

Living plan on the scratch branch. Design authority:
[0.1.0-alpha.19-design.md](./0.1.0-alpha.19-design.md). When this plan and the
design disagree, fix the plan.

## 0. Metadata

- **Task:** implement alpha.19: `fulltexts/`, dense/vector query substrate,
  schema references, computed evidence, and the full supported-platform
  code-output lane.
- **Implementation worktrees:** one branch/worktree per slice under
  `~/memoria-vault/worktrees/alpha19-<slice>` off `origin/main`.
- **Release/checkpoint:** `0.1.0-alpha.19`.
- **Non-goals:** beta.1 UI, seed corpus, multi-device topology, raw dataset
  bundling, non-Linux/WSL code execution without a platform proof.

## 1. Workstreams

| WS | Scope | Delivers | Minimal check |
| --- | --- | --- | --- |
| A | `fulltext/` -> `fulltexts/` | Template, folder schema, policies, docs, fixtures, search paths, tests updated together. | No writer emits `fulltext/`; existing bundle-root tests expect `fulltexts/`. |
| B | Query schema | `passages`, FTS5 table, `sqlite-vec` table, `file_index_state`, `concept_edges`, schema version bump. | Schema smoke creates all tables and capability checks fail closed. |
| C | Indexing and status cascade | Backfill/rebuild passages, query-time stale-file reindex, same-transaction `check_status` cascade. | Status demotion changes matching passage rows before transaction commit. |
| D | Hybrid retrieval | Filter-before-rank SQL, FTS5 lexical, vector lookup, RRF fusion, topic denominators, fixture preregistration/run. | BM25 remains default unless fixture clears the bar. |
| E | Computed evidence | Evidence item union gains `code-warrant`; `computed` derived type; verify complete/incomplete states. | Missing/changed code output flips computed evidence to incomplete. |
| F | Code artifact records | Project code artifact markdown record, companion source/output directory, `code_artifacts` rows. | Code artifact survives keep-test as files plus typed record. |
| G | Sandbox runner | Linux/WSL `bwrap` runner, argv-only commands, approved-command gate, env scrub, mounts, network/secret denial tests, resource caps. | Network and secret canaries fail closed; unsupported platform refuses execution. |
| H | Code run verification/export | `code_runs` rows, output hashing, rerun verification, draft/export gate integration for code warrants. | A computed claim verifies by rerunning its artifact; export refuses broken warrant. |
| I | Docs and beta gap sync | Reference docs updated; beta.1 gap list shows alpha.19 ownership and remaining blockers. | Docs describe actual shipped state, not beta aspirations. |

## 2. Execution Order

1. A first. The path rename is mechanical but touches many tests; doing it
   first prevents every later slice from landing new `fulltext/` references.
2. B and C next. The query substrate is not useful until freshness and status
   cascade are correct.
3. D after B/C. Hybrid ranking over stale or wrongly checked rows is worse than
   no hybrid ranking.
4. E before H. Code runs cannot warrant claims until evidence knows how to
   reference them.
5. F/G/H together but in small PRs: records, runner, verification.
6. I with each PR, then one final drift sweep.

## 3. Validation

Run the smallest useful checks per PR, then the full gate before release:

- path/folder tests for `fulltexts/`;
- schema smoke and migration smoke;
- FTS5 and `sqlite-vec` capability checks;
- retrieval fixture report comparing BM25, FTS5, vector, hybrid, and graph
  variants;
- stale-file lazy reindex test;
- status-cascade transaction test;
- code-runner canary tests: no network, no secret read, no shell command,
  timeout, max-output cap;
- computed evidence complete/incomplete tests;
- crash/recovery tests for index writes and code-run materialization;
- `python scripts/verify pr` before merge.

## 5. Risks

- `sqlite-vec` is a new runtime dependency. Keep vectors derived and rebuildable;
  BM25 remains the fallback.
- `bwrap` is argument-sensitive. Treat the proof tests as the feature gate, not
  as optional security decoration.
- The path rename is wide. Do it mechanically, with tests, before touching
  retrieval internals.
