# 0.1.0-alpha.20 Design

## Package Seed And Vault-Template Retirement

Alpha.20 retires the root `vault-template/` tree. The product keeps Memoria as a
standalone engine and moves the minimum runtime seed into the installable Python
package under `src/memoria_vault/product/workspace_seed/`.

The previous source template had stopped being a product contract. It preserved
empty directories, historical dashboards, markdown eval tasks, note templates,
adapter payloads, cron wrappers, and local navigation files mostly so checks
could prove they still existed. Alpha.20 removes that loop: files ship only when
runtime code reads them, and writable directories are created by `memoria init`
from `.memoria/schemas/folders.yaml`.

## Runtime Seed Inventory

The package seed contains only these authored files:

| File | Why it ships | Verified reader |
| --- | --- | --- |
| `__init__.py` | Makes `workspace_seed` an importable package resource namespace. It is not copied into the vault. | `importlib.resources.files("memoria_vault.product.workspace_seed")` in `src/memoria_vault/cli.py`. |
| `.githooks/pre-commit` | Provides the canonical schema/frontmatter pre-commit hook for initialized vaults. | `scripts/install.sh`, `scripts/install.ps1`, and sandbox refresh copy it to `.git/hooks/pre-commit`. |
| `.gitignore` | Keeps generated DBs, journals, indexes, blobs, caches, and local secrets out of vault git history. | `src/memoria_vault/cli.py` copies it through `SEED_FILES`; trusted-writer tests force-add it when asserting tracked seed behavior. |
| `.memoria/config/providers.yaml` | Supplies local provider and runner defaults without embedding secrets. | `src/memoria_vault/runtime/enrichment.py` and `src/memoria_vault/runtime/operations.py` read `.memoria/config/providers.yaml`. |
| `.memoria/eval/alpha15-seeded-errors.json` | Seeds the deterministic seeded-error verdict bundle. | `src/memoria_vault/cli.py`, `src/memoria_vault/runtime/worker.py`, and `src/memoria_vault/runtime/seeded_errors.py` require this path. |
| `.memoria/patterns/_preamble.md` | Shared prompt preamble for operation prompt assembly. | `src/memoria_vault/runtime/operations.py` reads `.memoria/patterns/_preamble.md`. |
| `.memoria/schemas/calibration.yaml` | Owns calibrated thresholds for matching, classification, and scoring checks. | `src/memoria_vault/runtime/subsystems/lib/schema.py` loads the schema directory used by linter, ingest, and tests. |
| `.memoria/schemas/folders.yaml` | Owns Concept homes, staging roots, quarantine root, and the `memoria init` skeleton. | `src/memoria_vault/cli.py` calls `schema.load_folders()["skeleton"]`; linter and worker tests use the same loader. |
| `.memoria/schemas/types/*.yaml` | Owns per-type frontmatter contracts. | `src/memoria_vault/runtime/subsystems/lib/schema.py`, pre-commit, linter, and schema tests load these files. |
| `steering.md` | User-editable standing program memory. | CLI `steering show/edit` and `src/memoria_vault/runtime/knowledge.py` read it. |
| `system/vocabulary.md` | User-editable controlled vocabulary. | CLI `vocab` commands, schema vocabulary loading, linter, and knowledge runtime read it. |

No `.gitkeep` files ship. No dashboards, templates, hidden adapter plugin
payloads, cron wrappers, design-system files, `AGENTS` files, `home.md`, `_nav.md`,
`troubleshooting.md`, `index.md`, `bibliography.bib`, or `system/manifest.jsonl`
ship in the package seed.

## Initialization Model

`memoria init` performs three separate steps:

1. Create writable runtime and content directories from the `skeleton` list in
   `.memoria/schemas/folders.yaml`.
2. Copy the package seed files and trees from
   `memoria_vault.product.workspace_seed`.
3. Generate local control/projection files: SQLite state, `.memoria/journal-head`,
   `.memoria/overrides.jsonl`, `system/manifest.jsonl`, `index.md`, and
   `bibliography.bib`.

Repair follows the same source of truth. `memoria doctor --repair` reinstalls
the package seed and generated files instead of reconciling against an in-repo
vault image.

## Sandbox Policy

The sandbox remains valuable as an inspectable, non-Python runtime vault. Its
value comes from being a real deployed vault, not from mirroring a source
template. Sandbox refresh should reinstall the current package into the
sandbox's `.memoria/.venv`, run `memoria doctor --repair`, and rewire the git
hook. It must not rsync, delete, or copy from a source scaffold.

## Removed Mechanisms

Alpha.20 removes mechanisms whose only purpose was preserving retired payloads:

- root `vault-template/`
- installer manifest copy layer
- dashboard/template drift gate tied to `system/dashboards/*.md`
- design-system drift detector tied to `.memoria/design-system.md`
- tests that asserted shipped note templates, installed policy plugins, empty
  directories, dashboard markdown, or historical eval markdown tasks

The remaining negative checks assert absence, not preservation: package-seed
tests, `alpha14_negative_gate.py`, and plugin provenance checks prevent deleted
payload categories from re-entering the baseline.
