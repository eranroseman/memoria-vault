# Alpha.15 Paper Adversarial Review

This review attacks the alpha.15 design using the root-level paper corpus at
`/home/eranr/memoria-vault/_papers/*.md` and the alpha.14 engine cut inherited
from main. It
is a companion to the existing `_papers/REVIEW-SUMMARY.md` / refutation posture:
the goal is not to restate the literature, but to find where the alpha.15 design
would fail if the papers are taken seriously.

Scope checked:

- Root-level `_papers/*.md` only: 380 Markdown files. Nested folders were out of
  scope.
- Design targets:
  - alpha.14 engine state as landed on main before the alpha.15 cut
  - `scratch/releases/0.1.0-alpha.15/0.1.0-alpha.15-design.md`
  - `scratch/releases/0.1.0-alpha.15/0.1.0-alpha.15-exec-plan.md`
  - alpha.15 ADR drafts 125-130
- Subagents:
  - Arendt: integrity, security, poisoned-read, persistence.
  - Gibbs: knowledge model, retrieval, memory, evidence.
  - Nietzsche: HCI, workflow, review burden, researcher control.
  - Feynman: feasibility, evaluation, overbuild, release slicing.

## Findings

### 1. Blocker: alpha.15 scopes itself as a platform before dogfood proves the loop

Alpha.15 has the right core shape, but the design still pulls paper framing,
`eval select-models`, MCP, local HTTP, per-Co-PI scoping, the future-adapter
thin-client contract, Tier-2 judges, and ADR adds into one alpha because they are "doable."
The papers push the opposite: first prove narrow real workflows, friction, cost,
and end-state behavior.

Pressure:

- Epstein et al. show personal-informatics systems fail on collection burden,
  abandonment, interpretation, action, and preparation, and recommend lower-burden
  evaluation of design ideas.
- Feng and Liu frame verification asymmetry as the central bottleneck in
  agent-assisted research: users skim when verification is expensive.
- ScienceAgentBench argues for evaluating individual real scientific workflow
  tasks before end-to-end automation claims; its best tested agent solved only a
  minority of tasks even with expert knowledge.
- Tau-bench shows agent reliability needs repeated final-state checks, not just a
  plausible transcript or successful tool call.

Recommendation:

Make the CLI dogfood checkpoint a hard scope gate. Second-wave surfaces should be
"candidate post-dogfood scope," not guaranteed alpha.15 scope. Exit criteria
should include per-work interaction count, review minutes, unchecked-backlog
slope, and final-state checks for the two real workflows.

### 2. Critical: all-vault CLI Co-PI is trusted local dogfood only

The alpha.15 write boundary is strong, but the read/conversation boundary is not.
Quarantine stops poisoned content from becoming checked state; it does not stop a
BYO agent from reading too much, exfiltrating via its own tools, or being steered
by malicious retrieved text.

Pressure:

- Greshake et al. show retrieved text can act like instructions/code inside
  LLM-integrated applications, including data theft, persistent compromise, and
  tool/API manipulation.
- AgentDojo shows agents over untrusted tool data remain vulnerable and current
  defenses are not foolproof; formal utility/security state checks are needed.
- Rabinovich and Anaby-Tavor show function calling is brittle under paraphrase
  and semantically related tool expansion.

Recommendation:

Do not present all-vault shell-agent Co-PI as a supported safe mode. Label it
trusted-local dogfood only until MCP identity plus read scope exists. Add poisoned
paper fixtures for read exfiltration, context steering, wrong-scope reads,
paraphrased tool calls, and task-tools-suffice attacks. The supported Co-PI path
should be scope-required from its first non-dogfood transport.

### 3. High: "data, not instructions" needs mechanical runner enforcement

Alpha.15 says returned text is data, not instructions. The papers say this is
not a property of text; it is a property of the runner, prompt assembly, tool
surface, and evaluation harness.

Pressure:

- Greshake et al. explicitly identify the data/instruction blur as the attack
  vector.
- AgentDojo says current techniques cannot guarantee security-critical
  distinction of data from instruction.
- The same risk applies to digest, ask, gap, contradiction, enrichment, and Co-PI
  context, not only to long-term persistence.

Recommendation:

Require operation manifests to declare untrusted fields. The runner should
serialize those fields into sealed data blocks and forbid raw source text from
being interpolated into instruction templates. Add negative poisoned-input
fixtures for every operation that reads full text, provider metadata, qmd hits, or
agent context.

### 4. High: stale descendants cannot stay machine-consumable by default

Alpha.15 improves alpha.14 by propagating demotion, but depth >= 2 stale
descendants remain checked and consumable with a flag. The memory papers make
that exact multi-hop stale reuse a primary failure mode.

Pressure:

- LongMemEval evaluates extraction, multi-session reasoning, temporal reasoning,
  knowledge updates, and abstention; fact-only compression loses context, and
  time-aware retrieval matters.
- TEMPO shows temporal retrieval requires time-period coverage, cross-period
  evidence, and hard negatives that are topically related but temporally wrong.
- Uddin et al. and Hu et al. stress forgetting/update failures in long-term agent
  memory.

Recommendation:

Stale descendants may remain readable, but should not feed promotion, gap
scoring, ask support, export readiness, or Co-PI context by default. Derived
outputs depending on stale evidence should inherit `stale` until rechecked. Add
stale-as-current, false-premise abstention, superseded-evidence, and partial
retrieval fixtures.

### 5. High: span anchors prove existence, not support, completeness, or faithfulness

Alpha.15's anchor and blockquote checks are necessary. They are not enough for
checked answers, digests, or export-ready paper claims.

Pressure:

- Wallat et al. separate citation correctness from citation faithfulness and show
  citations can be post-rationalized.
- Buchmann and Gurevych separate response failure from citation failure and show
  failures rise with complex response-evidence relations.
- FEVER shows verification often needs evidence sets across multiple sentences or
  pages; correct evidence retrieval is the hard part.
- Press, CiteME, and CiteGuard reinforce that scientific citation attribution is
  not solved by frontier LLMs or source-looking citations.

Recommendation:

Introduce checked evidence relation classes such as `direct_span`, `multi_span`,
`multi_doc`, `derived`, `background`, `supports`, `challenges`, `method`,
`data`, and `weak`. Export-ready claims need a complete evidence set or an
explicit missing-evidence marker. LLMs may propose support, but deterministic
provider metadata and anchored evidence records should own bibliography and claim
source identity.

### 6. High: ask/search needs planned retrieval and bounded-universe coverage

Alpha.14's qmd+graph posture survives only as a substrate. Alpha.15 narrows the
claim, but ask/gap/contradiction still needs an explicit retrieval plan and
coverage contract.

Pressure:

- BRIGHT shows strong retrievers collapse on reasoning-intensive retrieval where
  relevance requires more than lexical or semantic similarity.
- TEMPO shows temporal coverage is separate from topical relevance.
- KnowledgeBerg shows simple-looking questions can require complete bounded
  universes; failures split into completeness, awareness, and application.
- PaperQA, SciAtlas, RAGScholar, and reasoning-intensive retrieval surveys all
  push toward iterative evidence gathering, graph/topological reasoning, or
  source-attributed retrieval traces.

Recommendation:

Add an ask retrieval-plan phase: classify query type, decompose/rewrite, run
qmd+graph/RRF/propagation/rerank, emit the retrieval trace, and abstain when
coverage is incomplete. Add `bounded_universe_incomplete` as a gap kind. Ask JSON
should carry `coverage_scope`, `universe_source`, `known_count`, and
`missing_universe_risk`; forbid comprehensive language unless coverage is checked.

### 7. High: work-as-digest is too weak for agent-native research use

The four concept types survive, but the `work` object cannot be just digest plus
tags if alpha.15 expects agents to perform review, ask, gap, and paper framing
against works.

Pressure:

- Knows defines structured statements, evidence, relations, provenance, methods,
  limitations, assumptions, definitions, and freshness as the agent-native layer
  around papers.
- Intern-Atlas argues paper/citation-level graphs miss methodological evolution;
  method entities, typed causal edges, and verbatim evidence are needed.

Recommendation:

Keep four authored concept files. Add a checked read-model sidecar/table for
work aspects: `statements`, `evidence`, and `relations`, with anchored types for
`method`, `definition`, `limitation`, `assumption`, `result`, and `question`.
This should be a derived/read model, not more frontmatter.

### 8. Medium-high: evaluation gates are too fixture-local for agent/workflow claims

The gate list is strong as unit protection. It is weaker as evidence that the
system handles real workflows with stochastic agents and tool use.

Pressure:

- Tau-bench evaluates by final database state and repeated-run reliability
  (`pass^k`), not one successful trajectory.
- AgentDojo uses formal environment-state utility and security checks because LLM
  judges can be attacked.
- ScienceAgentBench uses executable artifacts, cost, subject-matter validation,
  contamination controls, and expert-validated tasks.

Recommendation:

Add one state-based evaluation harness for the two dogfood workflows. Run
repeated stochastic trials with final workspace-state assertions, cost/tool-call
budgets, poisoned-read/security checks, and a no-agent baseline. Make
`eval select-models` choose the cheapest passing model only after robustness,
abstention, attribution, stale-memory, anomaly, and repeatability slices pass on
held-out data.

### 9. Medium: Co-PI conversation and generated views are secondary surfaces

The read API as a boundary survives. Co-PI chat and generated views as primary
interfaces do not.

Pressure:

- Horvitz and Amershi emphasize invocation, dismissal, correction, uncertainty,
  expected cost, and user control.
- Shneiderman/Maes, Fekete, Hearst, Blackwell, Kery/Myers, and Rule et al. point
  toward direct manipulation, overview/filter/detail, low viscosity, visible
  dependencies, exploration history, and backtracking.
- Feng and Liu warn that plausible agent output plus deadline pressure encourages
  cursory review.

Recommendation:

Make static task/read views primary: faceted worklists, source-adjacent compare,
drilldown, batch preview, reversible enqueue actions, and table fallbacks. In
alpha.15 this is a read/view/action contract, not an Obsidian plugin
implementation; Inspector-style rendering belongs to a future adapter ADR. Keep
generated views shadow-only until they beat a static baseline, and require every
Co-PI exchange to expose plan, sources read, assumptions, unknowns, and pending
writes before enqueue.

### 10. Medium: hub-owned tags need scope, salience, and drift controls

Hub-owned tags are a good high-recall membership index. They are not argument
evidence and they are vulnerable to topic drift.

Pressure:

- SciAtlas rejects sparse/macroscopic concept-only structure and relies on scored
  reusable keywords plus graph structure.
- Wiki and information-visualization papers emphasize repair work, salience,
  freshness, norms, overview/detail, and support for multiple organizations.

Recommendation:

Each hub-owned tag should have a definition, inclusion/exclusion examples,
aliases/deprecated forms, membership provenance, salience/freshness, and orphan
warnings. Tag membership may seed retrieval; it should not count as evidence.

### 11. Medium-low: "cross-store ACID" overclaims the actual guarantee

The prepare/materialize/finalize protocol is defensible. The term "ACID" invites
database-grade semantics across SQLite, files, blobs, and git that the design
does not really provide.

Recommendation:

Rename the guarantee to "operation-level recoverability" or "observable
atomicity." Gate it with crash-matrix final states, fail-closed rebuild behavior,
and doctor detection. Defer history-rewrite repair beyond detection.

## Survivors

The adversarial pass did not refute these alpha.15 decisions:

- Standalone local Python CLI/engine with `uv`, Typer, Pydantic, stdlib
  `sqlite3`, `.sql` migrations, and no ORM/Alembic.
- Meaning-only frontmatter; DB-owned verdicts/derived state; fail-closed DB loss.
- Universal quarantine, read barrier, hash mismatch demotion, and no write on read
  path.
- "Checked" means checks/warrants passed, not approved/trusted.
- Sparse calibrated human checkpoints instead of item-by-item approval theater.
- Store contextual units and index atoms, rather than making atoms the canonical
  memory unit.
- Superseded/retracted material stays first-class and retrievable with temporal
  framing.
- Deterministic edge extraction as unchecked candidates only; no argument-edge
  auto-promotion.
- One `engine/api` with CLI/MCP/HTTP as thin skins; any future adapter must use
  that boundary, with no raw adapter reads.
- LLM contradiction judges as proposers only, after deterministic/NLI candidate
  handling and calibration.
- qmd/SQLite as a local retrieval substrate below the read barrier, not as truth.
- Git/journal operation history, stage-only-own-paths, and no `git add -A`.
- Deferring realtime watchers, multi-user/remote/OAuth, scale-out, and argument
  edge promotion tuning.

## Recommended Minimum Edits

1. Rewrite alpha.15 §S6 so the second-wave surface list is conditional
   post-dogfood scope, not guaranteed because it is doable.
2. Mark all-vault CLI Co-PI as trusted-local dogfood only, not a supported safe
   Co-PI surface.
3. Add operation-manifest untrusted-field enforcement to the runner contract.
4. Make stale descendants non-machine-consumable by default until rechecked.
5. Add evidence relation/evidence-set classes before export-ready claims.
6. Add ask retrieval plans and bounded-universe coverage fields.
7. Add work-aspect read models for statements/evidence/relations.
8. Add a state-based dogfood evaluation harness with repeated runs and baselines.
9. Rename "cross-store ACID" to observable atomicity/recoverability.
