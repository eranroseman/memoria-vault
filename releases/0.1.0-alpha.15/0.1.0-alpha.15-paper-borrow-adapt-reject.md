# Alpha.15 Paper Borrow / Adapt / Reject

Second pass over the root-level paper corpus. This complements
`0.1.0-alpha.15-paper-adversarial-review.md`: the first report asks what breaks;
this one collects ideas worth taking, reshaping, or explicitly refusing.

Scope:

- Corpus: `/home/eranr/memoria-vault/_papers/*.md`, root-level only, 380 Markdown
  files.
- Context: alpha.15 design index plus the alpha.14 engine cut inherited from main.
- Method: use `_papers/REVIEW-SUMMARY.md` and `_papers/REVIEW-REFUTATIONS.md` as
  orientation, then re-check source papers for the ideas below.

## Borrow

Take these ideas nearly directly, with Memoria's engine/read-barrier vocabulary.

| Idea | Source pressure | Alpha.15 move |
|---|---|---|
| State-based dogfood evaluation | Tau-bench, AgentDojo, ScienceAgentBench, EpiBench | Add final workspace-state assertions for the two dogfood workflows; include repeated runs, cost/tool-call budget, and no-agent baseline. |
| Evidence sets, not single warrants | FEVER, ALCE, Citation Failure, CiteME, CiteGuard, Correctness is not Faithfulness | Model `evidence_set` with `direct_span`, `multi_span`, `multi_doc`, `derived`, `background`, and `missing_evidence`; export-ready claims require complete sets or explicit incompleteness. |
| Contextual unit as stored value, atoms as index | LongMemEval, MemoryAgentBench, TEMPO | Keep authored/contextual note/work body as canonical; use atom/claim records for retrieval, routing, and evidence indexing only. |
| Structured work aspects | MASSW, Knows | Add checked work-aspect read-model rows for context, key idea, method, outcome, limitation, assumption, definition, and question. Keep files simple. |
| Untrusted data sealing | Greshake, AgentDojo, InjecAgent | Operation manifests declare untrusted fields; runner serializes source text into sealed data blocks and forbids prompt-template interpolation. |
| Ask retrieval trace | PaperQA, BRIGHT, TEMPO, KnowledgeBerg | `ask` emits query type, decomposition, retrieval sets, rerank basis, temporal frame, coverage gaps, and abstention reason. |
| Bounded-universe coverage | KnowledgeBerg | Add `coverage_scope`, `universe_source`, `known_count`, and `missing_universe_risk`; forbid "all/most/best" language when the universe is incomplete. |
| Static review cockpit | Horvitz, Amershi, Shneiderman/Maes, Fekete, Blackwell, Rule/Kery | Make faceted worklists, source-adjacent compare, batch preview, and reversible enqueue the primary PI surface before generated views. |
| Model-written eval bootstrap | Perez et al. | Let a model draft adversarial fixtures, then PI-validate a small sample; generated fixtures never become authority without validation. |
| Deterministic metadata resolution | Rao and Callison-Burch, OpenAlex/PubTator-style provider papers | Use DOI/citekey/provider records for bibliography and identity; LLMs may suggest, never construct canonical BibTeX/CSL. |

## Adapt

Good ideas, but only after reshaping them to Memoria's local, checked,
PI-directed model.

| Idea | Source pressure | Adaptation |
|---|---|---|
| Knows sidecars | Knows | Adapt to DB/read-model sidecars, not author-facing YAML as the source of truth. |
| Method evolution graph | Intern-Atlas, SciPaths | Keep local method/outcome/limitation relations and evidence quotes; reject a giant global KG requirement. |
| Iterative scientific QA agent | PaperQA, RAGScholar | Adapt into `ask` retrieval planning and traceable answer contracts; no autonomous literature agent. |
| MASSW five aspects | MASSW | Use as optional work-aspect rows. Do not force all five into frontmatter or block capture when an aspect is absent. |
| Tau-bench / EpiBench benchmark style | Tau-bench, EpiBench | Build one tiny Memoria-specific harness for dogfood, not a general benchmark suite. |
| Constrained decoding / prompt programming | LMQL, Shin semantic parsing, Agrawal extraction | Use constraints for parse/shape validity, then validate content separately. Do not treat forced-valid JSON as correctness. |
| Self-consistency | Wang self-consistency, refutation pass | Use low agreement as an uncertainty router on closed-label steps. High agreement is not trust. |
| Personal-informatics lifecycle | Epstein, Rogers | Convert to dogfood metrics: preparation friction, review burden, action completion, lapse/backlog slope. Do not build an analytics product. |
| Provider knowledge bases | OpenAlex, PubTator, Semantic Scholar-style papers | Cache provider fields with provenance and drift checks; providers are enrichment, not authority. |
| Context-aware computing | Dourish, Bell/Dourish | Treat graph and facets as per-occasion projections. The authored file plus provenance is canonical, not any one context model. |
| Mixed-initiative Co-PI | Horvitz, Amershi | Co-PI must expose plan, sources read, assumptions, unknowns, and pending writes before enqueue; chat is additive, not primary. |
| Visualization defaults | Heer/Bostock, Bostock D3, Hearst | Use stable-key projections, simple position/length encodings, and tables/facets first; small spatial views must beat the table baseline. |

## Reject

These are tempting because papers make them look powerful, but they conflict with
alpha.15's integrity or scope.

| Idea | Source pressure | Reject reason |
|---|---|---|
| End-to-end autonomous scientist as product posture | AI scientist / SciAgents / auto-research cluster, ScienceAgentBench, Bisht | Alpha.15 is PI-directed. Agent autonomy stays bounded to proposals and checked operations. |
| All-vault shell Co-PI as safe supported surface | Greshake, AgentDojo, InjecAgent | Trusted-local dogfood only. A shell agent with its own file/network tools bypasses Memoria's read scope. |
| Agent-generated raw UI / HTML / app surfaces | Generative UI and agent-app papers | Executable UI is another untrusted code path. Use declarative, graph-bound, fixed components only. |
| Per-item human approval as quality oracle | Jacobs, Feng/Liu, AutoResearchClaw | Dense approval becomes rubber-stamping and can reduce correctness. Use sparse, calibrated checkpoints. |
| Numeric confidence or fluent rationale as trust | Jacobs, Ye/Durrett, Wallat | Explanations and confidence can increase overreliance. Show source spans, routing reason, and unsupported claims instead. |
| LLM-authored bibliography or metadata | Rao and Callison-Burch, CiteME/CiteGuard | Bibliography identity must come from resolvers/providers; model text is candidate-only. |
| Atomic facts as canonical memory | LongMemEval, MemoryAgentBench | Facts are good keys and bad values. Store context; index atoms. |
| Spatial graph as default knowledge surface | Hearst, Fekete, Cognitive Dimensions, refutation pass | Dense spatial layouts mislead and destroy findability. Tables/facets are the default. |
| Arbitrary external agents/plugins/tool marketplaces | AgentDojo, InjecAgent, MCP/App ecosystem analogs | Violates MCP-only sandbox and expands the lethal-trifecta surface. Only Memoria's own engine tools are in scope. |
| Self-modifying prompts, skills, or weights | STaR/Toolformer/self-improvement cluster | Fine as offline research; not a runtime alpha.15 mechanism. Prompt/eval changes must be explicit, journaled, and held-out-gated. |
| Provider-scale search stack as baseline engine | PubTator-style large literature resources | Borrow entity/relation enrichment only. Do not add a server-shaped dependency before local qmd/SQLite proves insufficient. |
| "No contradiction found" as a truth claim | FEVER, BRIGHT, TEMPO, NLI/HANS papers | Only say "none found in scanned set"; candidate recall and coverage are part of the result. |

## Immediate Design Feed

1. Add a "BAR feed" paragraph to the alpha.15 Scope Boundary: borrow the core mechanics now,
   adapt surfaces after dogfood, reject autonomy/platform expansion.
2. Turn the dogfood checkpoint into a state-based harness with final-state,
   cost, tool-call, stale-memory, and poisoned-read assertions.
3. Add work-aspect read-model rows and evidence-set classes before paper export.
4. Make `ask` return retrieval trace and bounded-universe coverage, not only an
   answer with citations.
5. Mark all-vault CLI Co-PI as trusted-local dogfood only.
6. Rename "cross-store ACID" to operation-level recoverability / observable
   atomicity.
