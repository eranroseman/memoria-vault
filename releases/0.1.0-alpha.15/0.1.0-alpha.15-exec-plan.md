## 1. Purpose

Alpha.15 hardens the alpha.14 engine into a defensibly-correct one and pins the
clean-slate stack. Over alpha.14 it lands:

- **Stack**: `uv` distribution, Typer CLI, stdlib `sqlite3` with **no ORM**, WAL,
  `.sql` + `PRAGMA user_version` migrations (design §S1/§S2/§11).
- **Universal quarantine-and-verify invariant** with a consumption-hash read guard
  that **refuses + enqueues** on mismatch (no read-path write; scan demotes under
  the lock); promotion is the **shared verification pipeline** (operation lifecycle
  for engine writes, `workspace scan` as the foreign-edit ingress), and detection
  covers **all tracked paths** (§2/§9).
- **Cross-store recoverability contract (observable atomicity)**: 2PC lifecycle + prepare→materialize→finalize (the
  materializing commit SHA is recorded after `git commit`), write-ahead fsync
  ordering, workspace-scoped lock, scan-side demotion (no read-path write),
  crash-at-each-boundary recovery (§6).
- **Git/journal division + commit contract**: git owns file history + the
  `.memoria/journal-head` anchor; the journal stores git refs; one
  stage-only-materialized-paths commit per operation; file rollback = git restore
  + DB inverse (§6/§11).
- **Storage recovery classes**: authority-vs-recovery split, fail-closed-to-
  `unchecked` on DB loss, blob-replay for provider-derived records, and the
  human-override recovery log `.memoria/overrides.jsonl` (§3).
- **Enrichment merge** by static field authority with deterministic replay (§8).
- **Native gap analysis** with an ordinal severity scale (§10).
- **Runner policy**: two OpenAI-compatible providers (`local`, `gateway`),
  per-operation test/live model pin, `--mode` (§4/§7).
- **Contradiction Tier-1**: NLI + abstain + HANS acceptance gate (§9).
- **Deterministic edge extraction (raw)**: explicit-typed links → unchecked
  candidate edges (§4/§9).
- **Engine read-API + verdict-tagging + CLI agent**: `engine/api` single module,
  read commands emit `--json` carrying `check_status`, untrusted text returned as
  data (§2/§5, §1/§2).
- **§S5 integrity principle**: no "checked = approved"; frontmatter is meaning-only
  — `check_status` (and all verdict/derived state) is **not written to frontmatter
  at all**, and a gate asserts frontmatter carries no verdict fields.

Observable outcome: a fresh `uv tool install`ed `memoria` that runs the minimum
loop (`init` → `work add` → `work enrich` → `work interview` → `work digest`
(+ `new hub <tag>` when a topic earns a hub) → `new note`/`check` (candidates via
`attention resolve --apply`) → `project gaps` → `ask` → `project export` (Markdown)
→ `workspace recover`), survives a crash injected at each lifecycle boundary,
fails closed to `unchecked` on DB loss, and passes the seeded-error bars — with
the stack, recoverability, and quarantine guarantees above. (Command names per design §5 —
the verb-first scheme; `note propose`/`note accept` do not exist in 15.)

## 2. Context — delta on alpha.14

- alpha.15 is a delta; the design index Scope Boundary is the authority.
  Everything outside that in-15 boundary is deferred (see §6 of this plan).
- The alpha.14 CLI spine, SQLite engine, request envelope, worker lifecycle,
  read barrier, enrichment substrate, and qmd wiring are
  the base. alpha.15 sharpens them; it does not rebuild them.
- **Verified baseline facts** (checked against `origin/main` @ `88b7f0f5`,
  2026-07-02 — so slices claim the right work):
  - CLI is stdlib **argparse** with `note propose/accept/reject`,
    `work capture`, `attention resolve/worklist` present → PR-A's Typer cutover
    and PR-B3's verb renames are real changes, and the retirement gate has
    live targets.
  - `state.py` sets only `PRAGMA foreign_keys = ON` — **no WAL, no
    `user_version`**; the schema is inline `CREATE TABLE IF NOT EXISTS` Python
    → PR-A's WAL/`user_version`/`.sql` extraction is real work.
  - The journal **is** hash-chained (`prev_hash`/`row_hash`, GENESIS), but
    **no tracked `.memoria/journal-head` file exists** — `journal_head()` is a
    DB query only → PR-E *creates* the git anchor, it does not merely keep it.
  - `commit_writer_changes` **already stages targeted paths**
    (`git add -- <selected>`; no `git add -A` anywhere) and
    `mark_materialized(..., commit=)` **already records the commit SHA after
    `git commit`** (a finalize-like step per output) → PR-D/PR-E formalize
    the `pending` pre-finalize window semantics, fsync ordering, workspace lock, and the
    foreign-dirty-file fixture; they are contract + gates, not a rewrite.
  - Frontmatter **does** carry `check_status: unchecked` + `standing: current`
    in shipped templates (reinforced by #1157) → PR-B's supersession is real,
    newly-enlarged work.
  - `providers.yaml` already lives at `.memoria/config/providers.yaml` ✓; qmd
    already materializes the checked-only tree at `.memoria/index/qmd/checked`
    with collection `memoria-checked` (the §10 spike's duplication target; an
    `include_stale` parameter already exists on `checked_search_documents`).
- Two things alpha.15 changes that touch existing alpha.14 code and need care:
  the **frontmatter `check_status`/`standing` supersession** (meaning-only
  frontmatter overrides the alpha.14 field contract that #1157 just hardened),
  and the **commit contract** (the writer already stages selected paths; the
  new binding parts are one-commit-per-operation, the control-file set, and
  the never-sweep-foreign-edits gate).

## 8. Risks

- **Commit path**: `git add -A` anywhere reintroduces the quarantine-bypass
  (finding #3). PR-E1 enforces stage-only-materialized-paths with a gate; PR-E2
  removes capability product code from the workspace tracked set.
- **fsync ordering**: durability/atomicity depend on payload-before-commit and
  commit-before-materialize; PR-D must pin and test it.
- **`engine/api` factoring**: if adapters grow logic outside `engine/api`, thin
  transport parity decays. PR-K's purity gate and PR-T's MCP wrapper tests
  prevent it.
- **NLI without HANS**: an NLI comparator that has not passed HANS is no better
  than cosine on the cases that justify it; PR-I gates on it.
