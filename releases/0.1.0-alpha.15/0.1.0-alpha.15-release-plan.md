Draft release parent issue body. Live release state belongs in the GitHub
milestone, the "Release 0.1.0-alpha.15" parent issue, and its gate/stage

authority is the design index Scope Boundary.

## Status

Draft sandbox release plan for the alpha.15 engine + integrity hardening cut. It
is a **delta on alpha.14**: Memoria inherits the alpha.14 standalone CLI + engine
whole and extends — does not rewrite — its lifecycle, read barrier, and rollback
model. Alpha.15 pins the clean-slate stack (`uv`, Typer, stdlib `sqlite3` no-ORM,
`.sql`/`user_version`) and lands the integrity mechanisms that make the engine
defensibly correct (quarantine-and-verify with a consumption-hash guard,
cross-store recoverability, git/journal division + commit contract, storage recovery
classes, NLI Tier-1 proposer), plus the minimum product loop (enrichment merge,
gap analysis, Markdown export, recover) and the CLI agent contract.

## Scope

Alpha.15 delivers, in `Memoria-test`, the **safety/integrity core plus the
minimum end-to-end loop** (design index Scope Boundary):

- Stack cutover: `uv tool install memoria` (`pip install -e .` fallback), Typer
  CLI, stdlib `sqlite3` with WAL and FK, `.sql` + `PRAGMA user_version`
  migrations, no ORM.
- Universal quarantine-and-verify invariant + consumption-hash read guard that
  **refuses + enqueues** on mismatch (no read-path write; scan demotes under the
  lock); promotion via the **shared verification pipeline** (operation lifecycle for
  engine writes, `workspace scan` = foreign-edit ingress); detection over all
  tracked paths.
- Cross-store recoverability: 2PC lifecycle, write-ahead fsync ordering, workspace-scoped
  lock, crash-at-each-boundary recovery.
- Git/journal division + workspace commit contract: git owns file history +
  `.memoria/journal-head`; journal stores git refs; one stage-only-materialized-
  paths commit per operation; rollback = git restore + DB inverse.
- Storage recovery classes: authority-vs-recovery split, fail-closed-to-
  `unchecked` on DB loss, blob-replay records, human-override log
  `.memoria/overrides.jsonl`.
- Enrichment merge by field authority with deterministic replay; native gap
  analysis with an ordinal severity scale; runner policy (two OpenAI-compatible
  providers, per-op test/live model pin, `--mode`); NLI Tier-1 with a HANS
  acceptance gate; raw deterministic edge extraction → unchecked candidates;
  engine read-API + verdict-tagging + CLI agent; the §S5 integrity-principle sweep.

The minimum loop runs end to end (command names per design §5, the canonical
verb table — `note propose`/`accept` and `work capture` do not exist in 15):
`init` → `work add` → `work enrich` → `work interview` → `work digest`
(+ `new hub <tag>` when a topic earns a hub) → `new note`/`check` (candidates:
`attention resolve --apply`) → `project gaps` → `ask` / `project ask` →
`project export` (Markdown) → `workspace recover`, with no
Obsidian/Zotero/Hermes/MCP/profile dependency.

## Known Limitations

- Limitation: sandbox-only; no installed-workspace migration and no
  backward-compatibility mode. Impact: alpha.15 starts from fresh disposable
  workspaces only. Workaround: test only in `~/Memoria-test`. Tracking: alpha.15
  parent.
- Limitation: the all-vault CLI agent is **trusted-local dogfood only**, never
  a supported safe agent surface (design §1/§2 — quarantine bounds writes, not
  reads/exfiltration). Scoped MCP now bounds reads through Memoria-granted tools,
  but a local agent with its own file/terminal tools is still trusted-local.
  Workaround: use `memoria mcp --read-scope <path>` for scoped cooperating
  agents; use `memoria … --json` only on your own vault.
  Tracking: trusted-local caveat in release notes.
- Limitation: Obsidian plugin implementation, rich UI frameworks, per-host
  wrapper packages, and generated UI/forms are excluded from alpha.15. Local
  HTTP and scoped MCP have landed as machine transports, not UI commitments.
  Impact: CLI, local HTTP, scoped MCP, and the read/view/action contracts are
  the supported alpha.15 surfaces; Obsidian plugin code, manifests/packages,
  command-palette wiring, Inspector/palette UI, adapter-specific UI, and
  wrappers are future-adapter work that need their own ADRs. Tracking: future
  adapter planning.
- Limitation: paper/outcome readiness is structural only until the aspect model
  exists. Impact: `project export --ready-only` proves required framing and
  checked support, not semantic sufficiency. Tracking: aspect-model slice.
- Limitation: re-enrichment cadence and export format beyond Markdown +
  `references.bib` remain open (§15). Impact: do not block the first checked work,
  digest, ask, project export, or recovery. Tracking: alpha.15 open questions.
- Limitation: real-time detection only runs while `memoria serve --watch` is
  up. Impact: without `serve`, direct edits are caught at scan time or fail
  closed at consumption — the guard makes quarantine immediate-on-consumption
  either way. Tracking: landed in PR #1184; no daemon install is planned.
