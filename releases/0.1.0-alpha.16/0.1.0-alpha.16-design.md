# Memoria — 0.1.0-alpha.16 system design (the READ cycle + knowledge-production substrate)

Date: 2026-07-05
Status: **alpha.16 design — implementable now, grounded in the Borrow/Adapt/Reject analysis**

This design builds **forward from the alpha.15 reset — it is not a clean slate.**
alpha.15 encoded the earned lessons; this increment inherits them, folds in the
genuine engineering and research contributions that postdate them, and restores
the knowledge-**production** core that the blind beta.1 draft deleted. Every
non-obvious choice below names its basis — a research finding, a documented
failure, or a Borrow/Adapt/Reject verdict — never "an ADR decided it" or "a
version had it." This design **implements** the governing
[0.1.0-alpha.16-requirements.md](./0.1.0-alpha.16-requirements.md) (requirements,
evidence, technical adopt/defer decisions, and acceptance) — that document names
*what must be true*, this one names *how*. Other governing inputs, read as the
design base, not background:
[`./00-redo-method.md`](../0.1.0-beta.1/resources/redo/00-redo-method.md) (method, inverted burden of proof),
[`./alpha15-borrow-adapt-reject.md`](../0.1.0-beta.1/resources/redo/alpha15-borrow-adapt-reject.md), and
[`./beta1-adopt-borrow.md`](../0.1.0-beta.1/resources/redo/beta1-adopt-borrow.md). Where the evidence does not

## 0. Identity & ruling ideas

**Memoria is an opinionated, single-person knowledge-production tool — an
AI-enhanced Zettelkasten utilizing the LLM-wiki framework.** Its mission (held
with a grain of salt, as the method requires) is to help one researcher **read**
(accumulate / compile / curate knowledge) and **write** (compose / create new
knowledge), with code and text as the main outputs. alpha.16 is the **READ
cycle plus the substrate the WRITE cycle will stand on**: the full writing and
coding *output* modules are legitimately post-MVP (§14). Everything the READ
cycle needs to reach real, feedback-producing use is *in* scope — scope is set
by user need, not by deferral convenience.

Six ruling ideas govern the rest, in priority order:

1. **Two graphs, one bridge.** A *catalog graph* (objective, relational, SQLite,
   deriving from the bibliography) and a *ZK knowledge graph* (subjective
   markdown: notes / claims / hubs joined by typed argument edges) are distinct
   representations bridged by the **source-note**. This is the spine (owner
   two-graph model). beta.1's deletion of the ZK-graph half was not
   research-justified — TEMPO, Memora, and Hearst *require* keeping it
   (alpha.15 BORROW-3/4/5).
2. **Propose, never dispose; honesty over a synthetic index.** Every machine
   recommendation renders as argument-for / argument-against / what-tipped-it /
   coarse certainty (≤3 levels). No score-with-a-threshold is ever presented as
   truth — the owner's named biggest pitfall, and Jacobs 2021 shows a wrong
   machine verdict drags the human *below* their own baseline (0.299 vs 0.357).
3. **Checked is not approved (no write-time oracle).** A deterministic or NLI
   check certifies a structural property, never fitness; a human never clicks OK
   on a machine verdict presented as fact. Human+ML ≈ human-alone ≪ ML-alone
   (Jacobs 2021) means the gate cannot be theater.
4. **Integrity is structural, not semantic.** Knowledge integrity is the
   structural integrity of the graph — links, anchors, and references resolve —
   verified on any change; it is **not** the per-node semantic validity of a
   single note or edge (owner alpha.15 model; single-node NLI recall .38,
   κ=0.68 make per-node semantic gating unreliable).
5. **Keep-test rules formats.** Everything the researcher must keep if Memoria
   dies is a markdown file or a PDF inside the vault, navigable with only
   `vim` / a file browser. Derived and verdict state lives in a single-file DB
   *outside* the synced tree (alpha.14 keep-test; beta.1's plugin-required UI
   reversed it, a DO-NOT-ADOPT).
6. **One writer, staged, run-scoped, revertible.** Exactly one code path writes
   machine content; writes stage then apply per run; reversibility is the
   demotion cascade **and** snapshot/revert together (beta.1 ADOPT-3/4;
   alpha.15 ADAPT-1 — cascade and snapshot are complementary, not substitutes).

## 1. Host & process model

**One standalone engine (`mem`) + on-demand `mem serve` + one UI surface. No
resident daemon.**

- **`mem` is the CLI survival + ops floor and the only durable artifact**:
  `capture` / `import`, `export`, `rebuild`, `backup` / `restore --verify`,
  `migrate`, `revert-run`, `cost`, and `serve`. The keep-test answer is
  stock-editor navigability of the files, not a CLI, so there is deliberately
  **no `mem ask`** — Ask is a judgment surface with no survival role.
- **`serve` is spawned on demand for a work session, not resident.** It starts
  when the UI connects or a job is enqueued and exits when idle; it is **not** an
  unattended always-on loop and runs **no background scheduler** (documented
  rejection: an unattended always-on loop conflicts with the no-daemon
  architecture; B/A/R: beta.1's resident plugin-supervised daemon is a
  DO-NOT-ADOPT regression). Periodic work — reconciliation, the citation monitor
  (§10) — runs at session start or on explicit trigger, never as a resident cron.
- **The process still splits in two, for safety, not residency.** A *supervisor*
  holds the BYOK key, the DB handle, Guard (egress broker), the Ledger, and the
  single VaultWriter; a per-run *engine worker* is credential-free and
  socketless, with the filesystem scoped to staging + a read-only vault. PDF
  extraction, embeddings, and NLI do not belong in an editor's process; a
  separate OS process gives crash isolation and one network choke point (beta.1
  ADOPT — the supervisor/worker split is a real safety gain, independent of the
  residency regression it was wrapped in).
- **Work survives restart without running unattended.** The job queue is
  DB-persistent so a batch-API digest can be resumed at the next `serve`;
  offline, catalog / search / notes / reading fully work and LLM jobs queue and
  fail cleanly, never half-write — every machine write is staged (§5, B/A/R:
  beta.1 ADOPT-12 staged writes + offline contract).
- **One UI surface, keep-test floor under it.** alpha.16 ships a single UI
  surface — provisionally an Obsidian plugin pane (§11) — carrying only the
  interactions files cannot render: **Ask** and **Review**. Capture, Compose,
  and Reading ride files, CLI commands, and native primitives, so the product
  runs unaided in any editor. **The host is a provisional position, flagged open
  in §16**, because the owner's own clean-slate note records Obsidian was
  "chosen as a default without evaluating fit" and "is not clearly the best
  option for a structured knowledge graph."

## 2. The two graphs & vault layout

The two graphs are the design's spine. They differ in kind and must not be
collapsed (owner: the source-note "connects the catalog graph to the knowledge
graph"; the entity-vs-note tension is a named hazard).

- **Catalog graph — objective, relational, SQLite.** Sources, authors, orgs,
  venues, references, cited-by: a deterministic representation that *derives
  from the bibliography*. SQLite is authority because the graph is relational by
  nature and an 867-entry spike + two red-team rounds grounded it (alpha.15
  BORROW-7; beta.1's markdown-authority `record.md` reverts that spike and is a
  DO-NOT-ADOPT). Keep-test projections travel in files: a portable
  `bibliography.bib` and a per-work `record.md` carrying full CSL-JSON in a
  fenced block (the fidelity + Zotero-re-import copy), from which the DB
  rebuilds.
- **ZK knowledge graph — subjective, markdown.** Notes, claims, hubs, and the
  source-notes, linked by **human-authored typed argument edges
  {supports, contradicts, extends}** (§3). This is the WRITE-side substrate the
  mission centers and beta.1 deleted; TEMPO (claim context), Memora
  (supersession as a *precondition* for durable state), and Hearst (faceted
  navigation) support keeping it (alpha.15 BORROW-3/4/5).
- **The bridge is the source-note.** One source-note per work links the
  objective catalog row to the subjective note graph. It is where "in your own
  words" begins (owner: adding a source assumes the user knows what it is about;
  an LLM writing the source-note is task delegation the user may rewrite).

**Vault layout** (`Memoria/` is the bundle boundary; the researcher's
pre-existing notes outside it stay untouched — zero migration, since absence of
all `mem-*` verdict markers = human):

```
<vault>/
  <existing notes>                    # untouched; zero markers required
  Memoria/
    inbox/                            # staging drop; imports mint on the user's
                                      # action, never silently
    works/<work_id>/
      record.md                      # catalog projection: full CSL-JSON fenced
      fulltext.md                    # anchored extracted text (machine, untrusted)
      digest.md                      # LLM digest (machine, untrusted, opt-in)
      source.pdf                     # immutable raw copy, sha256 recorded
    sources/<slug>.md                # source-notes (the bridge; human/gated)
    notes/<slug>.md                  # claim- & question-mode notes (human)
    hubs/<slug>.md                   # hubs: tag-membership + human salience prose
    projects/<slug>/
      project.md                     # question/thesis + outline (human)
      evidence.md                    # pinned evidence (machine, idempotent)
      gaps.md                        # gap report over BOTH graphs (machine)
    bibliography.bib                 # generated BibTeX (portable, keep-test)
    system/                          # visible (not dot-named): audit copies must
      manifest.jsonl                 #   survive device loss / sync of hidden dirs
      incidents/
      metrics/
```

The single-file **derived + operational DB** lives outside the synced tree at
`~/.local/share/memoria/<vault-id>/memoria.db` (WAL; FTS5 + sqlite-vec;
per-device, transactional, `VACUUM INTO` backup). Two DB classes, stated
because the keep-test depends on the split: *derived state* (chunks, vectors,
links, anchor geometry, catalog rows, check verdicts) rebuilds from vault files
+ provider payloads via `mem rebuild`; *operational ledgers* (runs, jobs,
events, costs, incidents, guard log, and the content-addressed provider
`payloads` store) are **not** rebuildable and are not claimed to be — they are
backed up and their safety/eval slices exported into `system/` (beta.1
derived-store two-class split — a real improvement; ADOPT).

## 3. Concept/type model & identity

**Identity: a stable `work_id` (ULID-class), never the citekey.** Citekeys
mutate — Zotero re-exports the same work under a new key when metadata changes —
so a mutable key cannot be identity (owner R-1). The citekey is a display /
citation attribute stored on the work; `work_id` is the primary key and the
directory name. This adopts the *mechanism* on a *fact*, rejecting the
"because a numbering principle exists" authority justification (alpha.15
REJECT-3).

**Types** (a small, opinionated set — the type is a design surface, not a
configurable one):

- **work** — a catalog entity. DB row (authority) + `record.md` keep-test
  projection + immutable `source.pdf` + anchored `fulltext.md`. Objective
  properties only.
- **digest** — an opt-in LLM distillation of a work (machine, untrusted); claims
  cite verbatim anchors (§8).
- **source-note** — the bridge (§2). **One source-note type**: item-note vs
  paper-note differed only in metadata and connections, not essence, so source
  type collapses to a single `source_type` property (owner: granular source
  types — book chapter, preprint, repository, dataset — without convoluting the
  vault).
- **note** with a **claim-mode / question-mode** distinction and first-class
  **temporal-context fields**. A claim's truth is determinate *only* relative to
  temporal scope + tense; a bare claim ripped from its paper loses its
  time-window, hedging, and scope (research: TEMPO — "require ISO temporal
  scope, tense, certainty/hedging, and originating section as first-class typed
  fields"). beta.1 dropped these *against* that evidence (alpha.15 BORROW-4).
  A question-mode note is an open question pointed at a thesis.
- **hub** — **tag-membership (mechanical: checked-tag membership) + human
  salience (prose).** Faceted/tabular membership is the validated form at scale
  (research: Hearst — faceted beats spatial); a hub is a structure note that
  organizes claims toward a topic or a piece of writing. beta.1's total drop had
  no research basis (alpha.15 BORROW-5).
- **project** — a question/thesis plus its artifacts (`project.md`,
  `evidence.md`, `gaps.md`). Optionally an independent knowledge bundle (owner
  Round 2 — open; §16).

**Typed argument edges {supports, contradicts, extends}** are the argument
substrate, **human-authored**. Deterministic edge extraction reads explicit
typed body links into *unchecked candidate* edges only; argument edges are
**never auto-promoted** (documented failure: auto-typing bare links corrupts the
argument graph; NLI/contradiction auto-writes are refuted — alpha.15 BORROW-12,
ADAPT-5). Edges are the argument layer, **not the retrieval representation**:
retrieval and synthesis run on BM25+dense, never graph traversal (research:
MemoryAgentBench put GraphRAG/Cognee/Zep *below* plain BM25 — alpha.15 ADAPT-5;
beta.1 ADOPT-2 reads this correctly).

**Faceting / controlled vocabulary** is borrowed as the topics substrate
discovery needs, but **machine classification is enrichment-only + human
certified** — no machine-materialized frontmatter facets (alpha.15 ADAPT-6;
consistent with meaning-only frontmatter, §4).

## 4. Provenance & frontmatter

**Meaning-only frontmatter; files own meaning, the DB owns verdicts.** This is
earned by a documented failure: `check_status` in frontmatter meant checking a
note rewrote it, hand-edits forged the status, and every consumer had to
distrust the file it was reading (alpha.15 BORROW-1). Corollary: any in-file
marker is display-only, never authority.

Files carry only meaning-bearing keys; the DB carries every verdict (check
state, taint, stage state, integrity, run attribution):

```yaml
---
# meaning-only — files own meaning, DB owns verdicts
type: source-note          # work|digest|source-note|note|hub|project|report|incident
work_id: 01J9Z8P4K3...     # stable identity; citekeys mutate, cannot be the key
citekey: doe2023attention  # mutable display/citation attribute, not identity
title: "Attention and working-memory capacity"
source_type: article-journal
topic: [working-memory, attention]   # human-certified facets (machine = enrichment-only)
project: [wm-review]
mode: claim                # note types only: claim|question
temporal_scope: 2015/2023  # ISO — TEMPO requires it as a typed field
tense: present
qualifier: "in adults under cognitive load"
certainty: reported        # human-set label, never a machine score
superseded: false          # researcher-set; Ask excludes by default, counted
reading: read              # researcher-set pass-1 disposition (skip|cite-only|read)
---
```

- **Fulltext anchors** are one Obsidian block ID per paragraph, assigned once at
  extraction, zero-padded in emission order — plain text that survives reopen,
  sync, and the death of the tool (`…under load. ^p0042`).
- **The in-file marker is a claim; the verdict is DB state** (§5). A block's
  check state is a `checks` row bound to the block's `sha256` at check time; a
  UI badge paints green only when marker, *current* block hash, and DB row
  agree. A human edit changes the hash → the block reverts to unchecked and
  re-enqueues; a `rebuild`/restore shows unchecked until the $0 local Checker
  re-earns each verdict — **fail-closed to unverified, never fail-open to a
  stale green** (beta.1 ADOPT — the DB-authority-with-three-way-agreement fix;
  and B/A/R: an in-file provenance marker is borrowed *only if a UI need is
  proven*, else DB-only — §16).
- **`mc` is named "citation-correct," never "grounded / verified / safe."** NLI
  entailment certifies the cited anchor supports the sentence; it cannot certify
  the sentence was *derived from* it (research: Wallat — up to 57% of RAG
  citations are post-rationalized). The byte-substring quote guarantee carries
  the real safety; the faithfulness gap is documented, not papered over (beta.1
  ADOPT-9).
- **Taint is a separate, monotone, never-green dimension.** Anything derived
  from untrusted origin is tainted at creation and inheritance is monotone;
  checking never clears taint. Untrusted source text never interpolates into an
  instruction template, and a manifest violating that fails to load (alpha.15
  BORROW-11; research: injection sealing narrows the channel).
- **Frontmatter validation stays permissive** (tolerate unknown keys) — the one
  place beta.1's looser stance is better grounded; a closed reject-undeclared
  validator was never earned by a documented failure (alpha.15 REJECT-2).

## 5. Integrity, review & gate

**Integrity = structural integrity of the graph, verified on any change**
(owner alpha.15 model). A validity incident can arise from three events: (1) any
change by a human or machine triggers a verification response; (2) the user
declares an error; (3) periodic (owner alpha.15, as far as the paste is
captured — the remainder is truncated and open, §16).

- **Quarantine-and-verify.** Any change — human, agent, or machine — is
  **non-consumable until verified** (alpha.15 BORROW-10). The verification
  *target* is structural: links, anchors, and references resolve. It is **not**
  a per-node semantic check, because single-node semantic checks are unreliable
  (NLI recall .38, κ=0.68 — alpha.15 ADAPT-4).
- **The demotion cascade AND snapshot/revert, together.** A two-tier
  stale-vs-invalid cascade propagates over the derivation graph so one demoted
  node does not flood a whole subgraph (earned build-system lesson — alpha.15
  ADAPT-1); beta.1's snapshot → apply → verify → commit batch protocol +
  pathspec-limited commit + one-click revert adds reversibility (beta.1
  ADOPT-4). They are complementary; dropping the cascade for snapshots alone was
  a beta.1 regression the redo corrects.
- **The single VaultWriter** is the sole code path that writes machine content;
  human-file protection, taint, exfil neutralization, and the hash manifest are
  enforced there once (beta.1 ADOPT-3 — a real fail-closed safety gain).
  Human-content blocks are byte-immutable to the writer; a stale-base check
  skips and requeues any file the user typed into during a run, so machine
  writes never clobber concurrent human keystrokes. Verify recomputes each
  touched file's human-span hash before-vs-after under the lock; a mismatch
  rolls the run back whole and files an incident. The one deliberate exception
  to whole-run atomicity is **per-item batch failure** (32 good digests are not
  rolled back because 2 timed out) — atomicity protects the *safety* property
  (no human text altered), not job luck.
- **The review gate is sparse and uncertainty-routed, never per-item.** Targeted
  intervention at a few checkpoints dominates dense per-step approval, which
  erases automation gains and which time-constrained users refuse (research:
  AutoResearchClaw HITL ablation; Jacobs 2021). Machine proposals render as
  **honesty cards** — action, argument-for, argument-against (the machine's
  strongest self-rebuttal, the information-bearing field), what-tipped-it,
  coarse certainty (≤3 levels) — **with no verdict line on a proposal** (a
  verdict there is vacuous and induces automation bias). Any report needing
  dozens of same-shape decisions renders as **one batch worklist**, never N
  drip-fed items ("forty cards train select-all-accept"). **A rejected engine
  output is auto-added as a regression case** to that operation's fixture — the
  human catching a bad output becomes a permanent guard (beta.1 review
  discipline — ADOPT; owner propose-not-dispose + anti-anchoring).

## 6. Machine judgment & LLM allocation

**Memoria is not an agent, and alpha.16 adds none.** A resident reasoning loop
would re-create the exited-framework failure with the loop owning state and the
write boundary, has no arithmetic worst-case price, and is an uncontrolled
variable inside a pipeline built on pinned prompts (beta.1 §13 — ADOPT the
doctrine). Any agent — a future first-party worker or the researcher's own
Claude Code / Cursor — is a **client**: read-API + request envelope,
universally quarantined, read-scoped, never granted file/exec/send tools.

**The operation test — which LLM calls belong in the product.** Memoria runs an
LLM call directly iff it is an *operation*: (1) pinned prompt + pinned model, so
it ships with a fixture and a seeded-error test; (2) deterministically checkable
output (schema, verbatim-anchor, byte-substring); (3) an arithmetic worst-case
price (`max_tokens` ceiling + a Ledger price-table row); (4) per-item,
corpus-scale, unattended shape. Failing any, the call belongs to the
researcher's own agent, on their bill, entering the vault only as envelope
writes that land unchecked. The alpha.16 roster: the **digest** (§8), the
**seeding relevance prefilter** (§8), optional **paid synthesis** (§9), and the
local pinned **embeddings + NLI** as Memoria-owned machine judgment.

**Local NLI is a calibrated, high-precision *proposer*, never an authority.**
Off-the-shelf NLI is high-precision / low-recall, miscalibrated, and
size-dependent (research: ExpertQA precision .88 / recall .38; Sainz — default
T=0.5 badly miscalibrated, ~15-point swing from dev-tuning). So:

- Thresholds are **dev-tuned on a tiny hand-labeled calibration set then
  frozen**, seeded with the known failure classes — same-topic/different-variable
  pairs (which NLI confidently mints as false contradictions → route to
  *unknown*) and high-overlap/opposite-meaning pairs (beta.1 ADOPT-9).
- The Checker must pass a **HANS/PAWS-style high-overlap negation +
  direction-reversal probe as a launch-blocking acceptance test** — a floor, not
  a certificate; if it fails, NLI is no better than cosine on the cases that
  justified it (research: Utama/HANS — median accuracy <10% on the
  lexical-overlap non-entailment subset).
- **`mentions` means *unknown stance*, never "no contradiction"**, and an empty
  contrasts list is **never** presented as absence of disagreement (research:
  MRMR — contradiction retrieval is near chance on negation; an empty list would
  be the implicit agreement meter the guardrail exists to prevent).
- **No LLM tension judge.** Contradiction surfacing stays on the local NLI
  proposer only; an LLM-judge on this path is refuted (CiteGuard recall 16–17%;
  ConCoRD ~28% wrong belief-flips — alpha.15 REJECT-6). Every candidate
  contradiction is a *proposal* the human dispositions; the human sets the
  directional argument edge.
- **Shadow-first discipline.** No score changes routing, promotion, or
  visibility until it is grounded on recorded dispositions; every tunable ships
  behind `production_enabled: false`. This *is* the owner's anti-synthetic-index
  guardrail, and every threshold in the research corpus was un-validated
  (alpha.15 BORROW-8). Borrow the discipline, not any specific constant.

Neither Memoria nor an agent ever confers a verdict, performs a rule-derivable
write, writes human-class text, or chooses an egress destination.

## 7. Catalog & enrichment

**SQLite is the catalog authority; it derives from the bibliography.** The
catalog graph is relational; the keep-test is served by the `.bib` + `record.md`
projections (§2). Compact schema, work-keyed on the stable id:

```sql
works(work_id PK, citekey, doi, title, year, source_type, csl_json,
      status, pdf_sha256, distrust_flags)      -- citekey is a mutable attribute
entities(id PK, kind, label, ext_ids_json, status)      -- authors/venues; unresolved by default
entity_mentions(work_id, entity_id, raw_name)
edges(src_work, dst_key, kind, source, retrieved_at)    -- references, capped cited-by, paper<->repo
payloads(sha256 PK, provider, fetched_at, json)         -- raw provider evidence, verbatim
checks(path, block_id, verdict, block_sha256, run_id, ts) -- mc-authority (§4/§5)
-- + chunks/chunks_fts/vec_chunks/anchors/links/runs/jobs/events/costs/incidents (derived + ledger)
```

- **Per-field best-source-wins merge with provenance.** Machine fills empty
  fields only; conflicts render as per-field left/right diffs through Review,
  never auto-applied (the JabRef rule). Earned by a documented failure: an
  867-entry spike + two red-team rounds proved each of Semantic
  Scholar/OpenAlex/Crossref incomplete (reference counts 151/129/146; ORCID
  present in OpenAlex, ~0 in S2) and rejected single-source; beta.1's
  single-source empty-field fill departed from this with no new evidence
  (alpha.15 BORROW-6). References = a DOI-deduped union.
- **Raw provider responses are preserved verbatim** as sha256-named JSON in
  `payloads`, and every enriched field/edge carries that hash — so a merge is
  replayable and auditable after a provider later mutates its record (beta.1
  ADOPT; retention-evaluation need).
- **Semantic Scholar is a conditional, default-on-when-keyed provider**
  (reference union, citation intents) — enrichment value, research-neutral;
  gated on key feasibility (alpha.15 BORROW-14; owner R-17).
- **Enrichment is corpus-scoped and $0**, identifiers-only egress, no background
  auto-refresh; refresh follows attention (a work page opening). Counts render
  with provenance and as-of dates ("41 citations — OpenAlex, 2026-07-02"), never
  bare.
- **Entities are DB-side; no machine-minted person/venue pages** (a standing
  reject — the graph is queries, not vault sprawl). Resolution ladder: a
  name-incompatibility veto blocks any merge; an identifier conflict (two
  ORCIDs) is a hard cannot-link; shared ORCID + compatible names auto-merges with
  recorded provenance; everything else rests **`unresolved` by default**, counted
  honestly in any consumer's denominator and adjudicated **lazily at point of
  use**, never as a standing clerical queue (a day-one backfill of thousands is
  the week-3 abandonment mechanism). Venues merge on ISSN-L only; researcher
  same-as decisions are recorded and never overwritten by a sweep.

## 8. Capture, extraction & digest

The capture process is grounded on academic papers but applies to **any digital
source** (owner R-1). Every capture: (1) keep an **immutable raw copy**
(`source.pdf` / page / clone), sha256 recorded; (2) extract content into
markdown; (3) extract metadata; (4) add source + immediate entities to the
catalog graph; (5) create a candidate source-note.

- **Land, then resolve.** Every input (file, DOI/arXiv id, BibTeX entry, URL,
  repo) lands immediately as a catalog stub with its raw artifact attached —
  capture never fails; resolution is a separate, retryable, visible step, and a
  failure is a triage stub, never data loss (beta.1 ADOPT — the offline
  contract).
- **Deterministic external grounding for metadata.** Metadata comes from
  authoritative records, not the LLM: Zotero CSL-JSON (native, ≥95% fidelity) as
  the primary; DOI content negotiation via a pinned `citation.doi.org` endpoint
  (redirects disabled) for direct ids. Where an untrusted PDF *nominates* an
  identifier (DOI/arXiv regex over pages 1–2), a trusted registry *authors* the
  metadata, accepted only on a ≥0.95 title fuzzy-match — injection-resistant,
  zero LLM calls. Research: Rao & Callison-Burch — "a reliable mitigation must
  bypass LLM-based metadata construction entirely" (beta.1 ADOPT-7). Two-stage
  reconcile against the authoritative record; below threshold ⇒ `needs_metadata`,
  a fixable chip, never a guess.
- **Extraction: pdfium alone.** BSD-licensed, native, embeddable, matching a
  single-static-binary install; PyMuPDF's AGPL + Python contradicted it, and a
  dual-engine fallback doubled the test surface for no benefit. **No OCR**:
  scanned PDFs fail named as `extract_failed:needs_ocr`. De-hyphenate →
  paragraph-segment → headings by font percentile → linearized markdown with
  `^pNNNN` per paragraph. A cheap text-coherence gate fails closed to
  `extract_failed:garbled` before any write — garbled text otherwise silently
  poisons FTS, vectors, and quote verification. Deterministic distrust flags
  (`truncated`, `abstract_only`) continue the honest-continuation rule (beta.1
  ADOPT-13); repo works bypass pdfium (docs are already markdown).
- **Digest (opt-in, always behind a dollar preview).** Pinned prompt; input =
  record core + anchored fulltext clipped by one clipping algorithm; structured
  JSON `{summary, claims:[{text,kind,anchors[]}], methods, key_terms}` with every
  cited anchor appearing verbatim in the input. Deterministic validation before
  staging: schema check; claims citing a non-existent anchor are dropped;
  >20% dropped ⇒ `digest_failed:anchor_hallucination`. The Checker then
  statement-converts each claim (a failed conversion degrades to *unknown*, never
  a guess) and NLI-tests it against the **concatenation of all anchors it cites**
  — multi-span support is real (research: FEVER — 16.82% of claims need
  multi-sentence evidence). Entailed → `mc`; else `m` with `nli_unsupported`;
  taint stays untrusted. Batches ride the provider Message-Batches endpoint at
  50%, poller survives restart, per-item partial success per §5. Every printed
  number is computed in code, never by the model (research: Zou — 37.1% vs 70.2%,
  a pipeline property — beta.1 ADOPT-8).
- **Reading is where the researcher already reads.** The primary read can happen
  in Zotero's PDF reader with append-only annotation sync bridged to `fulltext.md`
  by exact-quote match; `fulltext.md` is also the full reading path for
  non-Zotero works. Annotation *processing* is initiated by a dismissible card,
  not auto-run — the research is unambiguous that initiation, not execution, is
  where annotation-processing collapses. (Zotero-first is a provisional, host-
  adjacent choice riding §11's open host question.)

## 9. Retrieval, Ask & gap analysis

**Retrieval: hybrid BM25 + dense, RRF + MMR, no graph.** FTS5 BM25 top-100 ∪
vector top-100 fused by RRF (`Σ 1/(60+rank)`, no learned weights), per-work
cap, MMR to a working set. A research SURVIVOR: MemoryAgentBench put graph
systems *below* plain BM25 for synthesis, so the typed graph is the argument
layer, not the retrieval representation (alpha.15 ADAPT-5; beta.1 ADOPT-2). Local
pinned models bundled by the installer, never downloaded at runtime; pins live
in `meta` and a mismatch refuses. **Only `works.fulltext` (+ opt-in digest
chunks) is embedded** — embedding the user's whole personal vault buys no
feature and bloats the DB.

- **The stored/retrieved unit is the verbatim byte-exact span, anchored.** The
  atomic claim is an *index over* the span, not the stored unit — the correct
  reading of LongMemEval that the alpha line had inverted, and the single largest
  research gain (Omni-SimpleMem: +53% F1 from returning full text over summaries
  — beta.1 ADOPT-1, a vindicated divergence). Every quote is **asserted to be a
  byte-substring** of its anchored block (fail ⇒ drop + re-index).
- **Ask flow (all local, $0):** deterministic probe derivation for stance (no
  probe ⇒ every item is `mentions`, never a fabricated stance); hybrid retrieval;
  verbatim quote; stance via NLI at the frozen calibration thresholds (entail →
  supports, contradict → contrasts, else `mentions` = *unknown*). Contrasts
  render as "contradictions found *within the retrieved set*"; an empty list is
  never absence of disagreement (§6). `superseded` works are excluded by default,
  **counted in the denominator**, surfaced only under a per-Ask historical
  override with a visible superseded label (research: Memora — obsolete reuse is
  a named memory failure; keep superseded items retrievable, do not present them
  as current). Zero survivors ⇒ "nothing matched lexical/semantic search over N
  documents" — a searched-set claim, never a corpus-ontology claim (BRIGHT/
  HotpotQA: second-hop evidence is missed at high rates).
- **Optional paid synthesis** expands each evidence item **to its enclosing
  section** — a deterministic coherent slice, citability preserved — rather than
  a bare quote (research: Hu — top-k fragments degrade the integrative answers
  the payoff lives on; beta.1 ADOPT-10). Quotes + citekeys + sections only, never
  whole documents/notes/drafts, so local-only material is structurally absent.
  Uncited sentences get a visible `⚠ uncited` flag; the synthesis carries the
  same stance chips and coverage denominator as raw Ask, never a neutral
  narrative overview.
- **Abstention is a pipeline property**, not the model choosing to say "I don't
  know" — faithfulness comes from deterministic grounding (research: Rao &
  Callison-Burch; Shaikh — zero-shot CoT *cuts* refusal 78%→25%).

**Gap analysis runs over BOTH graphs and the gap between them** — the owner's
central WRITE-side driver, and the restored production core beta.1 deleted. It is
**honest coverage reasoning, never a synthetic saturation number** (owner
anti-synthetic-index; the charter's aggregate-score exclusion; "saturation
cannot be pre-registered as a number" — alpha.15 ADAPT-2). Three lenses,
every quantity computed in code and printed with a denominator:

1. **Catalog-graph coverage (source gap).** Which question facets have corpus
   `df = 0` or no surviving item; which works are unsearchable (no PDF /
   extract_failed), partial (truncated / abstract_only, counted as *suspect,
   not covered*), or superseded: "Searched 38 of 46 works fully — 3 unsearchable,
   3 partial, 2 superseded excluded."
2. **ZK-graph saturation (argument gap).** Which project claims lack a
   **support** edge and which lack a **counterpoint** edge — a claim wants ≥1 of
   each (research: Popper corroboration; Memora promotes contradiction/
   supersession from feature to *precondition* — alpha.15 BORROW-3). Rendered as
   named lists ("3 claims have no counterpoint: [[…]]"), never a band or score.
3. **The gap between the graphs (the interesting case).** Dense sources + thin
   notes vs dense notes + thin sources — a well-sourced topic with no claims
   developed, or a claim cluster with no source backing; a work with no
   source-note; a claim with no cited anchor (owner Round 2: "a mismatch between
   the graphs is the really interesting case").

## 10. Discovery

**Bounded citation-graph discovery only** — deterministic, identifier-only
egress, structurally separate from Ask (discovery output never enters gap or
coverage denominators). Open-ended "research the literature for me" is excluded:
it fails the operation test (§6) and is the echo-chamber risk the owner names.

- **Snowball (human-triggered).** Backward from stored reference edges + forward
  from the capped cited-by cache; depth ≤ 2, per-run candidate ceiling; ranking
  by **transparent occurrence counting** across seeds (visible arithmetic, never
  a black-box recommender — the echo-chamber counter and the LTR-suppresses-
  disconfirmation finding, Bisht 2026). Every line is cap-stamped ("appears on 6
  paths from 4 seeds, among top-200 citers per seed"). Results land as one capped
  digest note; dispositions (import / reject / defer) are logged; a seen/rejected
  ledger stops resurfacing; continuation past 1–2 rounds is a human action framed
  by per-run gain ("this round: 3 new"). Coverage wording is index-relative —
  "declining yield within OpenAlex as of <date>," never "of the literature."
- **Citation monitor (on demand, not resident).** "N new works cite M of your
  catalog papers since last run," run at session start or on explicit trigger —
  **not** a background cron, honoring the no-daemon architecture (§1). One capped
  machine note per run, written through the VaultWriter exfil linter — a poisoned
  citing-paper title is a permissionless ingress channel (§12) and must land
  neutralized.

## 11. Surfaces & workspace — *(the workspace/gate model is an OPEN question, §16)*

alpha.16 ships **the minimum surface the READ cycle needs**, with the keep-test
floor holding under all of it. The full multi-gate workspace design is **not
settled by the evidence and is not invented here** — the carried
`spaces/`+`_nav`+three-places+Maintenance-cadence model rests on authority with
no surfaced failure, beta.1's five-surface model is equally not adoptable on
authority, and the owner's notes richly *re-open* the question (gates for
system/library/ZK; deep-vs-task-oriented work; left-pane nav / right-pane Co-PI).
So the workspace is redesigned from job-to-be-done requirements when that work is
done; alpha.16 states only the provisional minimum (alpha.15 REJECT-1).

**Provisional minimum (all flagged open, §16):**

- **One UI surface — an Obsidian plugin pane (provisional host).** It carries
  only the two interactions files cannot render: **Ask** (streaming retrieval,
  stance chips, span links, the gap list + denominator) and **Review** (the
  honesty-card worklist, the classified diff, one-click `mem revert-run`).
- **Everything else rides files, commands, and native primitives.** Capture is a
  staging drop folder + `mem capture`/`import`; Compose is plain markdown —
  `project.md` (human question + outline), `evidence.md` (machine, idempotent
  per pinned item), `gaps.md` (machine, regenerated wholesale, opens with the
  denominator), plus a **minimal `draft.md` + `mem export`** — the researcher
  gathers pinned claims and evidence into sections (a *structured assemble*,
  **not** the beta.1 writing module) and export passes an **export-readiness
  gate** that refuses on an unresolved citation, a missing bibliography entry,
  or a cited superseded/retracted work (the gate **blocks, never edits**),
  closing the alpha.16 loop to a real, acceptance-testable output. Reading is
  stock markdown or the Zotero reader (§8).
- **Machine recommendations obey the guardrail everywhere** (§5): argument-for /
  argument-against / what-tipped-it / coarse certainty (≤3 levels; the owner's
  ">3 levels collapses to 2–3"); propose-not-dispose; no verdict line on a
  proposal.
- **Keep-test floor:** every surface has a no-plugin equivalent except Ask,
  which by design has no survival role. The files are navigable in `vim` / a file
  browser; the plugin is a convenience over them, never a gate in front of them.

**Rendering discipline (borrowed with condition).** A hub / argument-graph
renders as **tag-membership + prose by default** (Hearst: faceted beats spatial);
a JSON-Canvas / spatial map is an **analyst-only, small-graph option that must
beat a tabular baseline**, not inherit "adequate at scale" (research: Hearst
head-to-head studies — alpha.15 ADAPT-3). An in-file provenance marker is
adopted **only if a UI need is proven**; otherwise the DB stays the sole
authority and the display reads from it (beta.1 BORROW-WITH-CONDITION-16 — the
marker re-opens the forged-status/hash-churn surface alpha.15 closed).

## 12. Security

Threat model: (T1) injection-steered output exfiltrating vault content via
markdown that fetches a URL; (T2) untrusted PDF/metadata text becoming active
content; (T3) key leakage; (T4) engines contacting non-allowlisted hosts; (T5)
third-party content entering the vault with no researcher action (the citation
monitor's poisoned-title channel, §10). **Injection classifiers are excluded —
we contain outputs, we do not detect intent**: injection classifiers hit >90%
adaptive bypass, so containment is the necessary control (research: Greshake,
Debenedetti; beta.1 ADOPT-6).

- **Read-API is the sole read path; writes are enqueue-only; every agent is
  read-scoped** (alpha.15 BORROW-9). Least-privilege is *necessary blast-radius
  control, not sufficient* — the dominant threat is untrusted *data*, not code,
  and the legitimate read+reason path *is* the attack for a poisoned paper
  (research: Greshake — the wording is "necessary, not foolproof").
- **Untrusted-input sealing.** Raw source text never interpolates into an
  instruction template; it enters only inside nonce-delimited data blocks, and a
  manifest violating this fails to load (alpha.15 BORROW-11). Vault text is
  stripped of frontmatter, markers, taint tags, and CSL fences before any prompt;
  local-only exclusion is fail-closed (unparseable frontmatter ⇒ treated as
  local-only).
- **Brokered egress, not a proxy.** The credential-free worker gets no sockets;
  all egress is a typed IPC call to Guard, which performs the TLS request,
  attaches the key at the socket, streams usage to the breaker, and logs every
  request. Compiled-in allowlist, each entry requirement-driven: the pinned LLM
  host, `api.crossref.org`, `api.openalex.org` (identifier-only), `api.github.com`,
  pinned `citation.doi.org`, Zotero localhost. No OA-PDF fetcher (acquisition
  goes through a Zotero handoff). Guard fails closed when its policy cannot load;
  a deny-rate spike right after a PDF ingest is surfaced as an injection signal.
- **The exfiltration linter is VaultWriter-owned, default-deny, strip-never-flag**
  (beta.1 ADOPT-6), applied to every machine-writable class at apply and re-run
  at export: the only surviving link shape is a bare `https://doi.org/10.…`;
  every other URL → inline code; embeds neutralized; raw HTML stripped;
  untrusted-origin text (extracted fulltext, imported metadata) neutralized at
  vault-write, so a beacon in a PDF body cannot survive into a rendered view.
- **Keys** in the OS keychain per platform (Keychain / DPAPI / Secret Service,
  mode-0600 fallback + logged warning — beta.1 ADOPT-11), never in
  vault/exports/logs; a weekly canary drill proves the redaction path fires and
  scans all artifacts for the canary and key prefixes.

## 13. Cost

**BYOK, one normative price table, a circuit breaker** (beta.1 ADOPT-5 — no
alpha precedent, no research against, directly serving the "usable by a real
researcher" bar: spend visible and bounded).

- The **normative price table lives in one place — Ledger — and every component
  references it, never restates constants** (three components had diverged).
  Each operation is bounded by a `max_tokens` ceiling so its worst-case price is
  arithmetic, not hope; token previews are honestly labeled bounded estimates
  (the provider tokenizer is not public).
- **Meter/cap enforced three times:** at preview (refuse enqueue), pre-call
  (recheck), and the circuit breaker (Ledger decides on streamed/polled usage,
  JobRunner executes cancellation as one canonical `cancelled(budget)` state).
  Default soft budget $20/mo, researcher-set hard cap; confirmation only when a
  single action exceeds $1 or the projected month crosses the cap.
- Batch API on ingestion (50%, applied in previews; cache savings budgeted as
  zero); one pinned model tier for the whole increment. Every run records the
  exact model id **and prompt-template version**, and **any model or prompt pin
  change re-triggers validation** of the affected engine against its fixture — a
  prompt edit is an unevaluated model swap (research: Perez/Lu/Zhao — prompt
  variance dominates and does not transfer across sizes).
- Unattributable spend is never zero: provider usage that cannot be tied to a run
  becomes an explicit miss row, and the breaker treats missing usage as
  over-budget evidence.

## 15. Acceptance tests

*Authoritative acceptance lives in the [requirements doc](./0.1.0-alpha.16-requirements.md)
(functional + technical-substrate checks); this list mirrors it for design
self-containment and is checked against it.*

alpha.16 is done when a real researcher, on one real project, can:

1. Create and open a **Memoria-born vault**; pre-existing notes outside
   `Memoria/` are untouched.
2. Seed from a **Zotero collection** (works + `bibliography.bib`); DOI/arXiv/PDF/
   GitHub inputs **land as works without data loss** (land-then-resolve).
3. Extract at least one PDF to anchored `fulltext.md`; an opt-in **digest**
   produces anchor-checked claims; the work links to a **hub**.
4. Turn **≥5 Zotero annotations** into human-authored claim/question notes via
   the processing card.
5. See notes carrying **claim/question mode + temporal-context fields**, with
   **typed argument edges** connecting claims.
6. Run **gap analysis over BOTH graphs** — source gaps, argument gaps (claims
   lacking a support or a counterpoint), and cross-graph mismatches — every
   count with a **printed denominator**, no saturation score.
7. **Ask** and get span-cited answers with honest denominators and abstention;
   `mc` renders only on three-way agreement (marker + current hash + `checks`
   row).
8. Produce a structured **`draft.md` + `mem export`** that refuses unresolved
   citations / missing bibliography / cited superseded works.
9. **Revert a machine run** without changing human text; **rebuild the DB** from
   files + provider payloads.
10. Pass the **HANS/PAWS probe set** before any `mc`; confirm the **keep-test** —
    the vault is navigable as markdown + `.bib` with no plugin.
