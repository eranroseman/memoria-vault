# Memoria Information Flow — Synthesized Design (base: B-reading-first; grafts from A and C as noted; revised against the adversarial review)

## 0. Ruling ideas

The reading-and-notes loop is the spine: every upstream stage (import, sync, enrich, digest) is judged by whether it gets the researcher into a productive annotation session faster, and every downstream stage (mapping, gaps, discovery, projects) is computed only from what the researcher actually wrote plus typed links — the LLM scaffolds and asks, but never authors an idea note, because elaboration in the researcher's own words is the active ingredient. The pipeline is data, not a daemon: each work carries its own stage record in machine-owned frontmatter, so resumption after crash, offline period, or budget stop is a frontmatter rescan, and every honesty denominator is a plain vault query — **recomputed and reconciled on schedule, not "true by construction"**: machine frontmatter is user-editable vault text, so a periodic reconciliation pass (piggybacked on the keep-test mechanical check) recomputes stage state from the artifacts themselves and flags divergence into the queue surface *(graft from C, hardened per review)*. All automation is front-loaded into cheap deterministic stages upstream of one flow valve — the pass-1 stop/continue decision — so money and attention concentrate only on works that earn them, with paid stages halting visibly at a budget counter rather than degrading silently *(graft from C)*. Every machine write passes through a single writer module that enforces the append-only machine-write law, records a region-scoped inverse patch, and sanitizes third-party text against renderer-side exfiltration, so review and revert are structural, not conventional *(graft from C, hardened)*. Absence is always rendered as absence: three-valued fields **extended to digest-derived fields with dual denominators**, denominator-stamped gap claims, provenance-carrying suggestions, and a machine-only calibration replay whose result renders as provenance, never as a bare accuracy figure *(graft from A, rescoped)*.

**Standing laws every stage obeys.**

- **Machine-write law**: a machine may (a) create a new file, (b) append inside a fenced machine block, (c) patch machine-owned frontmatter. Never prose outside those regions. Enforcement is mechanical: **one writer module** performs all machine writes, journaling `{when, stage, file, provider-calls, vault_text: true/false, cost}` to `_memoria/journal.md`. Three hardening rules, each closing a demonstrated hole:
  - **Inverse-patch revert**: the journal records a per-write inverse patch scoped to the machine region written; revert applies that patch only. Git snapshots are kept as forensic evidence, **never as a restore mechanism** — a vault-granular restore would overwrite human prose written after the snapshot, the one outcome the constitution forbids.
  - **Output sanitization**: every machine-written region containing third-party or model text is neutralized before it touches the vault — `![` and raw HTML escaped, external URLs rendered as non-clickable code spans, only vault-internal wikilinks left live. Obsidian's reading view is an unguarded egress surface (a poisoned title containing an image embed beacons on open); one sanitization function at the one choke point closes it.
  - **Human-decision namespace**: `locked` bits, `same-as:`/`not-same-as:` constraints, and recorded reading decisions live under `memoria.human.*`, which the writer module **refuses to modify** except through an explicitly human-approved diff. Human judgments encoded as YAML get the same structural protection as human prose; "machines never overwrite human text" is not silently narrowed to prose. Still plain YAML — keep-test unaffected.
- **Pipeline-state law** *(graft from C, with judgment 2's rendering rule)*: every work carries `memoria.stages` (`{status, at}` per stage) in machine frontmatter. Automatic stages (import, enrich, entity-resolve) use `pending → done → failed` and a stage runner rescans for `pending`; human-gated stages record *achieved* states — plus, new in this revision, **explicitly-marked inferred states** (see §6 pass-1 valve), always labeled `inferred:` and never counted as human decisions. Stage state renders in aggregate views and queue notes only, never as a per-page wall of pendings. Denominator honesty is maintained by the scheduled reconciliation pass, not asserted by construction.
- **Identity law**: filename = Better BibTeX citekey; identity = frontmatter `zotero-key` + `doi`/`arxiv`/`pmid`. Provider IDs are revalidatable cache keys, never primary keys.
- **Provenance law**: every machine-written field carries `{source, source_id, retrieved_at, license}`. **Correction routing, revised**: for citation-facing fields Zotero holds, the correction affordance deep-links to the Zotero item ("fix at source; next sync confirms") — a vault-side lock on those fields would fork bibliographic truth away from `references.bib`, the keep-test artifact. Vault `locked` is reserved for fields Zotero does not store; the keep-test audit flags any locked citation-facing field as divergence-at-risk.
- **Egress law, rewritten as a checkable table**. The previous one-sentence law ("only the digest synthesis call egresses vault-derived text") was false by the design's own mechanisms. The law is now this enumeration, mechanically backed by the journal's `vault_text` tag on every provider call, so it is auditable rather than asserted:

  | Channel | Payload | Gate | Injection screen |
  |---|---|---|---|
  | Digest synthesis (paid API) | full text of the one work being digested | explicit human trigger | sanitize pass, fenced data, instruction-likeness flag on metadata |
  | Crossref `query.bibliographic` | title/author strings extracted from an untrusted PDF | import ladder step 3 only | length cap, strip URLs/markup, journaled verbatim |
  | OA PDF fetch | none outbound beyond the URL, but the URL itself is a beacon | **named-host allowlist only** (§1) | n/a — off-list hosts never fetched |
  | OpenAlex snowball/monitor calls | work identifiers only (`vault_text: false`) | human-run or scheduled, capped | none needed — no free text |
  | Deferred channels (mapping narration, keyword query drafting) | vault text | **must re-enter this table with a gate and screen before revival** | specified at revival |

  The digest lint runs **local-only** (no paid cross-work call), restoring the claim that the digest synthesis call is the only channel carrying document text. One-provider-at-a-time discipline governs expansion; the table governs accounting.
- **Budget law** *(graft from C, corrected)*: the honest worst case is the ceiling, $20/mo, because the counter is the only real enforcement — the forecast is a forecast and is labeled as one. Every paid call site appears in the budget table (§ scope cut) with a per-call cap **and a per-month count cap**; paid calls compute an arithmetic worst case before the call and debit `_memoria/budget.md`; at any cap the stage flips to `pending (budget)` and pauses visibly.
- **Attention law** *(new, symmetric to the budget law)*: attention gets the same arithmetic dollars get. Every queue surface carries an expected-items-per-week estimate and a cap (§ scope cut); the dominant feeder — author entity resolution — is defused by making the DBLP unresolved holding state the **default**: a candidate pair reaches human review only when a downstream consumer (hub, mapping run, discovery seed) actually touches that entity.

**Local compute plan** *(new, required before scope freeze)*. Every "local, free" stage names its model and hardware floor, and each is spot-checked for latency and quality on the pilot machine before the scope freezes: section pass + relevance gate and NLI check (qwen2.5:7b, 16 GB-GPU floor, target < 90 s/paper — a test-fixture-class model; the PaperQA2 numbers cited in evidence came from frontier models and are *not* assumed to transfer), embeddings (local sentence model, CPU-acceptable). **Failure branch, chosen now**: if the local relevance gate fails its validation spot-check, the section pass **collapses into the single paid synthesis call** — larger context, one egress point, per-work cap raised to $0.30 and the budget table restated for that branch — rather than becoming a second egress surface. Either branch keeps the egress table true; neither is decided silently.

**Vault anatomy** (plain markdown, keep-test-first):

```
catalog/<citekey>.md        one page per work (machine header + human body)
notes/ideas/<claim>.md      typed idea notes, human-authored
notes/fleeting/…            dated fleeting captures
hubs/<topic>.md             editorial hub pages (human body + machine candidates)
projects/<name>.md          outline notes with machine coverage footer
_memoria/journal.md         append-only machine action ledger (incl. PRISMA-S search log)
_memoria/budget.md          monthly paid-call counter + per-site count caps
_memoria/queues/…           generated queues — ONE shared actionable-line format (§0a)
_memoria/deferred.md        deferral ledger with written revive-triggers (graft from A)
references.bib              BBT auto-export — the keep-test bibliography artifact
```

## 0a. The Memoria plugin — one interaction pattern (first-class component)

The interactive layer is the largest engineering line item in this design and is priced as such, not treated as free adjectives. Everything previously described as "retry buttons," "checkbox-actionable lines," "one-tap promote/link/drop/snooze," "accept/keep buttons," and "click-throughs" is **one component**: a generic actionable-queue note format — a single line schema (`- [action-verbs] payload · provenance · date`) — plus one command set, "act on the line under cursor," rendered N ways. Import retries, dedupe decisions, sync renames/orphans, enrichment conflicts, annotation triage, fleeting dispositions, discovery dispositions, and gap-stub promotion are all instances of this one pattern; a new queue costs a template, not a surface. The two shipped acceptance tests (paste-to-landed < 2 s; highlight-to-typed-stub < 60 s) are **tests of this component** and are bound to it. Anything the pattern cannot express (the per-field left/right merge diff, the resumable reading position) is named individually in the scope cut with its own cost line.

## 1. Import

**Mechanism.** Two-phase ladder: *land, then resolve*. Phase 1 always succeeds — any input (file, URL, ID string, BibTeX entry) lands immediately as a catalog stub with `memoria.stages.import: {status: unresolved}` and the raw artifact attached; failure becomes a queue item, never data loss. Phase 2 is identifier-first resolution:

1. **Explicit IDs** (DOI/arXiv/PMID, batch-pasteable): DOI → doi.org content negotiation; arXiv → arXiv export API; PMID → NCBI ID converter.
2. **PDF**: Crossref DOI regex + arXiv pattern over pages 1–2 (~97% of live DOIs); fetch registry metadata per candidate; **accept only if the registry title fuzzy-matches the PDF text** (Ratcliff/Obershelp ≥0.95). Untrusted PDF text *nominates* identifiers; a trusted API *authors* metadata — injection-resistant with zero LLM calls.
3. **No identifier**: Crossref `query.bibliographic` with SBMV-style validation scoring; below threshold → `unresolved`, never guess. The outbound query string is extracted from an untrusted PDF, so it is length-capped, stripped of URLs/markup, and journaled verbatim (egress table entry). No local GROBID.
4. **BibTeX file**: each entry lands as a stub keyed on its identifiers, then runs the same ladder.
5. **URL**: structured origins only — DOI landing pages, arXiv abs pages, GitHub repos. No per-site scrapers, ever.
6. **GitHub repo**: `CITATION.cff` → `codemeta.json` → Zenodo DOI badge → GitHub API fields, recording which chain level supplied each field. Pin HEAD commit SHA + retrieval date; fire a Software Heritage Save Code Now POST as a logged, non-blocking side effect. Paper and repo are **two works joined by a typed edge**, never merged.

**Dedupe at land**: tier 1, normalized-identifier exact match — auto-match; identifier *conflict* vetoes merge. Tier 2, title + year ±1 + one surname — advisory, review-queued, never auto-merged. **Preprint↔published = one work, two identifier fields.**

**PDF acquisition, revised**: Zotero's captured attachment first. Otherwise the OpenAlex `best_oa_location` chain is followed **only when the resolved host is on a small named allowlist — arxiv.org, PMC/europepmc.org, and doi.org-resolved allowlisted hosts** — recording *which version* was stored. Any other `best_oa_location` resolves to `pdf: missing` plus a one-click "capture in Zotero" handoff (Zotero is already the designated capture surface; nothing new is built). Rationale, recorded: an unbounded auto-fetch at import time is both an allowlist violation and a zero-interaction beacon — anyone who can get a work into the import path (shared BibTeX, discovery acceptance, hijacked OA record) would get an HTTP request from the researcher's machine to a host they control, plus delivery of an adversarial PDF into the digest pipeline. No watch folders that silently mint entries. Retraction flags from a periodically pulled local Retraction Watch dataset — zero disclosure, banner only, never delete.

**Data shapes.** Stub page with `memoria` frontmatter; raw input preserved; dedupe candidates as actionable-queue lines.

**Human vs machine.** Machine: resolution, tier-1 dedupe, allowlisted OA fetch, SWH save. Human: tier-2 merges, unresolved stubs, manual metadata, off-allowlist PDF capture via Zotero.

**UX affordance.** One "Import" command accepting paste/drag of anything; per-work status line; unresolved queue in the shared actionable format. **Shipped acceptance test**: paste-to-landed under 2 seconds, zero required fields.

**Cost & egress.** Regex + registries; $0 LLM. Hosts: doi.org, api.crossref.org, api.openalex.org, api.github.com, archive.softwareheritage.org, NCBI converter, **plus the named OA-PDF hosts above** — the enumeration now matches what the ladder can actually touch. Shared politeness layer: ~1 r/s token bucket, honor rate-limit headers.

## 2. Sync (Zotero, one-way)

**Mechanism.** Layered read-only. **Freshness source**: Zotero 7+ local HTTP API — GET-only, no auth, offline-capable. Change detection is **full snapshot + local diff** on item key + `dateModified` (`?since=` is broken for never-synced local items, zotero/zotero#5011). **Durable artifact**: a Better BibTeX auto-export of a *designated collection* — the keep-test bibliography, never trusted for freshness: sync cross-checks item count and newest date against the local API and reports "export is N items behind." **Drift guard, new**: the designated-collection gesture fails silently when forgotten, so every sync report carries a library-vs-collection drift line — "your Zotero library has 12 items not in the Memoria collection (5 added this week)" — with a one-tap add-all/add-selected action. The catalog can no longer silently diverge from the researcher's actual corpus while every denominator stays mechanically "honest."

Diff semantics: new item → import ladder; changed metadata → patch machine frontmatter only; same item key + new citekey → **rename event**, reviewed as a diff; same citekey + different key → collision, queued; vanished item → `orphaned: true`, never deleted. Ingestion gates: refuse a snapshot whose count collapsed without confirmation; export path local, non-cloud-synced. Citekeys pinned at import. Zotero-stored PDFs ingested via local API attachment access. Annotations flow Zotero→vault append-only (§6); a designated Zotero tag/color additionally carries pass-1 decisions (§6). Nothing flows back. No `zotero.sqlite` reads; no Zotero Web API.

**Data shapes.** Per-sync manifest in tool state; sync-review queue in the shared actionable format; drift line in the sync report.

**Human vs machine.** Machine: snapshot, diff, stub creation, frontmatter patching, annotation append, drift arithmetic. Human: rename approvals, orphan dispositions, drift add-backs, collection membership.

**UX affordance.** "Sync now" + optional interval poll; explicit degradation line, never an opaque error.

**Cost & egress.** Localhost only. $0.

## 3. Enrich

**Mechanism.** Backbone + fill-in: **OpenAlex** singleton per DOI ($0) returns metadata, `referenced_works` (outgoing edges), OA locations, abstract, and a `cited_by_count` — **not the incoming citation list**. **Cited-by edges, mechanism stated honestly**: forward edges require one paginated `filter=cites:W…` query per work (max 200/page). They are ingested by an **idle background backfill job** with journal progress, structurally capped: **top-N citers per work ranked by citation count** (N a logged shadow knob, ~200), a hard per-run candidate ceiling, and **no forward-of-forward expansion in beta.1**. Every downstream surface that consumes these edges (Top Citing, snowball, monitor) stamps its output with the cap — "among top 200 citers per seed" — so the honesty contract survives the truncation. **Crossref** fills publisher-deposited facts (licenses, funders), never cited-by or abstracts. **Semantic Scholar is deferred** — the first named post-beta expansion, revive-trigger: "discovery recall or reading triage measurably suffers without TLDRs/recommendations." scite: link-out only, forever.

**Merge rule (JabRef, verbatim)**: machine fills empty fields only; conflicts become per-field left/right diffs; plausibility heuristics pre-select, never auto-apply. No background auto-refresh, ever. **Correction routing per the provenance law**: citation-facing fields Zotero holds → deep-link to fix in Zotero; vault-side `locked` only for fields Zotero doesn't store. **Freshness follows attention**: eager-once at import (batched ~50 DOIs per list call), lazy refresh on mapping entry or stale page open (shadow knob).

**Field semantics**: every enrichable field is **three-valued** — `present | absent-in-source | not-yet-fetched`. Abstract fallback chain with recorded winner. Counts never bare: `41 citations (OpenAlex, as of 2026-07-02)`. **Adversarial boundary**: abstracts/titles arrive partly from author-controlled text; they cross into Digest as fenced data only, with a cheap **instruction-likeness flag** on suspicious metadata fields, and they cross into the *vault* only through the writer module's markdown sanitization. License semantics table as before.

**Data shapes.** Machine frontmatter block: fields + per-field provenance + three-valued markers + completeness line; edge cache in `_memoria/` (regenerable).

**Human vs machine.** Machine: fetch, batch, fill-empty, backfill, flag conflicts and retractions. Human: conflict adjudication, at-source corrections.

**UX affordance.** Invisible when clean; conflicts and retraction flags as work-page callouts (the one bespoke non-queue UI besides the reading position: the per-field merge diff).

**Cost & egress.** Singletons and filter queries $0 (rate-limited, journaled); no vault text outbound.

## 4. Entities

**Mechanism, cut to the deterministic core for beta.1.** No ML, and now also **no standing review queue**: the DBLP-style `status: unresolved` holding state is the **default disposition** for every author mention that doesn't auto-resolve. Rules in order: (1) **name-compatibility veto** — parsed given/family incompatibility blocks any merge regardless of other signals, including the ORCID path; (2) **identifier-conflict veto** — two ORCIDs → hard cannot-link; (3) **identifier match = strong evidence, not gold** — shared ORCID + name-compatible → auto-merge with provenance; (4) everything else → `unresolved`, counted honestly by gap analysis, **surfaced for human review only when a downstream consumer (hub, mapping run, discovery seed) actually touches that entity** — lazy adjudication at the moment of use, not a clerical inbox. The arithmetic that forced this: ~4–5 authors/paper × 10 imports/week ≈ 45 mentions/week, and a 500-item backfill mints ~2,000 on day one; a standing queue is the week-3 abandonment mechanism wearing a rigor costume. Human decisions persist as `same-as:`/`not-same-as:` under `memoria.human.*` — writer-protected, so no sweep or refresh can lawfully rewrite them. **Merges keep the loser as a redirect stub.** Venues: ISSN-L auto-merge only. Deferred to the ledger: the full OpenRefine-style evidence-column review note and the housekeeping duplicates sweep (trigger: lazy adjudication measurably blocks mapping or discovery).

**Data shapes.** Entity pages keyed by local slug; external IDs as dated claims; constraints in the human namespace; redirect stubs.

**Human vs machine.** Machine: vetoes, auto-merge zone, unresolved bookkeeping. Human: lazy adjudication at point of use.

**Cost & egress.** Zero marginal.

## 5. Digest (the wiki-LLM phase)

**Mechanism.** The digest is a **reading map, not a summary-as-product**. **Trigger rule**: digest runs only on the researcher's explicit "Digest" action — never on import, never batch. No automatic `pending`; `digested` when done, otherwise absent.

Pipeline per work: (1) **Sanitize — best-effort, and honestly labeled as such**: span-attribute heuristics (sub-threshold fonts, off-MediaBox text) strip what they can; white-on-white and layered attacks defeat attribute checks, so **the security load does not rest on the sanitizer** — the defenses of record are the fenced-data boundary, the no-tool-authority rule for document-consuming calls, and the writer module's output sanitization. (2) **Section pass (local, per the compute plan)** — split; per-section contextual summary against a fixed question schema + relevance score; threshold gate (RCS-style). Runs on the named local model at the stated hardware floor, validated by spot-check before scope freeze; on validation failure the **declared failure branch** collapses this pass into the paid call (§0). (3) **Synthesis (one capped paid call)** — fixed question set: *problem, contribution, method, limitations* — **connects-to-here is struck from this call**: the egress table forbids sending vault context, and without it the model would fabricate relevance to a vault it cannot see; connection candidates come exclusively from the local linking stage and the human interview hook, labeled as suggestions, never as digest answers. **Quote-first**: verbatim span + offset, then the claim; no quote → `not reported`, first-class. Worst case computed pre-call, journaled, capped ≤ $0.15/work ($0.30 on the failure branch). **Over-cap policy, defined**: a work whose gated sections exceed the cap is digested **fit-to-cap by relevance rank**, with a recorded coverage denominator — `digested 14/22 sections` — never refused, truncated silently, or downgraded silently. (4) **Verification (local, free)** — NLI entailment of each claim against its cited span, rendered as **three explicit per-claim states**: `unchecked | span-matched | ⚠ contradicted`, where the span-matched microcopy says *match, not verified* — silence never reads as endorsement, because the design's own evidence says a meaningful fraction of unflagged claims are wrong. (5) **Linking (local, free)** — candidates constrained to the existing canonical hub/entity list; new mentions embedding-snap to nearest existing; new-tag creation capped per work; new hubs always human-approved. **Pre-snap raw novelty counts are recorded before the linker constrains them** — they feed the saturation diagnostic (§9), which would otherwise measure the linker's cap, not the corpus. Citations/authors/DOIs only from enrichment data, never model memory. (6) **Interview hook** — 2–3 machine why/how questions + a "connects to / conflicts with [[…]]" prompt seeding §6.

**Charting frontmatter is vocabulary-gated**: field values come from a **closed per-project vocabulary**; a digest proposing a new value queues it for one-tap human approval before it can appear in any matrix or downstream prompt. Structure produced by the exposed model is the same injection surface wearing a schema; the closed vocabulary is what actually shrinks it.

Page shape: machine frontmatter → machine digest block (Q→grounded-A with span links; skim anchors) → Prior/Derivative context from cached edges → **human section, initially empty**. Repo digests over README+docs+manifests only, stating "purpose not declared by authors" rather than paraphrasing it into existence. **The "machine draft — unreviewed" banner clears only via an explicit "mark reviewed" action** — writing takeaways in the human section is elaboration, not review, and must not strip the warning label. The periodic **lint** run is **local-only** (no paid cross-work call — the egress table depends on it), writing a capped suggestions queue, never auto-applied to human text.

**Cost & egress.** One paid call per digest, debited; at the cap, `pending (budget)`. This is the only channel carrying document text off the machine, and the journal's `vault_text` tags make that checkable.

## 6. Annotate (the first gate — this is the product)

**Mechanism.** Keshav's three passes fused with the barbell method. **Pass 1** (5–10 min: digest map + abstract + headings) ends in a recorded stop / continue / cite-only decision — **captured where reading already happens, not as a new ritual**: a designated Zotero tag/color maps to stop/cite-only and syncs like any annotation, and the machine may record an **explicitly-marked inferred state** (`cite-only — inferred: no highlights 30 days after enrich`), always labeled, never counted as a human decision. Recorded tension: the rendering rule says human-gated states are optional and absence is neutral — which left the "main valve" either mandatory friction or decorative; inferred states populate the valve's denominators without a per-work ritual, at the price of an inference the label makes visible. **Pass 2** is highlight-only reading in Zotero's reader. **Pass 3** is the **processing session**, a distinct sitting revisiting only marked passages.

**The starter motor, new — the collapse point is initiation, not execution**: when a sync appends ≥N new highlights to a work, the next vault open shows a single dismissible card — "Process 14 new highlights on [work] (~7 min)" — opening the triage view. One event, one card, no calendar, no new queue. Without this, the resurfacing loop (project footers ← mapping ← idea notes ← processing) is downstream of the very notes it is supposed to cause, and during the measured retention weeks highlights would pile up invisibly while zero idea notes got produced.

Annotations sync append-only, keyed by Zotero annotation ID with a dateAdded watermark. Each carries: verbatim quote, color, **page + `zotero://open-pdf` deep link** (keep-test pass), and — **rescoped** — an exact-quote anchor only: orphan = "exact quote not found verbatim," with a per-work orphan count, never silent drops. TextPosition/fuzzy re-anchoring is Hypothesis-grade engineering and goes to the deferral ledger (trigger: measured orphan rate exceeds X%). **Color→type is an optional pre-fill, not a per-highlight obligation**: the design's own citation (color hierarchies decay even for inventors) predicts regression to all-yellow, so type assignment is a **one-keystroke action in the processing triage view** — where a decision UI already exists and cognitive load is low; an all-yellow vault loses only the pre-fill, never the taxonomy.

Processing session: annotations as groupable scraps; machine-proposed clusters (reviewable); one-tap **promote / link / drop / snooze**. Promote opens a pre-typed idea-note stub — but **machine pre-fill stays machine-attributed** (`suggested-*` property keys and a fenced suggestion block) **until the human edits or individually confirms each element**; the title lint flags any promoted note whose body contains zero human-typed characters as `unauthored`. Rationale, recorded: one-tap promotion of machine-drafted sentences derived from adversarial document text would launder injectable prose into the human-owned region, where downstream analysis treats it as "what the researcher wrote" and the machine-write law forbids automated correction. The digest's interview questions appear here; typed human answers land as own-words material. Stance-nudging prompts pre-type tension/claim *candidates*, same attribution rule.

**Honest denominators, no guilt inbox**: per-work `processed 9/24`; global backlog surfaces only contextually. A **resumable reading position** per work survives across sessions.

**UX affordance.** "Process highlights" triage view on the shared queue pattern. **Shipped acceptance test**: highlight-to-typed-stub under 60 seconds.

**Cost & egress.** $0 paid; localhost sync; local model only.

## 7. Fleeting

**Mechanism.** One capture command → `notes/fleeting/2026-07-02-hhmm.md` with optional context link. **TTL with default-to-discard bias — clocked on vault-active days, not calendar days**: the inbox dims past-TTL notes (shadow knob, ~1–2 weeks of *days the vault was opened*), never auto-deletes, never auto-promotes, never machine-elaborates. A three-week conference absence must not greet the returner with a graveyard: absence is the normal case, and decay clocks that punish it are re-entry killers. One-tap **promote-to-idea / attach-to-work / drop**. Fleeting notes **participate in the link graph immediately**; resurfacing via local embedding + keyword match against new works' digest fields, surfaced as provenance-carrying suggestion lines. The TTL governs the inbox view; the graph governs resurfacing.

**Re-entry, new**: on first open after >N idle days, one machine-owned "since you were away" note: three counts, the accumulated monitor summary collapsed to one line, and a single suggested action — the last resumable reading position. Every other surface says "you fell behind"; this one says "start here."

**Cost & egress.** Zero.

## 8. Idea notes

**Mechanism.** The atomic unit. **Human-authored prose, machine-checked mechanics.** Five types on Chan's discourse-graph typology — `concept`, `claim`, `question`, `hypothesis`, `tension`. **Title-shape lint at save**: shape check, never truth check; failing titles flagged, never blocked; the lint also carries the `unauthored` flag from §6. **Typed edges as inline properties**: `supports::`, `opposes::`, `informs::`, `derived-from::`/`evidence:: [[work#annotation-id]]` — plain-text, keep-test-safe, queryable, the thing that makes gap queries computable. Notes are **concept-oriented, not source-oriented**. Provenance discipline: every claim/tension links evidence through to a page number. No folgezettel. Machine involvement confined to: attributed stub pre-fill (§6 rule) and a suggestions footer (local embedding neighbors, provenance shown). Hubs are editorial: human-curated body, machine candidates below a structural boundary; **cold-start bootstrap, new**: at first import of a backfilled corpus, the machine proposes hub *stubs* from OpenAlex topic/concept clustering over the imported works — human-approved per the existing hub law — so the analytics have scope objects on day one instead of waiting weeks for squeeze points that need notes that need processing that needs projects. Squeeze-point proposals continue thereafter; creation always a human act; sparse by design.

**Cost & egress.** $0; local and deterministic.

## 9. Corpus mapping & gap analysis

**Mechanism, demoted for beta.1 to queries over frontmatter** — generated tables and Bases views, no bespoke staleness machinery — while keeping the honesty semantics that are the product's differentiator. Separation of duties unchanged: deterministic structure makes gaps visible; the human certifies; the LLM narrates nothing in beta.1 (narration deferred with the egress-table re-entry requirement).

- **Gap matrix** (3ie/Campbell EGM): two facets crossed; each cell = work count + linked-idea-note count. **Dual denominators, the blocker fix**: charting frontmatter exists only for digested works, so every cell and every absence claim renders both populations — "0 of 37 charted works; 105 of 142 cataloged works uncharted for these facets" — and uncharted works appear as an explicit **unknown band** in the matrix, never as implicit zeros. Which works get digested follows attention; without the unknown band, "absolute gaps" would concentrate exactly where the researcher didn't look. **Enrichment-only charting tier, new**: OpenAlex topics/concepts populate an explicitly-labeled `enrichment-only (unverified)` tier so the matrix, hub diagnostics, and Top Cited/Citing run on day one over a backfilled corpus, honestly labeled.
- **Gap typing via Robinson reason-classes** → question/tension/hypothesis stubs in the existing note taxonomy.
- **Per-hub diagnostics**: work count, median year, Price index, influx bursts; **saturation signal computed from pre-snap, pre-cap raw novelty** (§5) and **stamped with the settings that shaped it** — "under snap-threshold T, cap K, 3 hub proposals pending approval" — displayed diagnostic, never auto-stop; a corrupted signal a human obeys is worse than none.
- **Missing-work arithmetic**: Top Cited / Top Citing from the cached, capped edge set, every line stamped with its cap ("among top 200 citers per seed").

**The honesty contract**: every absence claim is denominator-relative and provenance-stamped, dual-denominator per the rule above, three-valued for enrichment fields. Gap notes store their query + as-of date and offer one-click re-run; automated staleness badges are deferred (trigger: manual re-runs demonstrably don't happen). Recorded tension: the scope-cut blocker demotes mapping machinery while the cold-start blocker demands day-one mapping — resolved by making beta.1 mapping pure queries over enrichment + digest frontmatter with the enrichment-only tier, deferring only the *automation* (badges, narration), never the *semantics* (denominators, provenance, unknown bands).

**UX affordance.** "Map" command scoped to a hub, a project, **or — new for cold start — a Zotero collection or tag**; never the whole vault. Findings file into durable notes.

**Cost & egress.** $0, offline, in beta.1.

## 10. Discovery

**Mechanism, cut to snowball + monitor for beta.1.** Discovery answers gaps; it never free-runs. Seeds are explicit and curated (multi-seed, never single-paper, never behavioral tracking).

1. **Snowball** (in): backward from `referenced_works` + forward from the capped cited-by cache; round 1 is a local graph query; only unresolved frontier edges hit OpenAlex, within the per-run candidate ceiling; **forward-of-forward expansion excluded in beta.1** (§3). Ranking = transparent occurrence counting at depth ≤2; every candidate shows its arithmetic *and its cap* ("appears on 6 paths from 4 seeds, among top-200 citers per seed").
2. **Scheduled citation monitor** (in — the defended retention hook): a local job writes "N new works cite 3 of your catalog papers since last run" into the discovery inbox, through the writer module's sanitization (poisoned titles are a permissionless ingress channel; the monitor is an unattended write). Recorded tension: this survives the scope cut against the platform-bloat blocker because beta.1 *is* a retention experiment and the monitor is the one mechanism with measured retention evidence.
3. **Deferred, with triggers and corrected labels**: keyword query drafting + PRESS lint (trigger: snowball recall measurably insufficient; on revival, the drafting call re-enters the egress table and **only human-certified gap stubs — never raw machine facets — may feed it**); the local logistic ranker (trigger: inbox volume exceeds triage capacity); the local-model screening vote — **relabeled "model flag," never "second screener,"** because the ~13% dual-screening statistic assumes error independence a 7B model reading author-controlled abstracts does not have, and the "injection-immune" label is deleted from it (it belongs to the non-LLM logistic ranker only). The **solo-screening caveat itself still prints** in beta.1 — the honesty requirement survives the mechanism's deferral — worded: "single-screener triage; solo screening misses ~13% in comparable settings."

**Delivery**: a **capped digest note** (10–20 items), each item with full provenance; dispositions import / reject / defer, all logged; accepted items enter the normal Import ladder. Persistent **seen/rejected ledger**. **Logging and stopping**: every run appends a PRISMA-S line to the journal; per-run gain curves display; **continuation past 1–2 rounds is a human action**. Coverage statements: "declining yield — likely near-complete *within OpenAlex as of D*," never "of the literature." Google Scholar stays off the allowlist permanently.

**Cost & egress.** OpenAlex only; identifiers only outbound (`vault_text: false`); $0 paid in beta.1.

## 11. Project

**Mechanism.** A project note is an **outline assembling existing idea notes toward an output**. Shape: goal line, deliverable, ordered outline of `[[idea-note]]` links with human transition prose, a `works::` scope list, subtypes mirroring idea types. **Seeding**: mapping runs propose seeds when clusters mature — suggestion, human-accepted; at cold start, bootstrapped hub stubs (§8) and collection-scoped maps (§9) stand in until real clusters exist. Projects are the **scoping object** and the **friction router**. Machine-owned **coverage footer** — "cites 12 catalog works; 2 gap claims open; 3 unprocessed annotations in scope" — computed from `memoria.stages` and `evidence::` edges, all plain queries, rendered through the shared queue pattern; each number a click-through. Archival is a status flip. **Retention instrumentation**: works-annotated, notes-promoted, projects-advanced — never notes-created.

**Cost & egress.** $0; local queries.

## How the loops close

**The spine (per work).** Zotero sync / direct import → land (always succeeds) → resolve → enrich (free, batched, automatic; cited-by backfill idles in the background) → entities (auto-merge or unresolved-by-default) → the work rests until the researcher chooses → digest (explicit, one paid call, fit-to-cap with recorded coverage) → pass 1 records stop/continue (Zotero tag or labeled inference) → pass 2 highlights → sync appends **and raises the processing card** → processing session promotes survivors to attributed stubs the human makes their own. **The pass-1 valve is the flow's regulator**, its denominators populated without ritual and reconciled on schedule.

**The outer loop (corpus → world).** Idea notes accumulate typed edges → mapping (human-triggered, scoped to hub/project/collection) computes matrix + diagnostics + Top-Cited/Citing under dual denominators → gaps become typed stubs and saved queries → snowball runs bounded, logged rounds → accepted candidates re-enter Import through the same gate. Bounded at four points: the pass-1 valve, the discovery stopping rule, the digest gate, the budget counter's hard pause. Honest at four points: three-valued fields extended to digest-derived fields, dual-denominator gap claims with unknown bands, provenance-carrying suggestions, and the calibration replay's provenance-sentence caveat.

**The inner loop (notes → notes).** Import, digest, and mapping events resurface dormant idea notes, fleeting thoughts, and unprocessed annotations contextually; the sync-triggered processing card is the loop's starter motor; the "since you were away" note is its re-entry ramp. Hubs stay editorial; projects pull idea notes into outlines and push attention back via the coverage footer.

**Trigger table.** Zotero diff → import ladder + drift line. Import resolved → enrich → entity auto-resolve (then nothing). Annotation sync → append + processing card (≥N highlights). Human "digest" → digest + interview. Human processing session → idea notes. Human "map" → matrix + gap stubs. Human "run snowball" / scheduled monitor → discovery inbox (sanitized). Human accept → import. The only unattended writes are sync appends, monitor digests, backfill progress, and queue regeneration — all machine-owned files, all through the writer module (sanitized, inverse-patched, journaled).

**Where the machine-write law bites.** Machine-owned: digest blocks, enrichment frontmatter, queue notes, hub candidate sections, mapping/gap notes, coverage footers. Human-owned, machine-suggests-only: idea notes, hub bodies, project outlines, annotation comments, work-page human sections — plus, structurally, **human decisions in `memoria.human.*`**. Boundaries are file/region/namespace, enforced by the single writer module.

**Keep-test audit** — shipped as a doc and the periodic mechanical check that also runs stage-state reconciliation. If Memoria dies tomorrow the researcher holds: `references.bib` (whose truth was never forked by vault-side locks); citekey-named work pages with plain-YAML metadata and `zotero://` deep links; annotations with page citations and exact quotes; typed idea notes; hub and project outlines; gap notes with dual denominators; the PRISMA-S log. The only non-vault state is regenerable cache.
