# alpha.15 → beta.1 — consolidated gap analysis

Every material difference between the **alpha.15** design and the **beta.1** design, merged from all
prior gap analyses into one report. Sources folded in: the curated gap analysis (design doc + ADRs
124–130 + shipped `vault-template`), the *from-the-reports* lens (beta.1 research/information-flow
reports), the *design-history* lens (alpha.1→alpha.15 history), plus the review and the independent
verification pass — whose corrections are applied inline. **Gaps only** — each entry states the alpha.15
state, the beta.1 state, and the delta. No pros/cons, no recommendations.

Legend: **A15** = alpha.15 · **B1** = beta.1 · **⇒ Gap** = the delta.

---


## 1. Host / runtime / distribution

- **A15:** standalone CLI/engine — installs no Obsidian plugin, no Hermes profiles, requires no Zotero; runs with no Obsidian (keep-test bar: the corpus stays plain markdown navigable from a stock editor such as vim/nano). alpha.14 cut the entire `.obsidian/**` tree (PR #1124) and added an `rg` negative gate that fails any runtime/installer path still requiring Obsidian; alpha.15 "is an engine and integrity checkpoint, not an Obsidian plugin release," future editor adapters requiring their own ADR (PR #1203). **B1:** an Obsidian plugin + `mem` engine where "the Obsidian plugin is the only researcher UI" (`design §1`). ⇒ **Gap:** B1 re-introduces Obsidian as the *required* researcher surface — a standing Memoria held Origins→alpha.10, that alpha.12 began demoting and alpha.14 removed outright; the returning plugin still inherits alpha.11's enqueue-only "control panel, not a writer" rule (ADR-121 → ADR-130). Recorded, not silent: `design §11` item (10) labels the host decision "new alpha.16 scope," and `design §16` says it amends ADR-125/ADR-130 and "is a proposal, not product truth" until that ADR lands.
- **A15:** "there is no internal daemon" (ADR-125) — `serve` exists only as an on-demand backend (loopback HTTP + token + file watcher, launched while Obsidian is open, explicitly "not a daemon"); the resident-watcher form was rejected (ADR-127), and scheduled work runs as plain CLI invocations under external cron/systemd/Task Scheduler with `--schedule-id`. **B1:** the same `serve` promoted to a resident, plugin-supervised two-process daemon — `mem serve` = a supervisor (holds key, DB, Guard, Ledger, VaultWriter) + a per-run credential-free/socketless engine worker that owns no DB handle (reaching the DB only through the supervisor's typed DB-RPC channel) — under LSP lifecycle (port-file/per-install-token discovery, spawn-on-`onLayoutReady`, version/capability handshake, parent-death watch, crash-loop give-up), resident while Obsidian is open (`design §1/§2/§16`). ⇒ **Gap:** B1 adds residency-by-default + plugin supervision + the supervisor/worker split — not the existence of a `serve` backend, which A15 already had (correcting "A15 had none"). Rationale: compute (PDF/embeddings/NLI/egress/DB) kept out of Electron for crash isolation + one network choke point. Scheduling also moves into the daemon (JobRunner with heartbeat rows, idempotency keys, bounded retries), which reverses the ADR-95 no-daemon rejection of an always-on discovery loop (charter amended 2026-07-03). The supervisor/worker shape echoes the pre-alpha.11 Hermes dispatcher+worker; the `mem serve`/LSP form itself is new-in-B1.
- **A15:** the verb-first standalone CLI is "the product surface … not operator-only" (alpha.14 §5), carrying ADR-72's PI-direct-access principle; its roster includes `ask` alongside `new note`, `work`, `link`, `check`, `export`, and ADR-130 names the CLI "the no-daemon recovery surface." **B1:** the `mem` CLI is a "survival + ops + daemon entry point" floor — `serve/capture/import/export/rebuild/backup/restore/migrate/revert-run/cost/hat`, **no `mem ask`** (Ask is plugin-only) (`design §1`). ⇒ **Gap:** B1 demotes the CLI from product to survival floor and drops `ask`, reverting toward ADR-122's earlier "CLI is inspect/verify/administer only, not a PI action surface" posture and returning Ask to the plugin (as in the alpha.1–alpha.10 Obsidian ACP pane). Rationale: the keep-test answer is stock-Obsidian navigability, not a CLI, and Ask/Review are "the only surfaces whose interactions cannot be file-rendered" (`design §16`); Success Claim 4 locates survival in files, not `mem`.
- **A15:** installer installs no Obsidian plugin, no Hermes profiles, requires no Zotero (alpha.14 §11; ADR-125 one-command/no-runtime install). **B1:** ships a catalog-clean, desktop-only (`isDesktopOnly:true`) Obsidian plugin whose binary is acquired consented, never bundled — the catalog ships only `main.js`/`manifest.json`/`styles.css`, with two sanctioned paths (a package manager: `uv tool install memoria`/winget/brew, or the rust-analyzer model: explicit-consent, checksummed, version-matched release-page download), Windows builds code-signed, BRAT the beta channel (`design §16`). ⇒ **Gap:** B1 adds a plugin-distribution + binary-acquisition story A15 had removed; the plugin-install concept reverts to the alpha.1–alpha.10 installer, while code-signing has no alpha-line precedent (new-in-B1). Re-homed packaging discipline (`docs-history` Borrow "Doc & release epistemics") constrains it but does not prescribe the consented-download/code-signing mechanics.
- **A15:** distributed as a **Python 3.12+ package** — `uv tool install memoria` (with `pip install -e .` the contributor fallback), per-vault `.memoria/.venv`, no bundled local models, retrieval via the qmd subprocess (the last Node ≥22 dependency, earmarked for removal) (ADR-125). **B1:** distributed as a **single self-contained static binary** acquired via package manager or checksummed version-matched download, with installer-bundled ONNX embedding/NLI models (pdfium chosen to match) and FTS5+sqlite-vec removing Node from the baseline (`design §7/§16`). ⇒ **Gap:** the distribution artifact changes class from a Python package to a static binary — no alpha-line precedent (the ADR-26→ADR-76 wheel→ADR-125 line was Python-shaped throughout); only checksum-pinning continues (echoing ADR-76's SHA256-pinned wheel / alpha.15's content-hashed baseline manifest). `uv tool install memoria` is shared by both sides, so the install *command* is a convergence point; from the reports' POV this artifact change rides the §1 host decision (`design §11` new-alpha.16 scope) rather than being independently reasoned.
- **A15:** the design-index Scope Boundary explicitly reserves the host decision for a later version — "alpha.15 cannot close with Obsidian plugin implementation counted as release scope," and PR #1203 requires future editor adapters to have "their own ADR"; the daemon half was fenced identically (ADR-127). **B1:** `design §11` item (10) records "the host decision (Obsidian plugin + local daemon)" as **new alpha.16 scope** that alpha.15 "anticipates but does not yet center." ⇒ **Gap:** the host/runtime pivot is a beta.1 addition filling the alpha.16-shaped slot alpha.15's negative gate reserved. Framing tension: `design §11` calls it "back-pressure, not redesign" and `design §16` "a proposal, not product truth," whereas the gap analysis scores it as a re-introduction/addition; the regression review, which treats any §11–§17-recorded delta as deliberate, buckets it as a recorded (not silent) divergence.
- **A15:** `serve --watch` is an inbound stdlib FS-polling watcher over the `workspace scan`/`observe-pi-edits` path, and `--json` is a free schema behind the transport (no plugin/daemon/CLI version coupling). **B1:** the plugin↔daemon link is loopback HTTP + WebSocket with a daemon→plugin **write-event push** (motivated by Obsidian's watcher provably missing external writes ⇒ stale badge = honesty bug), plus a **version handshake across plugin/daemon/CLI that refuses on mismatch** ("`--json` is a versioned contract; the schema stays free behind it"). ⇒ **Gap:** B1 adds an outbound write-event push channel (replacing A15's inbound FS-polling) and lockstep single-version plugin/daemon/CLI coupling, reversing A15's free-schema/no-handshake stance (both flagged by independent verification as uncovered host/runtime deltas).
- **A15:** CLI-first, with no host-portability story — the read-API `view-spec` contract existed (ADR-130, "render known kinds only") but had only CLI/MCP/HTTP marshalling clients, no extracted host-free renderer package and no second client. **B1:** `design Appendix A` records a VS Code client as "client #2 of the same daemon" — a host-agnostic daemon plus a shared host-free `ViewSpecRenderer`/`DaemonClient` package, yielding an M×N distribution dividend, with binary acquisition kept to the single §16 story. ⇒ **Gap:** B1 introduces a second-client/host-portability design (host-agnostic daemon + extracted renderer package) with no alpha.15 counterpart (flagged by verification).
- **A15:** the binary is `memoria` (`memoria init`/`doctor`/`serve`/`mcp`). **B1:** the binary is renamed `mem`, the daemon folded in as `mem serve`, with **no separate `memoriad` executable** — "one artifact … one distribution story, one version" (`design §16`). ⇒ **Gap:** B1 renames the CLI and collapses engine+daemon into one artifact with no separate daemon executable; the `docs-history` "memoriad" operational-contract re-home is explicitly retired to `mem serve` (flagged by verification as an unstated delta).
- **A15:** no explicit offline operating contract — the standalone CLI/engine runs local operations directly, and LLM-dependent operations simply fail when offline. **B1:** a stated offline contract — catalog/search/notes/reading fully work offline, while LLM jobs queue and fail cleanly, never half-writing (every machine write is staged). ⇒ **Gap:** B1 adds a formal offline operating contract, a consequence of the daemon + staged-write model, with no A15 counterpart (flagged by verification; the staged-write spine it rests on is detailed in §5).

## 2. Vault structure / bundle layout

- **A15:** one bundle root `knowledge/` with `folders.yaml categories = [knowledge]`; the shipped `vault-template/` owns the *whole* tree (ADR-26 "the repo is the install unit," carried to `uv tool install`, ADR-125) and additionally ships `catalog/{sources,entities}/` (with generated `catalog/index.md`), five live `spaces/*.md` navigation notes, `journal/`, `system/`, and an in-vault `.memoria/` engine dir (SQLite DB, `blobs/`, `index/qmd/`, `plugins/`) — catalog/journal *records* are SQLite (ADR-122/124), so those folder trees are vestigial projection/skeleton shells while `spaces/` is authored markdown. **B1:** a `Memoria/` **subfolder** beside unrelated existing notes (two sibling roots, zero-migration); the top-level `catalog/` (incl. `catalog/entities/` author/venue records → DB queries, "never vault sprawl"), `spaces/`, and `journal/` trees are gone (records/navigation/logs live in the derived DB + `Memoria/` `index.md`/`log.md` projections), and engine-private state (the SQLite DB, provider payloads) relocates *out of the synced vault* to a per-device data dir (`~/.local/share/memoria/<vault-id>/`), with only audit copies kept in visible `system/`. ⇒ **Gap:** nesting Memoria as a subfolder has **no alpha-line precedent** (the workspace *was* the install from the Old Skeleton through A15); removing the now-empty `catalog/`/`spaces/`/`journal/` shells plus the in-vault `.memoria/` dir continues the alpha.12→15 trajectory of moving records/nav/logs into the DB. `Memoria/` (not the whole vault) is the OKF v0.1 bundle that must match A15's `knowledge/` for exchange-format parity (OKF held as a boundary/exchange claim, not the internal model — docs-history Reject #125); `alpha-regression-review` records OKF-bundle conformance and the keep-test as **HOLDING**, not a regression. No report addresses the dropped `folders.yaml categories` mechanism or the two-sibling-roots collision (an existing user `Memoria/`).
- **A15:** catalog sources/entities are **SQLite records**, not markdown (ADR-124/126, "files own meaning, the DB owns records"); `knowledge/works/<id>/` holds the subjective `work` Concept only, with extracted text and raw payloads under in-vault `.memoria/blobs/source-content/<work-id>/` (alpha.13/14), not beside the Concept; fulltext anchoring uses W3C TextQuoteSelectors. **B1:** per-work `works/<citekey>/` co-locates `record.md` (full CSL-JSON, the fidelity/Zotero-re-import **authority** from which the *derived* DB is rebuilt — *not* a "DB-projection" [review §2 item 9]), `fulltext.md` (anchored extracted text via a `^pNNNN` anchor bridge, §16), `digest.md` (LLM digest, opt-in), and `source.pdf` (blob, sha256); provider payloads move out of the vault (per §5/§13), and a failed/no-PDF import materializes a **visible dead-letter triage stub** in the works tree (design §7; docs-history Borrow #41) so no captured source is silently lost. ⇒ **Gap:** B1 re-materializes catalog-record *authority* as markdown and co-locates record + concept + extracted-text + blob in one folder, **reverting the alpha.12→14 catalog-to-SQLite arc** (ADR-122/123/124) and reprising alpha.11's `catalog/sources/<id>/{source.md, content.md, raw/}` co-location shape re-keyed by citekey; the fulltext anchor scheme changes W3C-selector→`^pNNNN`. Treated as **convergence** (design §11, "SQLite authority + markdown keep-set + blobs"), not a regression; the CSL-JSON and durable-anchored-fulltext choices are named adversarial survivors (Rao & Callison-Burch 2026; Wang & Luan 2026) and docs-history Reject #123 ("beta.1 deliberately reversed to `works/<citekey>/source.pdf` for the keep-test"). (Citekey-as-primary-key vs A15 `work_id` is owned by §7.)
- **A15:** `knowledge/hubs` ships in every alpha (hub is one of the four surviving types, ADR-126); a single `project.md` at `knowledge/projects/<id>/`; `knowledge/digests` removed (digest folded onto `work`, ADR-126). **B1:** no `hubs/` folder (the `hub` type is dropped — a **recorded** divergence, one of the fourteen in `alpha-regression-review`, analyzed in §3, not a silent drop); `projects/<slug>/{project.md, evidence.md, gaps.md, draft.md, export/}` plus an optional `ruled-out.md` (human-written dead-end decision log, never machine-written / never auto-promoted); a per-work `digest.md` restored. `draft.md` is human-class, `local-only: true` by default, never machine-written. ⇒ **Gap:** B1 has no `hubs/` (the type persisted/reshaped across the whole alpha line — dropping it has no precedent); B1 splits project artifacts into separate files (continuing alpha.15's "detachable sub-bundle with artifacts" framing, echoing alpha.5's `projects/<slug>/{code,drafts,exports}` after alpha.11's single-file interlude; re-homed via docs-history Adapt #77/#78/#88, Borrow #59) and adds `ruled-out.md`; B1 restores a per-work `digest.md`, **reverting the alpha.15 fold** toward the alpha.11–14 separate-digest state (now inside `works/<citekey>/`, not a top-level `digests/`).
- **A15:** `inbox/` is a **reserved transient prefix** for machine-emitted attention projections/prompts (`folders.yaml transient_prefixes: [inbox/]`, ADR-119; AGENTS.md "`inbox/` transient attention projections"; `spaces/inbox.md`) — a declared prefix, *not* a physically shipped folder; `system/` ships as a rich visible tree (`templates/`, `dashboards/`, `eval/`, `patterns/`, `logs/`, `exports/`, `metrics/`, `vocabulary.md`); no `reports/`. **B1:** `inbox/` is an import-**staging** drop folder *outside the OKF bundle* (mints only via the "N files awaiting import" chip — no silent minting; every input lands as an always-succeeds catalog stub); `system/` holds only `manifest.jsonl` (exported human-span hash-manifest audit copy), `incidents/` (`type: incident`), and `metrics/` (weekly events+costs CSV) — visible-not-hidden because Obsidian Sync ignores hidden folders and audit copies must survive device loss (design §16); `reports/` is new (machine report notes — discovery digests, monitor notes — `type: report`, **never indexed for Ask and never citable as compose evidence**, structurally enforced by fulltext-only embedding, design §3/§5). ⇒ **Gap:** `inbox/` is **repurposed** (attention-projection prefix → capture-staging; a four-era lineage circling back to the era-1 `10-inbox/` capture role), *not* re-introduced [verify high-severity correction]; `system/` is **repurposed** (its `templates/`/`dashboards/`/`eval/`/`patterns/`/`logs/`/`exports/` subtrees dropped, only `metrics/` survives) [verify medium correction] and is audit/metrics only — not a capabilities/prompts tree (docs-history Reject #116, "operations are product code") nor an observability/debug system (charter "Support-org machinery" exclusion); only `reports/` is genuinely new (no alpha-line precedent — closest ancestors `knowledge/_views/` alpha.14, `system/dashboards/` alpha.1–11). None flagged as regressions.
- **A15:** the portable bibliography ships as top-level `references.bib` (name stable alpha.11→15, `.memoria/memoria.bib` before; a generated tracked projection and keep-test takeout with a bibliography-atomicity rule, ADR-122, rendered from checked SQLite rows, ADR-123/124); no folder-level or per-file local-only marker (the alpha "local-only" is a *deployment-topology mode*, ADR-63 — a homonym). **B1:** `Memoria/bibliography.bib` (renamed, role unchanged, generated from stored CSL-JSON — not a BBT auto-export "second source of truth with built-in staleness," docs-bar line 123); a new `Memoria/local-only/` folder-level marker **and** a per-file `local-only: true` frontmatter key enforced in prompt assembly (draft.md defaults to it); plus a second, new per-export `projects/<p>/export/references.bib` containing cited works only, emitted at export time. ⇒ **Gap:** `references.bib`→`bibliography.bib` is a purely nominal rename against a continuous-role history; `local-only/` (folder) and `local-only: true` (per-file) are new local-only/egress markers with no alpha-line precedent (closest analog: gitignored disposable trees); the bibliography splits into two artifacts (root portable `.bib` + per-export cited-only `.bib`).

## 3. Concept / type model

- **A15:** four knowledge Concept types — `note` (with `mode: claim|question`), `work`, `hub`, `project` — each with a per-type YAML schema (`.memoria/schemas/types/*.yaml`) loaded and validated in product code (`schema.py`), hardened at alpha.15 to closed validation (rejects undeclared fields; `x: map` escape hatch); per-type schema files run continuously from the pre-alpha.1 Old Skeleton (18-type roster at alpha.1, accreted then collapsed 25→12 at alpha.11 and 12→4 by ADR-126 at alpha.15). **B1:** a flat frontmatter `type` **class enum** `work|fulltext|digest|note|evidence|gaps|project|draft|ruled-out|export|report|incident|conflict` (design §3), no per-type schema files, OKF-permissive tolerate-unknown-keys consumption. ⇒ **Gap:** B1 replaces the four typed-concept schemas with a flat file-class enum; per-type frontmatter+body schemas are absent — no alpha-line precedent (every era carried per-type contracts, and the enum mixes Concepts, catalog record, and machine projections as siblings that ADR-126 had refused to make into types). Recorded as a deliberate trade, not a silent one: the flat enum is stated verbatim in design §3, and the dropped reject-undeclared validator is adjudicated in `regression-review` R6 ("Low–borderline"), declined in design §18(6) ("OKF-permissive is deliberate"; machine writes stay conformant by construction). Only the validator/strictness aspect and the enum replacement are recorded; the drop of the schema *files* as artifacts is not individually named.
- **A15:** `hub` is a first-class type owning exactly one ID-backed controlled-vocabulary tag (required `tag:` field); membership = everywhere that checked tag appears (ADR-126). It descends continuously from the Old Skeleton / alpha.1 MOC (renamed `hub`, ADR-50). **B1:** no `hub` type; no hub concept anywhere. ⇒ **Gap:** the hub type is dropped — no alpha-line precedent (a structure/MOC/hub node existed continuously from before alpha.1 through every alpha release). Recorded as deferred, not killed: design §7 names "hubs and mapping views are post-beta.1 consumers," and `regression-review` classifies "hub/tag graph" as a recorded divergence. Rationale: the graph stays DB-side and machine-spawned pages are a standing reject (§17(5) "Entities stay DB-side"; `docs-history` reject "Entity note types" — "Four concept types are deliberate"). Conflict: `info-flow-research` argued for editorial hubs as the organizing layer (verdict adopt), but the design defers them.
- **A15:** `note` carries `mode: claim|question`, `question_status: open|resolved`, claim-context fields (`temporal_scope`/`tense`/`qualifier`), and a full claim substrate (`claim_text`, `quote`, `source_id`, `evidence_set`, `extraction_confidence`, `annotation`, `anchors`); ADR-126 made claim/question a *mode* not a type so the question→claim transition isn't destructive, with the mode creating gap-engine obligations (a claim without evidence = an `under-warranted` gap; an open question = a compose lead). **B1:** `note` = "atomic + fleeting notes (human)" (design §3); no `mode`, no claim/question distinction, no `question_status`, none of the claim/anchor substrate. ⇒ **Gap:** B1 drops the note claim/question mode and claim-context fields. Classed a recorded divergence (`regression-review` names "claim-context fields"), but no §11–§17 passage or charter exclusion actually locates it — a partial silence at the design-section level; the design body simply omits the fields. The consumer (note-mode gap obligations) is argument-graph gap analysis deferred to alpha.16; fleeting capture folds into a mode-less `note` plus the staging `inbox/`.
- **A15:** `project` is a typed concept absorbing the argument graph (thesis role, `outcome_frame`, `paper_plan`; ADR-77/78/79, with the saturation floor folded into the gap engine). **B1:** a `project/` folder of separate files (`project.md`/`evidence.md`/`gaps.md`/`draft.md`), `project` is one class value, and gap analysis is retrieval-coverage, not argument-graph. ⇒ **Gap:** B1 has no argument-graph project concept (see §8) — structurally rhyming with alpha.1's numbered per-project subfolders rather than the alpha.11→alpha.15 consolidation into one typed concept. Recorded and explicitly deferred to alpha.16 (`docs-history` borrows all Target alpha16); saturation additionally collides with `requirements` Deliberate Exclusion "Aggregate evidence scores" (`research-evidence`: a "saturation %" widget "would be Memoria's Consensus Meter — seductive and wrong"). Also dropped (verify): A15's deliverable-artifact target types `outline`/`section`/`figure`/`code` collapse to the `draft`/`export` enum values (outline folding into `project.md`); and the operation-level face of the same concept — the `project frame-paper --frame-file` operation, structural paper-readiness gaps, and the `project export --ready-only` paper-plan variant.
- **A15:** ULID `id` identity kind ("identity ≠ filename ≠ title", ADR-126) — a single-release-old invention (`schema.py` `ulid`/`is_ulid`) decoupling identity from path so renames/moves don't break references. **B1:** identity is `citekey` + file path (works keyed by citekey; no ULID; design §5/§7, citekeys pinned at import). ⇒ **Gap:** ULID identity is dropped; identity is path/citekey-derived again — reverting the alpha.15-only ULID layer to the path-derived regime of alpha.1→alpha.14 and re-promoting `citekey` to an identity key (reversing alpha.11's demotion of citekey to a display alias and alpha.14's ADR-124 `work_id`-as-primary). Consequence (verify): a citekey correction now renames the Work directory (reversing ADR-124's "citekey change no longer renames the Work") as a first-class revertible `rename` run (design §7). Silence: the ULID drop is recorded nowhere as a deliberate decision — not in R1–R6, not among the fourteen recorded divergences, not in the "checked and intact" sweep; only embodied in design §5/§7.
- **A15:** `links:` is a nested relation-typed frontmatter map validated against `{supports, contradicts, extends}` (`schema.py` `_check_links`, `LINK_RELATIONS`), continuous from ADR-52 (alpha.1) through alpha.15. **B1:** links are Obsidian wikilinks in the body + a DB `links` backlink table + `%%prov%%` block markers; no relation-typed frontmatter links map (even the supersession successor is a scalar `superseded-by:` key, not a typed links entry). ⇒ **Gap:** typed authored relation links are dropped from frontmatter — reverting *past* ADR-52 to the OKF prose-conveyed body-link shape the alpha line kept out of the internal representation (ADR-107). Not individually adjudicated: not among R1–R6 (R6's "nested-link invariants" concerns validator strictness, declined in §18(6)), and not among the recorded divergences. Conflict: `info-flow-research` holds typed edges to be the "denominator-honest gap analysis" differentiator ("untyped wikilinks alone cannot power" it) and wanted them as plain frontmatter; the design substitutes untyped wikilinks + a DB table and defers the argument-graph consumer to alpha.16.
- **A15:** a controlled-vocabulary substrate — `system/vocabulary.md` as "the single source of truth for the controlled values" with faceted vocabularies (`research_area` ~30 curated terms, `methodology`, note `topics`); a `vocab` command mints/manages ID-backed tag IDs; a `classify` operation applies `research_area` from OpenAlex topic scores (floor 0.6 / near-tie 0.15). Present continuously from the Old Skeleton controlled-vocab schema through alpha.1's `vocabulary.md` + `classify.py` and every subsequent redesign, implementing the surviving half of ADR-04 ("topics live in frontmatter facets, never in folders"). **B1:** no controlled-vocabulary / faceting model — no `vocabulary.md`, no `research_area`/`methodology` facets, no `vocab` command, no `classify` operation. ⇒ **Gap:** the entire controlled-vocabulary / faceting model is dropped (§3 otherwise only touches the hub's single vocab-tag) — no alpha-line precedent for its total absence, and topical classification is left with no frontmatter facet home. Correction (review §3, PLAUSIBLE): no primary B1 source defines a `tags` field — the design's Memoria frontmatter block declares no `tags` key — so a surviving degraded/freeform-tags field must not be asserted; the stated delta (faceting dropped) is unchanged. Disposition (addendum): deliberate and doctrine-driven, not a silent loss — `if-design` reframes OpenAlex topics/concepts as an `enrichment-only (unverified)` charting tier (never a curated vocabulary; only human-certified gap stubs may feed query drafting on any revival), and `docs-bar` rejects machine-materialized frontmatter facets as violating meaning-only frontmatter.

## 4. Frontmatter discipline

- **A15:** universal meaning-only frontmatter enforced in product code on **every** Concept (`id`/`type`/`title`/`tags`/`links`/`aliases`/optional `archived` + per-type); universality was a late arrival (alpha.14 PR #1157 first forced it — before that frontmatter was per-type, exempting `system/` infra and nav pages). **B1:** frontmatter appears **only on Memoria-created files**; a freehand human file created inside `Memoria/` carries none and converges via a one-tap proposal ("flags never block"); "absence of all `mem-*` markers = human" is the human/machine detection rule (replacing DB-recorded authorship) and the basis of zero-migration. ⇒ **Gap:** B1 re-scopes universal coverage from every-Concept to every-Memoria-created-file at the `Memoria/` bundle boundary (design §3) — a re-scope, **not a regression**: OKF exchange-format conformance held (`alpha-regression-review`, "no regression of the exchange format").
- **A15:** closed validation — `validate_frontmatter` **rejects undeclared fields** ("put custom data under `x`") with an `x: map` escape hatch on every type (PR-AC #1197); this closure was alpha.15-only, reversing the alpha.1→alpha.14 permissive "schema constrains, does not enumerate" posture. **B1:** no reject-undeclared validator; the contract is re-framed as an OKF v0.1 bundle (tolerate-unknown-keys consumption, closed `type`/class enum, recommended-key semantics, `okf_version` key); machine writes stay conformant by construction (VaultWriter serializes from typed fields), freehand files get non-blocking OKF proposals. ⇒ **Gap:** B1 drops the closed/reject-undeclared validator and the `x.*` extension namespace and re-frames the frontmatter contract as OKF-permissive — recorded as regression **R6** (Low-borderline) and deliberately declined in **§18(6)**, with `x.*` confinement **deferred** (`alpha-regression-review`).

## 7. Catalog / bibliographic authority

- **A15:** the SQLite catalog + source-content blobs are the citation authority (ADR-124, absorbing ADR-06); the relational catalog is working state that "fails the survival test," so only a compact citation-survival copy is mirrored to markdown (the alpha.12 keep-test flip, ADR-122). **B1:** full CSL-JSON is serialized into `record.md` as a fenced block as the fidelity/re-import authority — "the derived DB is rebuildable, so bibliographic fidelity and the Zotero re-import key cannot live only in the DB" (design §3); the DB is a rebuildable projection (design §5, "Derived store"). ⇒ **Gap:** authority for bibliographic fidelity returns to markdown — re-adopting the "frontmatter-only catalog" alternative ADR-124 explicitly named and rejected, and storing *full* CSL-JSON where alpha.12 kept only a compact survival copy — but design §11 books this under **Converges** ("SQLite authority + markdown keep-set + blobs," l.965), so it is a re-scoping (`record.md` owns only what "cannot live only in the DB"), not a wholesale SQLite→markdown reversal.
- **A15:** `work_id`/provider identifiers are the primary runtime keys and citekeys are aliases (ADR-124); the citekey was demoted to optional alias back at alpha.11 §6.1 precisely because "citekeys are metadata-derived and get corrected" (fixture-verified across a citekey correction). **B1:** the schema is `works(citekey PK, doi, title, year, csl_json, status, pdf_sha256, distrust_flags)` (design §5, l.374) — citekey is the primary key, pinned at import (inherited from Better BibTeX), with correction as a first-class revertible rename run (design §7); the immutable Zotero item key is demoted to `csl_json.custom.zotero_key` for idempotent upsert. ⇒ **Gap:** the primary key reverts past alpha.11's demotion all the way to the Old Skeleton / ADR-05–06 citekey-backbone, with pinning-at-import a new synthesis the alpha line never had; it also silently diverges from the superseded IF-design "identity law" (immutable Zotero item key + DOI = identity, citekey = filename only).
- **A15:** `references.bib` is generated from SQLite. **B1:** `bibliography.bib` stays Memoria-generated from stored CSL-JSON — complete across non-Zotero works (repos, direct DOIs) — with the researcher's own BBT auto-export demoted to an optional cross-check, "never the artifact of record" (design §7 l.584–587, §17.6 l.1261–1264); a BBT `.bib` *ingest* interface was rejected as "a second source of truth with built-in staleness and sync races" (docs-history l.123). ⇒ **Gap:** the generator is kept but its input flips SQLite → `record.md` CSL-JSON; the engine-generated `.bib` is the survival artifact satisfying charter Success Claim 4 ("the vault remains useful as plain markdown plus standard bibliography files if Memoria dies").
- **A15:** a single root `references.bib`. **B1:** the generated bibliography is renamed/relocated to `Memoria/bibliography.bib`, and a *second*, new per-export `projects/<p>/export/references.bib` (cited works only) is emitted at export time (design §8 l.838–839). ⇒ **Gap:** one root artifact splits into two — a complete vault-level `Memoria/bibliography.bib` plus a per-export cited-only `references.bib`.
- **A15:** ADR-124 makes the Memoria catalog THE citation authority; citation-facing fields are locked vault-side. **B1:** citation-facing fields owned by the researcher's Zotero are corrected AT SOURCE (deep link to the Zotero item; the next sync confirms) rather than vault-side. ⇒ **Gap:** citation authority is partially handed back to Zotero for Zotero-owned fields, partially reversing ADR-124's catalog-is-authority stance.
- **A15:** the vault ships `catalog/`, `catalog/sources/`, `catalog/entities/` as workspace folders with a generated `catalog/index.md` projection and author/venue entity pages (records live in SQLite, but the folder tree + nav home exist). **B1:** the top-level `catalog/` tree is dropped entirely — a work's `record.md` co-locates under `works/<citekey>/`, and author/venue entities are graph queries, "never vault sprawl." ⇒ **Gap:** the `catalog/`+`entities/` folder tree with its index/entity pages is removed in favor of per-work co-location and query-time graph views.
- **A15:** "Memoria does not require a live Zotero library" — Zotero/BBT are optional user tools (ADR-124, superseding ADR-05), surviving only as importers, not authority; the alpha.14 standalone cut removed the Zotero-specific import surface behind an `rg` negative gate. **B1:** Zotero import is day-1 (§11's "single largest missing item," back-pressure priority 1, l.974–975) over the Zotero 7 local HTTP API (`127.0.0.1:23119`, Guard-allowlisted); reading is Zotero-first (Zotero's PDF reader is the primary read, append-only annotation sync-back, exact-quote anchor bridge; `fulltext.md` demoted to evidence substrate and kept as the full path for non-Zotero works, §17.1); PDFs are copied into the vault at `works/<citekey>/source.pdf` (deep links rejected — they "make the corpus die with Zotero's install," docs-history l.123), and a work with no attachment rests at `no_pdf` with a one-click "capture in Zotero" handoff — a Memoria-side OA-PDF fetcher stays out as "both an allowlist violation and a zero-interaction beacon" (design §7 l.561–567, §17.3). ⇒ **Gap:** Zotero is re-centered as the primary reading/capture surface — reverting the alpha.14 demotion toward the founding Razlogova / ADR-05 backbone — but as *primary/default*, not a hard requirement (non-Zotero DOI/arXiv/PDF/BibTeX/repo paths stay first-class; the charter lists Zotero as a required Capture input and null-workflow baseline, not a Deliberate Exclusion); Zotero-reader-as-primary-read with annotation sync-back has *no* alpha precedent and runs opposite the standing alpha rule (alpha.11 §13) that Memoria owns its own quote/span/hash selectors.

## 8. Discovery / gap analysis

- **A15:** gap analysis is argument/knowledge-graph based — `note` modes (ADR-126's `mode: claim | question`) create obligations the gap engine feeds on (a claim without evidence = an `under-warranted` gap; an open question = a compose lead); `project` carries the argument-graph **saturation** floor (ADR-79: ≥1 addressed support AND ≥1 counterpoint, advisory/never-gating, over checked-only inputs), with a `gap_type` taxonomy (additive/conflict/fragility/structural/unstated-warrant/refutation) and {0,1,2} ordinal gap-scoring inherited from the alpha.5 Project gate (`structural_impact.py`, ADR-77/78/79), absorbed into alpha.15's native gap engine (ADR-129); OpenAlex Topics/controlled facets are part of the read model. **B1:** gap analysis = **retrieval-coverage** gaps itemized by cause (unsearchable/partial/superseded, with truncated/abstract-only counted as *suspect, not covered*) + content gaps (question facets with corpus df 0), with a printed denominator, an empty-contrasts refusal (an empty list would be an implicit agreement meter), and a `gaps.md` compose artifact regenerated wholesale opening on the denominator (`design §8`). ⇒ **Gap:** B1 has no argument-graph/saturation/thesis-health gap analysis, no note-mode-driven obligations, and no ordinal gap-scoring; gaps are retrieval-coverage-based only. The dropped argument-graph machinery is **deferred to the alpha.16 writing module, not deleted on the merits** (`docs-history B/A/R` Borrow rows — argument graph, gap-taxonomy kit, saturation with refutation floor — all target `alpha16`); the aggregate *saturation-score* component is separately barred by the charter's "Aggregate evidence scores" exclusion (a "saturation %" widget would be Memoria's Consensus Meter). Scope-wise this reverts gap analysis to its pre-alpha.5, coverage-only state, though the printed-denominator, itemized-by-cause framing is itself new in B1.
- **A15:** discovery informed by the imported topic/concept/keyword/citation graph — a late alpha.13/14 build routing OpenAlex `concepts`/`topics`, `authorships`, and Crossref `relation` edges into candidate generation, with ADR-92 deterministic relevance scoring over those candidates. **B1:** discovery = bounded **citation-graph only** (snowball depth ≤2 backward-refs + forward capped cited-by + a scheduled citation monitor), identifiers-only egress (`guard_log.vault_text = false`), ranked by transparent occurrence-counting across seeds; output never enters gap/coverage denominators (`design §7`). ⇒ **Gap:** B1 discovery is citation-graph-only, no topic/concept-graph-driven discovery — the discovery-side face of the §9 enrichment-edge contraction (B1's `edges` table stores only outgoing refs + capped cited-by). The topic/concept co-occurrence leg is re-derived-as-out on evidence (VOSviewer/CiteSpace field-scale mapping non-transferable to a single-researcher corpus, `info-flow-research` verdict **avoid**), not merely dropped; the citation-graph leg B1 keeps is positively grounded (Hirt et al. 2023; BibliZap occurrence-count scoring, verdict **adopt**).
- **A15:** discovery is **steered** by the researcher — the shipped `steering.md` is a first-class "program memory" priorities file (open questions / synthesis gaps / papers-to-prioritize), traceable unbroken from alpha.1's `research-focus.md` through the alpha.11 rename; a named agent persona, **"The Librarian"** (tracing to the pre-alpha.1 Old Skeleton, consolidated at ADR-48), reads it at session start and weights discovery by it ("the main lever you have over what gets surfaced"); shipped `system/eval/` rubrics score the Librarian's behavior, and ADR-92 uses deterministic `steering.md` relevance to rank candidates. **B1:** no `steering.md`, no Librarian, no PI-priority weighting; discovery is deterministic occurrence-counting over the citation graph (the per-batch yes/no LLM relevance prefilter is a **seeding/onboarding** step, §15, not part of discovery). ⇒ **Gap:** the steering-lever + Librarian-persona discovery substrate is dropped entirely (a high-severity omission surfaced by independent verification). The Librarian's external-API, gap-driven discovery is a **reject on the three-host egress allowlist** ("alpha.16+ decision, never silent creep", `docs-history B/A/R`); priority-weighted ranking is replaced by transparent occurrence-counting; `steering.md`'s program-memory function is left **un-re-homed for beta.1**. §11 records the agent boundary itself as *convergence* with ADR-130 and points here for this concrete Librarian removal.
- **A15:** ADR-95 (nightly/always-on discovery loop — the long-deferred ADR-61 proposal, unbundled at alpha.8) was **rejected** at the alpha.15 consolidation sweep — "an unattended always-on loop conflicts with the no-daemon architecture" (disposition map) — a rejection resting entirely on the alpha.14/15 no-daemon premise (ADR-125; §1 "there is no internal daemon"). **B1:** a resident two-process daemon (§1) **and** a scheduled citation monitor (§7, one capped machine note per schedule, batch-API digest polling across sessions) — together the two conditions (an always-on machine + an unattended scheduled loop) that ADR-95's own "When this matters" and ADR-61's adoption conditions demanded. ⇒ **Gap:** B1 removes the exact no-daemon premise the rejection stood on and reverses it; the charter's discovery exclusion is **amended 2026-07-03** to bar only open-ended search/recommendation while admitting bounded citation-graph discovery (snowball ≤ depth 2 + capped monitor, identifiers-only egress, output out of Ask/gap denominators) — `requirements` Deliberate Exclusions, `design §17(4)`.

- **A15:** the `--json` schema stays **free** behind the read-API (deliberate schema-free stance). **B1:** a **plugin/daemon/CLI version handshake** with refuse-on-mismatch (lockstep single-version coupling), and `--json` becomes a **versioned contract** ("the schema stays free behind it") — a reversal of A15's free-schema stance. ⇒ **Gap:** B1 makes the read-API surface a versioned, lockstep-checked contract across the three clients; A15 left it schema-free. (Also stated in §1.)
- **A15:** the read-API's clients are CLI/FastMCP/loopback-HTTP marshalling skins; the view-spec is engine-emitted data with no renderer and no second host. **B1:** a recorded **second host client (VS Code)** as "client #2 of the same daemon" (the M×N dividend), served by a **host-free `ViewSpecRenderer` + `DaemonClient` TS package** and a `DocumentLinkProvider` resolving evidence anchors (design Appendix A); plus a **deferred, post-beta agent-submitted view-spec *instance* path** — agents may land view-specs as data through the request envelope inside a mandatory proposed-view container (provenance banner, engine-clamped loudness, enqueue-only vocab), needing its own adapter ADR + deceptive-layout fixtures. ⇒ **Gap:** B1 factors the renderer host-agnostic (enabling the VS Code second client) and opens a governed agent→surface view-spec channel; both are absent from A15's rendererless, single-host read-API. (Host-portability rooted, cf. §1; the agent-instance path also touches §11/§13.)

## 12. Security / OS confinement

- **A15:** network access operation-scoped + allowlisted and hardened fail-closed (alpha.14 #1103; alpha.15 ADR-125/130) — **no agent tool grants** (never grants file/terminal/exec/send), untrusted work text returned as tagged data never instructions, loopback HTTP token-auth (`MEMORIA_HTTP_TOKEN`); injection-as-untrusted-data (lethal-trifecta mitigation); capability import quarantined; standalone **Python 3.12 / uv / Typer** runtime (Node ≥22 dragged in only by the `qmd` search subprocess, design-earmarked for removal), platform-agnostic — **no OS-kernel enforcement layer at any point in alpha.1→alpha.15** (the boundary walked Hermes MCP-only write-gate → alpha.9 egress-as-P0 → alpha.11 channel-separation + `allowed_network` → alpha.14/15 no-agent-tool-grants, always application-level). **B1:** brokered egress via Guard (engines get **no sockets at all**; every egress is a typed IPC call to Guard over a compiled-in, requirement-driven allowlist; Guard fails closed when policy can't load, workers refuse to start) **+ Linux network-namespace (`CLONE_NEWNET`, no interfaces) OS enforcement only**; the seccomp fallback, macOS `sandbox-exec`, and Windows-Firewall/UAC flows are **cut** (design §10, "near-zero enforcement for real code"), so on macOS/Windows egress restriction is "process-architecture discipline, not OS enforcement," carried as a named honest residual and stated per-OS in SECURITY.md. ⇒ **Gap:** only the **OS-enforced kernel egress guarantee (Linux `CLONE_NEWNET` netns)** is Linux-only — a beta.1 addition with no alpha-line precedent; the containment model itself (brokered-egress Guard split, linter, keychain, canary) is cross-platform, and macOS/Windows lose *only* OS-level egress enforcement (a factual continuation of the alpha.15 application-level model), not "full security." Recorded explicitly as a cut in design §10 (not in the charter exclusions table); not a regression — swept as held/strengthened and framed in §11 as a *sharpening* of the existing allowlist mechanism. (See also §14 — platform.)
- **A15:** "no-exfil" named as a *structural* boundary property at alpha.11 (agents can't exfiltrate because they hold no network and no write authority) but **no content-neutralizing linter run at vault-write**; no per-OS keychain — the sole credential was the loopback bearer token `MEMORIA_HTTP_TOKEN` (residual "any local bearer credential is readable by a trusted-local agent"); a redaction-self-test existed only on the alpha.8 *diagnostic* plane (ADR-105). **B1:** VaultWriter-owned **exfiltration linter** (one owner; default-deny strip-never-flag link/embed/HTML neutralization; only surviving link shape `https://doi.org/…` with no query/fragment/userinfo; applied at apply and re-run over the whole bundle at export, extended to extracted fulltext + imported metadata); **one canary spec owned by Security** (`MEMORIA-CANARY-<8hex>` minted beside the key; weekly + post-migration drill proves the redaction path fires *and* scans all artifacts/DB/logs/exports; any hit ⇒ `INC-canary-<date>` + engines disabled); **per-OS keychain** (Keychain / DPAPI / Secret Service + mode-0600 fallback + logged warning on Linux). ⇒ **Gap:** B1 centers write-time exfiltration neutralization + canary + keychain — but, correcting the "all new in B1" framing: the **linter is a convergence**, not new (design §11 converges with alpha.15 on it and the alpha-regression review sweeps it as already-present/**strengthened**), operationalizing the alpha.9 poisoned-data threat model into containment-at-write; the **canary generalizes** the alpha.8 ADR-105 redaction-self-test from the diagnostic plane to every artifact; only the **keychain is genuinely new-in-B1**, arriving coupled to BYOK (§13) since the alpha line held no long-lived provider key to protect. (Instrumentation silence: the beta.1 evaluation-protocol logs only `guard_log` + weekly metrics and schedules no canary-drill/redaction-path acceptance test, despite the paper B/A/R call to score against canary drills and egress-log audits.)
- **A15:** no third-party-plugin-interference threat — the standalone engine had **dropped Obsidian as a required surface** (alpha.14: "product must run on `vim`/file browser"), so the plugin surface and its whole apparatus (ADR-74 provenance-lock, ADR-12 obsidian-linter "do not install" prohibition, `vault-hash-drift` post-hoc detection) were **structurally absent, not merely unmitigated**. **B1:** names **third-party-plugin interference** a security corruption class — format-on-save plugins rewriting machine-owned spans / flipping markers / tripping byte-immutability rollback loops, and secret-shaped fields in plugin `data.json` — handled by startup detection of known-hostile plugins/settings + registering Memoria-owned paths as formatter-excluded + auditing plugin `data.json` for secret-shaped fields, with the human-span pre==post byte-manifest gate as the structural backstop (design §10). ⇒ **Gap:** the class is a **revival, not an invention** (a re-homed docs-history borrow) — it lived in the alpha.1–alpha.11 Hermes/Obsidian era and was *eliminated* at alpha.14/15, re-created as a direct consequence of the §1 host pivot re-introducing Obsidian as the required researcher UI. The formatter-exclusion mechanism is new (the alpha line either forbade the plugin outright or caught its effect post-hoc); the "secret-shaped `data.json`" facet has no alpha precedent, coupled to the new keychain/BYOK key-leak threat (T3).

## 13. Cost / snapshot / migration machinery

- **A15:** cost was *telemetry, never a control surface* — per-run token-cost journaling (alpha.14 PR-H pinned `runner.test`/`runner.live` per operation, journaled resolved model + token cost per run; alpha.15 ADR-129 carried it per `(operation, model, mode)`), a cost-join export plane (alpha.8 ADR-104/105/106), and an *unaggregated* `cost-misses.jsonl` writer (ADR-104: "writer but no aggregator"); "no threshold in the prior corpus was ever validated" (ADR-129). **B1:** BYOK; a **normative price table** that "lives in one place — Ledger — and every component references it, never restates constants" (design §9, recorded reason "three components had diverged"); `max_tokens` ceilings so worst-case price "is arithmetic, not hope"; batch API (50%, cache savings budgeted as zero); a circuit breaker **enforced three times** (preview→refuse enqueue, pre-call recheck, breaker); $20/mo default soft budget + researcher hard cap; an explicit **miss row** (unattributable spend treated as over-budget evidence); owner is Ledger (design §2), and an LLM call is an "operation" only if its worst-case price can be printed before it runs (design §13). Charter Cost-control row + Success Claim 3 (Tolerance); eval §1/§3 map it to the `costs` CSV that "survives DB loss." ⇒ **Gap:** B1 centers a full cost/metering/breaker machinery with no alpha-line precedent, reviving alpha.8's miss-row concept by name (now with the aggregator it lacked); grounded in research (opaque credits a top SciSpace abandonment trigger; ~$10–25/mo tolerable spend behind the $20 default).
- **A15:** two OpenAI-*compatible* runner providers (Ollama + an OpenAI-style gateway), pinned per operation via `providers.yaml`, driven through **pydantic-ai** as the typed runner. **B1:** the Anthropic **Message-Batches** endpoint with one pinned model tier; 50% batch savings and cache-zero applied in cost previews. ⇒ **Gap:** the runner's provider transport swaps from OpenAI-compatible dual-provider to a single Anthropic Message-Batches tier, and pydantic-ai is dropped as the typed runner (surfaced by verification).
- **A15:** a **relational cascade-rollback** over a derivation graph — lineage ADR-25 hash-paired audit log → alpha.11 `journal/<machine>.jsonl` PROV-DM events with the output-Concept as rollback unit → alpha.12 ADR-122 hash-chained SQLite journal + cross-store recovery + recursive-CTE cascade → **alpha.15 ADR-127** 2PC (journal states `pending`/`prepared`→`materialized` with the commit SHA recorded after `git commit`, write-ahead fsync ordering, one workspace write lock, `workspace recover` crash recovery, WAL + `synchronous=FULL`), "one commit per operation staging **only** materialized paths, never `git add -A`." **B1:** **external git-dir + `VACUUM INTO`** ("both, because they version different things"), the 4-step **batch protocol** (snapshot/apply/verify/commit) pathspec-limited to the run footprint, and **one-click revert** via rendered inverse-diff + a byte comparison of captured human-block bytes (any difference aborts) + a **footprint-scoped DB reconciliation** re-deriving every derived table (design §6; SnapshotService, design §2). ⇒ **Gap:** B1 keeps ADR-127's pathspec commit hygiene but replaces the cascade engine with git snapshots + footprint-scoped DB reconciliation; `VACUUM INTO` as the DB-snapshot primitive has no alpha-line precedent (design §11 records convergence on "one-writer + staging + run-scoped snapshots").
- **A15:** ADR-127's recovery triad — files own authored meaning (file-recoverable), provider records replay from content-addressed blobs, verdicts/journal DB-only accepted-loss; revert = journal cascade replay. **B1:** an explicit two-class split — **DERIVED** state (chunks/FTS/vectors, links, anchors, edges, entities, `works.status`, checks verdicts) rebuildable via `mem rebuild` (incl. anchors-geometry-rebuildable-from-PDF) vs **OPERATIONAL LEDGERS** (runs/jobs/events/costs/incidents/guard_log/payloads) *not* rebuildable, preserved by `VACUUM INTO` + selective export; revert re-derives DERIVED but leaves the operational ledgers untouched (history appends, "cost rows never deleted," "no `record.md` ⇒ no work row"); per-file dry-run classification (clean-restore | delete-created | conflict), inverse hunks overlapping a later human edit divert to `<name>.conflict.md` + raise an incident. ⇒ **Gap:** B1 reframes ADR-127's file/blob-replay/DB-accepted-loss triad into a DERIVED-vs-OPERATIONAL taxonomy with an intentional revert carve-out and per-file conflict handling (verify-findings).
- **A15:** the write unit is the **operation** (`operation run <id>`). **B1:** an enumerated **run** with a fixed run-kind enum (capture|extract|digest|import|ask-materialize|export|migration|merge|revert|discovery|rename); every machine write belongs to exactly one `run_id`, a `runs` table keys snapshots/revert/cost/Review/incidents, whole-run atomicity with a `completed_partial` exception, elevating revert/migration/rename/discovery/export to first-class run-kinds ("runs rule time"). ⇒ **Gap:** B1 renames operation→run and makes an enumerated `run_id` the single key threading the snapshot/revert/cost/Review/incident machinery (verify-findings).
- **A15:** migration **deliberately declined** — alpha.11 removed a schema-migration shim ("no production vault → no migration target … schema paths are removed"); alpha.15 ADR-125 fixed the posture as stdlib `sqlite3` with "`.sql` DDL gated by `PRAGMA user_version`," "ORM + migration framework rejected." **B1:** a **MigrationRunner** named a beta.1 **blocker** — "beta is the first version whose data must survive later versions" (research-evidence 255/260, quoted verbatim; design §11 item 9, "a process artifact to land during alpha.16"); first-class supervisor component (design §2); charter "Beta.1 data survives later versions through exercised, lossless migrations." ⇒ **Gap:** B1 reverses the alpha-line anti-migration premise; docs-history Reject row 130 ("migration machinery before a real user is speculative complexity already torn out once") is reconciled by its own trigger — beta.1 is the "vault worth migrating."
- **A15:** schema evolution tracked by a bare `PRAGMA user_version` integer gating `.sql` DDL (ADR-125). **B1:** a structured `migrations(n PK, name, kind, applied_ts, git_pre_tag, db_snapshot, status)` ledger recording a per-migration git pre-tag + `VACUUM INTO` snapshot + status, plus `meta.schema_version` (design §5; §6 snapshots for import/migration kinds only). ⇒ **Gap:** B1 replaces the bare `user_version` integer with a structured, self-describing migrations ledger table (verify-findings).
- **A15:** ADR-127 recovery included a git-tracked append-only `.memoria/overrides.jsonl` human-disposition log with **tiered DB-loss replay** — restrictive dispositions (archive, contested-block) auto-reapply, permissive ones (degraded acceptance, retraction resolution) return as attention items to re-confirm ("the journal witness is gone, so the log cannot be trusted to unblock") — plus a mandated three-store backup contract whose second leg was **SQLite streaming replication (Litestream-class)**, `doctor`-reported and "never a runtime dependency" (all alpha.15-native, ~6 weeks old at the cut). **B1:** researcher decisions live in frontmatter (`superseded`/`reading`) + an exported decide-events CSV; §18 **explicitly declines** DB streaming replication (R5: "the charter's backup/restore bar is met by `VACUUM INTO` + `restore --verify`; streaming replication is production hardening") — grounded by research-evidence on Litestream #1083 (a silent-replication failure with healthy-looking metrics) — and the backup-status report moves from a `memoria doctor` command to the read-only **inspector pane**. ⇒ **Gap:** `overrides.jsonl` + the tiered restrictions-reapply/permissions-re-earn replay is dropped (no B1 analog, no dedicated report defense), and the streaming-replication leg is an explicit decline; surviving stores are git-remote + vault-sync-subsumed blob, backed by periodic `VACUUM INTO` snapshots.
- **A15:** a hard-won cryptographic apparatus — alpha.8 ADR-104 *rejected* an in-file `prev_hash` chain ("the vault is already a Git repo — commit history is a Merkle chain") → alpha.12 reversed *inside the DB* (`journal_events(… prev_hash, row_hash UNIQUE)`, SHA-256, `GENESIS` seed, append-only triggers) → alpha.15 ADR-127 added the git-tracked `.memoria/journal-head` anchor as the tamper-evidence root ("narrows ADR-104's rejection … DB rows plus a git anchor"). **B1:** the cascade-rollback journal is replaced by the git-snapshot batch protocol; the **DB-row-level hash chain and the `journal-head` anchor are not carried** (a beta Mem-Run trailer replaces them). ⇒ **Gap:** the DB-row-level hash chain + `journal-head` root are dropped — but *not* tamper-evidence wholesale, since git's own commit chain (itself hash-chained) remains, landing B1 back near alpha.8/ADR-104's original "git commit history *is* the Merkle chain" posture — i.e. tamper-evidence is narrowed, not dropped wholesale.
- **A15:** the derived SQLite DB (`memoria.sqlite` at `.memoria/state/`, alpha.12 ADR-122) and content-addressed payloads are per-machine and gitignored but physically **inside the synced vault** (alpha.15 ADR-125 kept this shape), affordable only under alpha.12's relaxed "no multi-device sync (single machine)" constraint. **B1:** the DB (and payloads) move **out of the synced tree** to a per-device OS data dir `~/.local/share/memoria/<vault-id>/memoria.db`, with `<vault-id>` namespacing — the XDG placement rhyming with the alpha.8 ADR-105 diagnostics plane's XDG home, and downstream of the beta plugin+daemon operating over the user's *existing* vault. ⇒ **Gap:** the derived store is relocated out of the vault to a per-device path (distinct from the external git-dir relocation in the snapshot bullet above — that relocates git history, this relocates the derived store itself).

## 14. Platform

- **A15:** standalone CLI, cross-platform by construction — no OS-specific enforcement claimed. The alpha line's *only* platform split had ever been runtime-location (Hermes WSL2-only + Windows-as-editing-surface, ADR-26/55/64; native-Windows deferred then accepted at alpha.4), and alpha.14/15 had just collapsed it — standalone installers by default (`374a702a`/`4d761efe`) and Hermes dropped from the product outright (arc f), yielding "one install command, no external runtime, no always-on process" (design-history §14). **B1:** re-introduces a platform split on a *new axis* — security-guarantee strength. The *application* runs cross-platform (code-signed Windows builds, per-OS keychain); the containment model, VaultWriter exfiltration linter, per-OS keychain, and canary drill all run on **every OS**, and only the `CLONE_NEWNET` kernel-enforcement layer is Linux-only; the plugin is desktop-only; mobile is keep-test mode (synced markdown + Bases, no daemon). ⇒ **Gap:** B1 grafts an OS split with no alpha-line precedent onto a codebase that had just finished flattening its *old* (runtime-location) OS dimension; the split decomposes into the four facets below (design §10/§16; twin of gap §12's OS-confinement bullet, which closes "(See also §14 — platform.)").

- **Facet A — OS-enforced egress is Linux-only.** **A15:** containment lived only at OS-agnostic layers — the alpha.1 write-layer policy gate (`policy_hook.py`, ADR-22 "permissions enforced per-profile at the write layer") and the MCP-only L5 sandbox; no network namespace, no egress broker, no "Guard" anywhere in alpha.1→alpha.15. **B1:** the supervisor/worker split with brokered egress — engines get no sockets, all egress is a typed IPC call to Guard against a compiled-in allowlist — is the OS-independent form of the charter's allowlist bar; on Linux the worker *additionally* runs in a `CLONE_NEWNET` netns with no interfaces (cheap, kernel-enforced), while the seccomp fallback, macOS `sandbox-exec` profile, and Windows-Firewall/UAC flow are **cut**, with "on macOS/Windows, no OS egress enforcement" recorded as an honest-residual (design §10 lines 904–913, 959; SECURITY.md, per verify). ⇒ **Gap:** network-level OS egress enforcement is entirely new in B1 and gated to Linux only. Per review §14 (finding 19), this downgrade is *only* the OS-level kernel layer — §10 rates the cut macOS/Windows sandboxes as "near-zero enforcement" — not the containment model; the delta is "OS-enforced kernel egress is Linux-only," not "full security only on Linux."

- **Facet B — cross-platform app: per-OS keychain + code-signed Windows builds.** **A15:** no user-held API key as a first-class secret (model routing was a Hermes/OpenRouter deployment concern, ADR-30), so no keychain/keyring/secret-store abstraction existed anywhere in the alpha line (design-history §14). **B1:** BYOK keys live in a per-OS keychain (Keychain / DPAPI / Secret Service + a mode-0600 fallback with logged warning on Linux; "the key exists only in the keychain and Guard's memory," design §10 lines 946–949), and Windows builds are code-signed for per-identity SmartScreen reputation (§16 lines 1169–1171; flip: package-manager-only channel if signing proves untenable). ⇒ **Gap:** B1 adds a per-OS credential store and code-signing — both B1-new — to satisfy the charter Key-handling bar (keys never in vault/backups/exports/logs, req line 65; research-evidence 362–365, Electron safeStorage + Claude Code behavior); A15 needed neither. Gap §12 already logs the keychain as a centered B1 addition.

- **Facet C — plugin is desktop-only.** **A15:** standalone CLI-first with Obsidian already demoted to optional adapter (alpha.12, ADR-122) and frozen behind an explicit scope gate — ADR-130's "alpha.15 cannot close with Obsidian plugin implementation counted as release scope"; future editor adapters "require their own ADR" (#1203/#1208, `d5131811`); installers "install no Obsidian plugin" (alpha.14). **B1:** the plugin is catalog-clean and desktop-only (`isDesktopOnly: true`, §16 line 1184) and is the only researcher UI, kept off mobile because it fronts the daemon path while heavy compute (PDF extraction, embeddings, NLI, egress brokering, DB) sits in a separate OS process for crash isolation, not Electron; §16 flags it amends ADR-125 and "requires that ADR before implementation begins." ⇒ **Gap:** B1 builds the plugin as a desktop-only adapter tier over the cross-platform engine — the "own ADR / next wave" work alpha.15's Scope Boundary punted forward. Per design-history this *continues* the alpha.12→15 demotion; it does **not** revert to the pre-alpha.11 era when Obsidian was the required substrate (correcting the gap Appendix's "Obsidian as required UI, re-introduced-in-B1" framing).

- **A15:** the minimum loop ran a mandatory standalone `work interview` elicitation step before digest. **B1:** no standalone interview — the subjective layer comes from a single-pass pinned digest call plus Zotero annotation sync + processing-card promotion; a lightweight 2–3-question "interview hook" survives inside the digest pipeline (if-design step 6), and the mandatory version is a recorded `docs-bar` Reject ("Mandatory ingest interview … duplicates annotations→notes"). ⇒ **Gap:** B1 drops the mandatory `work interview` elicitation loop (unrecorded in §9/§16 per verify-findings), retaining only a 2–3-question hook seeding note-composition.

## Appendix C — verification & completeness

### Method

The gap analysis was checked by three independent layers, each verified against the on-disk ground truth rather than against memory or an earlier draft — on the A15 side the consolidation ADRs 125–130 plus the disposition map, the git-recovered alpha.15 omnibus, the older ADRs, and the shipped `main/vault-template/` end-state; on the B1 side the beta.1 design §1–§18 and the beta.1 reports:

1. **Independent per-slice re-derivation.** The A15→B1 delta set was re-derived from scratch across eight subsystem slices (host/runtime/CLI/daemon; vault-layout/concept-model; provenance/frontmatter; store/catalog/snapshot/migration; ingest/extraction/enrichment; ask/compose/discovery/gap/cost; integrity/security/agent-judgment; surfaces/onboarding/reading/platform) and diffed against the report. This surfaced sub-mechanism gaps the report had not stated — 74 "missing" and 108 "partial" — before any triage for materiality.

2. **Adversarial critics.** Four critics attacked the report from different angles — cross-cutting completeness, accuracy-of-claims, appendix/internal-consistency, and source-coverage — each required to ground every objection in a re-opened source with a quote and to rate it high/medium/low. These produced the load-bearing corrections below.

3. **Review pass.** A separate reviewer audited all 16 sections bullet-by-bullet (each A15 / B1 / ⇒ Gap claim re-checked against the source), adversarially re-verified each finding (nitpicks dropped, thin claims downgraded to "plausible"), then ran a three-lens completeness sweep and an appendix/consistency pass. Verdict: **substantially accurate — every delta direction holds, no high-severity or load-bearing error remains.** The 19 residual findings are A15-side specifics (a field, quote, or install path attributed to the wrong alpha version or reworded), a few off-by-one section locators, one false cross-reference, and one appendix misclassification — all either corrected or below the threshold that changes a delta.
