# alpha.15 → beta.1 — gap analysis

Every material difference between the alpha.15 design (the alpha.15 consolidation ADRs
125–130 + the foundational alpha.14 catalog ADR-124 + the alpha.15
design/exec docs + the shipped `vault-template` end-state) and the beta.1 design
(`0.1.0-beta.1-design.md` §1–§18). **Gaps only** — each entry states the alpha.15
state, the beta.1 state, and the delta. No pros/cons, no recommendations.

Legend: **A15** = alpha.15 · **B1** = beta.1 · **⇒ Gap** = the delta.

---

## 1. Host / runtime / distribution

- **A15:** standalone CLI/engine, installed via `uv tool install memoria` (with `pip install -e .` the contributor fallback) + `memoria init`/`doctor`; runs with no Obsidian, no Hermes, no Zotero — the keep-test bar is that the corpus stays plain markdown a bare editor/file-browser can navigate. **B1:** an Obsidian plugin + `mem` engine, where "the Obsidian plugin is the only researcher UI." ⇒ **Gap:** B1 re-introduces Obsidian as the *required* researcher UI; A15 had removed Obsidian as a required surface.
- **A15:** "there is no internal daemon"; a scheduler (cron/systemd/launchd/Task Scheduler) invokes normal CLI commands with `--schedule-id`. **B1:** a resident two-process daemon (`mem serve` = supervisor + per-run worker) supervised by the plugin with LSP lifecycle, resident while Obsidian is open. ⇒ **Gap:** B1 adds a resident daemon; A15 explicitly had none.
- **A15:** the standalone CLI is the product. **B1:** the CLI (`mem`) is a "survival + ops" floor; `serve/capture/import/export/rebuild/backup/restore/migrate/revert-run/cost/hat`, **no `mem ask`**. ⇒ **Gap:** B1 demotes the CLI from product to survival floor and drops `ask` from the CLI (Ask is plugin-only); A15 shipped a verb-first CLI incl. `ask`.
- **A15:** installer installs no Obsidian plugin, no Hermes profiles, requires no Zotero. **B1:** ships an Obsidian plugin (catalog-clean, desktop-only) with a consented/package-manager binary-acquisition flow, code-signed Windows builds. ⇒ **Gap:** B1 adds a plugin distribution + binary-acquisition story A15 had removed.
- **A15:** distributed as a **Python 3.12 package** (`uv tool install` / pip; ADR-125). **B1:** distributed as a **single-static-binary** acquired via a package manager or a checksummed, version-matched download (design §7/§16). ⇒ **Gap:** the distribution artifact changes from a Python package to a single static binary.
- **B1 §11 itself records:** the "host decision (Obsidian plugin + local daemon)" is listed as **new alpha.16 scope** that "alpha.15 … does not yet center." ⇒ **Gap:** the host/runtime pivot is a beta.1 addition on top of alpha.15.

## 2. Vault structure / bundle layout

- **A15:** one bundle root `knowledge/`; `folders.yaml categories = [knowledge]`; the shipped template also ships `catalog/{sources,entities}/` (with a generated `catalog/index.md`), `spaces/`, `journal/`, and `system/` as workspace folders (catalog *records* are SQLite, but the folder tree exists). **B1:** the user's existing vault with a `Memoria/` **subfolder** beside unrelated existing notes; `Memoria/{inbox,works/,notes/,projects/,reports/,system/,bibliography.bib,local-only/}`. ⇒ **Gap:** B1 nests Memoria as a subfolder inside an unrelated vault (two sibling roots), and drops the top-level `catalog/`, `spaces/`, and `journal/` folder trees A15 shipped (records/navigation/logs move into the DB and `Memoria/` projections); A15's workspace *was* the bundle.
- **A15:** catalog sources/entities are **SQLite records**, not markdown (ADR-124/126); `knowledge/works/<id>/` holds the subjective `work` Concept only. **B1:** per-work folder `works/<citekey>/` co-locates `record.md` (catalog record, full CSL-JSON), `fulltext.md` (extracted text), `digest.md` (LLM digest), `source.pdf` (blob). ⇒ **Gap:** B1 re-materializes the catalog record as markdown and co-locates DB-projection + concept + extracted-text + blob in one folder; A15 had the catalog in SQLite and no `record.md`.
- **A15:** `knowledge/works`, `knowledge/notes`, `knowledge/hubs`, `knowledge/projects`; `knowledge/digests` removed (digest folded into `work`). **B1:** `notes/`, `projects/<slug>/{project,evidence,gaps,draft,export}`, `reports/`, plus `works/…/digest.md`. ⇒ **Gap:** B1 has no `hubs/`; B1 splits project artifacts into separate files (evidence.md/gaps.md/draft.md); B1 restores a per-work `digest.md`.
- **A15:** `inbox/` is a **reserved transient prefix** for machine-emitted attention projections/prompts (`folders.yaml transient_prefixes: [inbox/]`; AGENTS.md "`inbox/` transient attention projections") — a declared prefix, not a physically shipped folder; `system/` ships as a rich tree (`templates/`, `dashboards/`, `eval/`, `logs/`, `exports/`, `patterns/`, + a `metrics/` skeleton dir); no `reports/`. **B1:** `inbox/` (import **staging** drop folder — mints only via the awaiting-import chip), `reports/` (machine report notes), `system/` (manifest.jsonl, incidents/, metrics/ only). ⇒ **Gap:** `inbox/` is **repurposed** (attention-projection prefix → capture-staging), not re-introduced; `system/` is **repurposed** and its `templates/`/`dashboards/`/`eval/`/`patterns/`/`logs/`/`exports/` subtrees are dropped (only `metrics/` survives); only `reports/` is genuinely new.
- **A15:** the portable bibliography ships as top-level `references.bib`; no folder-level local-only marker. **B1:** the bibliography is `Memoria/bibliography.bib` (renamed), and a new `Memoria/local-only/` folder is a folder-level local-only marker. ⇒ **Gap:** `references.bib` → `bibliography.bib` rename; `local-only/` is a new folder-level marker.

## 3. Concept / type model

- **A15:** four knowledge Concept types — `note` (with `mode: claim|question`), `work`, `hub`, `project` — each with a per-type schema (`.memoria/schemas/types/*.yaml`). **B1:** a flat frontmatter `type` **class enum** `work|fulltext|digest|note|evidence|gaps|project|draft|ruled-out|export|report|incident|conflict`, with no per-type schema files. ⇒ **Gap:** B1 replaces the four typed-concept schemas with a flat file-class enum; per-type frontmatter+body schemas are absent.
- **A15:** `hub` is a first-class type (owns exactly one ID-backed controlled-vocab tag; membership = where the checked tag appears). **B1:** no `hub` type; no hub concept anywhere. ⇒ **Gap:** the hub type is dropped.
- **A15:** `note` carries `mode: claim|question`, `question_status: open|resolved`, claim-context fields (`temporal_scope`/`tense`/`qualifier`), and a full claim substrate (`claim_text`, `quote`, `source_id`, `evidence_set`, `extraction_confidence`, `annotation`, `anchors`). **B1:** `note` = "atomic + fleeting notes (human)"; no `mode`, no claim/question distinction, no question_status. ⇒ **Gap:** B1 drops the note claim/question mode and claim-context fields.
- **A15:** `project` is a typed concept absorbing the argument graph (thesis role, outcome_frame, paper_plan). **B1:** a `project/` folder of files (project.md/evidence.md/gaps.md/draft.md), `project` is one class value. ⇒ **Gap:** B1 has no argument-graph project concept (see §8).
- **A15:** ULID `id` identity kind ("identity ≠ filename ≠ title"). **B1:** identity is `citekey` + file path (works keyed by citekey; no ULID). ⇒ **Gap:** ULID identity is dropped; identity is path/citekey-derived again.
- **A15:** `links:` is a nested relation-typed frontmatter map validated against `{supports, contradicts, extends}`. **B1:** links are Obsidian wikilinks in body + a DB `links` backlink table + `%%prov%%` block markers; no relation-typed frontmatter links map. ⇒ **Gap:** typed authored relation links are dropped from frontmatter.
- **A15:** a controlled-vocabulary substrate — `system/vocabulary.md` is "the single source of truth for the controlled values" with faceted vocabularies (`research_area` ~30 curated terms, `methodology`, note `topics`); a `vocab` command mints/manages ID-backed tag IDs; the `classify` operation applies a `research_area` from OpenAlex topic scores (floor 0.6 / near-tie 0.15). **B1:** no controlled-vocabulary/faceting facet at all — no `tags` facet, no `vocabulary.md`, no `research_area`/`methodology` facets, no `vocab` command, no `classify` operation. ⇒ **Gap:** the entire controlled-vocabulary / faceting model is dropped (this doc otherwise only touches the hub's single vocab-tag).

## 4. Frontmatter discipline

- **A15:** universal meaning-only frontmatter on **every** Concept (`id`/`type`/`title`/`tags`/`links`/`aliases`/optional `archived` + per-type), enforced in product code; alpha.14 "enforce universal concept frontmatter." **B1:** frontmatter appears **only on Memoria-created files**; user notes need none; "absence of all `mem-*` markers = human." ⇒ **Gap:** B1 drops universal-frontmatter-on-every-concept; human/freehand files carry none.
- **A15:** closed validation — `validate_frontmatter` **rejects undeclared fields** ("put custom data under `x`"); `x: map` escape hatch on every type (PR-AC #1197). **B1:** no reject-undeclared validator; a freehand file without frontmatter surfaces a one-tap proposal, "flags never block." ⇒ **Gap:** B1 drops the closed/reject-undeclared frontmatter validator and the `x.*` extension namespace.
- **A15:** supersession/verdict state (`standing`, `check_status`, `note_status`) was moved out of frontmatter to the DB (meaning-only). **B1:** re-introduces researcher-set `superseded`/`superseded-by` (and `reading`) as frontmatter keys. ⇒ **Gap:** B1 returns researcher-set supersession state to frontmatter that A15 had moved DB-side.
- **A15:** verdict/stage fields (`check_status`, `standing`, `note_status`) removed from frontmatter (meaning-only). **B1:** meaning-only is kept for machine state (stage state DB-side), but provenance/verdict is expressed via in-file `%%prov:h|m|mc%%` block markers + `mem-*` frontmatter keys. ⇒ **Gap:** B1 re-introduces in-file provenance/verdict markers (`mem-prov`, `mem-taint`, `mem-flags`, `%%prov%%`) on machine files; A15 kept verdicts DB-only with no in-file marker.

## 5. Integrity / review / gate model

- **A15:** quarantine-and-verify (**ADR-127**); **"checked ≠ approved"** (ADR-128); the review-queue **approval** surface and the honesty-card inbox-as-Obsidian-category mechanism were removed, **but the disposition surface was kept** — ADR-128 keeps "a consequence-routed worklist … arguments not verdicts; batch worklists; graded loudness" and ADR-130 keeps the span-first Elicit table with one-click `--apply`. **B1:** a **Review** surface (dedicated main-pane view, ribbon pending count) with batch worklists (promote/link/drop/snooze; apply/reject/defer) and one-click revert; annotation "processing card" triage. ⇒ **Gap:** *not* a re-introduction — B1 renders in Obsidian the same consequence-routed worklist A15 kept (only the *approval* semantics were dropped); beta.1 §11 itself lists "the Elicit-style span-first review surface" under **Converges with alpha.15**. The real delta is surface form (Obsidian main-pane view + ribbon count) plus the added one-click revert.
- **A15:** no-write-time-oracle (ADR-128). **B1:** keeps no-write-time-oracle (export gate "blocks, does not edit"), and adds a block-level human-content read-only rule + human-span hash manifest with a **pre==post byte gate and auto-rollback**. ⇒ **Gap:** B1 adds the pre==post manifest gate + block-level byte-immutability + auto-rollback (B1 §11(4) states this is stronger than A15's propagation-only demotion).
- **A15:** verdicts DB-only, no in-file marker. **B1:** in-file `%%prov:mc%%` marker is display shorthand; green requires marker + current block hash + DB `checks` row to agree. ⇒ **Gap:** B1 adds the in-file marker layer (kept as a claim, not authority) that A15 did not have.
- **A15:** ADR-127 carries a first-class DB **derivation graph** (consumed-at-verdict on every edge) driving **two-tier demotion propagation** — depth-1 machine consumers demote; depth-≥2 machine descendants go stale (still served, excluded from correctness-bearing consumers); human descendants get ask-routed flags and are never auto-destroyed; retraction propagates one level deeper. **B1:** per-block hash divergence (edit under an `mc` marker → amber + re-check) + footprint-scoped DB reconciliation on revert; no source-changes-so-consumers-demote cascade, no derivation graph. ⇒ **Gap:** the derivation-graph + two-tier stale/invalid demotion-propagation mechanism has no B1 analog and is dropped.
- **A15:** ADR-128 makes dispositions first-class **non-backfillable telemetry** with an **apply-rate rubber-stamp detector** (apply-rate persistently >0.9 or <0.2 flags the surface). **B1:** captures dispositions at the review action (accept/reject/defer; accept-unedited vs accept-after-edit) but ships no apply-rate rubber-stamp detector. ⇒ **Gap:** the apply-rate rubber-stamp / disposition-calibration detector is dropped.

## 6. Machine judgment / LLM allocation

- **A15:** layered judgment (ADR-129): a **Tier-1 NLI comparator** + a **Tier-2 LLM tension judge** (retrieval-grounded, abstain-first, temp 0, quote-grounded, proposer-never-authority, admissible on Tier-1 abstain/hard cases; on HANS-degraded inputs some candidate routing goes to *ask* rather than the judge). **B1:** LLM call roster = digest, seeding relevance prefilter, paid synthesis, + local embeddings/NLI; contradiction detection = NLI within the retrieved set. ⇒ **Gap:** B1 has no Tier-2 LLM tension/contradiction judge; contradiction surfacing is NLI-only within the retrieved set.
- **A15:** enrichment merge by **static field authority** — per-field best-source-wins with provenance, references DOI-deduped union, `provider_coverage=full|partial|degraded`, high-authority conflict raises attention; new `field_provenance`/`enrichment_runs`/`provider_payloads` tables (ADR-129). **B1:** OpenAlex singleton fills **empty fields only** (JabRef rule; conflicts render as per-field diffs through Review, never auto-applied); raw responses in a `payloads` store. ⇒ **Gap:** B1 has no per-field authority merge, no coverage=degraded state, no `field_provenance`/`enrichment_runs` tables; enrichment is empty-field-fill only.
- **A15:** shadow-first calibration generalized (every score reports/journals but changes no routing until grounded on recorded PI dispositions; `production_enabled:false` semantics). **B1:** NLI thresholds tuned on a small pre-registered calibration set then **frozen**, gated by a HANS/PAWS launch-blocking acceptance test. ⇒ **Gap:** B1 uses per-engine frozen calibration sets, not the generalized shadow-first-until-grounded discipline; no `calibration.yaml` floor framework.
- **A15:** operations declare `method_class` (deterministic/hybrid/generative) and runner policy or fail loading; operations are product-code capability manifests. **B1:** an "operation test" (pinned prompt+model, deterministically checkable, arithmetic price, per-item corpus-scale) decides which LLM calls ship. ⇒ **Gap:** B1 replaces the per-operation `method_class`/manifest declaration with a prose "operation test"; no capability-manifest registry.
- **A15:** ADR-129's gap engine "reads checked inputs only" and scores on **fixed {0,1,2} ordinals**, emitting a project **saturation** band. **B1:** gap analysis is retrieval-coverage counting with a printed denominator (§8); no fixed-ordinal checked-input scoring, no saturation band. ⇒ **Gap:** the {0,1,2}-ordinal checked-input gap scoring + saturation band is dropped.

## 7. Catalog / bibliographic authority

- **A15:** the SQLite catalog + source-content blobs are the **citation authority** (ADR-124); Zotero/BBT are optional; `work_id`/provider IDs are primary runtime keys, citekeys are aliases; `references.bib` generated from SQLite. **B1:** `record.md` holds **full CSL-JSON** as the fidelity/re-import authority; citekey **pinned at import** (inherited from Better BibTeX); `bibliography.bib` generated from stored CSL-JSON. ⇒ **Gap:** B1 moves bibliographic authority back into markdown (`record.md`) and makes the citekey the primary key; A15 made SQLite the authority with `work_id` primary.
- **A15:** "Memoria does not require a live Zotero library" (Zotero optional). **B1:** Zotero import is day-1; reading is **Zotero-first** (Zotero's PDF reader is the primary read, annotations sync back); PDF acquisition via a "capture in Zotero" handoff. ⇒ **Gap:** B1 re-centers Zotero as the primary reading/capture surface; A15 demoted it to optional.

## 8. Discovery / gap analysis

- **A15:** gap analysis is argument/knowledge-graph based — `note` modes create obligations the gap engine feeds on (a claim without evidence = an `under-warranted` gap; an open question = a compose lead); `project` carries the argument-graph **saturation** floor (from ADR-79); OpenAlex Topics/controlled facets are part of the read model for gap analysis. **B1:** gap analysis = **retrieval-coverage** gaps itemized by cause (unsearchable/partial/superseded) + content gaps (question facets with corpus df 0), with a printed denominator. ⇒ **Gap:** B1 has no argument-graph/saturation/thesis-health gap analysis and no note-mode-driven gap obligations; gaps are retrieval-coverage-based only.
- **A15:** discovery informed by the imported topic/concept/keyword/citation graph. **B1:** discovery = bounded **citation-graph only** (snowball depth ≤2 + scheduled citation monitor), identifiers-only egress; output never enters gap/coverage denominators. ⇒ **Gap:** B1 discovery is citation-graph-only; no topic/concept-graph-driven discovery.
- **A15:** discovery is **steered** by the researcher — the shipped `steering.md` is a first-class "program memory" priorities file (open questions / synthesis gaps / papers-to-prioritize), and a named agent persona, **"The Librarian,"** reads it at session start and weights discovery by it ("the main lever you have over what gets surfaced"); shipped `system/eval/` rubrics score the Librarian's behavior. **B1:** no `steering.md`, no Librarian, no PI-priority weighting; discovery is deterministic occurrence-counting over the citation graph (the per-batch yes/no LLM relevance prefilter is a **seeding/onboarding** step, §15, not part of discovery). ⇒ **Gap:** the steering-lever + Librarian-persona discovery substrate is dropped entirely (see also §11 — this is the concrete agent role removed, which §11 otherwise frames only as "no resident Co-PI").
- **A15:** ADR-95 (nightly/always-on discovery loop) was **rejected** — "an unattended always-on loop conflicts with the no-daemon architecture" (disposition map). **B1:** a resident daemon (§1) **and** a scheduled citation monitor (§8) — both halves that rejection rested on. ⇒ **Gap:** B1 reverses the ADR-95 rejection once the no-daemon premise is removed; the charter exclusion is amended.

## 9. Ingest / enrichment graph

- **A15:** DOI enrichment imports the **first-order citation/reference/topic graph**: OpenAlex `concepts`/`topics` (→ `research_area`) and `authorships` + institutions, plus Crossref `relation` rows, plus BibTeX/CSL reference edges → `work_graph_edges`. **B1:** stores outgoing reference edges + capped cited-by edges (`edges` table); OpenAlex fills empty metadata fields; Crossref used for retraction check + title fallback. ⇒ **Gap:** B1 imports a much narrower graph (refs + capped cited-by only) — no OpenAlex topic/concept/authorship edges, no Crossref relation edges.
- **A15:** extraction via the ported ingest subsystems. **B1:** extraction is **pdfium-only, no OCR** (`extract_failed:needs_ocr`); text-coherence gate; deterministic distrust flags (`truncated`/`abstract_only`). ⇒ **Gap:** B1 fixes a single extractor (pdfium) and explicitly ships no OCR.
- **A15:** the canonical minimum loop is `work add → work enrich → work interview → work digest` — an interactive elicitation step that builds the subjective work before digesting. **B1:** no interview — the subjective layer is a single-pass pinned `digest` LLM call plus Zotero annotation sync. ⇒ **Gap:** the `work interview` elicitation flow is dropped.
- **A15:** enrichment ships **Semantic Scholar** as a conditional (default-on-when-keyed) provider in the reference/citation-intent union (ADR-129). **B1:** no Semantic Scholar; enrichment is OpenAlex (+ Crossref for retraction/title). ⇒ **Gap:** the Semantic Scholar provider is dropped.

## 10. Surfaces / navigation / control

- **A15:** verb-first CLI (`note`/`work`/`hub`/`project`/`ask`/`operation`); `status --json` replaces dashboard glance surfaces; no board daemon; `knowledge/_views/` generated reports. **B1:** 5 Obsidian surfaces (Capture/Reading/Ask/Compose/Review), 2 rich panes (Ask, Review), a read-only inspector pane, `ViewSpecRenderer` over `view-spec.v1`, Bases for meaning-only views, `index.md`+`log.md` projections. ⇒ **Gap:** B1 builds an Obsidian pane/surface UI + a generative-UI view-spec renderer; A15 was CLI-first with `status --json`.
- **A15:** no hub navigation surface. **B1:** no hub navigation surface either — **but** B1 also has no navigation-spaces/hub concept at all. ⇒ **Gap:** neither ships hub navigation; B1 additionally lacks the hub concept that A15 retained.
- **A15:** Canvas (projected-canvas, ADR-103) was **not shipped** in the alpha line. **B1:** likewise no Canvas / JSON Canvas. ⇒ **Not a delta** (continuity): Canvas is absent in both — noted only to forestall re-listing it as a removal.
- **A15:** ships a whole navigation layer — a `spaces/` tree (`inbox`/`library`/`knowledge`/`project`/`maintenance`, each `projection: space` with embedded Bases like `![[knowledge.base#Works]]`), a `_nav.md` "Navigator" (Now / Places / Actions), and a "three places" mental model (Library / Knowledge / Project) in `home.md`; the **`spaces/` naming is governed by ADR-101** (kept accepted), while the `_nav` Navigator and three-places framing are shipped-template conventions rather than ADR-101. **B1:** none of it — five interaction-mode surfaces (Capture/Reading/Ask/Compose/Review) replace the places/queues model; no `spaces/`, no Navigator, no three-places framing, no Friday-ritual maintenance cadence. ⇒ **Gap:** the ADR-101 spaces + Navigator + three-places navigation model is dropped (this doc otherwise mentions "navigation-spaces" only in a hub parenthetical).
- **A15:** the shipped retrieval substrate is **qmd** — the §10 spike verdict recorded that sqlite-vec + local embeddings did *not* clear the bar, so "qmd remains the alpha.15 baseline" (FTS5 cleared lexical only); `.memoria/index/qmd/` ships. **B1:** in-DB **FTS5 + sqlite-vec + bge-small ONNX embeddings + deberta NLI**, no external retriever — exactly the substrate A15 declined, with qmd dropped. ⇒ **Gap:** a concrete qmd → sqlite-vec+embeddings substrate swap that beta.1 §11 (and elsewhere this doc) treats only as convergence (qmd otherwise surfaces only as a §12 security aside and in the appendix).

## 11. Agent / autonomy

- **A15:** ADR-130 read-API + agent-as-quarantined-client contract; operations are product code. **B1:** "Memoria is not an agent, and beta.1 adds none"; any agent (first-party or the researcher's Claude Code/Cursor) is a **client** under the same contract (read-API + request envelope, quarantined, read-scoped, never file/terminal/exec/send); chat additive, never primary. ⇒ **Gap:** *convergence, not divergence* — ADR-130's **body** already defines the agent exactly as B1 does ("Co-PI" appears only in the ADR-130 *filename*, never as a resident first-party agent; beta.1 §13 grounds this agent boundary in ADR-125–130 as continuous with alpha.15). The genuine A15→B1 removal is the shipped **Librarian** persona (see §8), a user-facing agent role the vault-template actually shipped.
- **A15:** ships a `.memoria/plugins/memoria-policy-gate/` hook plugin providing `pre_tool_call`/`post_tool_call` hooks that route any external-adapter vault write through the runtime policy core, **fail-closed** ("any error blocks the write") — A15's mechanism for forcing a BYO agent/adapter's writes through the envelope at the tool-call boundary. **B1:** no `.memoria/plugins/` tree, no per-tool-call hook; machine writes go through the daemon's single **VaultWriter**, agent writes through the request envelope VaultWriter applies. ⇒ **Gap:** the policy-gate hook system is dropped, superseded by single-VaultWriter + envelope.

## 12. Security / OS confinement

- **A15:** network access operation-scoped + allowlisted; injection-as-untrusted-data; capability import quarantined; standalone **Python 3.12 / uv / Typer** runtime (requiring Node ≥22 only for the `qmd` search tool), platform-agnostic. **B1:** brokered egress via Guard (engines get no sockets) + **Linux network-namespace (`CLONE_NEWNET`) confinement only**; seccomp fallback, macOS `sandbox-exec`, and Windows-Firewall/UAC flows are **cut**; on macOS/Windows egress restriction is "process-architecture discipline, not OS enforcement." ⇒ **Gap:** B1's OS-level egress enforcement exists only on Linux; macOS/Windows OS-sandbox profiles are cut. (See also §14 — platform.)
- **A15:** no exfiltration-linter-at-vault-write detailed. **B1:** VaultWriter-owned exfiltration linter (default-deny link/embed/HTML neutralization) at apply + export; canary drill; per-OS keychain (Keychain/DPAPI/Secret Service + mode-0600 fallback). ⇒ **Gap:** B1 adds the write-time exfiltration linter + canary + keychain machinery as a centered mechanism.
- **A15:** no third-party-plugin-interference threat (standalone engine, no Obsidian). **B1:** names **third-party-plugin interference** a security corruption class — format-on-save plugins rewriting machine spans, secret-shaped fields in plugin `data.json` — with startup detection + formatter-exclusion (design §10). ⇒ **Gap:** a new Obsidian-plugin-interference threat class appears in B1 (a consequence of re-hosting on Obsidian).

## 13. Cost / snapshot / migration machinery (new-in-B1)

- **A15:** cost/measurement present but not the centered machinery. **B1:** BYOK; a normative price table in Ledger; `max_tokens` ceilings; batch API (50%); circuit breaker enforced at preview/pre-call/breaker; $20/mo default soft budget; "miss row" for unattributable spend. ⇒ **Gap:** B1 centers a full cost/metering/breaker machinery.
- **A15:** cross-store recovery + journal. **B1:** git external git-dir + `VACUUM INTO`; the 4-step **batch protocol** (snapshot/apply/verify/commit); pathspec-limited commits; one-click revert with inverse-diff + DB reconciliation. ⇒ **Gap:** B1 specifies a git-snapshot batch/revert substrate distinct from A15's SQLite-journal cascade-rollback.
- **A15:** no migration runner centered. **B1:** a **MigrationRunner** named as a beta.1 blocker ("beta is the first version whose data must survive later versions"). ⇒ **Gap:** B1 adds a migration runner as a launch blocker.
- **A15:** recovery (ADR-127) includes a git-tracked append-only `.memoria/overrides.jsonl` human-disposition log with **tiered DB-loss replay** — restrictive dispositions (archive, contested-block) auto-reapply; permissive ones (degraded acceptance, retraction resolution) return as attention items to re-confirm — plus a **SQLite streaming replication (Litestream-class)** leg in the A15 three-store backup contract. **B1:** researcher decisions live in frontmatter (`superseded`/`reading`) + an exported decide-events CSV; §18 explicitly **declines** DB streaming-replication for beta.1 (VACUUM INTO snapshots + `restore --verify` only). ⇒ **Gap:** `overrides.jsonl` + the tiered restrictions-reapply/permissions-re-earn replay is removed, and the streaming-replication backup leg is dropped.
- **A15:** tamper-evidence via a **row-hash-chained append-only journal** (ADR-127), with the git-anchored `.memoria/journal-head` root contributed by ADR-104. **B1:** the cascade-rollback journal is replaced by the git-snapshot batch protocol; the hash-chain + `journal-head` tamper-evidence root is not carried. ⇒ **Gap:** the cryptographic tamper-evidence apparatus is dropped (this doc's §13 covers rollback, not this).
- **A15:** the derived SQLite DB and provider payloads live **in the synced vault** (`.memoria/…`). **B1:** the DB (and payloads) move **out of the synced tree** to a per-device OS data dir (`~/.local/share/memoria/<vault-id>/memoria.db`), with `<vault-id>` namespacing. ⇒ **Gap:** the derived store is relocated out of the vault to a per-device path (distinct from the external git-dir already noted above).

## 14. Platform
