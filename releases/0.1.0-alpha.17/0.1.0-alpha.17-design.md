## 0. Identity — what alpha.17 is

alpha.16 gave the researcher the **READ** half: capture → digest → notes/claims
→ gap analysis → grounded Ask, over the two-graph substrate. alpha.17 adds the
first half of **WRITE** — **producing verified TEXT output and closing the loop
back into the graph** — plus the two cross-cutting instruments the mission needs
early (an anti-suppression exploration channel and gate-efficacy measurement).

**The ruling scoping principle (owner directive): include everything we can
build now; defer only what we can't.** Applied honestly, the line falls in one
place — the **host/stack decision is unmade**, so the two features that depend
on it defer, and *everything else ships*:

- **In alpha.17 (design-resolved + host-neutral):** the writing spine
  (canvas → outline → draft), draft→graph write-back, text output verification,
  a minimal compose surface, the exploration/anomaly channel, and
  gate-calibration instrumentation.
- **Deferred (host-gated only):** code **execution** (needs sandbox hardening +
  adversarial validation) and the **full multi-gate workspace** (an unresolved
  design task) — both downstream of the **host/stack CRITICAL decision**
  (alpha.16 §16.1). See §6.

Why text escapes the host gate: text output runs entirely through alpha.16's
**host-neutral read-API + plain files** (`memoria ask --json`; `.md`/`.canvas`
artifacts). It neither renders a workspace nor executes code, so it is
buildable against any eventual host. The **keep-test holds throughout** — every
artifact is a plain file usable in `vim`/`nano`/a file browser.

All alpha.16 guardrails carry unchanged: **no-write-time-oracle** (checked ≠
approved), **no synthetic index**, **propose-not-dispose**, **graph *structural*
integrity** (not per-node semantic validity), single **VaultWriter** +
snapshot→apply→verify→commit + one-click revert.

## 1. The writing module — text output

The owner's named flow, made concrete: **argument graph → canvas → outline →
draft (TEXT)**. Every stage is a *proposal surface*: the system proposes with
its reasoning shown; the human — sole owner of judgment — disposes
(beta.1 §1).

### 1.1 Argument graph (consumed, not built here)

The writing module reads the alpha.16 argument graph: **claims-with-context**
carrying TEMPO fields (`temporal_scope`, `tense`, qualifier/certainty,
originating section — a claim's truth is determinate only relative to these),
connected by human-authored typed edges. Saturation is expressed as **honest
coverage reasoning** (which claims lack support, which lack a counterpoint) —
**never a synthetic saturation score** (anti-synthetic-index). This is the
substrate the thread is drawn from.

### 1.2 Canvas — the spatial sketch that connects notes into a thread

The **canvas** is the project-scoped spatial sketch connecting the relevant
notes/claims into a *thread* (rethink Class B / Q7):

- **Small-graph, offered alongside — never instead of — a list/table.** Spatial
  layouts degrade as density grows and node-link readability collapses above
  **~20 nodes**; a **list/outline is always the primary-or-fallback surface**,
  offered automatically past the threshold (Ghoniem 2004; Hearst 2009). The
  canvas is scoped to the *project slice* of the argument graph, not the vault.
- **Positions are stable, persisted, user-owned** — never auto-relayout of
  existing nodes; moving nodes destroys the mental map (Misue 1995). Positions
  are governed state, reversible via snapshot/revert.
- **Infer, don't force.** The system *suggests* which notes/claims belong and
  how they connect (with per-item reasoning); it never forces tags/edges upfront
  (premature formalization makes users reject the system — Shipman & Marshall
  1999). Candidate retrieval is **BM25+dense, never graph traversal** — typed
  edges are the *argument* layer, not the retrieval representation.
- Format: a plain `.canvas` (JSON-Canvas) file, editable without any plugin.

### 1.3 Outline — one-way derivation from the canvas

**The outline is a re-runnable, one-directional derivation from the canvas — not
a live two-way sync** (rethink Class B / Q6). Genuine two-way sync exists only
when map and outline are the *same node tree*; a free spatial arrangement →
linear order is a lossy many-to-one inference (spatial-hypertext linearization;
Shipman & Marshall). The canvas is the structural source of truth; the outline
is **derived, then human-owned** — edits stay local and do not propagate back.
Re-derivation is **explicit/opt-in**, supersedes the prior outline as a
reversible new version, and warns it discards outline-local edits.
*Measurement residual (§7):* whether heavy outline editing creates real demand
for back-propagation — measured, not assumed.

### 1.4 Draft — the TEXT output

The draft is composed section by section from the outline. It is the **output
artifact**, not a write into the graph. Synthesis is where the research bites
hardest, so the rules are concrete (beta.1 §1.4 + rethink Class B / Q8):

- **Feed coherent slices, not top-k fragments.** Each outline node maps to its
  canvas notes/claims; the **slice builder** emits: the **whole note if it fits
  the per-node token budget**, else the **section-expanded anchor** with the
  section/document header prepended; the **strongest anchor at slice top or
  bottom** (never padding the middle — U-shaped "Lost in the Middle"
  degradation, Liu 2023); the **total slice capped** well under the model window
  (context is non-monotonic — Databricks 2024); a cheap **Self-Route** falls to
  fragments only for narrow factual nodes (Li 2024). Notes are indexed at
  section/whole-note granularity (LongRAG). This is the single most important
  synthesis rule.
- **Every number computed in code; every citation resolved deterministically.**
  A printed number is produced by the engine's code path, never the model
  (abstention + numbers-in-code is a pipeline property: 37.1% → 70.2%,
  Zou 2024). Citations resolve against the catalog's deterministic metadata to a
  **verbatim span**.
- **Warrant = a typed evidence-*set*.** Each drafted claim carries an ordered
  evidence set of **items** — each a **source-span ref** (`citekey#^pNNNN` or an
  `ev-<8hex>` pinned-evidence id) — from which the engine **derives** the `type`
  (*single-span* / *multi-span* / *multi-hop* / *implicit*) and never asserts it.
  `state` is *complete*, or *evidence-incomplete* iff the type is implicit or any
  item fails to resolve; incomplete **routes to the PI** (§2). Durable form: a
  trailing inline marker on the claim block (`%%ev: <id> type=… state=…%%`, same
  pattern as `%%prov%%`) **plus** an `evidence_sets` DB row (id, block_ref,
  items_json, type, state, run_id) — inline for portability, DB for
  query/rebuild. A span match is labeled **citation-correct, not grounded** (up
  to 57% of RAG citations are post-rationalized — Wallat).
  *(alpha.17 scope: evidence items are source-spans only. The **code-warrant**
  item variant — a computed claim warranted by a re-runnable script — belongs to
  code execution and defers with it, §6. A claim that would need a code warrant
  enters **evidence-incomplete** and routes to the PI, exactly as an implicit
  warrant does — no fabricated number ships.)*
- **Abstain rather than fabricate.** Where the corpus does not support a passage,
  the draft emits an explicit gap/"Unknown" marker; faithfulness comes from
  deterministic grounding, not the model *choosing* to say it doesn't know
  (Rao & Callison-Burch; CoT degrades abstention — Shaikh 2023).
- **Prose, not a forced schema.** The draft is natural language; constrained
  decoding to a rigid schema corrupts negation and polarity — the worst error
  class here (Shin 2021). Semantic content is generated as prose first;
  structured extraction (IDs, links, tags, citekeys — closed-vocabulary,
  machine-checkable slots only) is a **second pass**, never at the point of
  semantic commitment (rethink Q13).
- **Propose-not-dispose.** The draft is a proposal with per-passage reasoning;
  the human composes and judges. Drafting **does not auto-write knowledge back**
  into the graph — that is the §1.5 gated action.

### 1.5 Draft → knowledge-graph write-back

Drafting is a *producer* of notes, not only a consumer (a manuscript is the
rearrangement of notes; writing reveals notes that don't yet exist —
Zettelkasten/Matuschak). The write-back is a **PI-gated action, never automatic**
(no-write-time-oracle) (beta.1 §1.5 + rethink Q9):

- **Extract to an atomic, concept-oriented permanent note.** A first-class
  in-draft action turns a selected passage into a new `note` (claim/question) in
  the graph — atomic and concept-oriented, detached from any single source.
- **Transclusion, not copy.** The draft keeps a **back-link/transclusion** to the
  one canonical note — never a pasted copy that silently diverges.
- **Provenance + maturity.** The note carries a **link** to its source record
  (never inlined source text) and a **maturity marker** (`seedling`),
  distinguishing a raw draft-spawned extraction from a vetted permanent note.
- **Squeeze-point trigger.** The system *offers* extraction when a draft section
  accumulates many loose references (the "mental squeeze point") — an
  event-driven prompt, never an auto-write.

Promoted notes re-enter gap analysis like any note, closing the post-draft loop.
*Measurement residual (§7):* the squeeze-point detection threshold, and whether
`seedling` notes need a distinct review lane before counting as compose evidence.

### 1.6 Close the loop

Drafting surfaces new gaps — claims that abstained, evidence-incomplete
warrants, sections with no counterpoint. These feed back into alpha.16's **gap
analysis over both graphs**, so writing drives the next READ iteration rather
than being a terminal export.

## 2. Output verification — TEXT

The "verify" step of the cycle, consistent with **no-write-time-oracle** and
**graph structural integrity**. Verification is **never a semantic quality
oracle** — human+machine review does not beat machine alone and a wrong machine
verdict drags the reviewer *below* baseline (Jacobs 2021). It produces
**deterministic flags + PI routing**, and proposes; it does not dispose
(beta.1 §4).

- **Provenance labels — no label ever means "true."** `human`; `machine`;
  `mc`/`span-matched` (= **citation-correct, not grounded**); `contradicted`;
  `unchecked`; `tainted` (depends on machine/untrusted content — monotone, never
  clears); `stale` (evidence/source changed after the check). Semantic truth
  stays the user's judgment (beta.1 §4.0).
- **Deterministic citation/number checks (necessary, not sufficient).** Every
  cited claim's evidence set resolves: citekey/DOI exists, the verbatim span is
  present in the source, every printed number **recomputes and matches**. These
  prove *span existence and arithmetic*, **not grounding** — gates on form,
  never a certificate of truth.
- **Abstain / evidence-incomplete surfacing.** Passages that abstained or carry
  implicit/multi-hop (or would-need-a-code-warrant) warrants are flagged and
  **routed to the PI** — the exact claims that cannot be auto-resolved.
- **Graph structural integrity.** The draft's references into the notes/claims
  graph must resolve (links, anchors, references) — *structural* checks, not
  per-node semantic validity.
- **Sparse, triaged, tooling-first.** Auditing costs the same expertise as
  authoring and floods under load (Feng & Liu). Verification routes **only
  high-uncertainty / high-leverage** items to the PI, invests in per-claim
  tooling (side-by-side source, citekey+span diff), and treats a
  **max-items-per-session / triage threshold as a first-class constraint** — not
  a throughput target.
- **Export-readiness gate (extends alpha.16 §11).** Export **runs** on a clean
  draft and **refuses** when unresolved evidence-incomplete claims or broken
  structural references remain — the refusal is a first-class, explained output,
  not a silent pass. Targets start at **markdown + Pandoc**, expanded on measured
  demand (rethink Q14/F).

*(Code verification-by-execution — beta.1 §4.2 — defers with code execution, §6.)*

## 3. Minimal compose surface

The owner's split is design intent: **separate deep work (the project document)
from task-oriented work (reviewing links) — distinct mindsets, don't mix**
(beta.1 §3). alpha.17 ships the **Project** deep-work job as a **minimal compose
surface**, not the full multi-gate workspace (which defers, §6):

- **What it hosts:** the deep-work text artifacts — **canvas, outline, draft** —
  as plain files, plus the read-API panes (Ask, evidence, gap report) alpha.16
  already exposes. It deliberately does **not** surface the inbox/recommendation
  stream (that lives on the task-oriented gates — mindset separation).
- **Host-neutral by construction.** The surface is a thin convenience layer over
  files + the read-API, never a gate on them: the product runs with
  `vim`/`nano`/a file browser (the keep-test), and the surface makes **no
  commitment to the eventual host** — so it does not wait on the host/stack
  decision. Preserves the **no-daemon** posture.

The five UI-framework-independent jobs (Read / Knowledge / Project / Review /
Upkeep) remain the requirement; alpha.17 realizes **Project** minimally and
inherits the others from alpha.16. The *full* gate topology is §6.

## 4. Exploration / anomaly channel

A feedback loop trained on what the PI up-votes suppresses disconfirmations (the
McNamara fallacy: consensus-priored models converge and down-rank anomalies).
alpha.17 reserves a **separate exploration/anomaly channel not subject to the
relevance ranker** (beta.1 §6 + rethink Class D / Q10). It ships now because it
is host-neutral (a read-API slate over the existing citation graph) and it must
exist *before* the loop can start suppressing:

- **A separate, salient, labeled slate** — never blended into ranked results
  (Google exploration slates; McNee 2006). Relegating diverse items below the
  fold means they go unseen (Munson & Resnick 2010), so the slate is **salient
  and interpretable**: each item says *why* it's here (the "sagacity" half —
  André 2009), with a **"consider the contrary"** framing that redirects
  attention (Lord 1984).
- **Populated by coverage/diversity, not relevance** — submodular
  facility-location coverage (provable (1−1/e)) or DPP over citation-graph
  breadth (co-citation, Swanson-style cross-silo bridges); MMR as the cheap
  baseline. This **reuses alpha.16 §10 bounded citation-graph discovery**, not a
  new recommender.
- **A first-class contrary-evidence lane** — the local NLI already surfaces
  REFUTES within the retrieved set; extend it as a standing "contrary evidence"
  channel (SciFact framing). **An empty contrary list is never presented as
  agreement** (guardrail).
- **Measured as its own objective, validated by action** — intra-list diversity
  + subtopic/coverage + honestly-labeled serendipity (unexpectedness × usefulness),
  **never a synthetic score presented as truth**. Validated by whether the
  researcher **acts on** the slate; if measurement shows it is never acted on, it
  is theater and gets thinned (same discipline as the gate, §5).

*Honest caveat (carried from the rethink):* the filter-bubble harm is genuinely
debated (Bakshy 2015; Flaxman 2016) — this channel is **insurance, presented as
debated, not settled fact**, and no controlled A/B yet proves "separate region
vs blended" for a *noticing* outcome, so it is measured, not assumed.

## 5. Gate-calibration instrumentation

The load-bearing risk of the whole system is that **human review is theater** —
human+AI is frequently *worse* than the better of the two alone, losing exactly
when the AI is more accurate (Vaccaro 2024, n=106, g=−0.23). alpha.17 starts
measuring this **a release early**, because the WRITE half is the first place
the PI reviews machine *prose* at volume (rethink Class A / Q3-Q5; beta.1
§4.3/§6). It ships now because it is pure instrumentation over the existing gate
— host-neutral, no new surface:

- **Standing seeded-error probe.** Inject a known low rate of deliberately-wrong
  machine proposals into the normal Review stream; measure the reviewer's catch
  rate and false-alarm rate (d′). **If seeded errors aren't caught above chance,
  the gate is theater** — surfaced, not hidden (the operational Green & Chen
  reliability test; Jacobs incorrect-recommendation arm).
- **Agreement re-judging, both directions.** Periodically re-judge a sample of
  accept *and* reject decisions against ground truth; track the
  **correct→wrong flip rate** (automation bias) vs wrong→correct. Healthy gate:
  wrong→correct ≫ correct→wrong. Apply-rate bands (≳90% or ≲20%) flag
  rubber-stamping/disengagement — but only the re-judged sample is decisive
  (accept/override rate alone is uninterpretable — Felisberto 2024).
- **No persuasive explanations.** Proposals show the machine's **self-rebuttal**
  (argument-against) + coarse certainty, never a polished rationale — explanations
  raise acceptance without raising accuracy (Bansal 2021), and an
  interpretable-but-wrong explanation does the most damage (Jacobs).
- **Load caps as first-class constraints.** Review in **short bounded batches**
  (≤~60–90 min, ≤~a few dozen units — code-review defect detection falls after
  60–90 min, SmartBear/Cisco; vigilance decays in 15–30 min), never an open
  streaming approve-queue.
- **Disposition capture, implicit.** The loop records accept/reject against the
  proposal's existing impact/certainty/reasoning — **no separate rating chore**
  (Rogers 2006; Epstein 2020). Every tunable ships behind
  `production_enabled:false`: **no score changes routing, promotion, or
  visibility until grounded on recorded dispositions** (anti-synthetic-index as
  an operational rule).

The **ABAB efficacy protocol** (gate-on/off reversed ≥3×, ≥5 measurements/phase;
abandonment tracked as a first-class outcome) is the *analysis* run over this
data; alpha.17 lays the instrumentation, and the full n=1 efficacy verdict is a
beta.1-completion output (§6).

## 6. What alpha.17 defers (tracked debts, each with its gate)

Each is a debt against the MVP with a named gate — not a silent deferral. **All
three host-gated items trace to one root: the host/stack decision is unmade.**

- **Code execution (Blocker 1 — security gate).** Running project code (the
  executable-warrant and code-deliverable roles, beta.1 §2) needs a **hardened
  sandbox + adversarial validation** before it can face a real corpus, because
  ingested untrusted content can influence a run (InjecAgent; AgentDojo). The
  *minimal* sandbox is designed (no-network credential-free worker,
  project-write-scope **+ secret read-denylist** `~/.ssh`/`~/.aws`/keychain/
  provider-keys/DB, approved-commands, resource caps — rethink Class C), but its
  per-OS backend (bubblewrap / Seatbelt / WSL2) and the InjecAgent/AgentDojo-style
  adversarial validation are real, uncertain work. *Path:* build + validate the
  sandbox; then the code lane (beta.1 §2), code-warrant evidence items (§1.4),
  and code-by-execution verification (beta.1 §4.2) land together.
- **The full multi-gate workspace (Blocker 2 — an unresolved design task).** The
  gate triad (system/maintenance · sources/library · knowledge/ZK), the
  deep-vs-task surfaces beyond the minimal Project one (§3), left-pane-nav /
  right-pane-Co-PI, and the active-assistance model are an **owner
  job-to-be-done design that has not been done** — both alpha.15's carried model
  and beta.1's five-surface model are rejected as authority-only inputs
  (alpha.16 §16.1). You cannot implement an undefined design. *Path:* the
  workspace design pass.
- **The host / stack CRITICAL decision (Blocker 3 — the shared upstream gate).**
  Whether the host is an Obsidian plugin + SQLite or something else is unmade
  (Obsidian/SQLite were "chosen as defaults without evaluating fit"). The
  workspace is *rendered by* the host and code execution *runs in* its process
  model — so this decision gates **both** of the above. Text output does not
  depend on it (host-neutral read-API), which is why the two cluster into the
  next version behind this decision. *Path:* a first-principles stack evaluation
  → a committed choice; **this is the prerequisite for the next version**, not
  something to discover mid-implementation.

**Inherited alpha.16 follow-ons that remain open** (unchanged by alpha.17):
sqlite-vec fusion behind the retrieval benchmark gate; fuzzy annotation
re-anchoring behind a measured orphan rate; the QLoRA adapter as a tracked
contingency; advanced backup hardening. **Deferred to beta.1 completion, not
lost:** the full onboarding wizard + ≤30-min time-to-first-answer bar (beta.1
§5), the complete feedback dashboard + ABAB efficacy verdict (beta.1 §6), and
the end-to-end MVP acceptance run on a real corpus (beta.1 §7).

## 7. Open questions & measurement residuals

alpha.17 invents no mechanism to fill a gap. It **starts collecting** the data
these need:

- **Coherent-slice per-node token budget** — the threshold above which a note is
  section-expanded rather than passed whole (rethink Q8) — tuned on the real
  corpus; behind `production_enabled:false` until grounded.
- **Canvas ~20-node behavior** — does the list fallback trigger at the right
  density; do users re-derive the outline or diverge and never re-derive (Q6/Q7).
- **Evidence-set PI-routing fraction** — what share of drafted claims land
  evidence-incomplete (Q2/F) — a measurement, not a target.
- **Squeeze-point threshold** — when to offer draft→note extraction, and whether
  `seedling` notes need a distinct review lane (Q9).
- **Gate efficacy (the big one)** — the seeded-error d′, both-directions flip
  rate, and the ABAB verdict (§5) — the honest answer to "is the gate theater,"
  resolved by measurement, not assumption.
- **Exploration acted-on rate** — whether the slate is used or is theater (§4).

**The one decision that gates the next version:** the **host/stack choice**
(§6, Blocker 3). Resolve it and the deferred code + workspace bundle becomes
buildable; leave it open and text output still ships, unblocked, now.

## 8. Acceptance — the text-output loop, end-to-end, host-neutral

alpha.17 is accepted when, **on a real corpus, via the read-API + files (no host
UI required)**, the text-output loop runs end-to-end with no core item stubbed
(extends the alpha.16 acceptance list):

1. **Canvas thread** — the system proposes a project-slice canvas with per-item
   reasoning; the human arranges it; positions persist and revert; the list
   fallback appears past ~20 nodes.
2. **Derived outline** — one-way from the canvas; re-derivation is explicit and
   reversible and warns before discarding local edits.
3. **Draft (text)** — composed from coherent slices (whole-note-if-budget else
   section-expanded anchor, top/bottom placement, capped); **every number
   recomputes in code**; every citation resolves to a verbatim span; unsupported
   passages **abstain** rather than fabricate; each claim carries a typed
   evidence set.
4. **Verified** — deterministic citation/number checks pass or route to the PI;
   evidence-incomplete claims are surfaced; structural graph references resolve;
   export **runs on a clean draft and refuses** (with an explained refusal) on an
   unclean one.
5. **Write-back** — a draft passage promotes to an atomic `seedling` note via
   transclusion (not copy), with source-link provenance, and **re-enters gap
   analysis**.
6. **Exploration slate** — a separate, labeled coverage/contrary slate surfaces
   at least one item the ranker would suppress, each with a *why*.
7. **Gate instrumentation** — the seeded-error probe, both-directions re-judging,
   and disposition capture are live and producing honest data (no synthetic
   score).

*(No separate alpha.17 requirements/governing doc is created here per the
"design only" scope. If we split governance out later, this §8 is the seed for
its acceptance list — kept single-authoritative to avoid the drift alpha.16 §15
warns about.)*

## 9. Trade-offs

- **Whole-note slices cost tokens** — mitigated by the budget threshold + a
  Self-Route to fragments for narrow factual nodes. Flips toward fragments only
  where measurement shows the budget is routinely blown.
- **One-way outline** means outline edits don't propagate to the canvas — the
  price of not building unbounded two-way sync machinery with no proven need
  (the residual is measured, §7).
- **Write-back as a gated action adds friction** — the deliberate price of not
  laundering machine prose into the human note graph (no-write-time-oracle).
- **Measuring the gate spends real attention on seeded fake items** — kept at a
  low rate and counted as calibration, not work; the alternative is not knowing
  whether the gate does anything.
- **Deferring code + workspace means alpha.17 delivers half the WRITE mission** —
  the deliberate, honest consequence of the host/stack decision being unmade;
  the alternative (merging them in) would either block ready text output on
  security hardening + an unresolved design, or ship code/workspace unsafe and
  undesigned. Text output shipping now, unblocked, is the correct call.
