# Insights — closing the day's dossier — 2026-07-09

Cross-cutting observations from the full arc (harness audit → alignment
plan → product statement/axioms → promise audit → architecture review →
roadmap → workflow/onboarding → OKF/storage constitution → schema
analysis). Patterns across the day, not facts inside it.

1. **The day was a manual run of Memoria's own methodology — and it
   worked.** Adversarial audits produced findings; the owner corrected
   them from unwritten design knowledge; corrections became recorded
   doctrine; a coherence sweep caught six stale statements the
   corrections orphaned. Capture → interview → digest → integrity-lint,
   performed by hand on the product's own design. The axioms, kernel,
   bundle boundary, and placement doctrine did not exist in writing this
   morning; all came out of interrogation. Strongest empirical evidence
   yet for the co-PI pattern — the product proved its thesis before it is
   built. Open thread — closed same day into roadmap item 19: Memoria
   lives as a project inside Memoria (decisions as claims, principles as
   warrants, reversals as blast radius, design-history as exports);
   prerequisites are Tier 0 + two item-20 source types, nothing from
   Tier 1.
2. **One decomposition line resolved every hard fork.** Truth/grounding;
   judgment/method; brain/protocol; agent-loop/fenced-operation;
   knowledge/trust-state; authored/judged/derived; bundle/engine. One
   split: *the fluent, judging half stays with the human (or their
   agent); the structural, inspectable half goes into the engine.* This
   is axiom 2's generative form and now a design test: a fork that
   resists resolution probably hasn't been cut along this line yet.
   (Promoted to `product-statement.md` as the master pattern.)
3. **The schema is the most truthful document in the repo.** It knew
   `'work'` was a concept type while the API couldn't list works; carried
   `file_index_state` and `schedule_id` before the designs that need
   them; its triggers encoded the catalog bridge no doc mentions —
   while docs repeatedly described unbuilt things in present tense.
   Heuristic (promoted to the alignment plan's facts AGENTS.md): **when
   layers disagree, trust order is schema → tests → code → docs.**
   Systemic cure for the docs' disease is already scheduled: aspiration
   gets a legal home in superpowers `specs/`, making zero-contradiction
   satisfiable.
4. **The warrant ontology is an undissipated decision, not an open
   question.** Four independent weights accumulated on the node side
   (typed propagation, methodology inventory, OKF exportability,
   frontmatter evolvability); zero new weights edge-side. Schedule the
   ratifying brainstorm as the *first* design session after Tier 0 —
   keystone decisions left ambient are how graphs get built twice.
5. **The uncomfortable one: eleven documents, zero lived-in vault.** The
   promise audit found most prior plan effort went to process, and today
   has the same shape — justified (it extracted a constitution), but the
   marginal value of another planning day is now far below Tier 0 code or
   the Zotero import. Treat any future session that produces only
   `workflow-audit/` commits as a signal, not a success.
6. **OKF changed what Memoria is.** From "a tool" to "an engine over a
   standard." The durability pitch — the knowledge outlives the tool;
   `cat` works — is likely the product's strongest adoption sentence and
   a healthy design constraint: every feature must answer "does this
   survive in the bundle without us?"
