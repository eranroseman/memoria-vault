# Project records

These are **out-of-compass records** — they sit outside the four [Diátaxis](https://diataxis.fr/) modes ([tutorials](../tutorials/), [how-to](../how-to/), [reference](../reference/), [explanation](../explanation/)). Rather than documenting the system itself, they document the project around it: the decisions made, the road ahead, what ships today, and the alternatives weighed and set aside.

- **[decisions/](decisions/)** — Architecture Decision Records (ADRs), one per non-trivial design choice. The index is gap-free 1–30; ADRs 1, 8, and 11 are recorded **index-only** (resolved in the index table without a standalone file), and retired ADRs keep their files as history.
- **[roadmap/](roadmap/)** — the phased plan: timeline, deployment options, autonomy progression, success metrics, design tensions, and future directions.
- **[implementation-status.md](implementation-status.md)** — the shipped / accepted-pending / deferred ledger reconciling these docs with the starter vault; the source of truth for what exists today.
- **[evaluated-alternatives/](evaluated-alternatives/)** — tools evaluated and not adopted (or held as future-migration targets), kept on record so the reasoning isn't lost.
