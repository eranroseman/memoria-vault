# Release plan — v0.1.0-alpha.7

**Current status: complete (internal checkpoint, untagged).** alpha.7 is the
**clean-slate Obsidian UI / navigation** checkpoint. It landed the buildable subset of
the UI redesign: Bases view layer, schema-generated capture forms, four persistent gate
dashboards (Inbox/Library/Knowledge/Project), Portals folder navigation with the core
file explorer retained as fallback, and the Memoria-tuned Obsidian config + CSS tier.

Implementation landed in
[PR #677](https://github.com/eranroseman/memoria-vault/pull/677), with closeout
evidence preserved in the [v0.1.0-alpha.7 validation log](validation-log.md). The durable UI decision is
[ADR-81](../../adr/81-persistent-gate-dashboards.md), which supersedes ADR-68 and records
the deferred future set. `released:` remains `false` because this checkpoint did not cut
a tag or GitHub Release.

**Scope-reconcile.** The alpha.6 roadmap penciled the shadow/instrument telemetry
harvest (#370 / #611 / #416 / #371) into alpha.7. alpha.7 closed as the UI release
instead; that harvest rolls forward to alpha.8 / the next telemetry checkpoint rather
than being silently dropped.

## 1. Scope — what shipped

alpha.7 shipped:

- **View layer:** `inbox`/`sources`/`catalog`/`claims` + hubs / `projects` /
  `patterns` Bases, the Inbox gate glance, and the existing `kanban.db →
  system/board` mirror.
- **Capture forms:** six Modal Forms (`fleeting`/`source`/`claim`/`hub`/`project`/
  `thesis`) generated by `scripts/gen-forms.py` from `.memoria/schemas/`, with pytest
  drift coverage.
- **Persistent gate shell:** four user-facing dashboards under `src/gates/`, switched
  by a wikilink nav row. This retires the ADR-68 workspace-swap mechanism,
  `load-workspace.js`, the QuickAdd workspace macros, and Workspaces Plus.
- **Portals navigation:** vendored Portals config and plugin files from the Linux
  `Memoria-test` artifact, provenance-locked under ADR-74, with `system/` and
  `.memoria` hidden while the core file explorer remains enabled as fallback.
- **Obsidian config + CSS:** shipped `app.json`, core-plugin toggles, snippet
  enablement, and value/state legibility updates.

It closed the scoped UI issues #659, #663, #664, #665, #666, #667, and #668.

## 4. Blockers

No alpha.7 blockers remain. PR #677 passed required CI and merged to `main`; the scoped

## 5. Out of scope (deferred)

The deferred set lives in
[ADR-81](../../adr/81-persistent-gate-dashboards.md). It includes:

- **The general projector engine** beyond the existing board mirror.
- **Projected telemetry bases** (`fleet-health` / `eval-trend` / `skill-lifecycle`),
  which depend on the projector engine.
- **The Canvas / argument-graph spatial axis**, including the future Studio gate.
- **The dedicated edge-authoring "relate" control**; alpha.7 continues to use
  `links:` frontmatter through agent proposals or hand edits.
- **Tab/title-bar prose title** via another plugin; alpha.7 instead ships
  `showInlineTitle:false` and title-led Bases.
- **Shadow / instrument telemetry harvest** (#370 / #611 / #416 / #371), rolled
  forward to alpha.8 / the next telemetry checkpoint.

## 6. Known limitations

- **No PI-direct edge-authoring control.** `links:` frontmatter remains the edge system
  of record, but the PI can originate an edge only through agent propose→confirm or by
  hand-editing frontmatter.
- **No Canvas / argument graph.** The spatial axis ships nothing in alpha.7.
- **Telemetry dashboards are absent.** `fleet-health`, `eval-trend`, and
  `skill-lifecycle` remain projector-dependent.
- **Empty-on-arrival.** Most dashboards render blank on a fresh vault; empty-state copy
  mitigates the first impression, but the vault genuinely starts near-empty by design.
- **Portals is a gated core-surface replacement.** If it breaks on an Obsidian upgrade,
  navigation degrades to the retained core file-explorer fallback.

## 7. Cut procedure

This internal checkpoint followed the untagged cut path:

1. Merge the implementation PR with required CI green.
2. Preserve closeout evidence in the [v0.1.0-alpha.7 validation log](validation-log.md).
3. Keep ADR-81 as the durable decision record for the gate shell and deferred UI set.
4. Skip release-please, tag, and GitHub Release.
5. Set frontmatter to `status: complete`, `released: false`.
6. Prune tracked `tmp/` scratch after durable decisions are captured, carrying only
   unresolved future design notes into the next checkpoint.
7. Close the scoped GitHub issues and milestone.
