# Memoria 0.1.0-alpha.11 — execution plan

Date: 2026-06-28
Status: implementation plan for the `0.1.0-alpha.11` reset. Companion to `0.1.0-alpha.11-design.md` (the design SSOT) and the M0 evidence files in this folder. Release design scratch — promote durable decisions into ADRs before release close.

## 1. What alpha.11 ships (and what it doesn't)

**Goal.** The **basic knowledge cycle, end to end, in Obsidian, on a *test* bundle**, on the integrity spine:
`open project → seed → gap analysis → source → capture → conversation → digest + hub compounding → notes → re-run gap analysis` (design §3/§15).

**The two bets (settle separately so failure localizes — design §16):**
- **Bet A — storage/trace works** (plumbing): a real source captured into files, trace + index rebuilt from scratch, every write carrying a coupled derivation.
- **Bet B — integrity recall holds** (the thesis): trace + check + cascade-rollback keep the base trustworthy *without* per-write approval — settled only by the **seeded-error verdict**.

**Deferred (not built in alpha.11):** the **writing module** (draft/verify/paper-export), draft/export compatibility work, and the **coding/autonomy lane**. Also excluded: automatic Link/contradiction promotion, graph viz beyond Canvas, nightly discovery at scale, multi-user, any sync-capable/authoritative DB.

**The gate.** Optimistic writes stay on a disposable **test** bundle until the **seeded-error bar** clears (§14). There is no migration target in alpha.11; any later non-sandbox use starts from a fresh workspace (§20).

## 2. Standing guardrails (every work package inherits these)

- **Test-bundle-only** until the seeded-error bar passes. `Memoria-test` is the sandbox; no production/private install is in scope.
- **Read barrier is real, not prose:** machine writes land in `.memoria/staging/{catalog,knowledge,capabilities}/`, are checked, and **promoted only on `check_status: checked`**; consumers fail closed (design §4.4).
- **generator ≠ checker:** the operation that generates is never the one that checks (§11).
- **Everything traced:** machine writes/promotions go through the trusted worker; PI edits are observed/backfilled with a coupled `derived` event; foreign/untraced changes are quarantined by scan.
- **Baseline discipline:** every capability declares a cheap baseline (BM25, no-checks, unranked, long-context) and must beat it before it counts (§14).
- **Acceptance = a runnable check + evidence file** in this folder (mirroring the M0 `*-results.md` pattern), not a claim.
- **Projections regenerate + drift-check:** `index.md` / `ai-catalog.json` / `references.bib` are generated; a drift check fails if regeneration changes the tracked file. Hand-edited projection ≠ acceptance.

## 4. Work packages (sequenced)

### Phase 1 — Foundation (settles **Bet A**)

**WP1 — Schema + bundle skeleton.**
*Build:* the canonical schema/folder owners `vault-template/.memoria/schemas/types/*.yaml` + `folders.yaml` (design §7.1); the three bundle roots (`catalog/`, `knowledge/`, `capabilities/`) under a workspace parent; Concept types `source`/`person`/`organization`/`venue`/`digest`/`note`/`hub`/`operation`/`skill`/`mcp`/`workflow`; `check_status` + required `title`/`description`/`resource` fields; two validators — `okf-core` (permissive shape) and `memoria-profile` (strict, pre-promotion).
*Accept:* an empty workspace validates; `memoria-profile` rejects a malformed Concept; M0 schema-reset fixture passes; round-trip holds.
*Depends:* —

**WP2 — Trusted writer + worker + journal + staging/promote.**
*Build:* the per-machine worker + one local queue (`.memoria/`); the trusted writer (write-request → validate → stage → check → **promote on `checked`** → one run = one commit; foreign/untraced → `.memoria/quarantine/`); the append-only content-hashed **journal** (PROV-DM events `derived`/`run`/`model_call`/`check-fired`/`resolved`); PI-edit observe-and-backfill (`inputs = prior upstream ∪ prior-HEAD`); rebuild-from-journal.
*Accept:* a Concept created via the writer has a coupled derivation; the staging→promote path is the *only* way machine content enters `knowledge/`; a foreign commit is quarantined; trace + state rebuild from the journal. (Generalize `git_writer_smoke.py`.)
*Depends:* WP1

**WP3 — Disposable index (qmd) + rebuild.**
*Build:* the query-index interface (BM25 + vector + hybrid over the knowledge graph) with **qmd** as the backend (M0 verified disposable-bundle BM25); SQLite FTS5+sqlite-vec as the dependency-free baseline/fallback; rebuild-from-scratch; **index only `checked` content**.
*Accept:* a checked-only qmd BM25 index rebuilds from the record; retrieval never returns `unchecked`/`quarantined`; an eval harness exists for the later Ask/retrieval baseline-beat checks.
*Depends:* WP2

**WP4 — Capture pipeline (catalog).**
*Build:* capture-event-first → immutable `raw/` → content extract behind a coherence gate (parser per D2) → metadata (CSL-JSON, ACM minimum; Zotero Local API import; ambiguous identity → forced flag) → `source` + entity Concepts (immutable `source_id`, citekey alias); `check_source_metadata` (bibliographic-hygiene lint).
*Accept:* **one real source → catalog `source` + `content.md` + `raw/`, fully traced; the `# Citations`/`references.bib` projection regenerates; a first annotated-PDF fixture exists** (M0 gap). **→ Bet A settled.**
*Depends:* WP2, WP3, D2

### Phase 2 — Integrity loop (settles **Bet B**)

**WP5 — Operation framework + conversation + synthesis (digest + hub compounding).**
*Build:* the **operation** policy contract (operation_id, allowed tools/paths/network, runner+model+routing, prompt version, I/O schema, risk class, required checks, audit events); the **runner split** (direct-API default; Hermes adapter, fresh `AIAgent`, `quiet_mode`/`skip_memory`/bounded, no canonical-write tools); the **Co-PI interview** grounded in wiki+project → machine-owned `digest` compile + the 5-15 `digest`/`hub` nodes it touches (the compounding effect); curated `hub` changes land as **suggestions** and never overwrite human curation. Born `unchecked`.
*Accept:* a source → a checked machine-owned `digest` + touched hub suggestions, staged→checked→promoted where applicable; compounding observable across 5-15 nodes; `model_call` events log runner/route/model. (Build on the M0 deterministic-wiki fixture.)
*Depends:* WP4, D5

**WP6 — Notes (knowledge construction) + gap analysis.**
*Build:* the conversation continues → candidate **anchored notes** + **claims** (`evidence_set` = supporting Citation links) the PI accepts/edits/rejects/links (the *whether* is the PI's); **gap analysis** over *both* graphs + their mismatch (thin/thin = new-topic; source-dense/note-thin = undigested; note-dense/source-thin = under-warranted), thesis adds the argument-health lens; cold-start handling.
*Accept:* an agent emits note candidates; the PI curates; gap analysis names a real gap and the proposed seed; the Ask/Query answer carries sources + unknowns/staleness/contradictions.
*Depends:* WP5

**WP7 — Detection layer + flag routing + cascade rollback.**
*Build:* the structural checks (citation/identifier resolves; span entailment [model-free first, NLI behind a HANS gate]; counterfactual faithfulness; contradiction candidates; supersession consistency; orphaned warrant; near-duplicate review-only; retraction; trace integrity); **shadow-first error-budgeted calibration**; **`act`/`ask`/`drop`** flag routing; **trace → cascade rollback** (machine downstream auto-reverts, human-directed flagged; the inverse commit appends `resolved`/`derived`).
*Accept:* a **seeded error is flagged**; the PI traces and cascade-rolls-back; the rollback is itself traced (not flagged foreign); checks run in shadow before they route. **→ Bet B mechanism built.**
*Depends:* WP6

### Phase 3 — Surface

**WP8 — Obsidian plugin (control panel, Option A).**
*Build:* the Co-PI conversation pane (Agent Client / Local REST bridge); knowledge-graph browsing (Bases / Canvas `argument.canvas` / graph view); project/gap + flags(integrity) panels; the trace→rollback panel; **enqueue-only mutations** through the narrow worker API (`enqueue_operation`, `acknowledge`/`resolve_attention`, `open_trace`) — the plugin **never** writes Concepts/journal/projections/`check_status`; static/provenance guardrails (D1).
*Accept:* the full cycle is **usable in Obsidian**; every plugin mutation is a recorded worker event; static checks deny direct vault/journal/projection/`check_status` writes. (Generalize the start-blocker plugin smoke to real panel activation.)
*Depends:* WP2 (bridge) + WP5–WP7 for live content; can start in parallel after WP2.

### Phase 4 — Documentation and release readiness

**WP-Docs — System documentation + release plan.**
*Build:* update the public/system documentation that describes alpha.11 behavior (workspace layout, Concept types, trusted writer, journal, read barrier, operation policy, qmd index, capture, Co-PI conversation, notes, gap analysis, checks, rollback, plugin boundary, seeded-error verdict); add/update ADRs for durable decisions; maintain `release-plan-0.1.0-alpha.11.md`; dispose or move `tmp/` scratch before checkpoint close.
*Accept:* docs describe only built behavior or clearly labeled planned scope; generated references/projections have drift checks where they mirror machine-readable contracts; release plan gates/stages match this ExecPlan; docs checkers pass except unrelated pre-existing drift.
*Depends:* WP1-WP8 for built behavior, WP-Gate for final verdict wording; starts in parallel after WP1 and closes last.

### Phase 5 — Verdict gate (the release gate to non-sandbox use)

**WP-Gate — Seeded-error integrity-loop verdict.**
*Build:* a frozen **gold test bundle** with injected known errors — including the blind classes (crafted injection; well-formed-but-unwarranted claim) and the structural ones (poisoned span, wrong extraction, false Link, planted contradiction, stale-as-current); the **batch runner**; per-error-class measurement of **detection recall** (make-or-break), time-to-detection + false-positive rate, **rollback completeness**, **residual error vs a no-checks baseline**; **calibrate the human checkpoint itself** vs a no-checkpoint baseline.
*Accept:* the bar (D4) is set and met per class → **cleared for continued alpha use or fresh non-sandbox initialization**; any class below bar → re-introduce gating for that class, by evidence. (Build on the M0 seeded-error harness shape.)
*Depends:* WP7 (+ WP3–WP6 for a populated bundle)

### Phase 6 — Fresh initialization only

No migration, upgrade path, backwards-compatibility mode, or data-preservation checklist is alpha.11 scope. If a non-sandbox workspace is ever needed after WP-Gate, initialize it from the alpha.11 skeleton and seed it through the alpha.11 capture/conversation pipeline.

## 5. Cross-cutting tracks (run alongside, not at the end)

- **Projection generators + drift checks** (WP2+): one worker-owned generator for `index.md`/`ai-catalog.json`/`references.bib`; CI diff-against-regeneration.
- **Eval + CI** (WP3+): checked-only qmd BM25 eval harness; state-based + `pass^k` engine certification; projection-drift + schema-valid-template + boundary-fires tests as CI gates. Ask/retrieval features prove hybrid/query-expansion/rerank beat BM25 or long-context baselines before those features count.
- **System docs + release prose** (WP-Docs): public docs, ADRs, release plan, and `tmp/` disposition stay in sync with the built alpha.11 behavior.
- **Calibration** (WP7): shadow-first thresholds; per-model prompt validation (a real, recurring labeling cost — budget it).
- **Evidence discipline:** each WP closes with a `*-results.md` in this folder.

## 8. Top risks

- **R1 — Detection recall on the blind slice** (crafted injection / unwarranted claim) below bar → gateless model needs gating back for that class. *Mitigation:* WP-Gate measures it explicitly; read barrier + staging shrink the window; export gate + provenance-triggered checkpoint stand by.
- **R2 — Compounding cost/quality** (5-15 nodes/source) makes ingest slow or noisy. *Mitigation:* model-quality wiki eval is an alpha.11 implementation test; cap fan-out; suggestions-not-overwrite protects human work.
- **R3 — PDF span fidelity** (no local extractor at M0) → annotation provenance unreliable. *Mitigation:* D2 spike before WP4; the parser-output contract (page/span/bbox) is parser-agnostic.
- **R4 — Plugin trust boundary** (Option A = trusted plugin JS, not an OS sandbox). *Mitigation:* static/provenance guardrails + enqueue-only API; escalate to Option B/C only if approval requires a hard sandbox.
