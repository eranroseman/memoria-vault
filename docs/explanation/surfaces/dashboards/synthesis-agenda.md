# Synthesis agenda

The named Library and Knowledge views are planned optional-adapter groupings
over shipped catalog, note, and link reads. They would surface what remains to
read, discuss, and reconcile. The actor-split rationale is in [Surfaces and
dashboards](README.md).

## Reading pipeline

The planned Reading pipeline would show Library work in flight: checked Work rows awaiting
reading, Works discussed but not yet distilled, and claim-bearing notes grouped
by linkage status. It prevents intake from looking healthy when reading is not
becoming synthesis.

It is not a board view. It reads note state, not cards.

## Discuss queue

The planned Discuss queue is the narrower read-but-not-distilled slice. A long queue means
intake is outrunning thought.

## Open questions

The planned Open questions view would surface current claim-bearing notes with no inbound links. These
are notes captured but not yet integrated into hubs or other notes.

It does not propose links or track a separate resolved state. Integration, not a
dashboard disposition, resolves the question.

## Contradictions

The planned Contradictions view would surface current claim-bearing notes with confirmed `links.contradicts`
relationships. It is a synthesis starting point, not an error list and not a
truth judgment.

A contradiction is an open tension between current notes; supersession is a
resolved temporal replacement.
