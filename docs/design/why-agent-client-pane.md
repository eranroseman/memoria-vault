# Why the Agent Client pane exists

Most Memoria work belongs on the board because it produces a reviewable artifact.
The Agent Client pane exists for a different kind of work: synchronous,
exploratory thinking where the useful output is a clearer PI.

That split keeps conversation and production separate. The pane is for asking,
branching framings, reading context, and deciding what should happen next. Once
the result should become durable, it leaves the pane through a command, a
PI-authored note, or a delegated board card.

Only the Co-PI lives in the pane. That makes the write contract simple:
read-only conversation in one place, scoped artifact production on the board.
Specialist boundaries remain process boundaries, not UI tabs a human has to
operate correctly.
