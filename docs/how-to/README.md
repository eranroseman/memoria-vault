# How-to guides

Goal-oriented recipes for getting a specific task done. Each guide assumes you already know the shape of the system — if you don't, start with the [tutorials](../tutorials/), which teach the same ground from zero, in order.

The guides are grouped by what you're acting on:

- **[workflows/](workflows/)** — the operational pipelines: upstream (source → durable knowledge), downstream (knowledge → written output), and cross-cutting maintenance. One recipe per named workflow.
- **[obsidian-ui/](obsidian-ui/)** — driving Memoria from inside Obsidian: the command palette and the home / front-door note.
- **[operations/](operations/)** — running and recovering the system: installing a profile, and detect / fix / verify recipes for failure modes.
- **[coder/](coder/)** — setting up and using the external coding-agent workspace.

For exact field values, command names, and schemas, see [reference/](../reference/); for the *why* behind a workflow, see [explanation/](../explanation/).
