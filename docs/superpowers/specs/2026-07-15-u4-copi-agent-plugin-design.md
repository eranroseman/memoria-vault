# U4 · Co-PI agent plugin — Design

Date: 2026-07-15. Status: **design (PI-approved in session), pre-plan**.
Plan 23 LOOP.12 output. Consumes `2026-07-15-surfaces-bootstrap-design.md`
(bundle seeding, write perimeter, MCP wiring, skew detection). The
load-bearing part of the agent artifact is the **hook** (the write
perimeter, owned by the bootstrap spec); this spec adds the **method** and
the two operations it rides on. PI ruling of record: a plugin, not a bare
skill — without the hook, any agent could edit vault markdown directly and
its edits would masquerade as the PI's to the provenance watcher.

## 1. The surface split, concrete

**Engine authors the method; the user's agent voices it.** Engine-authored
artifacts (generated, regenerated by `memoria upgrade`, stamped in
`vault.json`, never PI-edited):

- **Operation vocabulary** — when to reach for `answer-query` vs
  `analyze-gaps` vs `compare-and-contrast` vs `surface-tensions` vs
  `red-team-argument`, with the one-line contract of each.
- **Grounding discipline** — no truth assertions ever; every voiced claim
  about vault content carries resolvable sources; the honest-empty wording
  verbatim; "citation-correct is not grounded; grounded is not true."
- **Disposition etiquette** — machine proposes, PI disposes; how to submit
  proposals (attention cards through the normal path), never bypassing.
- **Toulmin question taxonomy** — grounds-seeking, warrant-challenging,
  rebuttal-probing, qualifier-testing; vocabulary aligned with the merged
  grounds contract (`2026-07-14-evidence-set-grounds-contract-design.md`).
- **Argument-health reading guide** — how to interpret
  `analyze-project-argument` output *for* the PI, never re-scoring it.

Agent-owned: phrasing, dialogue flow, follow-up choice. The skill text
teaches the method and the refusals; it never grants judgment.

## 2. Bundle layout

Consumes bootstrap §5 (perimeter, including the `.codex/hooks.json` deny
mirror) and §6 (skew), plus the bootstrap-seeded wiring (`.mcp.json` with
`operation_run` as the only write tool — U1 baseline; `CLAUDE.md`
`@AGENTS.md` loader). This spec **owns exactly two files** inside the
vault-embedded bundle:

- `.claude/skills/memoria-copi/SKILL.md` — the method (§1), generated.
- `.claude/hooks/session_status.py` — SessionStart: runs
  `memoria doctor --json --quick`; injects context lines for engine-missing
  / skew / credential status, plus the one-line method pointer.

Codex receives the condensed method as ungated `AGENTS.md` prose (the
generated projection, R1NG.4); Codex *enforcement* is bootstrap-owned.

No enforcement is defined here — the perimeter is the bootstrap spec's;
this bundle adds method only.

## 3. `generate-questions` operation

Engine-side manifest, new operation id `generate-questions`: scope (a
concept, project, or slice) → Toulmin-taxonomy questions emitted as
**proposal cards** through the normal attention path (loudness `notice`;
machine proposes, PI disposes). Mechanics: a fenced one-shot
prompt-operation on the existing runner machinery, with the manifest
enumerating: call-site id `generate-questions.v1` (versioned prompt
variables in the call-site ledger); runner branches `deterministic-fixture`
(returns fixture questions; the tests/CI path) and the standard live branch
through the resolved provider profile; shadow-first discipline
(`production_enabled: false` until promoted against the call-site gold
set). Required structural checks on every
output (never a truth judgment): it is a question; it carries its taxonomy
role tag; it references a resolvable target (concept path or `work_id`).
Cards land in the same triage surface U3 renders — the co-PI "asks the hard
questions" without any agent-side generation authority.

## 4. `conversational-ask` grounding contract

The skill's Q&A mode is a **voicing of `answer-query`**, never a bypass:

- The agent must call the MCP read tools for any question about vault
  content; answering from its own priors is refused with the skill's
  scripted wording.
- Every claim in the voiced answer carries the same resolvable source refs
  the raw payload returned — rephrasing allowed, ungrounded additions
  forbidden.
- Retrieval-empty is voiced as the honest-empty message, verbatim.
- Reads flow through the read API, so I1's `read-observed`/staleness
  telemetry fires exactly as for any other consumer.

## 5. Out of scope

Any write tool beyond `operation_run`; autonomous or looping agent behavior
(fenced one-shot doctrine holds); memoria-in-memoria dogfood (beta.2
register); Codex enforcement (bootstrap-owned) and any Codex parity beyond the prose
method; the onboarding interview flow (O1 gate).

## 6. Acceptance criteria

The write path is exclusively `operation_run` (probe: an agent session in a
seeded vault cannot modify any `.md` via file tools or recognized bash,
and the watcher flags subprocess writes); the SKILL.md regenerates on
`memoria upgrade` and its version is stamped in `vault.json`;
`generate-questions` outputs pass the structural checks and arrive as
attention proposals; a `conversational-ask` transcript shows resolvable
sources on every claim and the honest-empty wording on a no-hit query; the
SessionStart hook injects skew/engine/credential truth on session open.
