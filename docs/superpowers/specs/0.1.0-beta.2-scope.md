# Memoria 0.1.0-beta.2 — deferred scope of record

Date: 2026-07-12. Status: **derived scope** — redeems the consolidation's §4
promise that every deferred unit gets "a home in a beta.2 spec". Derived from
`2026-07-12-beta.1-consolidation.md` §4 and the pre-registered decision rules in
`0.1.0-beta.1-empirical-use-action-plan.md` §4. Milestone: `0.1.0-beta.2`.

## The rule this spec operates under

Nothing here was dropped from beta.1 by preference. Each unit carries a
**proven blocker** (usually an evidence gate whose data beta.1 itself will
produce), a **beta.1 precursor** already in scope (so the substrate exists the
day the gate opens), and — where the empirical plan pre-registered one — the
**activation rule** that converts beta.1's recorded events into the go/no-go
decision. When a gate opens during beta.1, the unit may be pulled forward;
nothing waits for ceremony.

## Deferred units

### 1. Reactive daemon (Tier A/B/C) + SSE events + live on-save badges
- **Blocker:** staleness-data-gated. The alpha.19 same-transaction cascade keeps
  a daemon never load-bearing; "no daemon before `staleness_hit` data justifies
  its priority — the read barrier keeps correctness meanwhile."
- **beta.1 precursor:** `staleness-instrumentation` (I1), `observe-file-event`
  manifest, poll-based plugin status (U3).
- **Activation rule (pre-registered):** collect `staleness_hit` count,
  index-refresh latency at 1000 works, on-read wait time, all sessions.
  *Frequent staleness or painful refresh promotes the daemon (Tier A) up the
  roadmap; silence defers it behind Tier C nightly only.*

### 2. Dense/hybrid retrieval default activation (+ community detection, ANN index)
- **Blocker:** BM25 stays default until a pre-registered spike beats it on the
  real corpus. "No retrieval default switch before the preregistered fixture
  beats BM25."
- **beta.1 precursor:** full dense substrate + `real-embeddings` + the fixture
  harness (R3).
- **Activation rule:** preregistered recall/grounding/latency/cost fixture;
  shadow-first; incumbent stays until beaten. Community detection and ANN
  index activate only if the fixture shows curated grouping under-performs /
  brute-force KNN exceeds ~200ms at observed corpus scale.

### 3. Code-execution lane (generated source, sandboxed runs, executable warrants)
- **Blocker:** large security surface with no MVP-loop dependency; requirements
  and design both descope it post-beta.1. "No code execution data collection
  for beta.1; SRD is the code-side output."
- **beta.1 precursor:** `code_artifacts`/`code_runs` schema (built), the SRD
  document lane (C1), sandbox-hardening decisions (ratified: canonical-path
  allowlist, `--` delimiter, adaptive red-team).
- **Activation rule:** first real SRD that needs an executable acceptance
  check; enters via its own security review, never silently.

### 4. Warrant-node reification (default ontology)
- **Blocker:** evidence-gated on the warrant touch budget; expensive to reverse
  after accumulation. "No warrant-node schema work before the touch-budget
  observation."
- **beta.1 precursor:** roles-carried-by-relations with promotion-ready edges
  (`concept_edges` gains `edge_id` + attributes, G4); a warrant reifies to a
  node only when explicitly authored/shared.
- **Activation rule (pre-registered):** track under-warranted findings raised,
  "state the warrant" demands, PI minutes per explicit warrant, over one full
  project loop. *If explicit warrants stay under budget and get demanded,
  ratify hybrid-with-nodes; if never demanded, keep demandable-only and defer
  nodes.* See `warrant-ontology-brief.md` for the full decision frame.

### 5. Autoresearch overnight loop (instrument + prompt autoresearch, Tier-C scheduler)
- **Blocker:** needs a scheduler (not built) and gold tasks harvested from real
  beta.1 use; `eval_dispatch` is a stub.
- **beta.1 precursor:** seeded-error battery (built), instrument single-file
  extraction, `experiments/program.md`, `empirical_event` plumbing (I1),
  LLM call-site ledger (E1).
- **Activation rule:** enough real-use gold tasks to freeze an eval suite;
  instrument-only fence stands (throwaway vault, `production_enabled:false`,
  never optimizes vault content).

### 6. memoria-in-memoria (dogfooding Memoria's own design as a project)
- **Blocker:** gated on Foundation (trust repairs), the real corpus being
  loaded, the seeded-error license gate, and the repo-at-pinned-commit source
  type.
- **beta.1 precursor:** everything — it dogfoods the whole machine; plus
  `software`-work + version-qualifier support (O2/S1).
- **Activation rule:** seeded-error battery passes its bars (recall 1.0,
  false-clean 0) — "the license for real use."

### 7. 1000-scale seed-corpus load + staged-import acceptance test
- **Blocker:** disposition telemetry must exist first (non-backfillable —
  "every empty day is permanently lost baseline"); needs the real corpus.
- **beta.1 precursor:** import pipeline + 10→100 staging + `bulk-admission-mode`
  (O2); **schema-before-corpus**: every cheap-while-empty reshape lands before
  this import.
- **Activation rule:** telemetry live + Foundation landed; the staged import
  measures Shape-1/Shape-2 query latency per stage and onboarding triage
  minutes per 100 admitted works (bounded batches guardrail).

### 8. Facility-location / DPP exploration diversity
- **Blocker:** fixture-gated; only MMR is proven at baseline.
- **beta.1 precursor:** MMR baseline + diversity-channel measurement (I1).
- **Activation rule:** exploration acted-on rate shows MMR under-serving
  coverage on the diversity fixture.

### 9. Two-way canvas↔outline sync
- **Blocker:** excluded until measured demand; one-way derivation is the
  committed shape.
- **beta.1 precursor:** one-way outline derivation (W1).
- **Activation rule (pre-registered, scoped ADR-103 reopen):** track
  outline-composition friction — list reorder sufficed vs reached for spatial
  arrangement, every outline session. *Repeated spatial reaching admits
  canvas-as-authoring; silence keeps canvases projection-only.*

### 10. Entity-resolution catalog (`catalog_entities`)
- **Blocker:** specified-but-unbuilt until a real duplicate/entity-browse need
  appears.
- **beta.1 precursor:** free `work_graph_edges.target_id` join for
  gap-discovery.
- **Activation rule:** observed duplicate-entity incidents in real use.

### 11. Multi-device write topology
- **Blocker:** freeze-blocker decision defaulted: single-writer local-first +
  read-only secondaries.
- **beta.1 precursor:** single-writer enforcement (built).
- **Activation rule (pre-registered):** track second-device attempts,
  conflicts, sync workarounds, all sessions. *Single-writer unless repeated
  real handoffs demand more.*

### 12. Bundle merge (two Memoria knowledge bundles)
- **Blocker:** owner ruling 2026-07-12 — an exception path, not the common
  case. OKF export = folder copy; OKF import of a foreign bundle = the normal
  source-import path (both beta.1, K1).
- **beta.1 precursor:** export-unit + permissive OKF import (K1).
- **Activation rule:** a real second vault whose knowledge must merge (not
  just be imported as sources).

### Also riding beta.2 unless pulled forward
Spatial-graph-at-scale and a local fine-tuned/QLoRA adapter remain excluded by
the design's own §7 ("legitimately out"); the frozen local model stays a test
fixture. Embedded chat panel earns back only on measured two-window friction
(pre-registered: context switches between editor and agent, dropped handoffs,
`context.read` misses — repeated friction triggers the earn-back).

## Exit posture

beta.2 is scoped by evidence beta.1 produces, so its freeze happens *after*
beta.1's exit verdict (adopt / targeted redesign / kill). If beta.1 exits
"kill", this register is moot; if "adopt", the open gates above are read
against their recorded data and this spec graduates into the beta.2
requirements document.
