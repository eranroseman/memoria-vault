# Memoria beta.1 ‚Äî folded scope consolidation

Date: 2026-07-12. Status: **draft for owner approval** (nothing filed or built from it yet).

## Why this document exists

The `docs/superpowers/` corpus (a governing requirements doc, a design-of-record,
two mechanism analyses, decision ledgers, an empirical-use plan, and a stack of
2026-07-09/10 audits + a tiered roadmap) carries a large amount of invested
design thinking that was **never implemented**. This document **folds all of it
into one place** so nothing is lost, deduplicates the ~250 raw units the corpus
describes, marks each against what alpha.20 already ships, and assigns each a
release.

### The rule (owner-set)

> Everything in the corpus is in scope and gets **implemented**, not archived.
> **Default is beta.1.** A unit moves to **beta.2 only when there is a proven
> blocker** ‚Äî after analysis it is genuinely impossible to land in beta.1. A
> deferred unit is never dropped: it carries a named blocker, its beta.1
> *precursor* (the substrate/instrumentation we still build now), and a home in
> a beta.2 spec.

So beta.2 is the rare, justified exception. Where a capability is data- or
security-gated, we split it: the **substrate + instrumentation + shadow mode**
is beta.1; only the **gated activation/decision** defers.

## 1. The alpha.20 baseline reframes the work

The requirements + design were written **2026-07-05 against alpha.16**; the code
is now **alpha.20**, and four releases shipped a great deal the design assumes it
must build. A code baseline pass confirms **much of the substrate already
exists** ‚Äî beta.1 is closer than the corpus implies.

**Already built (do not rebuild):** the full writing pipeline (argument-analyze ‚Üí
canvas ‚Üí slice ‚Üí draft ‚Üí promote), verify + export-refusal gate, `evidence_sets`
(marker-driven, ~alpha.17), `passages` + FTS5 + BM25 + RRF fusion, a
code-execution warrant substrate (`code_artifacts`/`code_runs`), Guard-brokered
egress, the checked-file read barrier, single-writer enforcement (workspace
lock + one trusted-writer path), the seeded-error verdict harness, `memoria init`
scaffolding, and the append-only hash-chained `event_log` (write side).

**The real gaps (narrower than the corpus suggests):**

- `concept_edges` exists but is a **permanently empty stub** (`indexing.py`
  returns `[]`; wiped on every reindex) ‚Äî the designed graph join point is dead.
- **No forward DB-migration path** (`state._init` hard-fails on any
  `user_version` except 0/8) ‚Äî blocks every schema change.
- **No journal-chain verifier** (chain is written, never validated).
- **sqlite-vec / embeddings are a stub** (16-dim hash test-embedder; dense
  retrieval non-functional for real content).
- **No cost ledger**, **no feedback instrumentation** module at all.
- "Wired-at-edges-dead-in-middle" bugs: `list --type work` and `note --mode work`
  are advertised but unreachable; `_emit` prints `ok` on FAILED ops.
- Provenance `actor` is split (`operation_requests` defaults `'pi'` with no
  CHECK; `derivations` CHECK forbids `'agent'`); context threading reaches only
  ~3 of ~40 operations, so several ops hardcode `actor:"pi"`.

### Owner corrections (2026-07-12) ‚Äî applied

1. **Blast-radius propagation fires on _any_ knowledge-base change or addition**
   (new/edited claim, new/changed edge, retraction), not only a "decided-wrong"
   disposition ‚Äî that is one trigger among many. (G5)
2. **Derive-on-write is primary; derive-on-read is the fallback.** Write-time
   derivation keeps the KB always-current and gives the user immediate feedback;
   reads can come weeks or months later, and even a few hours' staleness can
   matter. Read-time derivation covers only what write-time did not. (G5, S1)
3. **Prior ADRs, designs, and implementations are reference only** ‚Äî never a
   justification or evidence. History is a teacher, not a prison; the
   "reverses ADR-N" notes carried from the feeders are context, not blockers. (¬ß6)
4. **Living design-history replaces ADRs as the _history-preserving_ mechanism**
   ‚Äî not a "decide once, never change" lock. The decisions in ¬ß6 are honored but
   revisable on merit. (¬ß6)
5. **beta.1 includes the coding SRD document and excludes code execution.** (C1, ¬ß4)
6. **OKF export = copy-paste of the bundle folder; OKF import = the normal
   source-import path** (permissive, re-enters as unchecked). **Merging two
   Memoria bundles is an exception and is out of beta.1** (beta.2). (K1, ¬ß4)
7. **"Zones are states, not places" is a foundational axiom that predates OKF**
   ‚Äî not an OKF-derived rule. (X1 / foundations)
