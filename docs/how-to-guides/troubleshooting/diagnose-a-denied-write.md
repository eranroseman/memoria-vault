# Diagnose a denied or blocked write

**Symptom:** a CLI command or optional adapter reports a write, but the expected
workspace file or projection is missing.

**Diagnosis:** separate policy denial from runtime failure:

1. The runtime policy gate denied or dry-ran an optional adapter write.
2. The CLI worker request failed before materialization.
3. A valid direct Concept edit under a bundle root was observed and marked
   unchecked by `workspace scan`.
4. A changed tracked projection was quarantined and regenerated by
   `workspace scan`.

## Steps

**1. Check request state.**

```bash
memoria status --workspace <workspace> --json
memoria request list --workspace <workspace> --json
```

Inspect the request payload. Retry a `failed` request after fixing its cause;
resume a `pending` request. If its arguments are wrong, create a successor with
`request amend` and a fresh idempotency key only when scope stays the same. A
changed ID, reference, path, or target requires a new original operation.

**2. Check the audit log.**

```bash
memoria journal tail --workspace <workspace> --json
```

For optional adapter writes, also inspect `system/logs/audit.jsonl`. A matching
`deny` or `dry_run` means the gate worked; read `policy_rule` and `reason`.

**3. Scan Concept edits and tracked-projection drift.**

```bash
memoria workspace scan --workspace <workspace> --json
```

For valid Markdown Concepts under `notes/`, `hubs/`, `projects/`, `digests/`,
and `fulltexts/`, the scanner journals direct PI edits and marks the Concept
unchecked until the relevant checks pass. It does not scan arbitrary workspace
files such as `steering.md` or `system/vocabulary.md`.

Changed tracked generated projections follow a separate path: the scanner
quarantines the changed projection and regenerates it from authoritative state.

**4. Check generated projections.**

```bash
memoria workspace rebuild --workspace <workspace> --json
```

Generated files such as indexes, bibliography, and search input mirrors are rebuilt
from SQLite and checked Concepts. A direct edit to a generated target is not
authoritative.

## Common Outcomes

| Outcome | What to do |
| --- | --- |
| `deny` | Use the correct CLI operation or narrow adapter policy; do not bypass the gate. |
| `dry_run` | The target is gated or missing a recorded promotion path; inspect the staged output. |
| failed request | Fix the payload/provider/runner error, then retry the request. Use `request amend` for changed non-scope arguments; submit a new original operation for a scope change. |
| observed Concept edit | Review and check the now-unchecked Concept through the normal workflow. |
| changed tracked projection | Inspect the scan's projection quarantine and regeneration result; edit authoritative inputs instead of the generated file. |

## Verify

- `memoria request list --workspace <workspace> --json` no longer shows the request as failed or stuck
- `memoria workspace scan --workspace <workspace> --json` reports the expected
  Concept paths and no unexpected projection quarantine
- The expected file, projection, or journal event is present
