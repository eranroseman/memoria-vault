# Memoria documentation

Four sections, each with a distinct purpose.

| Section | Purpose | Start here when… |
|---|---|---|
| [explanation/](explanation/) | Understanding-oriented — why the system is built this way | You want to understand how something works or why a design choice was made |
| [reference/](reference/) | Information-oriented — exact values, schemas, commands | You need a specific field name, command, or configuration value |
| [how-to-guides/](how-to-guides/) | Task-oriented — step-by-step recipes | You know what you want to do and need the procedure |
| [tutorials/](tutorials/) | Learning-oriented — guided first experiences | You are new and want a guided walkthrough |

If you are new: start with [explanation/what-memoria-is.md](explanation/what-memoria-is.md).
