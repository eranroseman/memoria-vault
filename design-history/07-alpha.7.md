## alpha.7 ‚Äî Clean-slate Obsidian UI & navigation (the persistent gate shell)

**Theme.** A ground-up redesign of the Obsidian front end: retire the ADR-68 three-workspace layout-swap model and re-implement navigation as **four job-named dashboard notes in a single persistent shell**, backed by a consolidated **Bases view layer**, **schema-generated capture forms**, and **Portals** folder navigation. This is the "UI/navigation release."

**What drove it.** Three shipped-UI issues (#659 "no single clear title," #665 "the same base shown 3‚Äì4√ó," #667 "using a base to navigate is unintuitive"), plus #663/#664/#666/#668, converged on one verdict: *Bases-as-navigation was rough and the workspace-swap machinery over-implemented the requirement.* The team ran a clean-slate design pass (four scratch design docs under `tmp/`, reviewed by a non-author reviewer and sandbox-verified against Obsidian 1.12.7), then shipped only the **verified, low-risk substrate** and explicitly **deferred** the speculative half (a general projector engine, projected telemetry bases, the Canvas/argument-graph spatial axis, and a dedicated edge-authoring control) in ADR-81. It closed as an untagged internal checkpoint (`released: false`), and the alpha.6-penciled shadow/instrument telemetry harvest (#370/#611/#416/#371) was consciously *not* absorbed and rolled to alpha.8.

### Vault / concepts & schemas

- **What:** New top-level `gates/` folder and a new `gate` category/type. `folders.yaml` gains `gates` in `categories`, maps `gate: gates` in the type‚Üíhome table, and adds `gates` to the `skeleton:` build list (`src/.memoria/schemas/folders.yaml`); a new `src/.memoria/schemas/types/gate.yaml` defines the type (`enums.gate: [inbox, library, knowledge, project]`, `lifecycle: [current, archived]`, `gated: false`, required `type/lifecycle/title/gate`). ‚Äî **Why:** gate dashboards are now *vault content* (notes), not plugin layout state, so they need a first-class folder, type-home, and Linter-checkable schema. ADR-81 Decision: "Gate switching becomes vault content instead of plugin layout state‚Ä¶ easier to diff, restore, and test." The four-gate enum hard-codes the derived JTBD set from `workspaces-design.md` ¬ß3 (Inbox/Library/Knowledge/Project; Studio deferred).

- **What:** New `worker-card` type schema (`src/.memoria/schemas/types/worker-card.yaml`): `category: system`, home `system/board`, `status` enum `[triage, todo, ready, running, blocked, retrying, done]`, required `task_id/lane/status/title`, optional `as_of/review_status/retry_count/reason/expected_outputs/created`. Detectors add `gate: gates/` to `TYPE_HOME`, `gates` to `KNOWN_TOP_DIRS`, `NOTE_FOLDERS`, and the design-scan prefixes (`src/.memoria/operations/integrity/linter/detectors.py`). ‚Äî **Why:** the board mirror is now read by a real Base (`board.base`) and must be schema-legible like any other collection. Previously the projected cards carried an ad-hoc `type: board-card` with no schema; the redesign makes the *one projection that already exists* conform to the type system so the Linter and Bases treat it uniformly (`ui-architecture-alpha7.md` ¬ß2‚Äì¬ß3: "authored bases vs the one projected base"). The detector edits are the mechanical consequence of adding a permitted top-level folder ‚Äî without them a stray-folder detector would flag `gates/`.

- **What:** `board_export.py` rewrites card frontmatter from the old key list (`task_id/status/assignee/review_status/retry_count/reason/last_updated` + `type: board-card`) to the full `worker-card` schema shape ‚Äî `title`, `type: worker-card`, `lifecycle: current`, `lane` (renamed from `assignee`), `status`, `as_of` (renamed from `last_updated`), `created` (`src/.memoria/mcp/board_export.py`). ‚Äî **Why:** to satisfy the new `worker-card` schema and let `board.base` sort on the payload-native `as_of` rather than `file.mtime` (all cards in a pass share a generation mtime, so mtime is meaningless ‚Äî `ui-architecture-alpha7.md` ¬ß2: "Sort on `as_of`, not `file.mtime`"). The `assignee‚Üílane` rename aligns the projection with the schema's `lane` grouping key used by the base's "By lane" view.

### Retrieval / view layer (Bases)

The core design bet of the release: **Bases is the universal tabular view layer, Dataview shrinks to a single cross-collection surface, and every `.base` view leads with `title`.**

- **What:** Every authored base's `order:` now leads with the `title` property instead of `file.name` (`inbox.base`, `sources.base`, `claims.base`, `catalog.base`, `patterns.base`, `project-gate.base`; catalog entity views lead with `name`). ‚Äî **Why:** the direct fix for **#659** ("one clear title"). The design deliberately *keeps a stable slug filename* (agents/API write raw `links:` wikilinks that must not churn on rename) but leads the visible column with the readable `title` property, paired with `showInlineTitle:false`, so collections read as titles while links stay stable (`ui-architecture-alpha7.md` ¬ß5: "Keep a *stable* filename‚Ä¶ Make the slug human-readable‚Ä¶ lead every Bases `order:` with the `title` property"). No migration was needed because the only vault is the rebuildable `Memoria-test` sandbox.

- **What:** `inbox.base` is consolidated into **four non-overlapping views** ‚Äî "Needs me" (`proposed`), "Drift watch" (flag/alert), "Loose ends" (notice-loudness flags), "All cards" ‚Äî with a new `loudness_rank` formula (block‚Üí0, alert‚Üí1, notice‚Üí2) driving priority sort and an `age_days` formula (`src/inbox/inbox.base`). ‚Äî **Why:** kills **#665** ("same base shown 3‚Äì4√ó") *by construction* ‚Äî the triplication was an artifact of the old layout; one base with disjoint views deletes it (`ui-architecture-alpha7.md` ¬ß3, ¬ß5: "The four views are non-overlapping‚Ä¶ deletes #665 by construction"). The `loudness_rank` formula orders the queue by urgency (block-first) since Bases can't sort an enum by semantic severity directly.

- **What:** `sources.base` renames "To read & distill" ‚Üí **"Reading pipeline"** (now `proposed || provisional`, grouped by lifecycle) and adds a **"Discuss queue"** view (`provisional`, oldest-first via `file.ctime` ASC) plus an `age_days` formula; adds `evidence_level`/`research_area` to the order (`src/system/dashboards/sources.base`). ‚Äî **Why:** these are the composed surfaces of the Library gate (`ui-architecture-alpha7.md` ¬ß3/¬ß5). "Discuss queue" (oldest first) drains read-but-not-distilled sources through a Co-PI pass; the reading pipeline distills into claims in the Knowledge gate. Fields shown are the judgment-relevant ones a reader triages on.

- **What:** `claims.base` replaces the old By-maturity/Retracted pair with **four views**: "By maturity" (now excludes `superseded_by`), "Open questions" (orphans via new `is_orphan: file.backlinks.isEmpty()` formula), "Contradictions" (`!links.contradicts.isEmpty()`), and "Retracted (lineage)"; adds `is_orphan`/`age_days` formulas (`src/system/dashboards/claims.base`). ‚Äî **Why:** the Knowledge gate is the synthesis surface (`ui-architecture-alpha7.md` ¬ß3). "Open questions" = current claims with zero inbound = the synthesis backlog; "Contradictions" surfaces open tensions before advancing a claim. Both filters were **live-verified native** at 300- and 10k-note scale (¬ß7: nested-map `!links.contradicts.isEmpty()` and orphan `file.backlinks.isEmpty()` work without a materialized field), which is *why* the design could drop its `has_contradiction`/`inbound_count` materialization fallback. The "Retracted (lineage)" flat table is flagged as a stopgap ‚Äî proper DAG lineage is deferred to the spatial axis.

- **What:** New `hubs.base` (`src/notes/hubs/hubs.base`, one "Hubs index" view) and new `projects.base` (`src/projects/projects.base`, views "Active projects"/"Saturation"/"Project gaps"), plus new `board.base` (`src/system/board/board.base`, "By lane," `age_min` on `as_of`). ‚Äî **Why:** the Knowledge and Project gates need their own composed collections; hubs fold into a single view rather than a heavyweight dedicated base ("avoid a dedicated base for a handful of files," ¬ß3). `projects.base` is a clean-slate replacement for `project-gate.base` that also folds in `gap` cards; `board.base` is the read layer for the one shipped projection. The projected *telemetry* bases (`fleet-health`/`eval-trend`/`skill-lifecycle`) were deliberately **not** created ‚Äî they depend on the deferred projector engine (ADR-81 deferred set).

- **What:** The home Dataview status strip is the single surviving cross-collection surface; everything else is Bases. ‚Äî **Why:** the primitive-selection rule (`ui-architecture-alpha7.md` ¬ß1/¬ß6): a Base row is exactly one markdown file and can't join across collections, so the "reviews pending ¬∑ blocked ¬∑ HIGH/CRITICAL" one-liner is the *one* thing that must stay Dataview. (Its exact composed queries were left as an "open before merge" item, #664.)

### Discovery / navigation chrome (Portals)

- **What:** Portals is adopted as the left-sidebar folder navigator ‚Äî added to `community-plugins.json`, vendored under `src/.obsidian/plugins/portals/` (main.js/styles.css/manifest/data.json), and provenance-locked. Config: six folder portals (`inbox`, `catalog`, `notes/sources`, `notes/claims`, `notes/hubs`, `projects`), `replaceFileExplorer: true`, `hiddenItems` routing `system/` + `.memoria` to Hidden, `tagNotesFolderPath: system/_tag-notes`, and trimmed side modules (`recent/bookmarks/context-notes/trash`) ‚Äî dropping `journal` and `properties` (`portals-design.md` ¬ß4/¬ß6, `docs/reference/obsidian-plugins.md`). ‚Äî **Why:** the direct fix for **#667** (raw type-first file tree is unintuitive) and **#663** (`system/` shouldn't be user-facing). The design pins Portals to a strict lane: it *reaches* folders, it does **not** classify (that's frontmatter), does **not** switch gates (verified: it pins folder/tag "spaces," not notes, and has no public API), and does **not** query (that's Bases) ‚Äî the "three orthogonal layers" boundary (`portals-design.md` ¬ß5). No tag-portals because Memoria classifies by typed frontmatter, not `#tags`, so tag-portals and the auto `_Tag Notes` folder are dead weight (¬ß1); `journal`/`properties` modules dropped because there's no daily-note workflow and properties are redundant with the Bases/badge tier (¬ß4).

- **What:** Portals *replaces* the file explorer only via its own `replaceFileExplorer: true`; the core `file-explorer` plugin stays enabled as fallback. ‚Äî **Why:** Portals replaces a *core Obsidian surface*, so if it breaks on an upgrade, navigation breaks ‚Äî disabling the core plugin would also break reveal-in-explorer, context menus, drag-and-drop, and Portals' own integration (`portals-design.md` ¬ß7, `ui-architecture-alpha7.md` ¬ß10). This is a gated adoption under ADR-74, not a free "adopt."

### UI / navigation shell (the headline change)

- **What:** Gate navigation is re-implemented as **four dashboard notes in one persistent shell**, retiring the ADR-68 workspace-swap. New `src/gates/{inbox,library,knowledge,project}.md`, each `type: gate`, embedding its gate's Bases views via `![[base#View]]`, carrying empty-state `[!brief]` copy and a **wikilink nav row** header (`**Inbox** ¬∑ [[gates/library|Library]] ¬∑ ‚Ä¶`). ‚Äî **Why:** a gate is a *job*, not a saved pane layout; implementing it as an Obsidian workspace over-implemented the requirement ‚Äî a layout swap tears down the PI's open notes (violates "preserve context"), must be re-vendored as plugin state (fights ship-in-vault), and needs an extra plugin (`workspaces-design.md` ¬ß0‚Äì¬ß4; ADR-81 Context/Decision). The switch is *"opening a different note,"* not *"rebuilding the window."* Base-embed + per-view targeting (`![[claims.base#By maturity]]`) and tab-reuse on a plain wikilink click were both **sandbox-verified** (Obsidian 1.12.7), which is what made the zero-plugin nav row shippable.

- **What:** The gate set is **Inbox / Library / Knowledge / Project**; the triage gate is renamed **Desk ‚Üí Inbox** and **Studio is deferred**. ‚Äî **Why:** derived from JTBD verbs against surfaces that actually ship (`workspaces-design.md` ¬ß3). The ADR-68 "room metaphor" (Desk/Library/Studio) *broke* once Knowledge and Project gates were added ‚Äî those aren't rooms ‚Äî so "Desk" became the odd label; "Studio" was the spatial-synthesis/Canvas mode, and Canvas is deferred, so shipping a Studio gate would violate the "no gate whose content is deferred" rule (R7). ADR-81 explicitly **overrides ADR-68's rejection of "Inbox"**: the collision with `inbox/` is coherent (the gate *is* the inbox queue) rather than harmful, so it's the one-name-per-thing rule, not a violation.

- **What:** `src/home.md` is gutted from a ~133-line control panel (Dataview status strip + Act buttons + Workspaces buttons + Dashboards suggestion callouts) down to a single `[[gates/inbox|Open Inbox]]` link plus a "Homepage opens Inbox; this is a fallback" note (`src/home.md`). The Homepage plugin's target changes `home` ‚Üí `gates/inbox` (`src/.obsidian/plugins/homepage/data.json`). ‚Äî **Why:** the PI should land on "what needs me?" every session, not a separate control panel (ADR-81: "launch opens the working gate rather than a separate control panel"). Global actions moved from `home.md` Buttons into the Commander ribbon so they're reachable from any note, fixing #664's column-of-buttons.

- **What:** Commander loses its three workspace-switch ribbon entries (`open Desk/Library/Studio workspace`) (`src/.obsidian/plugins/cmdr/data.json`); the ribbon keeps capture √ó3 + delegate + resolve. ‚Äî **Why:** the gate switcher is now the dashboard nav row, so the ribbon carries only the cross-cutting *global* actions (capture/delegate/resolve), which are gate-agnostic (ADR-72 action layer; `workspaces-design.md` ¬ß4 "capture is cross-cutting chrome, never a gate"). Gate-switch ribbon buttons were made *optional* (only if the QuickAdd open-note choices are adopted for hotkeys) and not shipped.

- **What:** Six schema-generated capture forms replace the two hand-maintained ones. Modal Forms `data.json` grows from `source`+`project` to `fleeting/source/claim/hub/project/thesis`; a new generator `scripts/gen-forms.py` (194 lines) emits the form stanzas from `.memoria/schemas/types/*.yaml` + `system/vocabulary.md`, with an extended `tests/test_modalforms.py` drift test. `source` gains `evidence_level`; `entity`‚Üínote-picker scoped to `catalog/`; `sources`‚Üícatalog-sourced multiselect; `output_mode`‚Üíradio (`src/.obsidian/plugins/modalforms/data.json`, `scripts/gen-forms.py`). ‚Äî **Why:** Modal Forms reads its own `data.json`, not the YAML schemas, so the two silently drift; generating the form from the schema (with a failing pytest on drift) makes the schema the single source of truth (ADR-71 structured capture; `ui-architecture-alpha7.md` ¬ß4 "The schema‚Üíform consistency mechanism is load-bearing"). Control-by-cardinality (radio ‚â§4, dropdown 5‚Äì12, note-picker for vault refs) kills broken-wikilink entry *at the source* (¬ß4 clean-slate deltas). Forms deliberately **don't** cover inbox cards (engine-written), catalog entities (ingest engine), or projected rows.

- **What:** `src/.obsidian/app.json` ships Memoria-tuned editor settings (was `{}`): `showInlineTitle:false`, `readableLineLength:true`, `newLinkFormat:absolute`, `alwaysUpdateLinks:true`, `newFileLocation:folder`‚Üí`notes/fleeting`, `attachmentFolderPath:attachments`, `trashOption:local`, `propertiesInDocument:visible`. ‚Äî **Why:** each is a targeted fix (`ui-architecture-alpha7.md` ¬ß10 table): `showInlineTitle:false` kills the filename+H1 duplication (the direct #659 fix); `newLinkFormat:absolute` gives links the folder prefix the link-color CSS keys on; `alwaysUpdateLinks:true` makes readable-slug renames safe; `newFileLocation:folder` sends a raw Ctrl-N note to fleeting instead of root clutter; `trashOption:local` feeds Portals' Trash module. *(Note: `newLinkFormat:absolute` was flagged "must-verify" against nested-map backlink resolution ‚Äî an accepted open risk, since the orphan/contradiction views depend on that backlink behavior.)*

- **What:** Core-plugin toggles: `tag-pane`, `daily-notes`, and `templates` all flipped `true ‚Üí false` (`src/.obsidian/core-plugins.json`). ‚Äî **Why:** `tag-pane` is low value (Memoria classifies by frontmatter, not `#tags`); `daily-notes` is unused (capture is form-based `fleeting`); core `templates` is superseded by vendored Templater (`ui-architecture-alpha7.md` ¬ß10). Lean disables, no back-compat cost given the sandbox-only vault.

- **What:** Empty-state copy is treated as a **release decision**, not a footnote ‚Äî every gate dashboard carries a `[!brief]` "nothing here yet?" callout, and G6 gates it. ‚Äî **Why:** alpha.7 ships a near-empty vault; first impressions of the whole UI form on empty Bases, so "empty must read as intentional, not broken" (`ui-architecture-alpha7.md` ¬ß8, release plan G6). This was elevated from an open-items line to a top-line release risk in the honest-reframe pass (commit `954c74bf`).

### Removed / retired (and why)

- **`load-workspace.js` and `open-project-gate.js`** deleted (`src/system/scripts/`), plus the QuickAdd workspace macros and the three Commander workspace ribbon entries. ‚Äî **Why:** the workspace-swap model is retired *entirely*, with **nothing** replacing it ‚Äî the switcher is now vault-native dashboard nav rows. This is the net-simplification argument for the whole redesign (`workspaces-design.md` ¬ß6; ADR-81 Consequences).
- **Workspaces Plus: never adopted.** ‚Äî **Why:** it was the planned mechanism to make ADR-68's swap "shippable"; the dashboard-note model removes the need for a workspace-switching plugin, resolving **#666 by elimination** ("there is no workspace-switching mechanism to build," `workspaces-design.md` ¬ß6).
- **`workspaces.json` collapsed** from three per-gate layouts (Desk/Library/Studio) to a **single "Memoria" reset layout** (`src/.obsidian/workspaces.json`), with the core Workspaces plugin kept enabled only to *reset* a disarranged window, never per-gate (`saveOnSwitch` off). ‚Äî **Why:** reproducibility (ADR-55) without the per-gate swap; a mode change is a change of focus, not a window teardown (`workspaces-design.md` ¬ß4).
- **`home.md` control-panel** (status strip, Act/Workspaces/Dashboards button clusters) removed. ‚Äî **Why:** replaced by launch-into-Inbox + Commander ribbon + gate dashboards (ADR-81).
- **`map:graph-claims`/`map:canvas-hub` removed from the deferred list** in `hermes-cli.md` ‚Äî the graph-visualization pair now ships (#381); only the calibration-gated `map:score-writability`/`map:score-readiness` remain deferred.
- **Deferred (not retired), recorded in ADR-81:** the general projector engine beyond the board mirror, projected telemetry bases (`fleet-health`/`eval-trend`/`skill-lifecycle`), the Canvas/argument-graph spatial axis (and the future Studio gate), the dedicated edge-authoring "relate" control, and a tab/title-bar prose-title plugin (replaced for now by `showInlineTitle:false` + title-led Bases). ‚Äî **Why:** for a near-empty alpha vault, "fewer surfaces + more Dataview + accept manual refresh beats building a reconciler"; each deferred item is gated on a real triggering need (vault size, demonstrated demand), not a release deadline (`ui-architecture-future.md` ¬ß0 gating-triggers table).
