## alpha.15 — The clean-slate cut: standalone engine, four-type model, quarantine-and-verify, "checked ≠ approved"

**Theme:** alpha.15 is the release where Memoria stops pretending to be a Hermes/Obsidian product and becomes what it shipped as: a standalone local CLI + SQLite engine whose files own *meaning* and whose database owns *verdicts*. Two "cut over" PRs (#1164 concept model, #1165 CLI surface) collapse a 12+ type zoo into four Concept types and rename the whole command surface, while six consolidation ADRs (125–130) retire ~70 fine-grained ADRs into subsystem-sized decisions. The driving pressure was twofold: (1) the alpha.14 architecture cut had landed on `main` with *no decision-authority ADR*, so the corpus still described the retired Hermes runtime as current; and (2) an adversarial literature pass refuted the founding epistemic premise — that a human reviewing each machine write improves correctness — which, combined with alpha.14's now-reversible errors, let the team replace the structural write-gate with quarantine-and-verify plus calibrated, thinned human checkpoints. The result is the mature alpha end-state that beta.1 later diverged from.

What drove it (2–3 sentences): the release-plan frames alpha.15 as *"a delta on alpha.14 … pins the clean-slate stack and lands the integrity mechanisms that make the engine defensibly correct."* The ADR-128 Context section records the epistemic reversal from the `_papers/` adversarial pass (Jacobs 2021; AutoResearchClaw; Feng & Liu verification-asymmetry). The scope was deliberately fenced by a *negative* gate: "alpha.15 cannot close with Obsidian plugin implementation counted as release scope" (design index Scope Boundary).

---

### 1. Vault structure & concept model

- **What:** The type roster collapsed from 12+ types across three bundles to **four knowledge Concept types** — `note`, `work`, `hub`, `project`. Nine type schemas were deleted outright: `source.yaml`, `person.yaml`, `organization.yaml`, `venue.yaml`, `digest.yaml`, `adapter.yaml`, `operation.yaml`, `skill.yaml`, `workflow.yaml` (all `REMOVED` at 5b1ea8bc; `git show 88b7f0f5:…/types/` confirms they existed prior). A new `work.yaml` was added. (PR #1164, commit `71ec0dc4`; `vault-template/.memoria/schemas/types/`.) — **Why:** the roster "grew by accretion … each with folder homes, lifecycle fields, and per-type rules. The multiplicity itself caused drift and confusion" (ADR-126 Context). The discriminator is now "a fundamentally different artifact"; a flippable check regime is a *mode*, and a thing on a different layer (a source record) is not a knowledge type. Rejected alternative: keeping claim/question/thesis as types — "flippable states and roles, not artifacts; separate types make the question→claim transition destructive" (ADR-126 Alternatives).

- **What:** `note` gained `mode: claim | question` (declared intent), `question_status: open | resolved`, and claim-context fields `temporal_scope`/`tense`/`qualifier` + `anchors`; it lost `check_status`, `note_status`, `standing`, `resource`, `quote`, `text_sha256`, `source_id`, `annotation_ref`, `extraction_confidence` (note.yaml diff). — **Why:** a mode is "declared intent creating obligations the gap engine feeds on — a claim without evidence is an `under-warranted` gap; an open question is a compose-flow lead" and "field-derived typing could not tell a plain thought from a claim missing its evidence" (ADR-126). The excerpt/quote fields relocate to a W3C `TextQuoteSelector` `anchor`, not a type (ADR-126 "Relocations").

- **What:** `hub` now **owns exactly one ID-backed controlled-vocabulary tag** via a required `tag:` field; it dropped `members: list`, `confidence`, `contested`, `project` (hub.yaml diff). Membership = "everywhere that checked tag appears"; only checked tags confer membership. — **Why:** hand-curated hub membership "rots (every relevant new note requires a hub edit)"; the owned checked tag is maintained at write time (ADR-126 Alternatives). This splits *membership* (mechanical, inclusive) from *salience* (human, curated in prose), directly answering ADR-19's recorded hazard "a generated list that looks curated but isn't" (ADR-126, disposition map).

- **What:** `project` gained `outcome_frame: map` and `paper_plan: map`, kept `thesis: str`, and dropped `scope: map` (project.yaml diff). Project is defined as "the compose-flow working frame: a nested, detachable sub-bundle with one-way project→corpus references … deliverable targets (`outline`/`section`/`figure`/`code` — **artifacts, not Concepts**)." — **Why:** absorbs ADR-77 (project gate), ADR-79 (argument graph/warrant/saturation floor → the gap engine's saturation block), and ADR-78 (thesis is now a *role* link, not a type; "no note is born thesis-designated-and-checked; a falsified thesis is a preserved result via the refuting subgraph") (ADR-126).

- **What:** `work` is a **new** type: "a source's subjective digest + tags, 1:1 with the catalog work (`work_id`)"; the objective record stays in SQLite. `digest` materialization moved onto `work` (work.yaml has `work_id` required and optional `aliases`/`archived`/`description`/`x`; `citations`/`evidence_set` existed on the deleted `digest.yaml`, not on `work`). — **Why:** the split-authority rule — files own meaning, the DB owns records/verdicts (ADR-124/125). The old `digest` type conflated the human-authored subjective take with the machine-owned bibliographic record.

- **What:** `folders.yaml` rewritten: `categories`/`bundle_roots` cut from `[catalog, knowledge, capabilities]` to `[knowledge]`; the `catalog/*` and `capabilities/*` folder-homes and staging roots removed; `knowledge/works` added, `knowledge/digests` removed (folders.yaml diff). — **Why:** catalog sources/entities are now SQLite records (not markdown Concepts), and operations became product code shipped in the package, so there is no workspace `capabilities/` tree (ADR-125: "Operations are product code"; ADR-126 relocations).

- **What:** **Meaning-only frontmatter** enforced: the universal contract is ULID `id`, `type`, `title`, `tags`, typed `links`, `aliases`, optional `archived`, plus per-type fields; `check_status`, `standing`, `note_status`, and every derived/verdict field were removed from *all* type schemas and the templates (`note.md`, `hub.md`, `project.md` diffs; `initial_check_status` key deleted everywhere). (PR #1164 + PR #1179 terminology sweep.) — **Why:** "`check_status` lived in frontmatter, so checking a note rewrote it, hand-edits forged status, and every consumer had to distrust the file it was reading" (ADR-126 Context). Rejected alternative: keeping `check_status` as a generated-but-ignored projection — "hash churn plus a forged-status foot-gun; meaning-only is stronger" (ADR-126). Note this *retires* the enforcement PR #1157 landed the same day (`id`/`standing`/`check_status` across 74 files) — the retirement negative gate had to cover #1157's just-added surface (exec-plan §Relationship to alpha.14).

- **What:** ULID identity introduced as a schema kind (`id: ulid`); `schema.py` added `ulid` and `links` field kinds and `is_ulid` validation (schema.py diff). — **Why:** "identity ≠ filename ≠ title" (ADR-126 meaning-only section) — decoupling identity from the path so renames/moves don't break references.

- **What:** Frontmatter validation flipped from **permissive to closed**: `validate_frontmatter` now rejects undeclared fields ("undeclared field: … (put custom data under x)"); every type schema gained an `x: map` extension escape hatch (schema.py diff: "Unknown extra fields are rejected"). (PR-AC #1197.) — **Why:** closes the field-closure part of ADR-119 (schema-as-single-declarative-contract), now enforced in product code rather than as a gated content edit.

- **What:** `links:` became a **nested, relation-typed map** validated against a fixed relation set `{supports, contradicts, extends}`, rejecting scalar values, absolute/URL/mail targets, and traversal-like (`..`) targets (schema.py `_check_links`, `LINK_RELATIONS`). (PR-AE #1199.) — **Why:** ADR-52's distinction carried verbatim — authored `links:` are the human/argument layer; given `relationships` are engine-built DB records (ADR-126). Guarding target shapes prevents a link from escaping the workspace or smuggling active content.

### 2. Profiles / runtime / stack

- **What:** The Hermes agent runtime, installed profiles, board lanes/fleet, and the MCP policy plugin were **dropped from the product, not deferred**; the stack is pinned to Python 3.12+, `uv tool install memoria` (with `pip install -e .` contributor fallback), Typer CLI, stdlib `sqlite3` (WAL + FK, **no ORM**), `.sql` DDL gated by `PRAGMA user_version`, pydantic-ai as the sole typed LLM runner (ADR-125 Decision; PR-A #1160/#1161/#1162; enforced Python 3.12 in PR #1216/#1217). — **Why:** alpha.14 cut Hermes out but "its decision-authority ADR was never written, so the corpus still presents the Hermes-era architecture as current" (ADR-125 Context). Rejected: staying on any agent framework — "the framework owned state, workflow, and the write boundary; every integrity mechanism had to be threaded through it"; ADR-22's "never build your own runtime" warning is "acknowledged and accepted as a cost, bounded by ADR-127's explicit contracts" (ADR-125 Alternatives). ORM + migration framework rejected: "the integrity schema is the product; single-user scale needs `.sql` + `user_version` only."

- **What:** SQLite schema formalized as a loaded `.sql` DDL file (`runtime/schema_v1.sql`, `PRAGMA user_version = 3` by release end) with tables for `operation_requests`, `journal_events`, `concepts`, `concept_verdicts`, `concept_flags`, `outputs`, `materialization_payloads`, `catalog_sources`, `external_ids`, `enrichment_runs`, `provider_payloads`, `field_provenance`, `work_graph_edges`, `work_aspects`, `derivations` (`git show 5b1ea8bc:…/schema_v1.sql`). At `88b7f0f5` the schema was inline `CREATE TABLE IF NOT EXISTS` Python with only `PRAGMA foreign_keys = ON` — no WAL, no `user_version` (exec-plan Verified baseline facts). (PR-A #1160; `be2ea1b9` "load sqlite schema from ddl".) — **Why:** SQLite is "the authority for records, operations, queue, journal, and verdicts," explicitly overturning ADR-49's rejected "SQLite: never the source of truth" (ADR-125 State).

- **What:** Console script routed through a Typer entrypoint (`typer_cli.py` added, `afa83639` "route console script through typer"); the argparse CLI with `note propose/accept/reject`, `work capture` was retired. — **Why:** verb-first canonical surface (ADR-125/design §5); the argparse surface with propose/accept was a live retirement target (exec-plan baseline facts).

- **What:** **Runner policy** — `providers.yaml` gained `runner_providers:` with two OpenAI-compatible connections (`local` → `127.0.0.1:11434/v1`, `gateway` → OpenAI with `KILOCODE_API_KEY`); operation manifests must pin per-operation `runner.test`/`runner.live`, selected by `--mode test|live`; resolved model + actual token cost journaled per run; the legacy root `providers.local/gateway` fallback was removed (PR-H #1175; providers.yaml diff; PR #1218 `c3fd37a3`). — **Why:** "no threshold in the prior corpus was ever validated"; the manifest is the hard ceiling and non-sandbox use is licensed only by the **live** model's seeded-error verdict per `(operation, model, mode)` (ADR-129 Runner policy). Removing the fallback forces explicit config so a silent default can't run unlicensed.

- **What:** `eval select-models --operation --mode` added as a build-time seeded-error-bar selector that "pins the cheapest candidate that passes the operation's bar — human-triggered and inspectable, never an online optimizer" (PR-Q #1185; `88ec2ddb`). — **Why:** an explicit **amendment to ADR-11**: whole-system vault-eval stays diagnostic and never gates work, but "`eval select-models` gating a build-time model choice is a different act and is permitted" (ADR-129 Amendment).

- **What:** Agent posture carried without the profile system — "Memoria never grants an agent file, terminal, code-execution, or send tools"; substantive coding stays delegated to external agents as the alpha.16 module target (ADR-125). — **Why:** the surviving invariant of ADR-07/32/46/48; ADR-35/43 (skill insights/governance) marked dead.

### 3. Gate / provenance / epistemics

- **What:** The founding epistemic was **reversed**: "Correctness is not conferred at write time by any approver, human or machine … A human-authored note and an LLM digest are epistemically equal." The structural per-write human gate (ADR-03) was superseded; `checked` now means only "checks passed and warrants resolve — never approved," enforced by `checked_terminology_gate.py` blocking approved/verified/trusted drift (ADR-128; PR-L #1179 `2ad8e886`). — **Why:** the adversarial literature pass refuted the premise that human-reviewing-each-write improves correctness: Jacobs 2021 (N=220) found clinician+ML accuracy 0.356 ≈ clinician-alone 0.357, both far below ML-alone 0.667, and an incorrect recommendation dragged the human *below* their own baseline (0.299, p=0.0018); AutoResearchClaw's HITL ablation showed targeted 3–6 checkpoints dominate per-step approval; Feng & Liu's verification asymmetry means the gate degrades to a cursory nod under load (ADR-128 Context, inlined because `_papers/` is gitignored). Rejected: keeping the structural gate — "it charged the scarcest resource (PI attention) for correctness it measurably did not buy"; LLM-reviewer-as-gate — "a probabilistic verdict is never a structural control" (ADR-128 Alternatives).

- **What:** The human checkpoint is now **calibrated or thinned**: humans are routed to act/ask/log only where checkpointed items are "measurably more correct than the same items un-checkpointed"; a checkpoint that can't beat its no-checkpoint baseline "is theatre and is thinned" (ADR-128). — **Why:** deliberately re-admits the consequence/uncertainty routing ADR-21 foreclosed, "with the theatre-check as the new guard in place of the blanket rule." The precondition that justified gating creation (irreversibility) "no longer holds" because ADR-127's quarantine/rollback restored reversibility.

- **What:** **Disposition telemetry** made first-class and non-backfillable: every `attention resolve` journals the routing class (act/ask/log) and outcome (apply/reject/defer) with decision time, from first ingest; apply-rate bounds (>0.9 or <0.2) are the rubber-stamp detector (ADR-128; PR-B3 #1165 §S5.6). — **Why:** "This is what makes 'calibrated' falsifiable" — without recorded dispositions the theatre-check cannot be evaluated. The borrow-doc's "state-based dogfood evaluation" (Tau-bench, EpiBench) and "personal-informatics lifecycle → dogfood metrics" pressures.

- **What:** The review queue as an **approval surface was removed**; replaced by a consequence-routed attention worklist keeping the honesty-card obligations (arguments not verdicts; batch worklists; graded loudness quiet/notice/alert/block) (ADR-128 Consequences; ADR-130). — **Why:** ADR-51's honesty card and ADR-121's enqueue-only rule carried verbatim; the inbox-as-Obsidian-category mechanism is dead.

- **What:** From ADR-57, "the writer half survives; the disposal half does not": any rule-derivable write is engine code, never an LLM ("engines write"); "agents judge, PI approves" replaced by "checks route, propagation protects, calibrated humans decide where they measurably help" (ADR-128). — **Why:** pinned models "bound drift, they do not buy reproducibility," so deterministic engines must own rule-derivable writes.

### 4. Integrity, quarantine & recovery

- **What:** **Universal quarantine-and-verify invariant**: every mutation to tracked content, regardless of origin, is non-consumable until a verification pass dispositions it act/ask/log; one shared check suite, two ingresses (operation lifecycle for engine writes, `workspace scan` for foreign edits); detection covers **all tracked paths** including `references.bib` and `.memoria/overrides.jsonl` (ADR-127; PR-C3 #1168 moved the projection change detector into `runtime.projections`). — **Why:** alpha.14's engine "leaves the guarantees implicit"; ADR-128's epistemics are only safe if "errors become detectable, attributable, and reversible instead of gated at creation" (ADR-127 Context/preamble).

- **What:** **Consumption guard (refuse + enqueue)**: at machine consumption the on-disk hash must equal the recorded `checked` hash; mismatch refuses to serve and enqueues scan — no write on the read path; scan demotes under the lock (`runtime/read_barrier.py` added; PR-C1 #1166 enqueues `observe-pi-edits` with `schedule_id=read-guard`). — **Why:** synchronous demote-on-read rejected — "a write on the read path races the workspace lock; refusal alone is already fail-closed" (ADR-127 Alternatives).

- **What:** **Two-tier demotion propagation**: depth-1 machine consumers flip to `unchecked`; PI-authored descendants raise an ask-routed flag and stay live (never auto-destroy human work); depth-≥2 machine descendants get a DB `stale` flag (a derived signal, excluded from correctness-bearing consumers but still served to `ask` with mandatory stale framing); retraction-driven demotions propagate one level deeper before going stale (ADR-127; PR-C2 #1167). — **Why:** the build-system stale-vs-invalid lesson, "so one demoted foundational node cannot cascade a whole subgraph into an attention flood"; hard exclusion rejected because "hard exclusion breaks cross-period queries."

- **What:** **Cross-store recoverability (observable atomicity)**: 2PC with journal states `pending`→`materialized` (materializing commit SHA recorded after `git commit`), write-ahead fsync ordering (payload fsynced before the DB commit that references it), one workspace-scoped write lock per operation, crash-at-any-boundary recovery via `workspace recover`; connections pinned WAL + `synchronous=FULL` (ADR-127; PR-D1 #1169, PR-D2 #1170; `worker.py` +306 lines). PR-D3 kept the deployed `pending` name rather than adding a separate `prepared` enum (exec-plan PR-D3). — **Why:** ADR-22 warned against owning durable state/retries/crash-recovery; ADR-125 accepts the cost and "ADR-127 is where that obligation is paid" (ADR-125 Consequences).

- **What:** **Git/journal division**: git owns file-tree history + the tamper-evidence root via the tracked `.memoria/journal-head` anchor (newly *created* — at `88b7f0f5` `journal_head()` was a DB query only, no tracked file); the SQLite journal stores git refs (never file bytes) and is row-hash-chained; one commit per operation staging **only** materialized paths + control files, **never `git add -A`** (a swept foreign edit would bypass quarantine) (ADR-127; PR-E1 #1171). — **Why:** narrows ADR-104's rejection of hash-chaining (which targeted an in-vault log file) — "DB rows plus a git anchor get row-level tamper evidence without that cost." Git-as-sole-history rejected: "verdicts and the records graph are relational, high-cardinality, and deliberately not file-recoverable."

- **What:** **Storage recovery classes**: files own authored meaning (file-recoverable); provider-derived records replay from content-addressed blobs; verdicts/journal are DB-only accepted-loss; **DB loss fails closed to `unchecked`** (rebuild ignores any forged frontmatter status); human `work update` dispositions live in the git-tracked append-only `.memoria/overrides.jsonl`, **tiered on DB-loss replay — restrictions reapply, permissions re-earn** (ADR-127; PR-B #1163, PR-D2). — **Why:** "the journal witness is gone, so the log cannot be trusted to *unblock* anything"; "No human-authored datum is ever in the accepted-loss class." A three-store backup contract (git remote, SQLite streaming replication à la Litestream, blob file-sync) is documented and `doctor`-reported, "never a runtime dependency."

- **What:** `golden_restore.py` **removed** (155 lines deleted; `b12ac8a1` "Remove golden restore product path", PR #1206); operation/capability policy relocated out of the workspace `capabilities/` tree into packaged product data, `memoria init` no longer seeds `capabilities/`, capability-index output is a local `.memoria/index/` cache (PR-E2 #1172, `7d071d8f`; `capabilities/` tree and `capability-index.json` 1421-line file deleted). — **Why:** package reinstall is the restore path for product files (supersedes ADR-26/55/74 repo-as-install-unit + golden-copy); behavior changes are now PR-reviewed code, "a stronger review than a gated content edit" (ADR-125).

### 5. Pipeline / ingest / enrichment

- **What:** **Enrichment merge by static field authority**: per-field best-source-wins with provenance; references are the DOI-deduped **union** across providers; author sub-fields never paired by index across sources; `provider_coverage = full|partial|degraded`; high-authority conflict marks `degraded` and raises attention (never silent overwrite); `update-work` cannot mark degraded coverage checked (ADR-129; PR-F #1173, `runtime/enrichment.py` ~+133; new `field_provenance`/`enrichment_runs`/`provider_payloads` tables). — **Why:** supersedes ADR-30's tiered-ingest merge; "no single provider is complete"; "provider output is claims, not ground truth" (ADR-129). ADR-30's spike lessons (capture-commits-first, scriptable-before-LLM) are binding.

- **What:** Two operational drift policies bound: **work-ID merges raise attention and never auto-merge** works/digests (historical provider IDs kept as aliases; per-field drift journaled); anchors are **re-anchored, never silently rebound or dropped** across extractor upgrades (exact-then-fuzzy; failures → orphaned-anchor attention) (ADR-129). — **Why:** "providers drift, extractors upgrade" — "the Hypothes.is lesson" (borrow-doc "untrusted data sealing"/provenance pressures).

- **What:** Semantic Scholar is **conditional, default-on when keyed** (reference union, citation intents as free Tier-0 contradiction candidates, TLDRs, SPECTER embeddings; never promotion-required); scite.ai is optional opt-in (ADR-129). — **Why:** "Like every provider and judge: proposers, never authorities."

- **What:** **Untrusted-input sealing**: operation manifests declare untrusted fields; the runner serializes them into sealed data blocks — "raw source text never interpolates into an instruction template (a manifest violating this fails to load)"; every text-reading operation carries a negative poisoned-input fixture (ADR-129 Runner policy; `classify.py`, `extract.py`, `runner.py` diffs). — **Why:** borrow-doc "untrusted data sealing" pressure (Greshake, AgentDojo, InjecAgent) — sealing "narrows the channel without making the model unsteerable."

- **What:** **Deterministic edge extraction (raw)**: on materialization, explicit-typed body links extract to **unchecked candidate edges** with zero LLM calls; bare wikilinks yield at most weak `mentions`/`related`; argument-class edges (`supports`/`contradicts`/`extends`) are **never** auto-promoted; act-promotable allowlist is `{mentions, related, cites, version-of}`, tuning deferred (ADR-129; PR-J #1177, `link.py` diff; writes `edge-candidate` Inbox prompts). — **Why:** auto-typing bare wikilinks rejected — "corrupts the argument graph; a bare link means 'see also'" (ADR-129 Alternatives).

- **What:** **Work-aspect read model**: schema v3 `work_aspects` table populated from CSL aspects + Markdown full-text sections (context/key_idea/method/outcome/limitation/assumption); included in checked Work search documents (PR-AB #1196, `5b426316`). — **Why:** ADR-99 (MASSW aspects) accepted "via the DB `work_aspects` read model … Keep files simple" (disposition map; borrow-doc "structured work aspects," MASSW/Knows) — adapted as DB sidecars, *not* author-facing YAML.

### 6. Retrieval / derived store

- **What:** Retrieval stays on **qmd for alpha.15** after a baseline-gated spike; the committed verdict lives in `tests/fixtures/retrieval-substrate-spike.json` and `runtime/retrieval_substrate.py` (new); SQLite FTS5 covers lexical cases but "the packaged product does not yet carry sqlite-vec plus a local embedding pipeline, so the semantic/hybrid replacement bar did not clear" (ADR-125; PR-A #1162, `d502ac20`). — **Why:** the honest premise that no threshold was ever validated — the replacement had to *beat a declared baseline* on a committed fixture, and it didn't (ADR-129 "each tier must beat a declared cheap baseline or it is disabled").

- **What:** Checked-only retrieval enforced through the read barrier: `include_stale` parameter on `checked_search_documents`; stale content served to `ask` only with mandatory stale framing, excluded from promotion inputs/export-readiness/gap scoring (ADR-127; `search_index.py` +89, `retrieval_substrate.py`). — **Why:** "hard exclusion breaks cross-period queries" — stale is a derived signal, not a verdict change.

- **What:** Generated `index.md` bundle projections described as "OKF-style" (`projections.py`, `BUNDLE_ROOTS = ("catalog", "knowledge")`). ADR-107 (OKF interchange bundle) stays **proposed/deferred**, but ADR-126 records a *new* decision: "the project layer's internal OKF sub-bundle is a new decision … deliberately beyond 107's boundary-only stance" (disposition map). — **Why:** OKF's convergence with Memoria's shape "is validation of ADR-126/125," but "OKF's defining trait is the inverse of Memoria's thesis" (mandatory permissive consumption vs. quarantine), so full OKF import/export stays deferred (ADR-107 Context).

### 7. Discovery, gaps & machine judgment

- **What:** **Native gap analysis**: checked-only inputs, canonical gap kinds (`full-text-missing`, `citation-neighborhood`, mapped `argument-*`), fixed `{0,1,2}` ordinal severity (`score 0..8`, bands ≥4/≥2/else), `--json` findings with `why`/`next_actions`/`candidate_work_ids`, project `saturation` block, discovery candidates as unchecked attention rows (never promoted Works) (ADR-129; PR-G #1174, `3fa82bc6`). Saturation = a claim with ≥1 addressed support AND ≥1 counterpoint. Advisory, never gating. — **Why:** absorbs ADR-79's saturation floor; "gap analysis reads checked inputs only" so unverified content can't drive the compose-flow phase signal.

- **What:** **Discovery relevance scoring** (ADR-92 accepted): `analyze-gaps` annotates provider-discovered candidates with deterministic `steering.md` relevance, sorts ranked ahead of a separate exploration channel; "model embedding similarity remains out until candidate abstracts/full text are reliably present" (PR-AA #1195, `79b2e6a6`). — **Why:** the cheap deterministic path clears the bar; ML deferred until data exists.

- **What:** **Tag candidates from gaps** (ADR-93 accepted): `analyze-gaps` runs a deterministic stdlib phrase pass over checked Work text, suppresses existing vocabulary/frontmatter terms, emits unchecked `inbox/` tag-candidate cards, never writes tags — "KeyBERT or YAKE stays unnecessary until real corpus misses prove it" (PR-W #1191, `1977cd14`). — **Why:** simplest-design-that-meets-the-requirement; avoids adding ML deps speculatively.

- **What:** **Record-linkage attention** (ADR-94 accepted across #1190/#1192/#1193/#1194): `check-source-metadata` flags checked sources sharing exact external IDs, title/year/first-author blocks, duplicate person ORCID/OpenAlex IDs, and normalized entity name blocks as `record-linkage` findings with stable `inbox/` prompts — **never merges records** (`detectors.py` +66). "fuzzy thresholds stay out until real corpus misses justify them." — **Why:** deterministic ID + string-blocking clears the bar; automatic merges are irreversible and unvalidated (ADR-94 disposition).

- **What:** **Contradiction Tier-1 (NLI proposer)** (PR-I #1176): `surface-tensions` runs a pinned local NLI comparator, tri-class SUPPORTED/REFUTED/NOTENOUGHINFO with first-class abstain and an FP budget, canonical-id dedup; it may gate candidates **only after passing a HANS-style overlap-but-opposite acceptance set**; failing HANS is a **loud degraded mode** (cosine/BM25 → ask, attention raised) — never a silent no-op; no automatic `contradicts` writes (ADR-129; `retraction.py`, `eval_score.py` diffs). — **Why:** "NLI shares cosine's lexical-overlap blind spot"; "NLI silence is never presented as 'no contradiction.'"

- **What:** **Tier-2 tension judge** (PR-AF #1200, `0a1c1b09`): an LLM judge admissible **only** on Tier-1 abstain/HANS-degraded hard cases, retrieval-grounded, abstain-first, temperature 0, requires quote grounding from both consumable checked files, refuses dirty DB-checked files through the read barrier, never writes checked `contradicts` links — "a proposer, never an authority; its verdict never promotes or demotes anything by itself." — **Why:** LLM-judge-as-authority rejected (GBrain's failure; "~28% wrong belief-flips in ConCoRD-class systems") (ADR-129 Alternatives).

- **What:** **Shadow-first calibration generalized** (ADR-66 → the rule for every score): a new score reports/journals but changes no routing/promotion/visibility "until its thresholds are grounded on recorded PI dispositions with a named fixture and pinned model"; the **clustering-config surface was removed from `calibration.yaml`** (PR #1204); the file itself and its `entity_resolution`/`classify`/`hybrid_scores` floors **survive** at 5b1ea8bc (ADR-129). — **Why:** "Every tunable number ships behind `production_enabled: false` semantics; the first grounded thresholds arrive with beta-era disposition data." None of calibration.yaml's constants were ever validated, so all scoring stays shadow-first; only the stale clustering-config section was removed (PR #1204 "remove stale clustering config surface").

### 8. Surfaces / UI / navigation / distribution

- **What:** The engine's **JSON read-API is the sole read path** for every Memoria-built surface and agent; alpha.15 implements CLI (`--json`, transport 0), scoped **FastMCP** stdio server, and **loopback HTTP** server as "pure marshalling skins" over one `engine/api` module; "a wrapper holding logic is a gate failure" (ADR-130; PR-K #1178 `engine/api`, PR-R #1186 HTTP, PR-T #1188 MCP behind `.[mcp]` extra, PR-U #1189 view-specs). — **Why:** "Under ADR-125 the engine is the product and every surface is a client"; MCP-as-product-surface rejected — "MCP is one transport over `engine/api`; the CLI remains the no-daemon recovery surface" (ADR-130 Alternatives).

- **What:** **Per-agent read scoping** enforced (`read_scope` filtering in `engine/api` before transports see payloads; out-of-scope reads return the not-found shape; MCP requires a non-root scope; HTTP parity added) (PR-S #1187, PR-AD #1198, `52ab9ad7` "enforce http read scope"). The first-rung all-vault CLI agent is "trusted-local dogfood only, never a supported safe surface." — **Why:** the lethal-trifecta / exfiltration mitigation; threat model stated: "read scoping is a policy boundary for tool-limited/cooperating agents; a local agent with its own file tools is in the trusted-local class regardless of transport" (ADR-130). Direct-vault-access for agents (obsidian-local-rest-api-class) rejected — "bypasses read scoping and verdict tagging."

- **What:** **All writes enqueue-only**: carried verbatim from ADR-121 — "a UI mutates only by enqueueing — it never writes concepts, verdicts, frontmatter, or commits"; the agent contract exposes plan/sources/assumptions/unknowns/pending-writes before any enqueue (chat transparency), chat additive not primary (ADR-130). — **Why:** ADR-121 carried verbatim as "the universal UI write rule" (disposition map).

- **What:** **Declarative verdict-carrying view-specs** from a fixed component vocabulary (table/card); clients render known kinds only; agent-generated components, shared mutable agent↔UI state, and rendering untrusted text as active content are rejected (ADR-130; PR-U #1189). — **Why:** carries ADR-102/116 "one view definition, projections disposable and never a second truth"; framework-hosted UX (CopilotKit-class) rejected — "inverts the boundary via shared mutable agent↔UI state." ADR-103 (projected Canvas spatial axis) **rejected**: "spatial-at-scale refuted; tabular/faceted is the validated default; small-graph view must beat a tabular baseline" (disposition map; borrow-doc visualization defaults, Heer/Bostock).

- **What:** **Honesty-card + code-review ergonomics** (ADR-51 carried, extended): attention items present argument-for/against, what-tipped-it, and certainty (never a verdict on a proposal); high-cardinality decisions become one batch worklist; loudness graded; sessions are bounded, presentation span-first with source evidence adjacent (the Elicit table pattern), `--apply` one click — "review effectiveness collapses beyond ~60 minutes/bounded batches" (ADR-130; PR-O `--ready-only` export, attention view-specs). — **Why:** "Making each review cheaper is the other half of ADR-128's bargain" (borrow-doc "static review cockpit," Horvitz/Amershi/Shneiderman).

- **What:** **Serve-hosted file watcher**: `memoria serve --watch` (stdlib polling over the existing `workspace scan`/`observe-pi-edits` path; `--once` for deterministic gating) — "the worker remains the correctness boundary, so the watcher adds no new promotion or demotion path" (PR-P #1184, `13411f3e`). — **Why:** watcher-as-required-daemon rejected — the accepted form is "real-time when the on-demand process runs, scan + consumption guard otherwise, guard never retired" (ADR-127 Alternatives), preserving the no-daemon architecture.

- **What:** **PI direct access carried from ADR-72** — every routine action reachable by the researcher directly without conversing with an agent; in alpha.15 the direct-access surface is the CLI plus the read-API transports; the CLI command scheme cut over to verb-first (`new note`, `work add/enrich/interview/digest`, `new hub <tag>`, `link`, `check`, `show`, `list`, `ask`, `export`; `project`/`workspace`/`vocab`/`attention`/`request`/`journal` groups); candidate instantiation via `attention resolve --apply`; `note propose`/`accept` do not exist in 15 (ADR-130; PR-B3 #1165, `16131e72`). — **Why:** ADR-72's "never agent-only" carried; the propose/accept surface was destructive-transition modeling now unified under one id via journaled mode flips.

- **What:** **Distribution** = `uv tool install memoria`, one command, no external runtime, no always-on process; scheduled work is plain CLI invocations under cron/systemd/Task Scheduler; docs reframed accordingly (README/AGENTS.md rewritten — "shared instruction layer for any human, local tool, or delegated agent"; cron scripts `--no-hermes`-style flags dropped) (ADR-125 Consequences; PR-AG #1201–#1222; vault AGENTS.md diff). — **Why:** the Hermes/board/profile install unit is gone; scheduled jobs are "operator-managed wrapper invocations rather than installer-wired cron."

### 9. Evaluation / measurement

- **What:** **Seeded-error verdict ported to the four-type schema**: `alpha15-seeded-errors.json` added; `alpha12-seeded-errors.json` (and alpha.11) **deleted** (`git diff …/system/eval/`); verdicts keyed by `(target operation, bundle, mode, provider, model, params)`; a live-only `non_sandbox_licensed` flag; the seeded stale-source fixture moved from CSL `standing` into DB Concept flags (`seeded_errors.py` diff; PR-M #1180, PR #1219 `f47fcf88` required the alpha15 bundle as the only one). — **Why:** ADR-10 (PR 10 harness) already existed on main, so this was a **port not a build** (exec-plan §Relationship to alpha.14); the old bundles referenced retired types/`standing` frontmatter and had to be deleted for the four-type cut.

- **What:** **CLI dogfood checkpoint reporter**: `scripts/alpha15_dogfood_checkpoint.py` (read-only workspace-state reporter — request counts, checked/unchecked backlogs, attention counts, per-work interaction count, unchecked-backlog slope, model/tool-call budget proxies) executed on `~/Memoria-test`; evidence in `dogfood/` (baseline + after-CLI JSON, provider/interview replay, Markdown export) (PR-N #1181; dogfood report shows the minimum loop end-to-end with a `missing-thesis` structural gap correctly surfaced). — **Why:** borrow-doc "state-based dogfood evaluation" (Tau-bench, AgentDojo, ScienceAgentBench, EpiBench) — "Build one tiny Memoria-specific harness for dogfood, not a general benchmark suite"; ADR-11 diagnostic-never-gates stays intact.

- **What:** ADR-11 amended (not superseded) to permit build-time model selection via `eval select-models` (see §2). — **Why:** whole-system verdicts stay diagnostic; a build-time knob choice is a different act.

### 10. Security

- **What:** **Fail-closed enforcement** as an architecture invariant: the request envelope, checks, and read barrier fail closed on any internal error — "a logged denial is policy, silence is wiring" (ADR-125; `read_barrier.py`, `trusted_writer.py` +185). — **Why:** ADR-28's load-bearing lesson — "a gate that proceeds on its own failure is not a gate."

- **What:** **No agent tool grants**: Memoria never grants an agent file/terminal/exec/send tools; untrusted work text returned as tagged data, never instructions; loopback HTTP binds only loopback hosts, token-auth (`MEMORIA_HTTP_TOKEN`), no raw SQLite/file routes; MCP behind an optional extra with a closed tool roster (ADR-125/130; PR-R/PR-S/PR-T). — **Why:** surviving ADR-32/46 invariant + lethal-trifecta mitigation; cross-channel credential-reuse fixture documents the residual (any local bearer credential is readable by a trusted-local agent).

- **What:** **HTTP read-scope enforcement / declared-frontmatter enforcement / prompt-injection sealing** (see §5, §8) with negative poisoned-input fixtures per text-reading operation (ADR-129; `52ab9ad7`, `19d7d006`). — **Why:** borrow-doc "untrusted data sealing" (Greshake/AgentDojo/InjecAgent).

- **What:** **DB-loss fails closed to `unchecked`**, ignoring forged frontmatter status (see §4). — **Why:** a rebuild "cannot be trusted to unblock anything" (ADR-127).

### 11. ADRs (consolidation sweep)

- **What:** Six consolidation ADRs (**125** standalone-cli-engine, **126** four-type-knowledge-model, **127** quarantine-and-verify, **128** no-write-time-oracle, **129** layered-machine-judgment, **130** read-api-surfaces-and-copi) landed in `docs/adr/` (PR #1158/#1159); ~70 fine-grained ADRs tombstoned as **superseded** (text kept, `status: superseded`, `superseded_by`), four **rejected** (95 nightly loop, 97 Writer claim-notes, 103 Canvas spatial axis, 106 Hermes cost-capture), several **stay-proposed/deferred** (16, 88–91, 98, 107, 108); ADR-124 absorbed ADR-06; the ADR index was regenerated and the README now directs new decisions to the six subsystem ADRs by default (`1a8664da`, `a3b3f1f4`). — **Why:** "a corpus of ~90 fine-grained ADRs has repeatedly caused confusion and contradiction — decisions were re-litigated, superseded-in-practice ADRs read as current, and reviews missed collisions. The unit of decision is now the **subsystem**, not the mechanism" (disposition map Consolidation rationale). Rejected-alternatives memory is never pruned — "numbers are permanent."

- **What:** In-15 proposed ADRs flipped to **accepted** as slices landed: **92** (discovery relevance, #1195), **93** (keyphrase tags, #1191), **94** (record-linkage dedup, #1190/#1192/#1193/#1194), **99** (MASSW aspects, #1196) — each on a deterministic path *without* the ML dependency the original ADR contemplated (disposition map). — **Why:** "Correct, not cheap" but "simplest design that meets the *stated* requirement" — the cheap deterministic path cleared the bar and ML was deferred until real corpus data justifies it.

- **What:** ADR-64 lost stale `-WithHermes`/`--with-hermes` installer-flag claims (#1207); ADR-130 lost stale CLI+plugin/second-wave wording (#1208); future editor adapters now **require their own ADR** rather than inheriting an alpha.15 commitment (#1203, `d5131811`). — **Why:** the negative scope gate — "adapter-specific UI needs its own ADR before it is scheduled" and "alpha.15 cannot close with Obsidian plugin implementation counted as release scope" (design index Scope Boundary; ADR-130).
