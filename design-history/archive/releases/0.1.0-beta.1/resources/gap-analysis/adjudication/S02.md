Owner rulings R-4/R-9 contest this (relational catalog, .bib as the survival artifact); on the merits the alpha line's own alpha.12 record — 'references.bib is portable but lossy relative to the catalog DB' — is the decisive counter, since only full CSL-JSON in a keep-test file satisfies the charter's Survival and Rebuild bars; the mandated re-examination pass is noted. Citekey-as-primary-key (owner hard ruling R-1) is owned by the §7 adjudicator, not this entry.

- **Recorded:** Design §3 L224-226 (CSL-JSON-in-record.md decision with rationale) and L286-292 (anchor rationale); design §11 L965 (convergence: SQLite authority + markdown keep-set + blobs); docs-history Reject #123 (source.pdf co-location for keep-test) and Reject #125 (CSL-JSON/W3C selectors as boundary subsets); §17.1 (fulltext.md as evidence substrate).

The alpha.12 catalog-recovery spike failure (periodic export loses DB-only catalog changes) argues for beta.1's file-canonical records, and alpha's own '.bib is lossy relative to the catalog' finding is the reason record.md carries full CSL-JSON.
