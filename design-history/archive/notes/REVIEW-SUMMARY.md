## Executive summary

# Executive summary

Across 401 papers (378 with adoptable/borrowable ideas in 11 categories), the literature does not merely permit Memoria's design — it independently re-derives it. The strongest cross-cutting signals, ordered by impact:

- **The "engines write, agents judge, PI approves" gate is the single most-validated decision.** HCI converges on augmentation-over-delegation (Horvitz, Shneiderman-Maes, Amershi); agent papers show that durable, schema-governed vault state + a human checkpoint is *what makes multi-agent reliable* and that removing it collapses performance (Chen File-as-Bus, Zhou harness). Treat the gate as load-bearing infrastructure, not UX polish.

- **Faithfulness is engineered, never assumed — and prose safeguards fail.** Closed-domain "no info not in source" contracts, gate-existence-before-extraction, forced verbatim source citations, temperature=0, and multi-sample self-consistency (disagreement *becomes* the uncertainty flag) are the cross-corpus recipe. Search-enabled frontier LLMs still emit ~49% wrong BibTeX; bare LLM judges err 16–46% on citation/claim calls. Verify deterministically (citekey resolves, source-span entails), never by instruction.

- **Retrieval quality — not model size or agentic search — drives grounded output.** One-line BM25/title baselines repeatedly beat long agentic deliberation. Spend the local budget on the retriever, exemplar selection, and offline index enrichment (hypothetical-question enrichment, reasoning-trace query expansion, a small-k LLM-judge reranker). This directly funds the retriever/scout profile and pre-file similarity gate.

- **Lexical/embedding cosine must never stand in for semantic equivalence.** Contradictions, supersession, and dedup belong on a local NLI/entailment comparator with Tversky-style shared-vs-distinctive weighting and an explicit false-positive budget; high embedding similarity routinely coexists with *flipped meaning*. Test every comparator on adversarial word-overlap-but-opposite-meaning fixtures.

- **Forgetting, not recall, is the universal memory failure.** Across five memory benchmarks, atomic-unit retrieval beats raw-log/summary retrieval (validating the atomic-claim store) — but every memory agent silently surfaces superseded facts and degrades over horizon. Promote claim supersession from a dashboard to a first-class **deterministic retrieval filter** over an append-only trace, tested with a forgetting-aware metric.

- **Ingested paper text is untrusted code; the sandbox is the only sound defense.** Indirect prompt injection hijacks agents and self-poisons persistent memory (Greshake, AgentDojo, AI-Scientist self-modification). Least-privilege MCP allowlists and capability denial — not prompt filters or a bigger model — are the validated answer, confirming ADR-32/46/28/27. Every engine prompt needs an explicit instruction/data split with data ordered last and output caps.

- **Cheap + local is sufficient — if you constrain the output, not the model.** A frozen small model with version-pinned prompts and a closed schema/label set matches scale at single-researcher size; a 355M classifier beats GPT-4o zero-shot. The brittleness is structured-output adherence, so invest in strict schemas, constrained decoding, self-consistency routing, and abstain paths — not fine-tuning.

- **Abstention is a feature, and evaluation must reward it.** "Reject-until-grounded / choose unknown when underinformed" is the cross-paper consensus (DOCBENCH, BBQ, TEMPO where no-retrieval beat all RAG). Certify stochastic local engines with state-based regression tests, the pass^k consistency metric, capped tool/cost budgets, and LLM-judges validated against the PI's own approve/reject labels.

## Cross-cutting themes & design implications

These themes recur across categories rather than within one, and each pressures or validates a specific Memoria design area.

**1. The human-in-the-loop gate is architecture, not decoration.** HCI/CSCW (Horvitz mixed-initiative, Shneiderman-Maes, Amershi), LLM-agent (Chen File-as-Bus, Zhou harness, Tang, Long), and claims/argumentation papers all converge on: engines produce, a verifier judges narrow spans, the human decides. Empirically, removing the durable inspectable state + checkpoint collapses agent performance. *Implication:* the structural review gate and "engines write, agents judge, PI approves" split are validated as core infrastructure. Make the gate an explicit **expected-utility act/ask/drop router** (low-risk/high-confidence auto-applied, ambiguous batched-and-ranked) — never an alert firehose. *Pressures:* gate routing, structural review gate, profiles.

**2. Engineered faithfulness over generated prose.** Across NLP, RAG, claims, KG, and domain papers: closed-domain "no info not in source" contracts, gate-existence-before-extraction, forced verbatim grounding, temperature=0, self-consistency, and deterministic hard checks (citekey resolves, source-span entails). Bare LLM judges err 16–46% on citation calls and must act only with retrieved/NLI evidence in hand. *Implication:* every claim/relation edge carries a verbatim warrant validated by a *model-free* checker (substring + entity-type + year-order + opposite-edge) at the gate; semantic correctness is deferred to agent/PI, structural validity is machine-checked. *Validates:* atomic-claims backbone, Chain-of-Evidence provenance, claim-sentence classification, the review-gate warrants.

**3. Contradiction / supersession / dedup must be NLI-based, precision-first.** Plain cosine collapses on the exact cross-paper reasoning-based relevance the argument graph targets, and high similarity coexists with opposite meaning. The dominant hard case is NO-RELATION / NO-EQUIVALENCE, so conservative defaults with a **tunable abstain threshold** prevent over-linking. Tri-class SUPPORTED/REFUTED/NOTENOUGHINFO with evidence sentences; a false "fabricated"/"duplicate" flag is the costly error, so route uncertainty (POTENTIAL) to the PI. *Implication:* contradictions dashboard, record-linkage dedup, and supersession run on a local NLI comparator with adversarial word-overlap fixtures and an explicit false-positive budget; dedup keyed on canonical IDs (LLMs reliably mint near-duplicates). *Pressures:* contradictions dashboard, claim supersession, record-linkage entity dedup, pre-file similarity gate.

**4. Retrieval quality is the dominant lever.** Across RAG, retrieval, and memory categories, retriever quality beats model size and agentic search; BM25/title baselines beat long deliberation; temporally-incomplete retrieval actively *harms* output (TEMPO no-retrieval beat all RAG). Recover reasoning-based relevance via index-side hypothetical-question enrichment, reasoning-trace query expansion, and a small-k LLM-judge reranker — not off-the-shelf cross-encoders. *Implication:* invest the local budget offline (enriched index, exemplar/selective-annotation) and keep online retrieval deterministic and cheap (union dense+sparse, token-budgeted pyramid expansion, tight top-k) to fit qwen2.5:7b. *Validates/pressures:* retriever-scout profile, discovery relevance scoring, learning-to-rank triage, nightly discovery loop.

**5. Memory's failure mode is forgetting; time is a first-class field.** Atomic-unit retrieval beats raw-log/summary (validates the atomic-claim store), but memory agents silently surface superseded facts. A newer in-context claim overrides stale belief ~85% of the time (Si) — so *feed superseding claims into the prompt and flag the conflict* rather than trusting memory. Stamp each claim with ISO temporal scope + tense + confidence (TEMPO) so supersession ordering and the contradictions dashboard reason over explicit windows. *Implication:* make supersession a deterministic retrieval **filter** over an append-only trace, not just a dashboard; add cross-period coverage metrics to the scout. *Pressures:* claim supersession, temporal scoping on claims, contradictions dashboard.

**6. The sandbox is the security architecture.** Ingested papers are untrusted code that hijacks agents and self-poisons persistent memory (Greshake, AgentDojo, InjecAgent, AI-Scientist self-modification). Least-privilege tool allowlists and capability denial — not prompt filters, not a bigger model — are the only sound defense. *Implication:* MCP-only agent sandbox is validated (ADR-32/46/28/27); every engine prompt needs an explicit instruction/data split, data-ordered-last, output caps, and perplexity-based input cleaning. *Validates:* MCP-only sandbox, profiles compiled, skill governance.

**7. Cheap-and-local works when output is constrained.** A frozen small model + version-pinned prompts + closed schema matches scale at single-researcher size (Schick-Schütze, SciLitLLM); a 355M classifier beats GPT-4o zero-shot; relation extraction/entity typing go training-free via NLI/cloze verbalizers with type constraints. The brittleness is JSON/tool adherence on small models. *Implication:* invest in strict output schemas, constrained decoding, self-consistency routing, deterministic pre-filters (MinHash/n-gram dedup, PMI keyphrase, transfer-trained quality classifiers) *ahead of* every LLM pass — not fine-tuning. *Validates:* deterministic ingest, keyphrase tag candidates, local-model-first constraint.

**8. Evaluate process and abstention, not output prose.** Score every atomic claim on grounding-to-cited-span and warrant; reward abstention ("reject-until-grounded", "choose unknown"); certify stochastic engines with state-based regression tests, pass^k consistency, capped tool/cost budgets, and random/trivial-control baselines so triage gains aren't distributional shortcuts. Every threshold/prompt/judge tuned on the PI's notes is spent supervision — freeze and version-pin, and validate LLM-judges against the PI's own approve/reject labels. *Pressures:* discovery relevance scoring, triage ranker, engine test suites, prompt versioning.

**9. Surfaces: render over inspectable artifacts, encode one priority signal.** The visualization strand (Healey-Enns, Fekete/D3, Cognitive Dimensions, Hill) says: render over native inspectable artifacts, diff-re-project by stable key, encode the single priority signal preattentively, and turn the PI's own read/edit history into a salience signal. Confident wrong outputs and tidy single-rationale explanations harm experts — surface evidence weight, provenance, dissent, a no-suggestion baseline, and *expose what the ranker down-ranked*. *Validates/pressures:* disposable projection engine, projected canvas, dashboards/gates, read-only inspector.

### What to build next (priority, grounded in strongest adopt ideas)

1. **NLI/entailment comparator as a shared service** for contradictions, dedup, supersession, and citation verification — with adversarial word-overlap fixtures and a tunable abstain threshold (Themes 2, 3). Highest-leverage: it underpins three existing pillars at once.
2. **Supersession as a deterministic retrieval filter** over an append-only claim trace, plus ISO temporal-scope stamping on every claim, tested with a forgetting-aware metric (Theme 5). Closes the universal memory failure mode.
3. **Model-free warrant checker at the review gate** (substring + entity-type + year-order + opposite-edge + citekey-resolves) so cheap local engines cannot post-rationalize provenance (Theme 2). Pure deterministic win, no model cost.
4. **Expected-utility gate router (act/ask/drop)** with auto-apply for low-risk/high-confidence and ranked batching for the rest, replacing any undifferentiated alerting (Theme 1).
5. **Offline index enrichment + small-k LLM-judge reranker** (hypothetical-question enrichment, reasoning-trace expansion) for the retriever/scout, with cross-period coverage metrics (Theme 4).
6. **Engine certification harness**: state-based regression tests, pass^k consistency, capped cost budgets, LLM-judges validated against the PI's approve/reject labels, and trivial-control baselines (Theme 8).

# Category deep-dives

## NLP Methods (summarization, classification, dedup, keyphrase)  (87 papers)

**Most significant papers**
- [Zhang et al. - 2024 - MASSW a new dataset and benchmark tasks for AI-assisted scientific workflows.md] — the literal namesake of Memoria's MASSW-style aspect pillar; supplies the canonical five-aspect schema (context/key-idea/method/outcome/projected-impact), the key-idea-vs-method disambiguation FAQ, and a hallucination-aware "N/A for absent aspects" rule the ingest engine should implement verbatim.
- [Agrawal et al. - 2022 - Large language models are few-shot clinical information extractors.md] — the core deterministic-ingest recipe: a guided prompt fixes OUTPUT FORMAT (content correctness barely matters) so a <10-LOC resolver enforces the schema; plus gate-existence-before-extraction to kill hallucinated claims and multi-prompt sampling for uncertainty flags.
- [Wang et al. - 2023 - Self-consistency improves chain of thought reasoning in language models.md] — sample-and-marginalize (5–10 paths, unweighted majority vote) gives Memoria both a faithfulness boost on brittle local-model extraction and a free, logprob-independent uncertainty/calibration signal routing low-agreement items to the review gate.
- [Schick and Schütze - 2021 - It's not just size that matters.md] / [Ouyang et al. - 2022 - InstructGPT.md] — empirical backbone for the local/cheap-model bet: a small well-framed model matches a 1000x-larger one; InstructGPT adds a ready-made per-output judge rubric (faithful? hallucinated? satisfies-constraint?) and a closed-domain hallucination-rate metric that concretize "agents judge."
- [Li et al. - 2025 - SciLitLLM.md] — Qwen2.5-based pipeline validating Memoria's exact stack: LLM-based PDF parse-error correction, a lightweight transfer-trained quality classifier to drop low-value segments, and the finding that a small curated corpus beats brute size (faithfulness-over-flash).
- [Shahi and Hummel - 2026 - Automating categorization of scientific texts.md] — local-Ollama hierarchical prompt-chaining (predict domain→subject→topic, each step constrained to children) beats flat ICL and BERT; directly buildable for taxonomy/space/tag assignment with fuzzy-match scoring.

**Adopt**
- **Constrain output to a schema/grammar, then resolve deterministically.** A one-shot example fixing format + a tiny resolver (Agrawal 2022); grammar-/trie-constrained decoding so structured output is valid by construction (Shin 2021, Schucher 2022); pin output to an explicit template with named slots (White 2023 catalog, Liu 2021 cloze+verbalizer). This is the deterministic-ingest contract for claim/relation/MASSW/frontmatter extraction.
- **Adopt the MASSW five-aspect schema + key-idea/method disambiguation + mandatory "N/A" for absent aspects** (Zhang 2024, reinforced by STRUDEL's per-slot N/A convention, Wang 2022). Per-aspect prompts (one targeted prompt per slot) over one mega-prompt so each aspect is independently gate-checkable.
- **Self-consistency / multi-prompt sampling for faithfulness AND uncertainty flags.** Sample 5–10 paths, unweighted majority vote (calibration is poor, so skip logprob weighting), and use sample-disagreement to populate the uncertainty flag and route low-agreement claims to the gate (Wang 2023, Agrawal 2022, Schick&Schütze 2021 ensembling).
- **Guard against hallucinated extractions.** Gate-existence-before-extraction chaining (Agrawal 2022); closed-domain "no info not in source" contract with a tracked hallucination-rate metric (Ouyang 2022, Xie 2022 faithfulness taxonomy); force in-text citations to a source claim/citekey on every synthesized sentence (White CliniDigest 2023). temperature=0 for all extraction/summarization calls (CliniDigest 2023).
- **Hierarchical prompt-chaining for taxonomy/tag/space assignment**, constraining each step to the children of the prior node, with tolerant match scoring (exact / normalized-Levenshtein>0.7 / embedding-cosine>0.7) so semantically-correct-but-lexically-different labels aren't penalized (Shahi 2026). Deep/fine-grained tags route to the PI as candidates, never auto-write.
- **A structured engine/judge rubric** scoring each output on overall quality + binary flags (faithful / hallucinated / satisfies-constraint / unwarranted-claim) (Ouyang 2022), plus a multi-aspect LLM-judge (clarity/complexity/correctness/usefulness, drop avg<4) for gate write-decisions (Li 2025).
- **Cheap deterministic pre-filters before any model call:** MinHash-LSH / n-gram-collision near-duplicate detection (Brown 2020), ROUGE-L/Levenshtein dedup-before-semantic-check ordering (Wang 2023 Self-Instruct, Li 2025), and heuristic invalid-generation filters (length, repetition, output-echoes-input, missing required slot) (Hämäläinen 2023, Jin 2022, Self-Instruct 2023). These feed the pre-file similarity gate / record-linkage dedup.
- **Train one small local classifier from an LLM/PI-labeled set** for content-quality scoring or claim-sentence classification, then run it cheaply at scale (Li 2025 quality classifier; Agrawal 2022 weak-supervision distillation; Du 2022 bootstrap-from-4k-labels). Keeps data on-vault, removes per-doc API cost.
- **Constrained closed-set classification:** present the explicit label set (OPTIONS suffix) and rank-classify over it; fix verbalizers to high-frequency natural label words (Wei FLAN 2022, Webson 2022, Liu 2021). Counterfactually perturb the source sentence to confirm the extractor tracks text, not priors (Shin AutoPrompt 2020). Reframe prompts via itemizing/decomposition/restraining/specialization (Mishra 2022a).

**Borrow**
- **Weak-supervision aggregation over correlated prompts** (learn per-prompt accuracy + dependency structure, not majority vote) when running several classifier variants (Arora 2022 AMA) — a step beyond plain self-consistency for high-stakes labels.
- **Contextual calibration**: feed a content-free input (N/A), measure bias, apply a diagonal affine correction; report worst-case/variance across prompt+seed variants, not just average (Zhao 2021, Min 2022 channel, Holtzman 2022 PMI_DC). Cheap reliability for similarity-gate and triage thresholds.
- **NLI/entailment as the backbone for contradictions + supersession + dedup**: cast "does new claim entail/contradict/neutral existing claim" as Matching, needing no verbalizer; use a small local NLI model for record-linkage and step-equivalence (Sun 2022 paradigm survey, Li DIVERSE 2023, Chen AdaPrompt 2022). MRC framing (relation-specific question → answer span) grounds every relation in source text for the gate (Sun 2022).
- **Coding-then-clustering** (distill each note to compact codes before embedding → UMAP → HDBSCAN → centroid-nearest-code labels) reduces embedding noise and yields human-readable cluster names; cluster over a single aspect's embeddings (Hämäläinen 2023, Zhang MASSW 2024, Ben-David DRF/MI signature terms). Feeds the BERTopic clustering MCP and projection spaces.
- **PMI / MI / class-TF-IDF n-gram extraction** for keyphrase tag candidates and overlapping-cluster labels — deterministic, local, no LLM (Diao 2023, Ben-David 2022); prune generic terms via relevance>1 and frequency refinement (Hu KPT 2022).
- **Cascading map-reduce summarization** with proportional length budgets to beat local context limits, "thesis/argument" keyword steering to force cross-claim synthesis, and a source-coverage metric as a faithfulness signal (White CliniDigest 2023).
- **Meaning-changed vs cosmetic edit classification** across paper versions to decide when a stored claim must be superseded vs left intact (Du 2022) — a concrete supersession signal on re-ingest.
- **Adversarial/challenge eval fixtures**: word-overlap-but-opposite-meaning claim pairs and near-duplicate-but-distinct claims to test the similarity gate and contradiction engine, since lexical/embedding overlap ≠ semantic equivalence (Utama 2021, Min 2022 demonstrations); report distribution across prompt variants, never a cherry-picked best (Sanh 2022, Logan 2022, Du 2022 — automatic metrics correlate poorly with human judgment).
- **Staged/least-to-most decomposition** for argument-graph and typed-relation extraction (list claims → per-claim find supporting/attacking), authored per-pipeline-stage since decompositions don't transfer (Zhou 2023, Bursztyn CFT 2022, Tan MSP 2022). Rationale-before-verdict ordering and low-temperature decoding for warrants (Wei CoT 2023, Zelikman STaR 2022).
- **Robustness/cost guidance**: cap discovery/retrieval fan-out (top-k is a relevance/noise tradeoff — Chen AdaPrompt 2022, Zhou APE 2022 staged-budget filtering); prefer stable hand-written discrete prompts over soft-prompt tuning, version-pinned in compiled profiles, since wording and few-shot ORDER swing accuracy on small models (Lu 2022, Liu P-Tuning 2023, Zheng 2022, Zhao 2021).
- **Note**: two files are the same prompting survey (Liu 2021 / Liu 2022) and are a live dedup/supersession test case for the pre-file gate.

**Theme & implication for Memoria**
This corpus overwhelmingly validates Memoria's central architectural bet — a frozen, cheap, local model (qwen2.5:7b) driven by well-engineered, version-pinned prompts beats fine-tuning or cloud-scale models at single-researcher scale — and it converges on one discipline: **constrain the model to a schema/closed label set, then enforce structure deterministically.** The clearest design move is to standardize every ingest engine as decomposed, per-aspect, schema-constrained prompts with a tiny resolver, wrapped in self-consistency sampling whose disagreement populates the uncertainty flag and gates low-confidence items to the PI (claims, review-gate, uncertainty ADRs). For contradictions/supersession/dedup, the body of work says route everything through a local NLI/entailment comparator and never trust lexical or embedding overlap as semantic equivalence — back that with adversarial challenge fixtures and worst-case (not average) metrics (contradictions, dedup, retrieval/triage ADRs). Finally, cheap deterministic pre-filters (MinHash/n-gram dedup, length/repetition/echo heuristics, PMI keyphrase extraction, quality classifiers) belong ahead of every expensive LLM pass, matching the "engines filter cheaply, agents judge, PI approves" gate and the content-light diagnostics plane.

## HCI / CSCW / Ubicomp  (85 papers)

- **Most significant papers**
  - **[Horvitz - 1999 - Principles of mixed-initiative user interfaces.md]** — Gives the formal expected-utility backbone for Memoria's gate routing: act / ask / drop chosen by inferred-probability bands weighed against the cost of being wrong. This is the decision rule for auto-write vs. surface-to-gate vs. discard.
  - **[Amershi et al. - 2019 - Guidelines for human-AI interaction.md]** — 18 validated, observable guidelines (state capability + error rate, explain why, support efficient correction, scope when in doubt, update cautiously) that translate almost one-for-one into Memoria's inspector and gate UX.
  - **[Jacobs et al. - 2021 - How machine-learning recommendations influence clinician treatment selections...md]** — Hard evidence that wrong confident recommendations drag an expert *below* their unaided baseline, and that tidy feature explanations make overreliance *worse* — the empirical case for the review gate, no-suggestion baselines, and provenance-over-rationale.
  - **[Shneiderman and Maes - 1997 - Direct manipulation vs. interface agents.md]** — The "agent runs below the table, user-facing layer stays predictable/controllable" split *is* "engines write, agents judge, PI approves"; also supplies the overview→filter→details-on-demand grammar for every surface.
  - **[Ackerman - 2000 - The intellectual challenge of CSCW...md]** — The social-technical gap + critic/augmentative architecture (many small advisory critics, none autonomous) is the canonical justification for Memoria's many-small-engines + gate design over one monolithic agent.
  - **[Bernstein et al. - 2010 - Soylent a word processor with a crowd inside.md]** — Find-Fix-Verify and the finding that "verifiers are far more reliable than generators" directly underwrite using cheap local models as judges/gates rather than trusted authors.

- **Adopt**
  - **Expected-utility gate routing**: take an autonomous action only when its expected value beats inaction, with probability bands choosing drop / ask-at-gate / auto-write (Horvitz 1999; Horvitz et al. 1998; reinforced by Sundar 2020 "reactive over proactive," Weisz et al. 2024 deliberate friction). The cheap pre-gate classifier guarding expensive extraction maps to Memoria's pre-file similarity gate / discovery relevance scoring.
  - **State capability + expected error rate, and explain *why*** on every engine surface (Amershi G1/G2/G11; Luger & Sellen 2016; Sundar 2020). Build into the read-only inspector and gate: claim-extraction confidence, ranker calibration, and the provenance/warrant behind each ranked discovery, relation, or supersession.
  - **Verifier/judge over generator** — separate the writing engine from an independent review pass that votes/flags (Bernstein et al. 2010 Find-Fix-Verify; Kittur & Kraut 2008 propose-then-curate; Wang et al. 2024 ReelFramer; Chen et al. 2023 human-in-the-loop). Cheap local models are reliable as judges, not as trusted authors — the structural review gate's core.
  - **Efficient one-click correction + scope/abstain when uncertain** (Amershi G9/G10; Yang et al. 2020 graceful failure / enumerate FP-FN; Weisz "Design for Imperfection" — surface uncertainty, offer fix/regenerate/replace). Engines should abstain or scope rather than write a confident wrong note.
  - **Show evidence weight + dissent, not a single persuasive rationale** (Jacobs et al. 2021 — feature explanations worsen overreliance; Govers et al. 2026 — balanced prose hides lopsided literature). The contradictions dashboard must surface actual support/contradict *counts with provenance*, never a tidy "X% agree" or even-handed synthesis that anchors the PI.
  - **Declarative PI-defined "situations"** wired to triggers (Dey 2001) — let the PI declare rules like "a new paper contradicts an accepted claim in active project X" that fire the contradictions dashboard / gate, rather than hand-wired logic.
  - **Computational wear** — accumulate the PI's own read/edit/revisit history on notes and claims and render it as salience/wear marks in the inspector, feeding learning-to-rank triage (Hill et al. 1992).
  - **Capture provenance/assumptions at the write moment** as first-class typed fields, not buried metadata (Kandel et al. 2012; Brandt et al. 2010 machine-readable source links re-checked on open → claim supersession/staleness). Structured capture forms intercept the annotation point and require typed provenance.
  - **Path of least resistance + low-threshold/high-ceiling + predictability over cleverness** (Grudin/Myers et al. 2000). The easiest gate action must be the well-formed one; zero-config ingest reaching deep capability via progressive disclosure; deterministic, inspectable engines over opaque end-to-end generation.
  - **Pre-built, tested prompt-backed controls beat freeform chat** (Yuan et al. 2022 Wordcraft; Jiang et al. 2022; Wang et al. 2022 Wordcraft) — surface compiled-profile skills as capture-form/gate actions in the Obsidian UI rather than expecting the PI to craft prompts.
  - **Versioned regression test set for engine prompts**, scored each revision and gating prompt changes (Jiang et al. 2022 PromptMaker; Strobelt et al. 2022 verbalizer/answer-choice tuning; low/zero temperature for reproducibility). Matches deterministic-ingest discipline.
  - **Self-disclose and let the PI correct the user model** from the Obsidian UI (Bardzell 2010 self-disclosure; Sundar 2020 ground-truthing; Gajos et al. 2008 revealed-behavior beats stated-preference) — PI accept/reject at the gate becomes the labeled signal tuning discovery/triage.

- **Borrow**
  - **Risk/importance-tiered, batched alerting**: auto-handle high-confidence low-risk, escalate only ambiguous/high-impact items, suppress trivia to avoid rubber-stamp fatigue (Alsufyani et al. 2025; Friedman et al. 2013 minimal-distraction; Waldman 2020 overchoice; Chordia et al. 2023 alert-spam anti-pattern; Brown et al. 2007 coarse "less-is-more" glanceable digests).
  - **Contrastive, on-demand explanation** — justify only when an output *diverges* from the prior standard ("what changed and why" = supersession/contradiction diff); make provenance findable, not pushed into every view, to reduce gate fatigue (Jacobs et al. 2021 sociotechnical; Alfrink et al. 2023 explanation-vs-justification).
  - **Multi-sample self-consistency / agreement-thresholding** as the single-agent analogue of crowd voting: run the judge N times at low temperature, keep only consistently flagged claims/contradictions; report confidences as calibrated distributions (Bernstein et al. 2010; Hightower & Borriello 2001; Coopamootoo & Groß 2017 codebook + kappa; Ali et al. 2019 invert-to-validate faithfulness check).
  - **Generate divergent candidates for the PI to compare/curate** rather than iterating one (Dow et al. 2010 parallel prototyping; Weisz generative variability — but reject variability in the core write path; Faste et al. 2013 tight, time-boxed shortlists). Caps similarity/discovery candidate lists.
  - **Track a "seen/dismissed" set so discovery never re-surfaces judged items; gray out visited, push toward unexplored** (Faste et al. 2013; Wattenberg & Kriss 2006 road-less-traveled; Bryant et al. 2005 watchlist-diff-since-last-check).
  - **Fuse independent weak signals** (keyphrase overlap, embedding similarity, citation overlap) for triage and record-linkage; edit-distance/regex candidate generation + a judgment pass for tag/entity variant canonicalization (Kandel et al. 2012; Hightower & Borriello 2001; Chancellor et al. 2016; Brennan 1991 establish-and-reuse one canonical label; Wang et al. 2023 NER+NP-root keyphrase hierarchy).
  - **Outlier-on-similarity for faithfulness/anomaly**: flag a note whose claim text and cited-source embedding diverge beyond a threshold; BERT/MiniLM coverage-highlighting of which source claims a summary drops (Wang et al. 2024 ReelFramer; Wang et al. 2023 DiffusionDB; Chancellor et al. 2016 — distance alone insufficient, require a judgment step).
  - **Perceptual encoding discipline for dashboards/canvas**: encode the single highest-priority signal (contradiction status, uncertainty, supersession) in a preattentive channel; precompute conjunctions ("contradicted AND stale") as their own mark; place related claims spatially close; cap categorical color (Healey & Enns 2012; Fekete et al. 2008 Gestalt/economic-value model; Zhicheng Liu et al. 2008 epistemic actions; Card & Moran 1986 mental-model surfacing).
  - **Native, inspectable substrate + data-join re-projection**: render over Obsidian markdown / JSON Canvas / static HTML, and diff projections by stable key (citekey/claim-id) on re-project to preserve layout (Bostock et al. 2011 D³; Beaudouin-Lafon 2000 instrument/object separation; Abowd 2012 design-time matches run-time).
  - **Bookmarkable view-state tokens** for self-resumption across projected views (Wattenberg & Kriss 2006); **interruption-and-resumption** surfaces showing pending review/triage state (Abowd & Mynatt 2000; Karusala et al. 2021 starred reply queue; O'Leary et al. 2018 carry-forward open threads).
  - **Scaffolded "moves" in capture forms** (state-claim / state-warrant / note-contradiction) to cut blank-page friction (O'Leary et al. 2018; Edelson et al. 1999 embedded-expertise scaffolds; Vattam et al. SBF behavior-views as warrant template).
  - **Just-in-time, in-context feature surfacing** to counter single-PI feature decay and discoverability gaps (Beneteau et al. 2020; Bryant et al. 2005 progressive disclosure).
  - **Keep PI vocabulary editable, not a locked ontology** — typed relations, MASSW aspects, and tags must stay PI-bendable (Ackerman 2000; Brown et al. 2007 users re-purpose categories; Boehner et al. 2007 preserve ambiguity; Dourish 2004 relevance is occasioned — borrow the caution, keep typed relations as the durable backbone).
  - **Anti-pattern / faithfulness constraints**: no dark patterns, no forced action, no anthropomorphic over-promising; plain factual tone on ingest/judge surfaces (Waldman 2020; Chordia et al. 2023; Luger & Sellen 2016; Wong et al. 2024).
  - **Local on-device NLP as the privacy-vs-capability tradeoff already chosen** (Kocaballi et al. 2022; Rogers 2006 local data ownership; Abowd & Mynatt 2000 "keep the bits local").
  - **Evaluate the PI+vault unit in situ, not felt-usefulness or task-completion** — held-out ground-truth concordance, value-added single-feature A/Bs, longitudinal signals (Jacobs et al. 2021; Zhicheng Liu et al. 2008; Buchner et al. 2022; Fekete et al. 2008 — insight is hard to quantify; Brown 1992 design-experiment stance).

- **Theme & implication for Memoria.** This corpus is, almost in unison, a thirty-year argument for *augmentation over delegation*: from Horvitz's expected-utility math, through Ackerman's social-technical gap and critic architectures, to Rogers' "proactive people not proactive computing" and the Shneiderman–Maes settlement, the field converges on "the system surfaces, structures, and proposes; the human judges and decides." That validates Memoria's central pillar — "engines write, agents judge, PI approves" — and supplies the missing formalism: gate routing should be an explicit cost/benefit threshold (Horvitz), with low-risk/high-confidence items auto-applied, ambiguous items batched and ranked to the gate, and the rest dropped — never an undifferentiated firehose (Friedman, Chordia, Waldman, Brown). The empirical HAI papers (Jacobs ×2, Govers, Sundar) sharpen one design move with teeth: confident wrong outputs and tidy feature-explanations *actively harm* an expert, so the contradictions dashboard and inspector must show evidence weight, provenance, dissent, and a no-suggestion baseline rather than a persuasive single rationale — and the discovery/triage ranker must expose what it down-ranked so minority claims aren't silently suppressed. Finally, the visualization and notation strand (Healey & Enns, Fekete, D³, Cognitive Dimensions, Hill et al.) gives concrete build guidance for the spaces/projection/inspector ADRs: render over native inspectable artifacts, diff-re-project by stable key, encode the one priority signal preattentively, and turn the PI's own read/edit history into a salience signal — keeping every surface predictable, reversible, and directly PI-actionable from the Obsidian UI.

**Most significant papers**

- [Zhou et al. - 2026 - Externalization in LLM agents.md] — Names Memoria's whole architecture: capability lives in externalized memory + governed skills + protocols coordinated by a "harness" with sandboxing, approval gates, provenance, rollback. - [Debenedetti et al. - 2024 - AgentDojo.md] / [Greshake et al. - 2023 - Indirect Prompt Injection.md] — Ingested papers/web hits are untrusted code that can hijack agents and self-poison persistent memory; tool-filter/least-privilege drops attack success to ~7.5%, and deterministic state-based checks beat LLM-judges (an injection fools the judge too). The security case for the hard-deny sandbox and a deterministic gate.
- [Liu et al. - 2026 - AutoResearchClaw.md] — A near-exact instantiation of Memoria's gate: a numeric/claim registry the writer can read-but-not-modify, a post-hoc verifier rejecting any value not traceable to the registry, multi-source citation resolution, and the empirical HITL finding that targeted high-leverage approval beats both full autonomy and step-by-step approval.

**Adopt**
- **Verifiable claim registry + Chain-of-Evidence as a hard gate invariant.** Every atomic claim and dashboard number must trace to a grounding source-span/citekey, and the structural review gate must refuse any ungrounded claim; the writer reads but cannot modify the registry (Liu 2026; Meng 2026; Wang & Luan 2026; Wang et al. 2026 Sibyl). Maps onto the claims backbone + review gate.
- **Multi-API citation resolution before write** (DOI→CrossRef, title→OpenAlex, arXiv, Semantic Scholar) with an LLM near-miss disambiguation and Verified/Suspicious/Hallucinated tagging; "every reference originates from a resolved record, never model memory" (Meng 2026; Liu 2026; Lu 2024). Buildable as a pre-file gate check on the Zotero/citekey backbone.
- **Confidence-tiered routing via self-consistency** (extract k times, agreement count → auto-drop / route-to-PI / auto-accept), with a cheap yes/no eligibility filter BEFORE expensive numeric/claim extraction (Jacobson 2026 Elhuyar). Directly the gate triage + pre-file relevance gate, sized for qwen2.5:7b economics.
- **Deterministic-first, LLM-judge-second gating** (Ground→Critic→Resolve): deterministic field/schema/citekey checks first, narrow LLM judge only on specific spans, never whole-output grading; promote only a zero-violation draft (Meng 2026; Debenedetti 2024; Andriushchenko 2025; Long 2026). Reinforces deterministic ingest + the MCP-only critic with no write access.
- **Least-privilege per-profile tool allowlists.** Compile read-only MCP toolsets for retriever/scout; only the approval path may write; deny agents any fetch/URL/terminal egress that enables staged exfiltration or self-poisoning (Debenedetti 2024; Greshake 2023; Kang 2023; Lu 2024; Schmidgall 2025). External support for ADR-32/46/28/27 and the gate-plugin hard-denies.
- **Uncertainty/maturity status as first-class fields on claims AND relations**, promoted to "verified" only on corroboration; an evidence-maturity ladder (pilot → analysis-ready → audited) gates promotion (Long 2026; Wang et al. 2026 Sibyl; Xu 2025). Use a small calibrated scale, not binary — feeds the contradictions dashboard and supersession.
- **Append-then-compete supersession discipline.** Never overwrite an approved claim; create a candidate successor that must re-pass the gate (Gottweis 2025 Evolution agent; Wang & Luan 2026 versioned-with-history artifacts). Matches claim supersession.
- **Verifier-before-Memory architecture** (Planner/Memory/Action/Verifier blueprint; V1 self-critique / V2 agent critique / V3 PI / V4 deterministic-tool) as the named pattern for the review gate (Ren 2025; Wang et al. 2023 AutoGen grounding-agent; Wu et al. 2023). Separates the writing engine from the judging verifier.
- **Router + deterministic experts; LLM only extracts structured arguments, never computes.** Counting, dedup arithmetic, ordering, similarity, contradiction tallies run in engine code; the LLM emits typed fields (Karpas 2022 MRKL; Gao 2023 PAL; Merrill 2026 PHIA; Abbasian 2024). Core to deterministic ingest + faithfulness.
- **Structured-artifact (not free-chat) handoffs** with declared writer/reader permissions and per-artifact update modes (append-only logs / versioned-with-history / mutable control) (Chen 2026; Hong 2024 MetaGPT; Yue 2026). A concrete schema for vault governance.
- **Negative-results / "what never worked" store** so the discovery loop and triage never re-surface rejected directions; record failure/tacit knowledge the literature omits (Qu 2026 CORAL; Chen 2025 survey cognitive-loop; Bisht 2026). Complements supersession/contradictions.

**Borrow**
- **Entity co-occurrence store + randomized-waypoint graph walks** over the claim/relation graph to surface non-obvious cross-domain connections beyond nearest-neighbor, with scenario-typed retrieval modes (similar / contradictory / cross-domain / counter-intuitive) feeding the contradictions dashboard and nightly discovery loop (Baek 2025; Ghafarollahi 2024 SciAgents; Wang & Luan 2026 PARNESS). NetworkX-scale, deterministic-seedable.
- **Time-decayed, role-scoped "lesson" overlays** distilled from recurring critiques / PI accept-reject patterns and injected into compiled profile prompts (no fine-tuning); stale guidance fades, mirroring supersession (Gottweis 2025 meta-review; Liu 2026; Wang 2026 Sibyl routed-memory; Chen 2025 Reflexion/ExpeL). PI-reviewable prompt-amendment, not always-on autonomy.
- **Pairwise/Elo ranking and rubric-anchored multi-dimensional judging** (novelty/feasibility/soundness, position-swap debiasing, calibrate to ~10 PI labels) for triage/discovery scoring — but cap match counts and validate qwen-class judges against gold labels first, since quality collapses on weak models (Gottweis 2025; Tang 2025; Baek 2025; Li 2025 MLR-copilot; Weng 2025). Keep judge scores advisory, never auto-merge (Schmidgall 2025; AgentRxiv 2025).
- **Tiered review cascade**: cheap local triage / boolean filter first, deeper grounded check only on survivors; novelty-differentiation as a gate condition against retrieved priors (Gottweis 2025 reflection modes; Ren 2025 SciMON loop; Lu 2024 novelty filter; Xu 2026 NanoResearch). Maps onto pre-file similarity → structural gate.
- **Lightweight specialist roles over a shared typed graph** — a Contrarian (find contradictions) and Connector (cross-domain) profile, role-played sequentially on one local model, not a K-agent fan-out (Wang & Luan 2026; Qu et al. 2026; Liu 2026 Skeptic/Methodologist). Single-researcher scale.
- **Constrained/declarative decoding** to force schema-valid output (enums, citekeys, label sets) and a `distribute`-style closed-label classifier for claim-sentence/aspect tagging — guarantees parseable structure in one pass, cutting retries on the local GPU (Beurer-Kellner 2023 LMQL). Hardens deterministic ingest.
- **Thin-control / progressive-disclosure context management**: a compact "workspace map" + lazy skill loading so local 7B–14B models aren't drowned; pass handles/citekeys not blobs; compress between phases (Chen 2026; Zhou 2026; Huang 2025; Abbasian 2024 Data Pipe). Respects the scarce context budget.
- **Self-consistency / cross-representation consensus** (regenerate, measure semantic variance to flag fabrication vs. stable-wrong; two-route extraction agreeing) as a cheap confidence signal routing high-variance outputs to the gate (Shanahan 2023; Imani 2023; Madotto 2021 perplexity router; Betz 2021 — but use explicit NLI for negated relations, since embeddings miss contraposition).
- **Confirmation-on-ambiguity and SmartPause escalation**: when an ingest/triage instruction is underspecified, surface to the PI rather than guess; escalate only on high uncertainty (Ruan 2024; Liu 2026; Gridach 2025; Alkhatib 2019 — confidently-wrong novel cases also need escalation, not just low-confidence ones).
- **Per-phase cost/latency/tool-trace accounting and a deterministic regression harness** with synthetic ground-truth corpora and frozen tool outputs, asserting on resulting vault state (Schmidgall 2025; Jacobson 2026; Li 2023 API-Bank; Debenedetti 2024; OpenHands 2025). Feeds the content-light diagnostics plane and transferability evals (hold harness constant, swap qwen2.5:7b ↔ qwen3:14b — Zhou 2026).
- **Gap synthesis & schema vocabulary**: promote a limitation appearing across ≥3 papers to a field-level "gap" node; borrow typed edges (has_limitation / causes / solves) and the "5-WHY abstract-the-mechanism-then-translate-vocabulary" query expansion (Long 2026; Xu & Peng 2025 gap-highlighting). Builds on BERTopic clustering + record-linkage dedup.

**Theme & implication for Memoria**
This corpus converges hard on one verdict: autonomous research agents fail precisely where Memoria refuses to automate — literature review, grounding, and final judgment — and what makes agents reliable is not a bigger model but *externalized, inspectable, governed state* with a human at high-leverage checkpoints. Memoria's vault-as-File-as-Bus, MCP-only sandbox, and "engines write, agents judge, PI approves" gate are independently re-derived by the strongest papers (Chen 2026, Zhou 2026, Liu 2026, Gottweis 2025), and the security papers (AgentDojo, Greshake, Kang, AI-Scientist incidents) turn the sandbox from a constraint into a proven necessity — ingested papers are untrusted code that can self-poison the vault, so writes (not just reads) are an attack surface and deterministic gates must do the adjudication. The concrete design moves: (1) make Chain-of-Evidence a hard gate invariant — no claim/number enters the vault without a resolved-record provenance edge, with multi-API citation resolution as a pre-file check; (2) add confidence-tiered self-consistency routing and verified/maturity status on claims and relations driving the contradictions and supersession dashboards; (3) keep all quantitative/graph reasoning in deterministic engines and the LLM in structured-extraction-only, validated by a deterministic state-based regression harness; and (4) enrich the discovery loop with typed-edge scenario retrieval, randomized graph walks, cross-paper gap synthesis, and time-decayed PI-preference overlays — while capping every loop with bounded iterations and a negative-results store so it never perseverates.

## Evaluation, Benchmarks & Metrics  (33 papers)

**Most significant papers**
- **[Cobbe et al. - 2021 - Training verifiers to solve math word problems.md]** — The formal ML grounding of Memoria's "engines write, agents judge" gate: a *distinct, small* verifier scoring N candidates and selecting the top-ranked beats scaling the generator, and answer-only verification misses flawed reasoning (verify the warrant, not just the conclusion).
- **[Dong et al. - 2026 - EpiBench benchmarking multi-turn research workflows.md]** — Process-level, provenance-anchored scoring (Evidence Correctness on *correctly-answered* turns) catches "right answer, wrong reason," and its memory-reuse ablation shows a well-indexed evidence store is the high-value component — Memoria's exact thesis, made evaluable.
- **[Chen et al. - 2025 - ScienceAgentBench.md]** — Stage-decomposed rubric scoring (model-drafted, expert-refined) plus cost-as-first-class-metric, with a minimal self-debug loop beating a heavy agent framework at ~17x lower cost — direct empirical backing for the gate's rubric and Memoria's thin, cheap, MCP-only design.
- **[Yao et al. - 2024 - tau-bench.md]** — State-based evaluation (judge the final vault state, not the trajectory) plus the pass^k *consistency* metric — the right way to certify that a stochastic local extraction/gate engine behaves reliably across reruns before the PI trusts it unattended.
- **[Malaviya et al. - 2024 - ExpertQA.md]** — A ready-made per-claim attribution rubric (Informativeness, Factuality, Cite-worthiness, Attribution completeness, Source Reliability) and the finding that corpus quality dominates factuality while vanilla prompting hallucinates citations — justifies binding every claim to a citekey.
- **[Zhan et al. - 2024 - InjecAgent.md]** — Indirect prompt injection via retrieved external content is the precise attack surface for Memoria's ingest; the data-steal-then-exfiltrate pattern (transmission ~100% once extraction happens) is the empirical case for the hard rule that agents get no outbound/file/terminal tools, ever.
- **[Perez et al. - 2022 - Discovering language model behaviors with model-written evaluations.md]** — Model-written evals: a generator drafts label-conditioned items and a *separate* discriminator keeps only high-confidence ones — the generate-then-discriminate template for Memoria's eval-fixture bootstrapping, with the discriminator's confidence doubling as a per-item uncertainty/quality signal.

**Adopt**
- **Generator–verifier separation with sample-and-rank**: generate N candidate notes/extractions, score each with an independent verifier, return the top-ranked (majority-vote top-k for robustness); keep generator and judge distinct so the judge doesn't overfit. A cheap *small* verifier can gate a larger generator's output (Cobbe 2021; ScienceAgentBench 2025). Cap N — returns go negative and adversarial candidates start gaming the verifier past a point (Cobbe 2021).
- **Stage/component-decomposed graded rubrics, model-drafted then PI-confirmed**, replacing binary pass/fail at the review gate — drafted by a local model, refined by the PI ("engines write, agents judge, PI approves" operationalized) (ScienceAgentBench 2025; Mueller 2019; ExpertQA 2024; SciConvBench 2026).
- **Verify the reasoning/warrant, not just the conclusion**: an extracted claim can be "correct" yet unsupported; the gate must check derivation against the source span, feeding the argument-graph/contradictions machinery (Cobbe 2021; Zhang 2026 ResearchArena; Ji 2022).
- **Provenance/groundedness scoring as a gate metric**: score each claim on whether it cites and is entailed by the required source span (Evidence Correctness; intrinsic-vs-extrinsic hallucination; NLI/SUMMAC sentence-level entailment); a lexical span-overlap check is a cheap, deterministic, MCP-safe substitute when an NLI model is too costly (EpiBench 2026; Ji 2022 hallucination survey; Ye & Durrett 2022; Abbasian 2024).
- **"Reject-until-grounded" abstention loop**: gate a claim by whether its supporting span actually appears in the source; abstain / route to PI / mark uncertain when grounding fails — and *reward* abstention rather than fabrication (Ye & Durrett 2022; DOCBENCH unanswerable category; PaperMind Failure-to-Detect-Question-Error; Basar 2024).
- **State-based evaluation + pass^k consistency** for regression-testing the nightly pipeline: judge by resulting vault state vs an expected state (path-independent), and certify reliability with the probability that *all* k reruns succeed, not best-of-k (tau-bench 2024).
- **Typed, earliest-deviation failure taxonomy** for diagnostics (Retrieve / Perception / Reasoning / Reading-Memory; or PaperMind's eight-way tool-augmented taxonomy) feeding the content-light diagnostics plane and admin reports (EpiBench 2026; PaperMind 2026; KnowledgeBerg three-stage completeness/awareness/application).
- **Cost as a first-class metric** alongside quality, with a tool-call budget (~6 calls; gains saturate, ≥10 degrades via context dilution), validating minimal scaffolding over heavy agent harnesses (ScienceAgentBench 2025; PaperMind 2026; Zhang 2026 ResearchArena; tau-bench 2024).
- **High-detail, justification-seeking prompts (TELeR L4–L6)** for auditable engine tasks: explicit goals + enumerated sub-tasks + format/criteria constraints + a required justification of what was included/discarded — record the TELeR level as engine metadata so regressions trace to prompt vs model changes (Santu & Feng 2023; Super-NaturalInstructions 2022).
- **Definition + 2 positive (and a negative) exemplars per extraction/classification step**, frozen via profile compilation; drop chain-of-thought explanations for small local models (they hurt sub-large models and waste context) (Super-NaturalInstructions 2022; Efrat & Levy 2020).
- **"Choose unknown when info is insufficient" instruction + separate CONFLICT vs MISSING detection**, and *do not blanket-apply CoT* in extraction/contradiction/relevance steps where the answer is often "uncertain" — zero-shot CoT manufactures over-confident disambiguation (Shaikh 2023; SciConvBench 2026; BBQ 2022; Akyürek 2022).
- **Indirect-prompt-injection hardening of ingest**: treat free-text paper fields (abstract/body/notes — highest content-freedom = highest attack success) as hostile; isolate data from instructions with explicit delimiters; never expose outbound/send/file/terminal tools (InjecAgent 2024).
- **Layered, traceable disclosure**: every dashboard/space summary links down through atomic claim → source sentence → citekey/PDF; the gate verifies summary-to-source linkage, not mere citation presence (Gilbert 2025; Abbasian 2024 interpretability; Mueller 2019 contrastive why/why-not).
- **Silent-resolution as a tracked failure mode**: an engine that silently fills a missing field (guessed relation type, invented uncertainty flag) is an unauditable repair; require engines to flag assumptions, and have the gate distinguish "resolved + flagged" from "resolved silently" (SciConvBench 2026).
- **Bootstrap eval fixtures by model-writing labeled examples, then PI-validating a sample** (model-written evals matched/beat hand-written quality), and use the discriminator's confidence as a calibrated data-quality/uncertainty estimate feeding triage and the gate (Perez 2022; Cobbe 2021).

**Borrow**
- **Validate any LLM-judge against PI/human-scored cases before trusting it**, sample-3-and-average for stability, run robustness ablations (swap local judge model, paraphrase the judge prompt), and treat its score as a triage signal, not a certification — manuscript/prose-only judging over-credits polished framing (PaperMind 2026; EpiBench 2026; SciConvBench 2026; ScienceAgentBench 2025; Zhang 2026 ResearchArena; Webson 2023).
- **Composite metrics that pair a quality axis with a coverage/consistency axis** so a degenerate engine can't game the score (icat = capability × bias-balance; Composite Reliability = accuracy × streak-robustness) — applies to discovery-relevance and LTR triage (StereoSet 2021; ClawArena 2026).
- **Honest few-shot accounting**: every threshold/prompt tuned on the PI's notes is spent supervision — never report engine quality on the notes used to tune it; freeze and version-pin prompts; use leave-one-out CV (with its high-variance caveat) on the tiny label set (Perez 2021; Efrat & Levy 2020; Webson 2023).
- **Random/trivial-control baselines** (Naive-Bayes/majority-class) to expose when triage gains are distributional shortcuts (always-accept a venue) rather than content relevance; partition eval into "guessable from metadata" vs "needs content" (Zhong 2021; Parrish 2022 directional bias score).
- **Semantic, not exact-match, scoring** for entity/field correctness ("Miami, FL" vs "Miami, Florida"), and keep the per-task exposed MCP toolset small/disambiguated — confusable tools degrade selection (Rabinovich & Anaby-Tavor 2025; ClawArena MetaClaw skill-overlay).
- **Build internal eval fixtures with hidden ground truth + a contradiction map** for testing supersession/contradiction detection; the hard cases are *contradiction-forcing* updates that flip a prior claim, not paper volume (ClawArena 2026; KnowledgeBerg bounded-universe grounding; EpiBench machine-draft-then-human-curate protocol).
- **locate → extract → compute** staging for table/figure-grounded numeric claims with per-stage error flags, always feeding the figure *with* its caption and referencing paragraph; keep metadata/counts in deterministic ingest, never route counting to the LLM (DOCBENCH 2024; Ji 2022 numeral hallucination; PaperMind background-context injection).
- **Hybrid sparse+dense retrieval beats dense-alone; reranking adds little; retrieval + an explicit reasoning step is what works** (passages or CoT alone are weak); prefer self-consistency/self-verification over iterative self-critique, which amplifies errors on a wrong initial trajectory (KnowledgeBerg 2026).
- **Prediction-then-explain mental-model calibration and reversible "adjustable replay"** via the disposable projection engine: let the PI predict a triage/relevance verdict on sample papers, or re-run a case with a tweaked profile and see the diff — without agents executing code (Mueller 2019).
- **Structured aspect schemas as extraction targets**: SRQR's 21-item qualitative checklist (sampling, trustworthiness techniques, reflexivity, limitations), SciConvBench's component ontology, model-card fixed-layout — populate structured note fields and give component-level extractor read-outs; separate "is the note complete/well-formed" (engine/gate-checkable) from "is the science good" (PI judgment) (O'Brien 2014; SciConvBench 2026; Gilbert 2025).
- **Perceptual charting defaults** for dashboards/admin HTML: prefer position/length over area/pie/treemap for magnitudes; gridline alpha ~0.2, ≥8px spacing, chart height >80px buys no accuracy — no user study needed (Heer & Bostock 2010).
- **Minimality-gap / redundant-tool-call metric** and the "updating" task (insert/supersede claims grounded in retrieved references, carrying the citation; score only the changed region à la UpdateROUGE) (EpiBench 2026; EditEval 2022).

**Theme & implication for Memoria**
Collectively this body says the same thing in many voices: **process and provenance, not output prose, are what to evaluate** — separate the producer from an independent (and cheap, possibly smaller) judge, score grounding/warrant per atomic claim against the cited span, and let engines abstain rather than fabricate. The concrete moves: make the structural-review gate a stage-decomposed, model-drafted/PI-confirmed *rubric* that scores each claim on faithfulness-to-source plus attribution completeness (claims/contradictions/argument-graph ADRs), with a deterministic lexical-overlap grounding check as the cheap default and an NLI/judge as an optional precision filter that is validated against the PI's own approve/reject decisions. Certify the stochastic local engines with **state-based regression tests + pass^k consistency** and a capped tool-call/cost budget before they run unattended in the nightly loop (retrieval/triage ADRs), and treat silent field-filling, prompt-injection via free-text paper fields, and over-confident CoT disambiguation as named, gated failure modes rather than edge cases.

## Retrieval & RAG  (29 papers)

**Most significant papers**
- [Yu and Wang - 2026 - Knows agent-native structured research representations.md] — Memoria's note model written as a portable, schema-validated sidecar: typed claims + evidence + typed-relation graph + two-axis confidence + replaces/supersession chain + deterministic linter. The closest external validation of Memoria's whole structured-substrate design, with a concrete field schema to align to.
- [Gao et al. - 2023 - Enabling large language models to generate text with citations.md] (ALCE) — Gives the review gate a deployable faithfulness metric: NLI-entailment-based citation recall/precision over decomposed atomic sub-claims, validated against human judgment (kappa 0.70), so an automatic entailment check can stand in for PI review on routine cases.
- [Wallat et al. - 2024 - Correctness is not faithfulness in RAG attributions.md] — Correct ≠ faithful: cheap local models post-rationalize citations (token-match a source they didn't actually use). Supplies a deterministic counterfactual perturbation test Memoria can run offline to flag low-confidence attributions — directly hardens claim provenance.
- [Lála et al. - 2023 - PaperQA retrieval-augmented generative agent for scientific research.md] — Decomposes RAG into MCP-shaped tools (search / gather-evidence / answer) with an LLM-rerank "context library" and mandatory citation + abstention, achieving 0% hallucinated citations. A direct blueprint for the retriever/scout profile under the MCP-only sandbox.
- [Su et al. - 2022 - Selective annotation makes language models better few-shot learners.md] (vote-k) — Fast vote-k selects a tiny, diverse-and-representative label pool over existing Sentence-BERT embeddings (no model inference), then retrieves per-instance exemplars — exactly the cheap-annotation, deterministic, single-PI regime for seeding extraction/classification prompts.
- [Su et al. - 2025 - BRIGHT.md] / [Wei et al. - 2026 - A survey of reasoning-intensive retrieval.md] — Prove cosine similarity collapses (~59→~18 nDCG) on relevance defined by shared theorem/warrant/method — i.e. exactly Memoria's cross-paper argument-graph connections — and that LLM reasoning-trace query expansion (BM25 gains most) and offline index-side enrichment recover much of the gap.

**Adopt**
- Make the structured note layer (not the PDF) the agent's primary substrate: typed statements (claim/assumption/limitation/method/question/definition), evidence items, typed relations (supported_by/challenged_by/depends_on/supersedes/cites), per-claim source_anchor (page/section/figure), two-axis confidence (claim_strength vs extraction_fidelity), and stable claim IDs with `record_id#local_id` cross-note references (Knows). Adopt the singly-linked `replaces` chain + `record_status: superseded` + freshness/`stale_after` for the supersession dashboard (Knows).
- Add a deterministic schema linter to the structural review gate: schema validation, cross-ref integrity, ID uniqueness, dangling-relation and predicate checks that fail fast — structural validity verified by lint, semantic correctness deferred to agent/PI (Knows). Extend with citation-hallucination as a hard, no-LLM reject: a citekey must resolve in Zotero/vault and the source must support the claim (Saxena 2025, Schwappach 2025).
- NLI-entailment faithfulness check at the gate: decompose extracted claims into atomic sub-claims, mark supported iff the concatenated cited spans ENTAIL the sub-claim (local NLI model); citation precision prunes over-attached sources (a cited span is irrelevant iff removing it leaves the rest still supporting) (ALCE Gao 2023).
- Causal-faithfulness counterfactual test in ingest QA: inject a claim's span into a decoy/uncited source and re-run extraction; if the engine re-cites the decoy, mark the attribution low-confidence and route to the gate. Distinguish "source grounds claim" from "source merely co-mentions token" in the argument-graph warrant (Wallat 2024).
- Instruction-tuned dense retriever (GritLM/E5/Instructor with task-instruction strings) as the retriever/scout default over lexical-only, but index title+abstract+extracted atomic claims, NOT whole PDFs — empirically better and cheaper with small encoders, and PQ-compress the index to fit the 16GB GPU (LitSearch Ajith 2024, Atlas Izacard 2022). Keep a BM25/Lucene lexical path as a strong, source-attributable, drift-robust fallback (RAGScholar Neekhra 2026, Tam 2022).
- Expose RAG as discrete MCP tools (search / gather-evidence / answer) with mandatory citekey-anchored provenance and an explicit "I cannot answer / unsure" abstention; add an LLM chunk-summary-with-1-10-relevance-score rerank layer ("context library") on top of embedding retrieval as the learning-to-rank triage stage (PaperQA Lála 2023).
- LLM reasoning-trace query expansion in the nightly discovery loop: have qwen2.5:7b write a short "identify the essential problem, reason step-by-step, draft a hypothetical relevant passage" trace before hitting the BM25/dense index — recovers reasoning-intensive relevance Memoria most needs (BRIGHT Su 2025, MRMR Zhang 2026). Offline index-side enrichment: at ingest, attach "hypothetical questions this claim answers" as searchable metadata to shift LLM cost offline (Wei 2026).
- Fast vote-k selective annotation (graph degree-discount over existing Sentence-BERT embeddings, deterministic, no model passes) to choose which vault items the PI labels, then kNN-retrieve the most-similar exemplars per new paper for extraction/classification/aspect prompts — eliminating run-to-run label variance (vote-k Su 2022; KATE Liu 2021/2022).
- Decompose discovery evaluation into the three measurable stages — discovery / selection / organization — and lean on cheap deterministic retrieval + hierarchical clustering as the core, with LLM agents only labeling/judging; agentic search underperforms a one-line title query (ResearchArena Kang 2024). Score discovery with both modes: exact-match for "find the one supporting/contradicting paper" (deep) and IoU precision+recall for "find all related work" (wide), with an explicit "no qualifying paper" abstention outcome (AutoResearchBench Xiong 2026).
- Faceted navigation as an Obsidian space/Base over typed metadata (citekey, aspect, claim-type, uncertainty, relation) with live result counts / query previews before a filter is committed — the empirically validated win over every spatial visualization (Hearst 2009).
- Tag source-type (review vs primary vs preprint) as a first-class field and weight primary evidence over review restatements in the contradictions/argument graph (Schwappach 2025).

**Borrow**
- Two-pass post-hoc citation (P-Cite): engine drafts the note, a separate MCP-only attribution pass retrieves and attaches/verifies citations sentence-by-sentence before the gate — higher coverage, decouples drafting from grounding, maps onto "engines write, agents judge, PI approves". Reserve inline precision-first (G-Cite) mode for claim-supersession/contradiction checks where a wrong citation is costlier than a missing one (Saxena 2025).
- Citation-quality rubric for the inspector/admin reports: correctness, appropriateness, comprehensiveness, faithfulness; down-weight "inappropriate" citations that merely restate the query so dashboard metrics aren't inflated (Wallat 2024). Report retrieval recall@k stratified by query specificity (broad vs narrow) rather than a single number (LitSearch Ajith 2024).
- Reasoning-relevance vocabulary for typed relations/warrants: five-type taxonomy (deductive/analogical/causal/analytical/numerical) and citation-failure axes (single/multi-hop/intersection × implicit/explicit) to classify WHY evidence is relevant and flag low-confidence multi-hop/implicit provenance for review (Wei 2026, Buchmann 2026).
- Staged reasoning-reranker (JudgeRank-style query-analysis + document-analysis prompt with the BM25 score injected as a feature) over a small candidate set — but keep k small and validate before adding, since off-the-shelf MS-MARCO cross-encoders HURT on reasoning relevance and gave negative gains on tool retrieval (Wei 2026, BRIGHT Su 2025, Shi 2025).
- Cap aggregated evidence at ~3 documents for qwen2.5:7b-class models (top-5 adds noise smaller models can't exploit); don't over-engineer per-doc summarize-then-fuse — direct aggregated-evidence conditioning is simpler and stronger (RAGScholar Neekhra 2026).
- Bounded refine/retrieve↔reason loop with a hard iteration cap and deterministic STOP gate; longer trajectories / THINK mode / bigger reasoning budget do NOT reliably improve agentic search, and naive module-chaining amplifies "overthinking" drift (Wei 2026, AutoResearchBench Xiong 2026). pass@k rescues single-paper lookups but won't fix systematic recall gaps — use diversified retrieval seeds (MMR) for wide coverage, not repetition (PaperQA Lála 2023, Xiong 2026).
- Query-side-only adaptation if Memoria ever tunes a retriever: freeze the doc encoder so the vault index never needs re-embedding; parameter-efficient prompt tuning generalizes better out-of-domain than full fine-tuning, and ECE-monitored calibration is a proxy for off-domain trustworthiness of discovery scores (Atlas Izacard 2022, Tam 2022).
- Multi-sample-then-vote / self-consistency for noisy single-pass extraction, feeding an uncertainty flag; dedup near-duplicate evidence by structural-hint (section path) before merging — both cheap, deterministic complements to record-linkage (RECITE Sun 2023, ALCE rerank Gao 2023). kNN-agreement against the PI's confirmed corpus as a model-light "this claim is atypical, surface to PI" hardness signal (RetroPrompt Chen 2023).
- Contradiction Retrieval as a first-class mode: retrieve the rule/claim a new claim VIOLATES, not what supports it — feeds the contradictions dashboard directly (MRMR Zhang 2026). Sentence-level supporting-facts scoring (joint answer+evidence EM/F1) and bridge-entity multi-hop traversal over the citekey/relation graph (HotpotQA Yang 2018).
- Offline eval discipline: auto-generate a retrieval/triage eval set by LLM-rewriting the PI's own inline-citation paragraphs into queries whose target is the cited note (LitSearch Ajith 2024); a tiny citekey-grounded gold-set for L5 regression testing, but keep human/agent-judged scoring since ROUGE/BERTScore track human judgment only weakly (RAGScholar Neekhra 2026). Enforce temporal-leakage controls (only pre-dated, non-survey sources) when evaluating "what was knowable then" for supersession (ResearchArena Kang 2024).
- Generated-knowledge / generate-then-integrate pattern: turn an implicit judgment (does A contradict B; what aspect is this) into an explicit intermediate rationale with confidence-based selection before deciding, improving auditability; factuality strongly predicts helpfulness as a gating rubric (Liu 2022 Generated Knowledge, Galactica Taylor 2022). Confine the LM to bounded, auditable sub-question call-sites with a deterministic interpreter doing the rest (BINDER Cheng 2023).
- Knows+Fallback token tiering for the projection engine: a statements-only projection keeps ~88% comprehension at ~93% fewer tokens — escalate to full evidence/relations or anchored full text only on low confidence; reserve the strongest model (or PI) for structured extraction on complex papers (Knows Yu 2026).

**Theme & implication for Memoria**
Collectively this body of work says the same thing three ways: retrieval quality — not model size or fancier generation — is the dominant driver of grounded output, and surface/cosine similarity systematically misses exactly the reasoning-based, cross-paper connections (shared warrant/theorem/method) that Memoria's argument graph exists to capture. The concrete moves: (1) make the structured claim/relation sidecar (Knows-aligned, with source_anchors, stable IDs, two-axis confidence, replaces-chain) the substrate, gated by a deterministic linter plus NLI-entailment + counterfactual-faithfulness checks so "engines write, agents judge, PI approves" has teeth (review-gate, claims, supersession ADRs). (2) Build the retriever/scout as instruction-tuned-dense-over-claims with a BM25 fallback, an LLM reasoning-trace query-expansion front-end and a small-k LLM-judge reranker, evaluated in deep (exact-match + abstain) and wide (IoU) modes (retrieval/triage, discovery ADRs). (3) Spend the local-model budget on retrieval, exemplar selection (fast vote-k), and offline index enrichment rather than agentic search or long deliberation, which repeatedly lose to cheap deterministic baselines — faithfulness over flash, made empirical.

**Adopt:**
- LLM-as-rubric-judge (Rule-Based Reward Models): feed (engine-extracted note, source, written governance rubric) to a local model that classifies pass / needs-revision / reject — the concrete "agents judge" mechanism for the review gate, claim-faithfulness checks, and similarity decisions (GPT-4 technical report).
- Treat the vault as the model's external long-term memory; retrieve relevant notes/claims as context rather than baking knowledge into weights — validates the RAG/retrieval layer and the inspectable external store (Pre-Trained Models survey; Augmenting Human Intellect; Man-Computer Symbiosis; Licklider).
- Augment-not-automate with a human-approval altitude, and design the knowledge model (claims, typed relations, supersession) independently of its markdown encoding so the vault is one projection over a canonical knowledge layer (Augmenting Human Intellect; The Knowledge Level; Turing's status-tagged propositions).
- Efficiency/compute as a first-class evaluation metric and open-weights local models as the deliberate choice — a well-prompted small model is competitive, so qwen2.5:7b-class extraction is sound, not a compromise (Stochastic Parrots; BLOOM; FewVLM).
- Carry confidence + OOD/uncertainty signals on every engine output so low-confidence extractions are triaged at the gate instead of auto-filed — grounds the existing uncertainty-flag design (Pre-Trained Models survey; GPT-4 calibration finding).

**Borrow:**
- Hard prompt-injection defenses for ingest: wrap paper body in a delimited untrusted-data block, put fixed instructions *after* the data, cap output tokens, use stop sequences, and schema-validate/post-process outputs — cheap deterministic hardening that also improves ingest determinism (Perez & Ribeiro; Branch et al.; GPT-4 red-teaming).
- Ingest hygiene as an attack-surface reducer: perplexity-based outlier-word filtering to strip injected triggers and web-crawl boilerplate (HTML/JS junk) before text reaches any model — a deterministic-ingest cleaning step, not a security guarantee (Xu et al.; PaLM dedup).
- Better similarity functions and explicit false-positive budgets for the pre-file similarity gate and record-linkage dedup: Tversky shared-vs-distinctive-feature weighting, edit-distance/hashing near-dup detection, and tunable z-test-style thresholds instead of opaque cutoffs — and never let embedding cosine be the sole arbiter of "same claim," since high embedding similarity can mask flipped meaning (Kunda et al.; PaLM; Kirchenbauer et al.; Branch et al.).
- Reverse-inference / base-rate discipline for any latent label (is-CLAIM, supersedes, is-relevant): require reliability + specificity + generalizability + validity, flag low-specificity labels, and guard triage/dedup against labels that fire on almost everything (Barrett et al.).
- Staged decomposed prompting plus self-consistency for extraction engines: split aspect/claim work into classify→extract→flag sub-prompts (robust on a 7B model), sample multiple reasoning chains for high-stakes judgments, and treat disagreement as an uncertainty signal; serialize relevant argument-graph neighborhoods into the prompt to ground reasoning (Qiao et al.; Reasoning survey; Pre-Trained Models survey; PaLM CoT).
- ML evaluation hygiene for learning-to-rank triage and discovery scoring in a tiny single-user data regime: hold out PI accept/reject feedback, beware multiple-testing inflation, prefer simple interpretable baselines first, and label co-occurrence/similarity edges as *candidate* typed relations for PI review, not asserted warrants (Domingos; He et al.).
- Governance/design framings: conservative safe defaults at the gate ("needs review" on, uncertainty flags on) since the PI mostly accepts defaults, and avoid a wall of fine-grained toggles (Acquisti et al.); version and regression-test prompts as governed artifacts, keep them human-readable rather than opaque continuous prompts (CoOp; Qin et al.; FewVLM; Reasoning survey); per-engine "model cards" as static admin reports (PaLM; BLOOM); the agent-vs-device distinction maps onto engines (devices) / sandboxed agents / PI (the intentional agent) (Benyon).

**Theme & implication for Memoria:** This corpus is two strands — a foundational HCI/AI lineage (Engelbart, Licklider, Newell, Turing) that *names* Memoria's core stance (augment a single intellect through an interlinked, status-tagged, knowledge-level store that the human judges) and a modern LLM-pragmatics strand (injection attacks, RBRM judging, calibration, prompting, small-open-model viability) that supplies *buildable mechanism and threat models*. Collectively they tell Memoria to treat ingested external text as untrusted data and the local model as a fallible, possibly-injectable component — so the review gate (rubric-driven LLM-as-judge), uncertainty flags, and the MCP-only sandbox are exactly right, but ingest prompts need an explicit instruction/data split plus perplexity-based cleaning. The concrete design moves: implement the gate's "agents judge" step as a rubric-classifier over (note, source, governance-rubric); harden every engine prompt with data-after-instructions delimiting and output caps; and upgrade the similarity/dedup layer to Tversky-style feature overlap with explicit false-positive budgets, never trusting embedding cosine alone.

## Knowledge Graphs & Entity/Relation Extraction  (22 papers)

**Most significant papers**
- **[Wu et al. - 2026 - Intern-atlas a methodological evolution graph as research infrastructure for AI scientists.md]** — Closest blueprint to Memoria: method entities as nodes, typed causal edges, every edge carrying a verbatim warrant, plus a deterministic verbatim/year/contradiction validator and a one-sided LLM veto — directly operationalizes Memoria's warrants, faithfulness mandate, and "engines/checks judge" gate.
- **[Qiao et al. - 2026 - SciAtlas a large-scale knowledge graph for automated scientific research.md]** — End-to-end recipe for typed-graph retrieval: reusable-keyword extraction, tri-path hybrid recall, random-walk-with-restart over typed edges, path-based explanations, and claim-grounded idea evaluation — covers keyphrase tags, retriever/scout, projection, and the contradictions/supersession gate.
- **[Davis - What is a knowledge representation.md]** — The design axioms: representation as lossy surrogate (justifies uncertainty flags), ontological commitment in layers (justifies the relation/aspect schema as an ADR-worthy choice), constraints ARE the representation (the gate must validate edge semantics), and sanctioned-vs-recommended inference (justifies ranking/triage as core).
- **[Wei et al. - 2024 - PubTator 3.0 an AI-powered literature resource for unlocking biomedical knowledge.md]** — Validates the whole architecture: deterministic offline NER→normalize→typed-relation pipeline, precomputed indexed store, and tool-augmented RAG where agents call read-only APIs (mirroring the MCP-only sandbox) to cut citation hallucination.
- **[Sainz et al. - 2021 - Label verbalization and entailment for effective zero- and few-shot relation extraction.md]** — A buildable, training-free path for typed-relation extraction: relation-as-NLI-hypothesis with entity-type constraints and a tunable abstain threshold, where "detecting nothing is the hard part" — exactly the over-linking risk in the pre-file similarity gate.
- **[Talukder et al. - 2026 - Towards automated ontology generation from unstructured text a multi-agent LLM approach.md]** — Empirically shows front-loaded planning (not bug-fixing loops) drives quality, redundancy/dedup is genuinely hard for LLMs, and 30B-class local models suffice with the right decomposition — endorsing Memoria's plan-then-emit extraction, deterministic dedup, and local-model posture.

**Adopt**
- Deterministic offline pipeline: NER → normalize-to-canonical-ID → typed-relation extraction, precomputed and indexed, run on a nightly/weekly cadence rather than at query time (Wei/PubTator 3.0; SciLinker). Confirms the deterministic-ingest, engines-write split.
- Entity canonicalization as the cross-note join key — collapse synonyms/surface forms to one node via char-3-gram + ANN linking, type-by-type, with provenance preserved and a coarse-to-fine merge order (Liu/SciLinker; Xiong/PaperScope; Wei/PubTator). This is record-linkage entity dedup made concrete.
- Two-stage normalize/dedup: embedding retriever proposes ~sqrt(N) candidate canonical entities, then a local LLM under the gate picks the match (Berkowitz/RAGnorm). Keeps the LLM call small; fits cheap-local-model constraint.
- Typed causal/relation edges where **every edge carries a verbatim warrant**, validated deterministically: substring-grounding of the quote, year-ordering, and opposite-direction-edge contradiction checks before write (Wu/Intern-Atlas; Xu/DeepReport's pair+evidence+hypothesis quintuple). Direct content for the structural review gate and warrants/contradictions.
- Relation-extraction as NLI entailment: verbalize each relation as a hypothesis template, score with a local NLI model, gate candidates by linked entity types, and use a tunable threshold for the dominant NO-RELATION case (Sainz; Chen/KnowPrompt type-priors). The entailment score doubles as the gate's confidence/uncertainty signal.
- Random-walk-with-restart over the typed note/claim graph with per-edge-type weights for topological discovery the vector-only loop misses, returning path-based "why retrieved" explanations to the inspector (Qiao/SciAtlas). Deterministic, no LLM.
- Tri-path hybrid recall (lexical exact + vector + title fuzzy with thresholds) merged by max-score for the retriever/scout and pre-file similarity gate (Qiao/SciAtlas).
- Reusable-keyword extraction with an explicit anti-marketing / anti-paper-specific-name instruction plus an importance score, feeding keyphrase tag-candidate ranking at the gate (Qiao/SciAtlas).
- Per-aspect typed nodes: extract MASSW-style aspects as typed nodes (contribution, method, dataset, results, limitations...) linked to the paper via "belongs-to", so each note is a small graph the relation layer connects (Xiong/PaperScope; Talukder's artifact-driven roles).
- Plan-then-emit extraction (front-load which claims/relations a note needs, then encode them) instead of single-prompt extraction on the 7B model; surface unrecoverable cases to the PI rather than burning a self-repair loop (Talukder).
- Constrain extraction to a closed label set (yes/no/uncertain), keep an explicit uncertain class, and run at temperature 0 — annotation-free zero-shot beats supervised baselines a single PI cannot train (Workman/ChatGPT-4 HF).
- Graph-grounded scoring with a **one-sided LLM veto** (the LLM judge can only LOWER a deterministically-computed score, never raise it) to neutralize LLM positive bias in triage and the gate (Wu/Intern-Atlas).
- Editable datastore of PI-confirmed relation exemplars (FAISS/ANN over a single vault) so a gate correction becomes a new key-value that immediately shifts future extractions — personalization without fine-tuning (Chen/RetrievalRE).

**Borrow**
- Statistical edge gating: background-corrected co-occurrence (STRING-style) so a heavily-cited concept doesn't dominate, plus a hypergeometric test + BH-FDR to attach a p-value and threshold which relations surface (Liu/SciLinker). Note SciLinker's own caveat that at SMALL single-vault scale, source-quality/venue weighting is warranted since the large-N significance argument doesn't hold.
- A fixed, curated typed-relation vocabulary with directionality and a relation-presence > same-sentence-co-occurrence > distant-co-occurrence ranking signal (Wei/PubTator; Liu's 7-way schema as a template; Forbus's signed I+/I- and conditional-validity vocabulary for "increases/decreases" and context-scoped claims/supersession).
- Verbalize-the-schema and competency-question self-tests: render relevant typed-relation/entity schema as prompt context for the local extractor, and generate "can this note answer X?" probes to verify extracted claims actually support them, with a 2-hop graph-traversal probe distinguishing "knowledge absent" from "query-formulation failure" (Ye/OntoPrompt; Talukder; Xiong inverted-construction). Knowledge-noise principle: feed only narrowly relevant context, not the whole neighborhood (Ye).
- Rank-then-verify two-stage discovery — cheap ranker prunes the candidate space, expensive agent/PI judgment only on the top few — with selection-bias correction (model P(PI accepts) separately from value-if-accepted) and a hybrid heuristic: graph/embedding signals for densely-connected notes, LLM judgment for sparse/cold-start notes (Yu/ArtifactLinker).
- Cloze/verbalizer framing for low-data typing and class-transfer: copy the mention into a declarative template and read the [MASK]/label-word distribution; decompose label names into constituent words and synonym-expand them, so new entity/relation types need no new classifier head (Ding; Ma/EntLM; Chen/LightNER; Chen/KnowPrompt). Versioned, evaluated prompt templates matter (Ding) — supports prompt/profile governance.
- Concept co-occurrence layer with community detection over the vault and existing/emerging/potential link-prediction triage, scoped to a single vault with a lightweight scorer (Xu/DeepReport). Borrow the evolving-graph + triage labels, not the heavy BERT temporal model.
- Frames-with-slots as the schema language: typed terminals with marker constraints, weakly-bound defaults displaced (not silently overwritten) by better-attested extractions, and a match-fail-then-substitute mismatch signal routing to an alternative candidate (Minsky). Maps to MASSW slots, supersession, and dedup.
- ORWAS biased random walk and survey-then-expand traversal primitives (get_relations → get_triples) as a read-only MCP navigation surface, with evidence-grounding rejection (admit an answer only if it appears in retrieved observations) (Xiong/PaperScope; Xu/GraphWalker). Reinforces faithfulness-over-flash; skip the SFT/RL training.
- Instrumentation: tokens-per-decision as the engine efficiency metric (reframed to local compute/latency), and graded ontology-distance eval where a hierarchy exists (Berkowitz). Synthetic-snippet generation to test extractors without exposing private vault content to any model (Workman).

**Theme & implication for Memoria**
This body of work is near-unanimous that Memoria's core bet — a persistent, typed, claim/relation graph that deterministic engines populate offline and read-only agents query (vs. re-deriving knowledge per session) — is the right substrate (Wu, Wei, Qiao). The actionable convergence: (1) make every edge carry a verbatim-grounded warrant and run a cheap, model-free validator (substring/type/year/opposite-edge) at the structural review gate before any relation or claim is written; (2) implement typed-relation extraction training-free via NLI-entailment + entity-type constraints + a tunable abstain threshold, because the hard, default case is detecting NO relation — over-linking is the failure mode for the pre-file similarity gate and contradictions dashboard; (3) treat dedup/normalization as a deterministic retrieve-shortlist-then-LLM-pick step keyed on canonical IDs, since LLMs reliably mint near-duplicate entities. Across retrieval/triage, the pattern is hybrid-and-explainable: combine deterministic graph signals (RWR, co-occurrence-with-significance, tri-path recall) with a constrained, one-sided LLM veto and path-based explanations into the inspector — keeping ranking/triage a first-class, auditable layer (the "recommended inferences" Davis demands) rather than a cosmetic add-on.

## Claims, Argumentation & Scientific Discourse  (17 papers)

**Most significant papers**

- **[Press et al. - 2024 - CiteME can language models accurately cite scientific claims.md]** — Frames "claim → exact source paper" attribution as a task (humans 69.7% vs best agent 35.3%); almost a named Memoria feature for verifying/repairing every claim→citekey link in the argument graph.
- **[Choi et al. - 2026 - CiteGuard faithful citation attribution for LLMs via retrieval-augmented validation.md]** — Shows bare LLM-as-judge recall is 16–17% for citation verification and that retrieval-grounded judging (body-text snippet search, claim-context expansion) nears human accuracy at ~$0.005/sample on small open models — exactly Memoria's "judge only with evidence in hand" + local-cheap posture.
- **[Mitchell et al. - 2022 - ConCoRD.md]** — A frozen NLI model + MaxSAT factor graph dynamically estimates entailment/contradiction between statements with <20ms solve times — the deterministic engine Memoria's contradictions dashboard and supersession detection need at single-PI scale (no training, no hand-written constraints).
- **[Cornelio et al. - 2025 - The need for verification in AI-driven scientific discovery.md]** — Argues verification, not generation, is the bottleneck; validates Memoria's core stance that the structural review gate is the deliberately-widened verification stage behind an abundant discovery loop.
- **[Li et al. - 2026 - Source or it didn't happen (CITETRACER).md]** — Field-level citation taxonomy with a deterministic-rules-first / LLM-only-on-residual cascade (61.7% closed with no LLM call) and precision-first design (frontier judges over-flag real citations up to 41–46%) — directly maps "engines write, agents judge" onto bibliographic integrity.
- **[Bouras and Fellow - 2026 - CrossTrace.md]** — Step-level source-grounded reasoning traces (every step carries section/paragraph/verbatim quote) on a Qwen2.5-7B model — the auditable evidence-link shape Memoria's gate wants, on Memoria's own default model class.

**Adopt**

- **Claim→source attribution + verification engine**: given an extracted atomic claim/warrant, identify and verify the exact source paper rather than trusting the author's citation; flag adjacent-citation confusion and "selected the citing paper not the target" failure modes at the gate (CiteME, CiteGuard). Use body-text snippet search and claim-context (surrounding-paragraph) expansion as MCP retriever tools — no PDF/file read needed (CiteGuard).
- **NLI-driven typed-relation extraction**: run a pre-trained NLI model over pairs of atomic claims to auto-detect supports/refutes/entails/contradicts edges, replacing hand-written constraint sets; feed the contradictions dashboard and the argument graph (ConCoRD, Maieutic, FEVER). Adopt FEVER's tri-class **SUPPORTED / REFUTED / NOTENOUGHINFO** as the claim-vs-evidence verdict vocabulary, and require the verdict to carry its evidence sentence(s), not just a label (the FEVER 31.9%→50.9% evidence-vs-label gap) (FEVER).
- **MaxSAT / constraint-solve over the claim graph**: model claims as nodes and NLI relations as weighted clauses, then solve symbolically for the maximal globally-consistent subset to pinpoint which claims to flag as contradicted/superseded — surfaced to the gate, never an autonomous overwrite (Maieutic, ConCoRD).
- **Per-field bibliographic integrity check at ingest** with a deterministic-rules-first cascade (DOI canonicalization, author/venue/page normalizers, early-exit) and the local model reserved for the ambiguous residual; a precision-first verdict with a POTENTIAL middle ground that maps onto uncertainty flags (CITETRACER). Reject claims whose citations don't resolve in the Zotero/citekey backbone (Cornelio).
- **Structured claim schema** `h := (context, variables, relationship)` so two claims contradict when same context+variables but conflicting relationship, with a faceted Hypothesis Match Score (ctxF1 / varF1 / relacc with partial credit when one relation is broader) for dedup and supersession granularity (DiscoveryBench). Rewrite each claim as atomic/abstracted `[object]+[property]+[what it enables]`, name-free, to improve record-linkage matching (SciPaths).
- **Step-level source grounding**: every extracted claim/reasoning step carries `{section, paragraph, verbatim quote}`, enabling automated + human fidelity checks at the gate; deterministic setup (version-controlled prompt, temp 0, rule-based section detection, review-article exclusion); retain sub-0.6-confidence items in the DB for the diagnostics plane but don't promote them past the gate (CrossTrace).
- **Typed-relation controlled vocabularies** instead of a generic "cites" edge: dependency functional roles (Core Method, Conceptual Framework, Data Source, Eval Protocol, Tooling…) (SciPaths); citation intents (Background / CompareOrContrast / Extends / Uses / Motivation / Future) classifiable by a 7B model on ingest (CiteBench); and discovery-pattern facets (gap_fill, analogy_transfer, contradiction_exploit, swanson_abc ABC-bridge) feeding the contradictions dashboard and discovery scorer (CrossTrace).
- **Constructive-necessity test for dependency edges**: include a "depends-on"/"enables" edge only if removing it would prevent the target from being realized as claimed ("if I had to build this tomorrow, what would I still need?"), and record an explicit **unmapped / NONE** outcome rather than forcing a citation — which simultaneously surfaces a research gap for the discovery loop (SciPaths).
- **Multi-aspect gate checkers** scoring orthogonal axes — faithfulness/reality, novelty-vs-vault (a BM25/embedding pre-file similarity analog), clarity, and a four-axis novelty/relevance/significance/verifiability rubric with per-score justification for an auditable PI rationale (MOOSE, Qi et al., Höök & Löwgren's contestable/defensible/substantive criteria). Don't auto-accept LLM judgments: bare/frontier judges are optimistic and over-flag (POPPER relevance overestimate; CITETRACER FPR; CiteGuard 16–17% recall) — route uncertainty to the PI gate.

**Borrow**

- **Logical-integrity reliability flag**: re-prompt the local model on a claim and its negation; disagreement under negation marks the claim low-confidence before it enters the graph (Maieutic). Pair with abductive both-sides prompting (argue for and against a claim) to harvest warrant candidates and expose hidden assumptions (Maieutic).
- **Chain-of-Ideas spine**: build a citekey-anchored chain tracing a line of development backward (builds-on/baseline, terminating at a >1000-citation milestone) and forward (supersedes/cites), annotating each edge with "what changed" — renderable in the disposable projection engine as a navigable space (Chain of Ideas).
- **Budget-aware retrieval**: default to snippet retrieval (~half the tokens of full-paper read) with a token-budget switch to full-read; full-text beats abstract-only for attribution, but a weak local model may not exploit it, so degrade gracefully to PI-approval candidates (CiteGuard, CiteME). Multi-source discovery triad — semantic neighbors + KG one-hop neighbors + Zotero citation neighbors (SCIMON, SciPaths).
- **Iterative novelty / supersession check**: query the vault with a candidate claim; if any neighbor exceeds a similarity threshold, flag non-novel / supersession-candidate rather than filing a duplicate (SCIMON, Chain of Ideas, SciMON loop as a one-shot score).
- **Evaluation discipline**: prefer pairwise + ELO with order-reversed double scoring over absolute Likert (Chain of Ideas); trust judge rank-ordering, not absolute calibration, across local-model judges (CrossTrace); small (100–200 item) hand-curated, provenance-tagged eval sets with a human-accuracy reference line and memorization decontamination (CiteME, Press); temporal publication-date splits to anchor "novel/already-known" judgments and avoid leakage (Qi, SCIMON). Reject BLEU/ROUGE for open-ended claim/idea judging; use a hard/soft consistency formula for gate-judge-vs-PI agreement (MOOSE).
- **Falsification scaffolding**: decompose an abstract claim into falsifiable sub-implications linked by warrants, with a relevance checker pruning sub-claims not logically implied before they count as evidence, and a lightweight accumulate-evidence-across-independent-checks analog of e-value aggregation (POPPER).
- **Prompt-engineering guardrails** for 7B judges: force the `reason` key before the `overall` verdict, bake non-negotiable hard rules into the prompt ("author-set mismatch is ALWAYS hallucinated"); class-specialist router judges as distinct MCP-callable skills rather than one monolithic prompt (CITETRACER). Self-reference filter so a paper's "we present…" sentences aren't mis-attributed as claims about a cited work (CiteBench).
- **Generate synthetic eval sets** via controlled mutation operators (paraphrase, negation, generalize/specialize, per-field bibliographic mutation) with round-trip diff audits — a single-PI way to label test sets for the claim classifier, dedup, and contradiction detector without multi-user data (FEVER, CITETRACER).
- **Design cautions**: don't over-ground — few-shot exemplars and excessive similarity-weighting suppress novelty/gap-finding (Qi); LLM-authored ideas are incremental recombinations, so keep the PI as the idea source and use the machinery for grounding/retrieval/novelty-flagging (SCIMON); preserve speculative/contradictory claims with flags rather than auto-pruning, for serendipity (Cornelio); inject the PI's accepted-claim context at extraction time so candidates are born consistent, but keep a separate judging step (Cornelio/AI-Hilbert).

**Theme & implication for Memoria**

Collectively this body says: in scientific discourse, *verification grounded in retrieved evidence is the whole game, and a bare LLM judge is not trustworthy for it* — citation/claim judges repeatedly show 16–46% error when ungrounded, while deterministic-rules-first + NLI/retrieval-grounded pipelines approach human accuracy cheaply on small local models. The concrete design move is a **claim-verification spine** that runs on ingest and at the structural review gate: extract `(context, variables, relationship)` atomic claims with verbatim source grounding (claims ADR area), derive supports/refutes/NOTENOUGHINFO edges via a local NLI engine and resolve the global consistent set with MaxSAT (contradictions + supersession), verify every claim→citekey link with body-snippet retrieval and a precision-first per-field bibliographic check (review gate + dedup/record-linkage), and type every cross-paper edge with intent/functional-role/discovery-pattern vocabularies (typed relations + spaces). Throughout, judges only act with evidence in hand and surface POTENTIAL/uncertain cases to the PI rather than auto-acting — and evaluation of these engines uses small hand-curated, temporally-split, memorization-decontaminated sets scored by pairwise/rank rather than absolute LLM scores.

## Domain Applications (health, science, etc.)  (13 papers)

**Most significant papers**
- **[Rao and Callison-Burch - 2026 - BibTeX citation hallucinations in scientific publishing agents evaluation and mitigation.md]** — Proves search-enabled frontier LLMs still botch BibTeX (50.9% correct) and that a deterministic identifier-resolution tool plus a separate revise stage fixes it to 78.3% at 0.8% regression; this *is* Memoria's Zotero/citekey ingest path and its "engines write, agents judge" gate, with a ready-made field-level error taxonomy.
- **[Carrillo et al. - 2026 - Personal data flows and privacy policy traceability in third-party LLM apps in the GPT ecosystem.md]** — Empirically shows a fine-tuned 355M RoBERTa beats GPT-4o zero-shot (0.87 vs 0.61 Macro F1) and that prompt-level "please redact" does NOT stop data egress — direct backing for Memoria's local-small-model and MCP-only hard-deny posture.
- **[Shanmugam et al. - 2024 - Generative AI in medicine.md]** — Best-articulated case for RAG-grounded, source-transparent synthesis plus justification-per-decision literature screening and fine-grained (not top-1) evaluation — maps onto retrieval, discovery triage, and L5 quality criteria.
- **[Thirunavukarasu et al. - 2023 - Large language models in medicine.md]** — Independently motivates Memoria's three core commitments: human-in-the-loop sign-off, per-claim uncertainty flags, and source attribution as a trust requirement.
- **[Whelton et al. - 2018 - 2017 ACC/AHA hypertension guideline.md]** — Offers two concrete schema patterns: orthogonal strength-vs-evidence-quality labels and a "modular knowledge chunk" (claim + supporting text + local refs co-located) that maps onto Memoria's atomic-claim-plus-evidence unit and supersession.

**Adopt**
- **Deterministic citation/identifier resolution as the canonical ingest path** — resolve DOI/arXiv-ID against Zotero Translation Server (CrossRef fallback) for bibliographic fields, with LLMs never touching citation metadata; enforce DOI/citekey resolution as a hard precondition so fabricated provenance cannot be filed (Rao & Callison-Burch; De Angelis).
- **Two-stage generate-then-revise gate** — a separate stage that revises a candidate only against authoritative ground truth, exploiting the very low regression (0.8%) to auto-correct without breaking already-correct fields; this is the structural-review gate's safe-correction property (Rao & Callison-Burch).
- **Six-way field-level error taxonomy** (Correct / Missing / Fabricated / Partial / Substituted / N-A) as the vocabulary for the contradictions/quality dashboard and inspector, replacing binary correct/wrong (Rao & Callison-Burch).
- **Version-aware entity resolution** — link preprint/proceedings/journal versions of one work as a single entity with version-specific metadata, preventing false-duplicate and false-contradiction flags (record-linkage dedup + supersession) (Rao & Callison-Burch).
- **Per-field provenance with multi-source majority-vote/priority reconciliation** (OpenAlex/DBLP/PubMed/Zotero) plus a venue synonym normalization table as the deterministic layer before any LLM-judge escalation (Rao & Callison-Burch).
- **Fine-tuned small/local classifiers over LLM prompting** for structured ingest tasks (claim-sentence classification, MASSW aspect tagging, keyphrase candidates) — cheaper and more accurate (Carrillo et al.).
- **Per-claim uncertainty flags + orthogonal strength/evidence-quality labels** — surface confidence on every extracted claim, and model recommendation strength separately from evidence quality as two typed fields (Thirunavukarasu et al.; Whelton et al.).
- **Source attribution as a hard trust requirement** — every generated claim/answer cites the source sentence(s)/citekey it derives from, grounded only in retrieved vault content (Shanmugam 2024/2025; Tian; Thirunavukarasu et al.).
- **Justification-per-decision in discovery triage** — require the agent to emit an inclusion/exclusion rationale for each screened paper, feeding the warrant/inspectable-decision pattern of the review gate (Shanmugam 2024/2025).
- **Fine-grained evaluation discipline for L5** — separate the prompt/test set from explicit quality rubrics; score on faithfulness, guideline adherence, and input-order robustness, not top-1 correctness, to define "good enough to write" (Shanmugam 2024/2025; Tian).

**Borrow**
- **LLM-as-annotation-aid, small-model-as-production** — use an LLM (TextGrad-optimized) to surface rare-class examples and pre-annotate, with the PI as human-in-the-loop, then train an in-house small classifier; pair with cluster-guided sampling (HDBSCAN over embeddings) for class-balanced training sets and diverse discovery surfacing (Carrillo et al.).
- **Stated-vs-actual diff with three-level disposition** (complete/partial/broken) — structurally the contradictions dashboard; transfers to claim-vs-source faithfulness checks (Carrillo et al.).
- **Editable outbound-payload preview** before any external/MCP action commits — a UI pattern for the structural review gate / read-only inspector (Carrillo et al.).
- **Self-consistency at small k** — sample a local model a few times on an extraction/classification decision and treat agreement as the uncertainty signal (k=100 too costly at single-PI scale) (Liévin et al.).
- **Multi-axis annotation rubrics** — score extracted claims along orthogonal failure axes (reasoning / knowledge / comprehension) allowing co-occurring strengths and defects, rather than a single pass/fail, for LLM-judge evaluation (Liévin et al.; Tian).
- **Watch for label/positional bias** in any classification step — check predicted-label distributions against priors in claim-sentence classification and triage ranking (Liévin et al.).
- **Dual-coder-plus-arbiter against a controlled vocabulary** — engine proposes a taxonomy code, agent cross-checks, PI arbitrates — mirrors aspect tagging under the gate (Martinengo et al.).
- **Frequent-itemset / association-rule mining over already-extracted aspect/claim-type tags** to surface co-occurrence patterns for dashboards and the discovery loop — cheap, deterministic, single-machine (Martinengo et al.).
- **Tiered adaptive escalation with pre-specified numeric triggers** — run cheap deterministic engines first, escalate only items failing an outcome-based quality threshold (low-confidence extractions, borderline pre-file-similarity cases) to a more expensive LLM/agent pass; set go/no-go thresholds in advance (Zheng et al.; Whelton et al. revision-trigger cadence).
- **Modular knowledge-chunk unit + commissioned-review triggers** — co-locate each claim with its supporting text and refs so it can be superseded in place, and define explicit practice-changing-evidence triggers (not continuous churn) that flag a claim cluster for re-review — echoes deferred-ADR cadence model (Whelton et al.).
- **Faithfulness over self-explanation** — do not treat LLM-generated rationales as authoritative warrants in the argument graph; verify against source. Beware models amplifying training-data correlations (a frequently-stated association may be model-amplified, not evidence-weighted) (Shanmugam 2024; Cheng et al.; De Angelis).

**Theme & implication for Memoria**
Collectively this body — even the lower-significance medical surveys — converges on exactly Memoria's design bets: hallucination is endemic, prompt-level safeguards ("please redact," "cite correctly") do not hold, so trust must come from deterministic retrieval, source-grounded provenance, small/local classifiers, and a human (PI) accountable at the gate. The one paper that goes beyond restatement, Rao & Callison-Burch, gives Memoria buildable mechanism: make identifier resolution the canonical citation ingest path (never LLM parametric memory), adopt the generate-then-revise gate whose 0.8% regression makes safe auto-correction possible, and import the six-way field error taxonomy plus version-aware entity linking into the contradictions/dedup/supersession layers. The concrete near-term moves: (1) wire deterministic DOI/citekey resolution as a hard ingest precondition; (2) split ingest into engine-candidate then deterministic-revise stages; (3) enrich the claim model with orthogonal uncertainty/strength/evidence-quality fields and per-field provenance; and (4) require justification-per-decision rationales in discovery triage so the review gate has inspectable warrants.

## Memory & Personal Knowledge Management  (8 papers)

**Most significant papers**
- **[Maharana et al. - 2024 - Evaluating very long-term conversational memory of LLM agents.md]** — LOCOMO gives direct empirical proof that retrieving over a database of atomic *observations* beats long-context and raw-log retrieval, validating Memoria's atomic-claim store as a core pillar.
- **[Wu et al. - 2025 - LongMemEval benchmarking chat assistants on long-term interactive memory.md]** — A clean indexing→retrieval→reading architecture with four control points (value granularity, key, query, reading), plus fact-augmented key expansion and knowledge-update/abstention as first-class abilities — a buildable skeleton for Memoria's retrieval layer.
- **[Uddin et al. - 2026 - From recall to forgetting benchmarking long-term memory for personalized agents.md]** — Memora/FAMA give Memoria a concrete metric and memory-trace ground truth for the hardest failure: memory stores get *worse* over time because they never forget — directly motivating supersession as a retrieval filter, not just a dashboard.
- **[Hu et al. - 2026 - Evaluating memory in LLM agents via incremental multi-turn interactions.md]** — A four-competency rubric (Accurate Retrieval, Test-Time Learning, Long-Range Understanding, Selective Forgetting) that maps onto Memoria's retriever, synthesis, and supersession layers as separately-tested axes; no agent masters multi-hop forgetting.
- **[Liu et al. - 2026 - The last human-written paper agent-native research artifacts.md]** — The ARA four-layer ontology (logic/src/trace/evidence) with forensic claim→evidence bindings, dead-end nodes, and a staging→crystallization promotion gate is almost a vault-native restatement of Memoria's claims + argument-graph + review-gate design.
- **[Liu et al. - 2026 - Omni-SimpleMem autoresearch-guided discovery of lifelong multimodal agent memory.md]** — Multimodal Atomic Units with hot/cold storage and a deterministic, model-free hybrid retrieval recipe (union-and-append dense+sparse, token-budgeted pyramid expansion) that fits Memoria's small-model, faithful-ingest constraints.

**Adopt**
- **Atomic-observation/claim-granularity retrieval over raw logs or summaries** — LOCOMO and LongMemEval both show observation/per-round units beat log- and summary-level retrieval; validates ingesting papers into atomic claims and running triage over claims, not PDF text (Maharana 2024; Wu 2025).
- **Supersession as a first-class retrieval filter, not just a dashboard** — Multiple benchmarks show every memory agent fails to forget, and explicit stores get worse at longer horizons (18→29% reductions); retrieval must actively down-weight/exclude superseded claims, with a deterministic typed supersession edge gated by review rather than trusting an LLM to forget (Uddin 2026; Hu 2026; Wu 2025).
- **Append-only memory trace / supersession ledger** — Record full memory state before/after each add/update/delete as ground truth for valid-vs-invalidated; this is exactly the substrate for both the contradictions dashboard and forgetting-aware retrieval, and it is deterministic and local (Uddin 2026).
- **Four-stage indexing→retrieval→reading model with explicit control points** — Use value granularity / key / query / reading as the design skeleton for the retriever/scout layer over the vault (Wu 2025).
- **Fact-augmented key expansion** — Index the full claim/note keyed with extracted keyphrases/facts (document expansion, +9.4% recall) rather than replacing it with a lossy summary; directly informs keyphrase tag candidates and the pre-file similarity gate, and reinforces faithfulness-over-compression (Wu 2025).
- **Small, high-precision retrieved evidence sets** — Top-5 observations beat top-25/50; more context degrades the reader, so keep the pre-file gate tight and rerank aggressively before handing claims to an agent (Maharana 2024).
- **Deterministic, model-free hybrid retrieval** — Set-union of dense (FAISS) + sparse (BM25), keeping dense ranking and appending BM25-only hits (which beat score-fusion re-ranking that "disrupts semantic ordering"); a concrete, cheap recipe for the retriever and pre-file similarity gate (Liu — Omni-SimpleMem 2026).
- **Token-budgeted progressive pyramid expansion** — Summaries → full claim text → source paragraph, each gated by a similarity threshold and explicit token budget under deterministic rules; fits qwen2.5:7b's limited context and the projection-engine "show summary, drill on demand" pattern (Liu — Omni-SimpleMem 2026).
- **Hot/cold storage split via Multimodal Atomic Units** — Lightweight searchable record (summary + embedding + links + provenance pointer) as hot metadata = the markdown note; original PDF/paragraph = lazily-loaded cold store (Liu — Omni-SimpleMem 2026).
- **Text novelty filter before ingest** — Jaccard/near-duplicate overlap with recent summaries to refuse writing near-redundant notes; the text path of selective ingestion is exactly the pre-file similarity gate / dedup need (CLIP/VAD multimodal parts out of scope) (Liu — Omni-SimpleMem 2026).
- **Hybrid entity resolution + decayed h-hop expansion** — Cosine + Jaro-Winkler above threshold for record-linkage dedup, with distance-decayed neighborhood expansion for multi-hop "connect these two notes" discovery across the claim graph (Liu — Omni-SimpleMem 2026).
- **ARA four-layer ontology as vault note-types/relations** — Falsifiable claims with proof pointers, typed related-work dependencies, an evidence layer grounding every claim, and explicit dead-end/pivot nodes that keep negative knowledge queryable ("git log for research") (Liu — ARA 2026).
- **Cross-layer forensic bindings** — An explicit claim→evidence(→warrant) relation type with resolution checking as the argument-graph backbone (Liu — ARA 2026).
- **Staging → crystallization promotion gate** — Raw observations live in /staging and only promote to formal claims on closure signals (abandonment, affirmation, empirical resolution); a concrete mechanism for the structural review gate that avoids premature structure (Liu — ARA 2026).
- **Mutation-benchmark / planted-defect testing with deterministic structural checks** — Seed the vault with known defects (fabricated claim, missing falsification, orphan reference, over-claim, rebutted-branch leak) and measure catch rate; orphan-reference detection belongs in a deterministic check, not the LLM — a direct rule for the gate (Liu — ARA 2026).
- **FactScore / atomic-fact precision-recall acceptance test** — Decompose engine-written notes into atomic facts and score precision/recall against source as the faithfulness metric at the "engines write, agents judge" gate (Maharana 2024).
- **Knowledge-update and abstention as evaluated targets** — Represent an updated fact via versioned supersession without destroying the prior, and abstain/flag false premises rather than confabulate (Wu 2025; Maharana 2024).
- **Complementary-learning-systems framing** — The vault is the fast one-shot episodic store; the structural-review gate + supersession is slow consolidation — names exactly why "capture fast, judge/consolidate deliberately" is sound, and justifies content-addressable similarity retrieval at single-researcher (limited-experience) scale (Hassabis 2017).

**Borrow**
- **Cheap whole-corpus regression eval** — Inject a known corpus once and probe retrieval/supersession with many queries (cost-efficient one-context-many-questions design); build a typed question set — single-hop, multi-hop, temporal, knowledge-update, adversarial-abstain — for the read-only inspector / eval harness (Hu 2026; Maharana 2024; Wu 2025).
- **RAG-vs-long-context split of duties** — RAG wins accurate snippet retrieval; long-context/holistic passes win test-time learning and long-range understanding — so don't make discovery purely top-k; contradiction detection and argument-graph synthesis need structured whole-corpus passes (Hu 2026).
- **Structure-augmented retrieval helps forgetting** — Knowledge-graph/event-timeline memory (HippoRAG-v2) is strong on retrieval+forgetting, supporting investment in graph-based retrieval over Memoria's claim graph (commercial Mem0/Zep/Cognee themselves out of scope) (Hu 2026).
- **Temporal/causal event-graph representation** — Typed temporal/causal edges for tracking how a finding evolves over time (reject the generative-agent simulation framing) (Maharana 2024).
- **Time-aware indexing with LLM query-range extraction** — Timestamp-scoped retrieval for "which claim was current as of date X," but gate the time-range extractor or escalate to qwen3:14b, since small models hallucinate ranges (Wu 2025).
- **Chain-of-Note reading + structured-JSON item formatting** — Extract per-item notes then reason, presenting retrieved claims as structured JSON (up to ~10 QA points even under perfect retrieval); cheap and model-agnostic for the synthesis/note-drafting step (Wu 2025).
- **Structured prompting discipline for extraction** — Enforce JSON response_format and mind constraint positioning to stop verbosity from polluting fields when prompting qwen2.5:7b for claim extraction (Liu — Omni-SimpleMem 2026).
- **Read-only / concurrent retrieval indices** — So the nightly discovery loop and interactive queries don't serialize (Liu — Omni-SimpleMem 2026).
- **Generate→validate→fix loop + stated-vs-inferred provenance tags** — A schema-validation fix loop for extracted claims, with provenance tags distinguishing stated vs inferred knowledge; plus the ARA Seal lesson: have the LLM emit findings with severity + verbatim evidence spans and compute the verdict deterministically to dodge grade inflation (Liu — ARA 2026).
- **Withhold-conclusion auditing** — When a judging agent audits a claim's support, withhold the asserted conclusion so it reasons from evidence; a context-scoping rule for the MCP-only review profile (Liu — ARA 2026).
- **Artifact lineage as declared-parent + structured diff** — Track how the PI's notes derive from and supersede prior ones (self-maintaining upstream propagation is over-engineered for single-PI scale) (Liu — ARA 2026).
- **Atomic-criteria LLM-as-judge** — Binary atomic criteria for the "agents judge" gate; downscale the 3-frontier-model majority vote (88% human agreement) to a single cheap local judge and calibrate before trusting it (Uddin 2026; Wu 2025; Maharana 2024).
- **Grounding-check-then-spot-check fixture validation** — Automated grounding checks plus a small human spot-check, rejecting batches on inconsistency, for validating synthetic test fixtures (Uddin 2026).
- **Controller/external-memory + protect-prior-structure principles** — The NTU/DNC controller-with-external-store mirrors "engines write, agents judge" (agents hold no durable state); EWC and prioritized replay loosely inform anchoring prior structure on update and prioritizing salient items in learning-to-rank triage / the nightly loop; interpretability tooling aligns with the read-only inspector (Hassabis 2017).
- **Shared-vs-specific knowledge separation** — Loosely informs how relation graph + spaces could segregate cross-cutting concepts from project-specific notes; purely conceptual — Memoria has no shared-parameter interference problem (Wang 2022 — L2P).

## Temporal Reasoning & Knowledge Updating  (2 papers)

**Most significant papers**
- [Abdallah et al. - 2026 - TEMPO a realistic multi-domain benchmark for temporal reasoning-intensive retrieval.md] — First benchmark fusing temporal reasoning with reasoning-intensive retrieval; gives Memoria ready-made evaluation primitives (Temporal Coverage@k, step-wise plans, ISO time scopes) for claim supersession and the contradictions dashboard, and the blunt finding that temporally-wrong evidence is worse than none.
- [Si et al. - 2023 - Prompting GPT-3 to be reliable.md] — Shows in-context knowledge updating (~85% adoption of a conflicting in-prompt fact vs ~33% for supervised T5), retrieval grounding, and confidence-based selective prediction; it is the mechanistic backbone for Memoria's knowledge-updating, RAG grounding, and confidence-triaged review gate.

**Adopt**
- Step-wise retrieval planning: decompose a complex query into sequential, time-targeted steps with gold docs per step, keeping the original query as context (Query+Step beat Step-Only by ~12 NDCG, +18 for ReasonIR) (TEMPO). A supersession query ("how has claim X been revised") decomposes into a baseline-period step and a current-period step — exactly the multi-hop shape Memoria's discovery/triage and contradiction-detection need.
- Temporal Coverage@k / Temporal Precision@k as retriever-eval metrics measuring whether top-k spans both the baseline AND comparison time windows (TEMPO). Operationalizes "a good supersession answer covers the old and new claim's time windows, not just the most recent" — directly scores the retriever/scout profile and supersession surfacing.
- Stamp each atomic claim with an ISO temporal scope [ts,te] plus dominant tense and a confidence score during deterministic extraction (TEMPO passage-level annotations; Si confidence scoring). This lets the contradictions dashboard and supersession ordering reason about which claim is newer / in-force rather than guessing from filing date.
- Feed superseding/contradicting claims into the extraction and judgment prompt so updated knowledge overrides stale parametric belief, and flag the conflict instead of silently trusting memory (Si in-context knowledge updating, ~85% adoption; smaller local models gain more from grounding). This is the concrete mechanism behind claim supersession.
- Confidence-based selective prediction: tag each extracted claim with a confidence score (token-logprob or self-consistency frequency over ~10 samples), auto-accept high-confidence, queue the rest for PI review (Si calibration). Turns the structural review gate from binary pass/fail into a triaged worklist and powers the uncertainty flags.
- Abstain rather than inject low-confidence or temporally-mismatched evidence: in TEMPO, no-retrieval (77.3) beat every RAG config and oracle retrieval only reached 80.5, because wrong-period evidence actively misleads the generator (TEMPO). Strong justification for the pre-file similarity gate and review gate — "faithfulness over flash" is empirically cheaper than feeding bad evidence.

**Borrow**
- A lightweight subset of TEMPO's 10-class temporal taxonomy (trend / cross-period comparison, supersession, causation) to tag PI queries so triage routes hard cross-period questions differently; the full 10-class scheme is more than a single vault needs (TEMPO).
- Annotate queries with explicit temporal-intent tags before retrieval (the "Normalized" variant gave +8 NDCG for ReasonIR) — cheap, deterministic preprocessing that boosts local embedders rather than hoping they infer time intent (TEMPO).
- LLM-as-judge for temporal relevance / review-gate scoring, but swap TEMPO's Qwen-72B and the GPT-4o/Gemini hard-negative pipeline for local qwen2.5:7b / qwen3:14b, keeping the PI as the human kappa anchor (TEMPO; local-first constraint).
- PI- or template-supplied question decomposition steers multi-hop reasoning far better than self-generated CoT (Si) — feed structured sub-claims into argument-graph / warrant construction; fits the structured-capture-forms philosophy.
- Keep calibration measurement (Brier/ECE) separate from accuracy tuning when validating the local extractor: more in-context demos improved accuracy but NOT calibration (Si). Adopt the entity-swap counterfactual recipe (Si) and TEMPO's temporal-mismatch negatives to test, in the L5 quality track, whether the local model honors superseding claims over memorized facts before trusting the pipeline.

**Theme & implication for Memoria**
Together these papers say that knowledge-updating is achievable by grounding — a newer claim placed in-context will override stale model belief most of the time — but ONLY if retrieval is temporally complete; temporally-wrong evidence is measurably worse than abstaining. The concrete design move is to make time a first-class, deterministic field: stamp every atomic claim with an ISO temporal scope and confidence, decompose supersession/contradiction queries into baseline-vs-current retrieval steps, and gate evidence on a Temporal-Coverage check so the contradictions dashboard and supersession ordering reason over explicit time windows rather than inferred recency. Confidence scores then drive a triaged structural review gate (auto-accept high, queue low for the PI), which is exactly how Memoria turns "engines write, agents judge, PI approves" into a calibrated worklist while honoring the local-first, faithfulness-over-flash constraints.
