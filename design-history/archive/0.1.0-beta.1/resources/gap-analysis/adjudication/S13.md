### S13.3 — Relational cascade-rollback (ADR-127 2PC journal) -> git snapshots + VACUUM INTO + footprint-scoped reconciliation

Design s6 is an explicit decision-with-rationale: 'both, because they version different things' for git+VACUUM INTO, external git-dir to avoid collision with the user's vault repo, and 'Revert is DB-complete via reconciliation, not snapshots' because per-run DB snapshots 'would be write amplification in service of inverting ledgers the design defines as append-only'. Alpha's reason for the relational cascade - verdicts/records were relational and deliberately not file-recoverable - is dissolved by beta.1's two-class split, under which every derived table is re-derivable from post-revert files and checks re-earn via the local Checker.

- **Recorded:** 0.1.0-beta.1-design.md s6 L448-452 and L501-510; s11 convergence list L965-970; 0.1.0-beta.1-alpha-regression-review.md intact sweep (L73-74)
