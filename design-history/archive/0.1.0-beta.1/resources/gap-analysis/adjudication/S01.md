## 1. Host / runtime / distribution

### S1.1 — Obsidian plugin returns as the required researcher UI

**LEAVE — deliberate and recorded** · verification: confirmed (citation audited) · confidence: high

The delta is adjudicated in two authoritative records I verified: design §16's first bullet ('ADR obligation, named') states the host decision amends ADR-125/ADR-130, requires that ADR before implementation begins, and keeps docs/reference/obsidian-plugins.md unchanged so the proposal is 'not product truth'; design §11 item (10) records the host decision as new alpha.16 scope. The alpha.14 removal was scope discipline plus the keep-test, not a plugin-failure incident, and beta.1 preserves the keep-test substance: charter Success Claim 4, no-plugin floors on every surface except Ask (§1/§16), mobile as keep-test mode, and the plugin inheriting ADR-121's enqueue-only rule (§2: 'nothing in the plugin writes machine content').

- **Recorded:** design.md §16 'ADR obligation, named' (L1152-1158) + §11 item (10) (L987-991); regression review buckets it as recorded (L78-81)
- **History:** Alpha.14 cut the .obsidian tree because 'the product must remain fully operable if the editor is only vim, nano, or a file browser' (history L1930, PR #1124) and alpha.15's Scope Boundary plus PR #1203 fenced editor adapters behind their own ADR (history L2216) — scope and keep-test reasons, not a documented plugin failure. Beta.1 addresses both: the keep-test survives as charter Claim 4 with per-surface no-plugin floors, and the ADR fence is honored by §16's named ADR obligation.

### S1.2 — On-demand serve promoted to plugin-supervised resident two-process daemon with in-daemon scheduling

**LEAVE — deliberate and recorded** · verification: confirmed (citation audited) · confidence: high

Residency, plugin supervision, and the supervisor/worker split are explicit decisions-with-rationale in the design body: §1 L51-65 (compute out of Electron for crash isolation and one network choke point; parent-death watch; crash-loop give-up), §16 'LSP lifecycle, verbatim' with its evidence base (Smart Connections' in-Electron 500MB index/hangs as the failure archetype), and §11 item (10) naming 'plugin + local daemon' as the recorded host decision under the §16 ADR obligation. The scheduling move is governed too: §2's operational contract is the docs-history 'memoriad job contract' borrow (an adjudicated recorded decision), and the always-on citation monitor rides §17 item 4 plus the charter's amended discovery exclusion (2026-07-03). Beta.1 addresses each reason alpha held the no-daemon line: the daemon is session-scoped (dies with Obsidian), the consumption-guard family is intact per the regression sweep, and heartbeat-on-success plus bounded retries directly answer the silent-cron-failure risk.

- **Recorded:** design.md §1 (L51-65), §2 operational contract (L170-186), §16 LSP bullet (L1172-1177), §11 item (10) (L987-991), §17 item 4 (L1253-1257); requirements.md Deliberate Exclusions amendment (L84); docs-history Borrow 'memoriad job contract' (L34)
- **History:** The no-daemon stance was earned by shedding Hermes weight — ADR-125's goal 'one install command, no external runtime, no always-on process' (history L2190, L2304) — plus ADR-127 rejecting the watcher-as-required-daemon while keeping 'guard never retired' (L2186), and ADR-95's unattended loop rejected as 'conflicts with no-daemon arch' (L2233) with silent cron failure and inbox flooding the named risks (L547). Beta.1 addresses all three: one artifact with no separate always-on install unit, the guard family verified intact, heartbeat rows as the silent-non-execution detector, and review-queue back-pressure bounding flood.

### S1.3 — CLI demoted from product surface to survival floor; mem ask dropped

**LEAVE — deliberate and recorded** · verification: confirmed (citation audited) · confidence: high

The drop is explicitly ruled with rationale in the design body: §1 L89-93 ('No mem ask — Ask is a judgment surface with no survival role, and the keep-test answer is stock-Obsidian navigability, not a CLI') and §16's 'Two rich panes, floors everywhere' bullet restating that Ask alone has no floor 'by §1's own ruling'; the demotion itself rides the recorded §11 item (10) host decision. ADR-72's underlying failure — routine actions reachable only through an LLM round-trip with the Co-PI — is not re-opened: the plugin panes are direct, non-agent-mediated PI surfaces, so PI-direct-access is honored by a different mechanism. ADR-130's 'no-daemon recovery surface' role is kept in substance by the mem survival floor (serve/capture/export/rebuild/backup/restore/migrate/revert-run), installed by construction wherever the daemon runs (§16 'One artifact').

- **Recorded:** design.md §1 (L89-93) + §16 'Two rich panes, floors everywhere' (L1178-1183), riding §11 item (10) (L987-991)
- **History:** ADR-72 was earned by 'some actions were reachable only by asking the Co-PI... making routine actions depend on an LLM round-trip' (history L779) — an anti-agent-mediation principle, not a pro-CLI one; ADR-122 had already held 'CLI is inspect/verify/administer only, not a PI action surface' when Obsidian was the UI (L1896); alpha.14/15 made the CLI the direct-access product surface only because Obsidian was dropped (L2188). Beta.1 restores the direct-access surface to the plugin, so the reason the CLI was promoted no longer applies, and ADR-72's principle is honored by the plugin.

### S1.5 — Distribution artifact changes class: Python package + per-vault venv to self-contained static binary with bundled models

**RECORD — sound but unrecorded** · verification: confirmed (skeptic found no prior record) · confidence: medium

The delta is sound on the merits: the charter's ≤30-minute clean-machine install bar and hash-pinned bundled-model supply-chain row, plus §16's acquisition story (winget/brew and the rust-analyzer consented-download model presuppose a self-contained artifact), all require it, and removing Node/qmd completes what alpha.15 itself earmarked (§11 records FTS5+sqlite-vec as convergence). But no record adjudicates the reversal of ADR-125's pinned Python 3.12 + uv-tool + per-vault-venv stack as such — §11's divergence list omits it, §16's recorded rationale covers plugin+daemon+acquisition rather than the stack class, and the only design-body traces are presuppositions (§7's pdfium bullet 'matching the single-static-binary install; PyMuPDF's AGPL + Python contradicted it'). Per the calibration rule, 'amends ADR-125' without the recorded rationale covering this half is not a leave.

- **Recorded:** nowhere - silent (fragments at design §7 L633-636 and §16 L1159-1171 presuppose the binary but never adjudicate the stack-class change)
- **History:** The alpha line was Python-shaped by stack pragmatics, never failure-earned against binaries: ADR-76's wheel was recast because a wheel 'cannot carry SOUL.md/plugin.yaml/crons' (history L989) — a problem that dissolved when ADR-125 made operations product code — and ADR-125 then pinned Python 3.12 + uv tool install as the runtime (L2096, L2190). Beta.1 keeps the surviving disciplines (checksum pinning, one artifact, one-command install) while changing the artifact class, so no documented failure is re-opened; only the recording is missing.
- **Action:** Add to design §11's divergence list (after item (10)): "(11) the distribution artifact — beta.1 ships mem as a single self-contained static binary with installer-bundled hash-pinned ONNX models, superseding ADR-125's Python 3.12 + uv-tool + per-vault-venv stack pin. Forced by the §16 acquisition story (winget/brew and the rust-analyzer consented-download model presuppose a self-contained artifact), the charter's ≤30-minute clean-machine install and bundled-model supply-chain rows, and FTS5+sqlite-vec completing alpha.15's own earmarked removal of the qmd/Node dependency. The §16 host ADR must cover this stack amendment explicitly, not only the plugin+daemon shape."

### S1.7 — Write-event push channel + lockstep plugin/daemon/CLI version handshake replace FS polling and the free-schema stance

**LEAVE — deliberate and recorded** · verification: confirmed (citation audited) · confidence: high

Both halves are explicit decisions-with-rationale in the design body: the push channel in §1 L56-60 and §16 'Write-event push, not watcher faith' (Obsidian's watcher provably misses external writes; a stale check badge is an honesty-contract violation, not cosmetic), and the handshake in §2 L183-186 ('stale boundary code waves through what a newer runtime treats as mutating') reinforced by §15 (the lockstep handshake is load-bearing in the authored-chrome ruling) and standing negative deny-tests proving boundaries fire on the installed build. The handshake is additionally an adjudicated docs-history borrow ('Guard liveness + fail-closed contract'). Alpha's free-schema/no-handshake stance was not failure-earned — a single artifact made version skew structurally impossible — so making skew refusable when three components ship is the same invariant extended, not a reversal of an earned position.

- **Recorded:** design.md §1 (L56-61), §2 (L183-186), §16 'Write-event push, not watcher faith' (L1192-1195), §15 (L1080-1084); docs-history Borrow 'Guard liveness + fail-closed contract' (L35)
- **History:** serve --watch was stdlib polling kept deliberately on-demand so 'the worker remains the correctness boundary' and the guard is 'never retired' (ADR-127 Alternatives, history L2186); --json was transport 0 of the read-API with no second component to skew against (ADR-130, L2176). Beta.1 keeps the guard family intact (regression sweep) and adds push/handshake against two documented failure classes: Obsidian's watcher missing external writes (§16 evidence base) and the enforcement-theater pattern where deployed boundary code silently diverged from source (docs-history L144).

The one risk a folded artifact could raise — component version skew — is the exact failure the §2 handshake and one-artifact rule close, consistent with the enforcement-theater lesson in the docs-history patterns (L144).

The never-half-write property rests on the staging spine the regression review verified intact.

- **Recorded:** requirements.md Required Product Scope 'Offline' row (L35) + design.md §1 (L66-69, staged writes §6)
- **History:** None found as an earned alpha position — A15 simply ran local operations directly and let LLM-dependent operations fail offline, with no documented failure or decision behind the absence of a contract. The nearest precedent, cross-store recoverability and observable atomicity (ADR-127, history L2128), is the machinery beta.1's staged-write offline guarantee generalizes, so the alpha lineage is honored rather than reversed.
