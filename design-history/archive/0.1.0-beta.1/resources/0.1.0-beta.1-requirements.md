# Memoria — 0.1.0-beta.1 requirements charter

Date: 2026-07-03
Status: proposal. Evidence base:
[0.1.0-beta.1-research-evidence.md](0.1.0-beta.1-research-evidence.md). System design:
[0.1.0-beta.1-design.md](0.1.0-beta.1-design.md). Alternatives summary:
[0.1.0-beta.1-design-alternatives.md](0.1.0-beta.1-design-alternatives.md).

Beta.1 is a retention experiment for one real researcher on a real corpus, not a
feature milestone. It must prove or falsify whether Memoria beats the current
null workflow: Zotero + Obsidian + ad-hoc ChatGPT.

## Success Claims

1. **Parity:** capture is no slower than Zotero, note-writing is no slower than
   plain Obsidian, and Ask costs no more than an acceptable attention premium
   over ChatGPT while doing corpus-grounded work ChatGPT cannot do alone.
2. **Payoff:** at least one real compose moment is more useful than a same-day
   archived ChatGPT counterfactual and ends in a portable exported draft.
3. **Tolerance:** weekly upkeep, verification time, and paid API spend stay
   within the researcher's pre-registered budgets with no trust incident.
4. **Survival:** the vault remains useful as plain markdown plus standard
   bibliography files if Memoria dies.

## Required Product Scope

| Requirement | Beta.1 bar |
| --- | --- |
| Capture | PDF, DOI/arXiv ID, BibTeX-file, Zotero, and GitHub-repo import reaches catalog, full text, and optional digest with one initiating action and no mandatory per-item decisions. |
| Reading | Full text is readable in the vault; selecting a span creates an anchored note in at most two actions. |
| Ask | Questions search the local corpus, cite source spans, state coverage denominators, and say when evidence is absent. |
| Compose | A project can gather evidence, show gaps, produce a markdown draft, and export markdown plus BibTeX with resolving citations. |
| Review | Machine writes are visible, reversible by run, and reviewable without creating a catch-up queue after absence. |
| Provenance | Normal reading distinguishes human, machine-unchecked, and machine-checked content; source-trust remains separate from check status. |
| Offline | Reading, writing, catalog browsing, and local search work offline; paid engines queue or fail cleanly. |
| Knowledge format | The `Memoria/` tree is an OKF v0.1-conformant bundle: required typed frontmatter on every Memoria-created file plus generated `index.md` projections — parity with the alpha.15 vault, no regression of the exchange format. |

## Required Evaluation Scope

Owning design artifact: [0.1.0-beta.1-evaluation-protocol.md](0.1.0-beta.1-evaluation-protocol.md) (the protocol
skeleton the researcher instantiates into the frozen pre-registration).

| Requirement | Beta.1 bar |
| --- | --- |
| Pre-registration | Hypotheses, metrics, thresholds, phase dates, amendments, and failure criteria are written before day 1. |
| Baseline | The null workflow has enough measured history to make parity and payoff claims comparable. |
| Passive logging | Core workflow events, cost, run outcomes, and review/revert activity are logged locally without per-item bookkeeping. |
| Counterfactuals | Payoff claims compare Memoria output with archived same-day ChatGPT output for the same question. |
| Abandonment | No-use windows, fallback to null tools, and failed resumption are pre-defined and scoreable. |
| Calibration | Every numeric threshold is labeled as a pre-registered guess until beta.1 data calibrates it. |
| Checker calibration | Stance/Checker NLI thresholds are tuned on a small pre-registered labeled set before day 1 and frozen; any change is a dated amendment (a new arm). |
| Citation audit | The weekly random audit uses a recall/precision rubric, samples for evidence completeness, oversamples near-threshold NLI promotions and marginal items, is sized for a double-digit fabrication base rate, and probes whether Ask resurfaces superseded-flagged works unlabeled. |
| Verdict basis | Qualitative researcher judgment is primary; event-log metrics are evidence into it. A missed engagement number alone maps to targeted redesign, never kill. Sequential verdict arithmetic is pre-registered and anytime-valid; a verb-mix shift between project phases is an expected pattern, not drift. |
| Leading indicators | Relegation clusters (failed/empty Asks followed by narrowing to capture-only use) are tracked ahead of the abandonment trigger. Export refusals are scored as the citation gate working, not as failures. Synthesis-shaped Asks are instrumented separately so payoff failures can be attributed to evidence granularity vs grounding. |

## Required Operations Scope

| Requirement | Beta.1 bar |
| --- | --- |
| Install | Clean-machine install reaches first value from a quickstart in under 30 minutes. |
| Upgrade and migration | Beta.1 data survives later versions through exercised, lossless migrations. |
| Backup and restore | A full backup restores on different hardware and passes health verification. |
| Rebuild | Derived state is regenerable from canonical files with stated time and cost. |
| Cost control | Routine operations have fixed ceiling prices, monthly spend is visible, and large/cap-crossing work requires confirmation. |
| Key handling | API keys never enter vault content, backups, exports, or logs. |
| Failure handling | User-facing failures name the operation, data state, and next action. |

## Required Trust Scope

| Requirement | Beta.1 bar |
| --- | --- |
| Human text | Machines never overwrite or delete human-authored text; a violation stops the beta immediately. |
| Citation honesty | Emitted citations resolve to corpus spans; absent evidence is reported as absence. |
| Prompt injection | Untrusted text cannot control destination, tool choice, or vault egress. |
| Revert | A machine run can be reverted with a preview, preserving human text byte-for-byte. |
| Canary checks | Known key/canary strings do not appear in logs, exports, or model-bound payloads. |
| Overreliance | Trust-incident defenses are attributed only to vigilance-independent mechanisms (taint, manifest gate, revert, export gate) — never to the researcher noticing a marker (explanations do not prevent overreliance). |
| Model supply chain | Bundled local model weights (embeddings, NLI) are hash-pinned and verified at load; a poisoned checker must not be able to silently pass claims. |

## Deliberate Exclusions

| Exclusion | Reason |
| --- | --- |
| Open-ended external search and recommendation | Corpus-only answers keep gap claims honest. Amended 2026-07-03: bounded citation-graph discovery (snowball ≤ depth 2 + a capped citation monitor; identifiers-only egress; output never enters Ask or gap denominators) is in scope; open-ended search and recommender feeds remain out. |
| Aggregate evidence scores | Vote-counting style meters mislead more than they help for beta.1. |
| Deep code intelligence | Code files must survive safely, but code understanding is not needed to prove retention. |
| Concurrent multi-device writes | Sync conflict handling is a data-loss risk outside the first beta claim. |
| Import-back of exported works | It matters only after compose has already produced work. |
| Support-org machinery | SLAs, polished debug bundles, and continuous observability wait until there is a support surface. |
