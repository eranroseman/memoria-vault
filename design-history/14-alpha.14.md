## alpha.14 ŌĆö The standalone CLI + engine cut

**Theme: Memoria stops being an Obsidian/Hermes/Zotero/MCP/profile assembly and becomes one standalone `memoria` CLI over a SQLite engine, then builds out the DOIŌåÆOpenAlex/Crossref graph, full-text acquisition, gap/discovery, and universal Concept frontmatter on top of it.**

Alpha.14 is a full architecture cut, not a partial phase. The alpha.12 SQLite working-state boundary (ADR-122) and alpha.13 DOI enrichment (ADR-123) had already landed, so the engine existed; this release reconciles the surface/runtime drift by declaring the CLI+engine the product boundary and *dropping* ŌĆö not deferring ŌĆö Obsidian, Zotero, Hermes, MCP, installed profiles, lane/fleet, and `direct_api` as a product runner (`0.1.0-alpha.14-design.md` ┬¦0, ┬¦7; `0.1.0-alpha.14-release-plan.md` Status). The commit range splits into two waves: a foundational cutover (CLI spine #1045ŌĆō#1053, runtime port #1114ŌĆō#1121, dropped-surface sweep + `rg` negative gate #1118ŌĆō#1130) and a change-heavy feature wave (#1131ŌĆō#1157) that fills the graph-import, full-text, gap/discovery, retrieval, and frontmatter holes the cut opened. There is no production vault, so there is deliberately **no backward-compatibility mode, no migration path, and no legacy runtime branch** (`design.md` ┬¦0 Cutover rule).

### Vault / Concepts

- **What:** Universal Concept frontmatter is now *enforced* for every file under `knowledge/` and `capabilities/`: each Concept must carry `id` (= its bundle-relative path without `.md`), an approved `standing` (`current|superseded|retracted|archived`), and a map-shaped `links: {}` (new `apply_universal_concept_frontmatter` / `universal_concept_frontmatter_errors` in `src/memoria_vault/runtime/vaultio.py`; `UNIVERSAL_CONCEPT_BUNDLES={knowledge,capabilities}`). Trusted-writer validation and `note capture` now default and check these; shipped operation manifests, type schemas, and PI templates were backfilled (commit `88b7f0f5` / PR #1157; `trusted_writer.py`, `subsystems/lib/schema.py`, `vault-template/.memoria/schemas/types/{digest,hub,adapter}.yaml`, every `capabilities/operations/*.md`). **Why:** `knowledge/` and `capabilities/` are declared OKF Knowledge Bundles where "a Concept ID is the path without `.md`" and "every Concept is YAML frontmatter followed by a Markdown body" (`design.md` ┬¦3 OKF bundle contract, ┬¦12 "Universal Concept frontmatter ŌĆ” `id`, `standing`, and map-shaped `links`"). Without enforcement, `standing`-based supersession filtering (┬¦10 retrieval hides superseded/retracted) and map-shaped `links` graph traversal have no guaranteed field to read; the gate ┬¦14 explicitly requires "Universal Concept frontmatter requires `id`, `standing`, and map-shaped `links`."

- **What:** Type schemas were tightened to make the universal fields *required* rather than optional and to add the `standing` enum. In `digest.yaml`/`hub.yaml`, `id: str`, `standing: enum:standing`, and `links: map` moved into `required:`, and `links` was removed from `optional:` (was `links: list`), i.e. links became a map not a list (`88b7f0f5`). **Why:** the alpha.13 object model carried `links` as an ad-hoc optional list; alpha.14's model (`design.md` ┬¦4) makes `links: {}` a typed map on the frontmatter baseline so machine graph reads are shape-stable, and makes `standing` first-class so lint can surface superseded/retracted evidence (┬¦9 check surface: "stale-as-current evidence, retraction/contested evidence").

- **What:** New tracked generated projection `knowledge/_views/index.md` ŌĆö a worker-owned "Knowledge views index" carrying a `Generated by memoria_vault.runtime.projections; edit source Concepts instead` banner (PR #1156 `f9e56093`; `runtime/projections.py`, `vault-template/knowledge/_views/index.md`). It was wired into projection drift checks, direct-write scan quarantine, `init --dry-run` targets, runtime replay, package refresh, and shipped template evidence. **Why:** `status --json` and generated `knowledge/_views/` replace the retired dashboard glance surfaces (`design.md` ┬¦5 "Generated views under `knowledge/_views/` cover daily glance, structural health, operational health, and synthesis agenda as files, not UI cards"); ┬¦14 gate requires "`workspace rebuild` emits final generated paths: `knowledge/index.md`, `capabilities/index.md`, `knowledge/_views/`." Making it a *tracked* worker-owned projection means a direct editor write to it is a foreign write that must quarantine (┬¦2 foreign-write classification), which the PR also proved.

- **What:** Concept object model is now records-vs-Concepts-vs-projections with **no markdown Work/Author/Source/Institution Concepts** at all ŌĆö Works, Authors, OpenAlex Sources, Institutions, Topics, Publishers, Funders, and graph links are SQLite rows; a Work's only keep-set presence is its digest (`design.md` ┬¦3ŌĆō┬¦4). The legacy markdown Work source fallback was removed from digest, integrity, rollback, projection, and seeded-error paths (PR #1121 `ace45000`), and a `workspace check --assert no-legacy-work-markdown` assertion plus CLI test was added (`d55fb98f`). **Why:** carries forward alpha.13's clean boundary (┬¦4, ADR reference); the design forbids legacy markdown Work lookup in the runtime (┬¦12 "Dropped: Legacy markdown Work lookup or schema support") and ┬¦14 requires "Lookup, digest, references, attention, rollback, and recovery work by `work_id` without a legacy markdown Work path." Checked SQLite catalog rows plus blob content are now the sole source authority (ADR-124).

- **What:** Retired the shipped Obsidian workspace payload from `vault-template`: the entire `.obsidian/**` tree, QuickAdd command scripts/buttons, and the plugin provenance lock, plus the JS helper scripts `capture-note.js`, `open-inbox.js`, `quickadd-utils.js`, `record-exploration-trace.js`, `resolve-inbox-card.js`, `restore-memoria-shell.js` (PR #1124 `5a93b142`). **Why:** Obsidian is dropped as a required surface (`design.md` ┬¦0, ┬¦13 Non-goals); the parity table (┬¦5) re-homes every palette action to a CLI command (`Memoria: capture note`ŌåÆ`memoria note capture`, `open Inbox`ŌåÆ`attention list`, `record exploration trace`ŌåÆ`project trace`, `resolve/dismiss inbox card`ŌåÆ`attention resolve --outcome`, and "restore shell on startup" becomes adapter-only, not core). The product "must remain fully operable if the editor is only `vim`, `nano`, or a file browser" (┬¦1).

### Gate / Provenance / Integrity

- **What:** `workspace scan` now records `observed_external_edit` journal events and reports observed PI edits that "need checking" in `workspace scan --json`; a direct edit to a checked keep-set Concept is marked `unchecked`/non-consumable and re-promoted only through the `mark-checked` capability (not a new top-level command) (#1055 `6a3ab4bc`, #1133 `fa569367`; projections regenerated after quarantine #1088 `a1887e7c`). A change to a generated projection, worker-owned path, `references.bib`, or capability manifest is *quarantined*, never re-consumed, proven by a `direct-write-generated-projection` fixture (#1084 `7d7f1878`). **Why:** `workspace scan` is "the only bridge for direct editor/file-browser edits" and foreign-write classification imports a keep-set edit as a PI edit (preserving prior upstream inputs + prior checked hash for cascade rollback) but quarantines anything worker-owned (`design.md` ┬¦2). ┬¦14 gate: "an edit to a generated projection, worker-owned path, `references.bib`, or capability manifest is quarantined, not consumed." This is the structural read-barrier substitute for per-write approval (`why-review-gate-is-structural.md` lineage).

- **What:** New **live-runner proof gate**: `memoria doctor --check runner --live` performs an opt-in real pydantic-ai dispatch through the same model/network policy guard operations use, and `scripts/verify live` runs the live pytest marker (#1145 `1341786d`; `test_live_runner.py`). **Why:** "The local-runner check is explicit because a deterministic digest fixture does not prove model dispatch" (`design.md` ┬¦11); ┬¦14 gate: "Runtime-local installed gate proves local runner dispatch with an explicit `doctor --check runner` check; deterministic digest fixtures do not count as runner proof." Deterministic fixtures let the whole loop pass without any model ever being called, which would be a false green.

- **What:** New **runtime replay gate**: a focused `test_alpha14_runtime_gate.py` (462 lines) exercises the full alpha.14 command sequence (initŌåÆcaptureŌåÆenrichŌåÆupdateŌåÆinterviewŌåÆdigestŌåÆnote curationŌåÆrecoveryŌåÆrunner/qmd doctorsŌåÆscan quarantineŌåÆscheduled queue/checkŌåÆrebuildŌåÆask/project ask/exportŌåÆjournalŌåÆseeded-error verdict), and `scripts/verify runtime` runs it before the older runtime suites (#1134 `724e6927`). **Why:** "the alpha.14 product loop is complete only when the CLI can run a thin end-to-end cycle" (`design.md` ┬¦5); the runtime-local gate (┬¦11) must prove installed CLI wiring, write boundary, denied direct write, audit rows, recovery, and the seeded-error verdict before any non-sandbox use, using checked-in provider-replay and crash fixtures so it "does not depend on external providers or timing."

- **What:** The seeded-error verdict is wired as a first-class CLI gate (`memoria eval seeded-error-verdict`, #1048 `3ee961cc`; `seeded_errors.py` reworked, and `alpha11-`/`alpha12-seeded-errors.json` fixtures updated). Digest-journal operation aliases were mapped (#1086 `23f3cab0`) and a crash-before-materialization recovery fixture added (#1085 `22e7b27b`). **Why:** the seeded-error harness is "the gate to real use" ŌĆö no structural check proves semantic truth (`design.md` ┬¦9). The provisional bars stand: 100% structural detection recall, 0 false-clean, Ōēż10% false-positive, 100% rollback completeness; and "The seeded-error harness measures the human checkpoint against a no-checkpoint baseline" (┬¦9, ┬¦14) ŌĆö a checkpoint that isn't measurably better than none "is theatre and gets thinned." This is the correctness-via-integrity bet from the literature review (FEVER evidence-grounded NLI, generator/verifier sample-and-rank; `_papers/REVIEW-REFUTATIONS.md`).

- **What:** Journal-list filters added (by request, path, decision, operation, date, check) (#1105 `31afb9d7`); `doctor --repair` restores missing/corrupt packaged capability/skeleton files, plain `doctor` only reports (#1104 `9719fe06`); `init --dry-run` report expanded to cover skeleton/DB/baseline/generated targets/steering/vocabulary/qmd collection/provider config (#1106 `12ca88a4`). **Why:** "The journal is forensic truth; `request show` and `journal show` are user-facing summaries over it" (`design.md` ┬¦9); `doctor` "checks local prerequisites and never repairs without an explicit flag" (┬¦5), and repair "restores missing/corrupt packaged ŌĆ” files from the installed package baseline" (┬¦11).

- **What:** Network allowlist matching was hardened so adapter HTTP bypassing the shared allowlist fails closed (#1103 `7c843f5d`). **Why:** "Every HTTP call passes a shared operation allowlist check before network I/O"; "the operation manifest is the hard ceiling for network access" (`design.md` ┬¦8); ┬¦9 security: "Network access is operation-scoped and allowlisted." ┬¦14 gate: "Adapter HTTP bypassing shared allowlist fails closed."

### Pipeline / Ingest / Enrichment

This is the change-heaviest area. The DOIŌåÆOpenAlex/Crossref graph and full-text acquisition were built out across #1135ŌĆō#1155.

- **What:** Capture became **SQLite-only**: `capture_source`/`capture_bibtex_source`/`capture_url_source`/`capture_pdf_source` now stage into SQLite catalog rows plus `.memoria/blobs/source-content/<work-id>/ŌĆ”` payloads and no longer write legacy Work markdown, derived-entity markdown, or a capture-time `references.bib` (`runtime/capture.py` heavily reworked, #1110ŌĆō#1114). `references.bib` and the capability index render from checked SQLite rows only (#1113 `26552935`). **Why:** capture "Stores raw/content hashes ŌĆ” Does not update `references.bib` ŌĆ” Does not create markdown Work, Author, OpenAlex Source, or Institution records" (`design.md` ┬¦8); `references.bib` is a generated projection from checked catalog rows, not a capture-time or user-maintained file (ADR-124 Consequences).

- **What:** Full-text is now **required before a checked digest**: metadata-only/abstract-only digest attempts fail in the shared runtime path, create a stable full-text attention projection, expose the attention path in the failed-request error, and leave no consumable digest output (#1068 `2e6e4315`, #1069 `59b5ceb1`, #1135 `49bae16e`; `flag digest without full text`). **Why:** decided policy: "a checked digest is compiled only from acquired full work text or a user-supplied equivalent. Abstracts and metadata ŌĆ” cannot produce a checked digest. Full-text acquisition is therefore baseline alpha.14 functionality, not a deferred enrichment" (`design.md` ┬¦8, resolved ┬¦15). ┬¦14 gate: "no checked digest is produced from abstract-only or metadata-only input."

- **What:** Open-Access full-text acquisition was made robust with an ordered candidate retry loop: try Unpaywall `best_oa_location`, then `oa_locations` (#1140 `229249ab`, #1141 `b9703650`), then OpenAlex `open_access.oa_url` / `best_oa_location` / `primary_location` / `locations` (#1144 `bf47fe5f`), then Crossref `message.link[].URL` (#1151 `62011e1c`) ŌĆö each candidate URL/PEP passed through the same allowlist-before-I/O check, deduped, and skipped if denied/failed/extraction-failing/empty until one yields extractable text (#1070 `805bee06`). **Why:** because full text is now a hard digest precondition, a single dead OA URL cannot be allowed to block an otherwise-acquirable Work; the DOI-scholarly branch lists "Unpaywall/OA locator; acquired full text" as *required for checked promotion* (`design.md` ┬¦8 Branches). Paywalled bypass/credentialed scraping stays a non-goal (┬¦13), so acquisition is limited to legal OA locations.

- **What:** First-order citation/reference/topic graph import was expanded far beyond alpha.13. During DOI enrichment Memoria now imports: reference edges from portable BibTeX/CSL metadata into `work_graph_edges` (#1139 `c172a0de`); OpenAlex `topics` + `primary_topic` (#1142 `d32dca03`), `concepts` as `topic` edges (#1149 `dedccea8`), `keywords` as `keyword` edges (#1143 `ef99b11f`), `mesh` and `sustainable_development_goals` as topic edges (#1155 `a618b1ed`), and `authorships` + nested institutions + location Source metadata as graph edges (#1154 `cf9f23ef`); plus Crossref `relation` rows as `related` edges (#1150 `c3538d47`). Discovered candidate Works are marked unchecked attention/worklist rows, never checked (`runtime/enrichment.py`). **Why:** enrichment "Imports the first-order citation/reference/topic graph available from required providers and marks discovered candidate Works as unchecked attention/worklist rows, not checked knowledge" (`design.md` ┬¦8); the graph feeds retrieval query expansion and gap analysis (┬¦10 "First-order citation/reference graph, OpenAlex Topics, controlled facets ŌĆ” part of the read model"). Depth stays first-order by design ŌĆö "Exhaustive or provider-specific graph expansion" is a non-goal (┬¦13).

- **What:** Crossref full-text links are acquired and OA URLs preferred (#1144, #1151); enrichment tries fallback OA lists (#1140/#1141). qmd embedding-enabled rebuilds now persist `mode: hybrid`, `embeddings: true`, and the exact `qmd embed --chunk-strategy auto` command in the checked-search manifest (#1152 `e7e624ef`). **Why:** the DOI branch requires "embeddings" for checked promotion (`design.md` ┬¦8); recording the embedding manifest mode makes the retrieval build reproducible and gate-checkable (┬¦10 rebuild contract).

- **What:** BibTeX/CSL portable imports stage as unchecked Work rows with queued enrichment (#1073 `e4476012`); the Zotero-specific import surface (CLI `--format zotero-export`, worker operation, runtime helper, capability file `capture-zotero-source.md`) was removed (#1123 `7ad88a84`), though a Zotero *export file* still stages as an unchecked Work via generic import (#1077 `e3f10cad`). **Why:** "`work import --format zotero-export` is not preserved as a product alias. Portable bibliography import means generic BibTeX/CSL files only" (`design.md` ┬¦5, ┬¦8); Zotero is dropped as required Work authority (ADR-124: "Zotero and Better BibTeX are optional user tools ŌĆ” Memoria does not require a live Zotero library"). ┬¦14 gate: "`work import --format zotero-export` fails as an unsupported format."

- **What:** `work update` routes through the worker and owns metadata/lifecycle changes (archive/restore, retraction/contested standing, classification, identifier corrections), recomputing views/filters/`references.bib` and resolving the linked attention item in the same request (#1075 `1fd4be0c`). `work interview` records the guided conversation and returns stable Work IDs, and `work digest` consumes interview answers (#1051 `dcf06cbc`, #1080 `116fba4f`, #1109 `a720d0c9`). **Why:** `work update` consolidates "archive/restore, retraction/contested standing, human classification, and corrected identifiers ŌĆ” There are no separate archive, restore, or classify commands" and resolves classification review as "one auditable operation" (`design.md` ┬¦5, ┬¦8). Digest fails if it requires a missing/blocked interview (┬¦8).

### Retrieval / Store

- **What:** qmd is now the required CLI-first checked-only search backend. `memoria ask` uses qmd for checked retrieval (#1091 `e51039f5`), `project gaps` is backed by qmd evidence (#1093 `05b1c76d`), and checked Work graph context is indexed (#1072 `4f0fb1dd`). qmd runs with workspace-local `QMD_CONFIG_DIR`/`INDEX_PATH` under `.memoria/index/qmd/` but the **shared user/global qmd model cache** (stopped creating a per-workspace `XDG_CACHE_HOME`/qmd cache path, #1090 `e09c5c28`, #1092 `7ef0d722`); the qmd executable must resolve to an unambiguous absolute path (#1101 `c992e96b`) and the `memoria-checked` collection is verified in `doctor` (#1102 `0ab30f4c`); rebuild fails fast on missing binary/wrong collection/update/embed failure (#1089 `1ffafe11`). **Why:** "Memoria owns the checked-only input tree, read barrier, qmd collection registration, rebuild command, and result filtering; qmd owns keyword and semantic ranking" (`design.md` ┬¦10). - **What:** Generated qmd graph-neighborhood documents now include raw OpenAlex topic rollups and authorship/institution/Source metadata, so retrieval can match on graph context (#1138 `613487d5`, #1154 `cf9f23ef`; `search_index.py`). - **What:** Retrieval hides superseded/retracted content by default via a deterministic `standing`/claim-trace filter (enabled by the enforced `standing` frontmatter above). **Why:** "Retrieval hides superseded and retracted content by default; historical lookup is explicit. ### Discovery / Gap Analysis

- **What:** `memoria project gaps <project_path>` became **project-scoped**: it feeds checked project argument/Toulmin health into the shared `analyze-gaps` runtime path, while project-less `operation run analyze-gaps` remains the workspace-wide maintenance scan (#1136 `4b1a4dc9`; `knowledge.py`, `worker.py`, `analyze-gaps.md`). **Why:** "With a thesis, gap analysis also reads argument health from the project's Toulmin graph: a claim that is unsupported, uncountered, or fragile ŌĆ” is a gap even where work/note coverage looks dense" (`design.md` ┬¦10); the project half is "the directed/writing half of the loop, not just coverage" (┬¦4).

- **What:** Gap analysis now counts checked SQLite catalog source terms ŌĆö from `csl_json.memoria.tags`, `topics`, `research_area` ŌĆö deduped by catalog source ID and ignoring unchecked/archived rows (#1137 `d1af323a`), counts checked topic/keyword graph-edge terms as source evidence (#1138 `613487d5`, #1143 `ef99b11f`, #1149, #1155), and project gap analysis seeds checked project scope/facet terms + counts checked linked-thesis terms with no manual `--seed-term` input (#1148 `95e16dc3`). **Why:** gap analysis "reads both work coverage and knowledge graph coverage ŌĆ” topic/facet coverage" (`design.md` ┬¦10 Gap analysis reads); counting SQLite/graph terms replaces the removed legacy-Work-markdown scan and makes coverage measurable from the new authority (ADR-124).

- **What:** Missing-full-text is now flagged by gap scans: catalog rows that cannot produce checked digests/retrieval because full text is missing get PI-reviewable full-text attention flags (#1153 `0cb13abe`). SQLite-detected source/topic gaps carry checked source IDs into discovery candidate generation, so candidate Works are proposed from first-order reference/related graph edges without qmd rediscovery (#1147 `8283d07f`), and Crossref `related`/OpenAlex graph edges feed discovery (#1150). **Why:** "Full-text coverage" and "Discovery candidates not yet captured, enriched, or digested" are explicit gap dimensions (`design.md` ┬¦10); "Automated discovery is baseline but bounded ŌĆ” Discovery writes unchecked attention/worklist rows. It never silently imports a discovered Work as checked knowledge and never bypasses the provider allowlist" (┬¦10). Routing graph edges straight to candidates avoids redundant qmd search for edges the graph already knows.

### Security

- **What:** Network access hardened to fail-closed (see Gate/Provenance #1103); capability import remains supply-chain-quarantined; retracted/contested Works are blocked from checked promotion (retraction subsystem ported #1117); DB source rollback quarantine markers added and source-metadata/prompt-injection checks moved to DB source rows (#1121). **Why:** security controls (`design.md` ┬¦9): "External text is untrusted data. Data/instructions are separated. Raw work text is never system prompt material. Network access is operation-scoped and allowlisted. Generation and checking are separate operation roles. A retracted or contested work is never silently checked. Capability import is supply chain and starts quarantined." Injection-as-untrusted-code is the least-privilege-sandbox bet confirmed by Greshake/AgentDojo (`_papers/REVIEW-REFUTATIONS.md`, ┬¦Provenance).

- **What:** Capability supply-chain metadata is kept deliberately small (origin URL, SPDX license, content hashes, signed/unsigned, quarantine state); in-toto/SLSA attestations and RO-Crate/BagIt wrappers stay out. **Why:** "Add in-toto/SLSA only when third-party capability import becomes real" and "Add RO-Crate or BagIt only when there is an actual archive/deposit consumer" ŌĆö simplest design that meets the stated requirement (`design.md` ┬¦13 Deferred-boundary posture; ┬¦14 gate).

### ADRs

- **What:** New **ADR-124 "Standalone catalog is the citation authority"** (`docs/adr/124-standalone-catalog-citation-authority.md`), `supersedes: [5]`, `assumes: [55]`. Decision: the Memoria SQLite catalog + source-content blobs drive `references.bib`, digest eligibility, retrieval, and metadata checks; Zotero/Better BibTeX become optional tools; citekeys are aliases while stable Work IDs/provider identifiers are the primary runtime keys; full text remains required for a checked digest. **Why (from ADR Context):** "Alpha.14 makes Memoria a standalone CLI/engine. A required Zotero library or live Better BibTeX auto-export would violate that." Alternatives rejected in the ADR: keeping Zotero as required authority ("makes the standalone CLI depend on a desktop reference manager"), a frontmatter-only catalog ("alpha.14 needs durable worker state, provider payload provenance, full-text gates, and generated projections better represented in SQLite"), and live API-on-demand metadata ("repeated provider calls produce drift and cannot be the local source of truth").

- **What:** ADR-05 (Zotero as bibliographic backbone), ADR-06 (citekey naming), ADR-22 (build on Hermes runtime), ADR-121 (enqueue-only Obsidian control panel), and ADR-118 (dashboard consolidation) were updated/annotated as superseded or historical; ADR-30 (deterministic ingest), ADR-10 (claim supersession), ADR-62 (measurement harnesses), ADR-64 (native Windows), ADR-80 (ephemeral test env), ADR-122 (SQLite boundary), and ADR-123 (DOI catalog enrichment gate) were reconciled with the standalone cut; `docs/adr/README.md` index updated. **Why:** slice 1 of the release plan is "Decision authority ADRs (alpha.14 as the cut; supersede Hermes/lane/profile/MCP/`direct_api`/Obsidian/OpenAlex-style graph markdown ADRs). New ADRs start at 124" (`release-plan.md` Implementation Slices ┬¦1).
