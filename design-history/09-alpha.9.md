## alpha.9 — The truth-alignment checkpoint: enforce by mechanism, capture with less friction

**Theme:** stop crediting boundaries to labels, and stop describing behavior the code doesn't have. alpha.9 is the "UI/workflow and runtime-gate" checkpoint (`scratchpad/README.md`, `docs/releasing/0.1.0-alpha.9`). Three forces drove it: a **Hermes utilization + gate audit** that found the runtime capability wall was schema-hiding (a curtain, not a wall) with the policy plugin as the only real boundary — and that boundary was incomplete; an **ADR↔code audit** of all 75 accepted ADRs that found 20% not faithfully built and six ADRs crediting enforcement to a description rather than a mechanism; and a **401-paper literature review** whose adversarial refutation + on-box measurement passes forced a correction of the system's own framing (qwen is a *test fixture*, not the production engine). The shipped code is comparatively small (gate `process` fix + a contract doctor, a refined Obsidian capture flow, a runtime-package refactor, telemetry planes); the release's weight is in **verification, ADR corrections, and a new working principle** that changes how every future boundary claim is judged.

---

### Vault / concepts, note types & schemas

- **What:** The **`claim.md` template** gains an action-oriented `[!suggestions]` callout + QuickAdd buttons (Link claim / Capture source / Delegate task / Archive claim) — a ~30-line addition (`src/system/templates/claim.md`, #819). The template itself already existed at alpha.7/alpha.8 with `schema_version: 2`, `links.supports/contradicts`, and the `superseded_by` pointer, so those fields are **pre-existing** (now populated by the new `write-claim.js` path), not introduced here. **Why:** the capture workflow refinement moved claim creation into a template-driven QuickAdd path (`write-claim.js`) so a PI can produce a schema-valid claim and immediately reach its next actions without hand-editing YAML — reducing the friction that pushes work into ad-hoc notes. The `superseded_by` field is the deterministic, human-set pointer the ADR-10 supersession filter needs (see Retrieval below) (`design-update-recommendations.md` §3.4).
- **What:** **`fleeting.md` template reworked** from a prose-prompt stub to a `title`/`body` capture with a triage `[!suggestions]` callout and four action buttons (Promote to claim / Capture source / Delegate task / Archive done); field renamed `thought`→`title` (`src/system/templates/fleeting.md`, #812/#819). **Why:** fleeting capture is meant to be frictionless and *queued for later triage* — "capture should not steal focus" (`capture-fleeting.js` header). The buttons turn a raw capture into a one-click path to its disposition, matching the JTBD action-first inbox design (ADR-70).
- **What:** **Exploration-trace note kind** — a QuickAdd flow records a rejected direction / dead end beside a Librarian map report as an ordinary fleeting note under `notes/fleeting/maps/`, never auto-promoted (`src/system/scripts/record-exploration-trace.js`, #780; ADR-100). **Why:** to make dead ends "visible, searchable, and easy to archive" as project-local context rather than canonical knowledge — negative-results capture the recommendations doc later formalized as a proposal (§4.6). The ADR audit confirms it shipped exactly as decided (ADR-100 IMPLEMENTED).
- **What:** **Zotero Tier-0 catalog stub** materialized — a QuickAdd flow reads the current Zotero selection over the Better BibTeX JSON-RPC and writes a catalog stub + an `intake:source` Librarian card (`src/system/scripts/capture-from-zotero.js`, #782). **Why:** completes the ADR-05/06 citekey-backbone capture path; a code comment records the load-bearing quirk (Zotero's local server rejects any request carrying an `Origin` header, so `curl` is used instead of Obsidian's `requestUrl`) — an external-system idiosyncrasy the AGENTS.md comment rule now requires be documented inline.

### Profiles / runtime (Hermes)

- **What:** **Runtime package spine refactor** — the stdlib-only policy/runtime core moved from `src/.memoria/memoria_runtime/policy/` to a repo-root `memoria/` package (`memoria/runtime/{vaultio,jsonl,time,paths,policy,diagnostics}`), compat paths removed, and shared helpers (`load_json`, retry loops) extracted (#763, #764, #788; the deleted `src/.memoria/memoria_runtime/` tree). **Why:** ADR-76 step 1-3 (the reconciling-installer prerequisite) landed ahead of its own prose; `test_package_spine.py` now pins the old path's *absence*. The ADR audit flags ADR-76's Decision-2 path citation as stale-prose to repoint (`adr-implementation-gap-audit.md`, ADR-76 PARTIAL).
- **What:** **"Build on the installed version; upgrade by verifying, not by reading"** added as a platform-runtime rule in AGENTS.md, with evidence tags (`on-box ✓` / `claim-only` / `untested`) and the ADR-106 upgrade→verify→re-enable order (`AGENTS.md`, #820). **Why:** the audit repeatedly caught the system reasoning about Hermes features from release notes rather than the installed v0.14.0 (`hermes-014-utilization-audit.md`). The rule "blocks shipping on marketing; it does not block upgrading." This is the general form of the **qwen correction**: qwen2.5:7b was repeatedly mis-framed as the production model when it is only the local *test fixture* — the live engines call an LLM API, making on-box numbers pessimistic floors, ingest a per-document API bill, and vault text leave the box on every call (`design-update-recommendations.md` §0c; #818).
- **What:** **Utilization audit disposition of unused/mis-relied Hermes features** (design decisions, mostly deferred): Bitwarden secrets vs the 5×`.env` fan-out (fleet/sandbox win, not a lone-laptop slam-dunk — U1); `auxiliary.*` model routing to Haiku/Flash for cheap side-jobs (production cost lever, moot on the local-qwen box — U2/A2); `reasoning_effort` per lane (recommended in 4 profile comments, set in none — "set it or drop the comment", U3); `session_search` for **Co-PI only** as a deliberate privacy trade (A4); confirmations that `memory` Co-PI-only, native-kanban-bypass, and external-cron are *correct as-is* (A5-A7) (`hermes-014-utilization-audit.md`). **Why:** to separate genuine gaps from deliberate divergences and stop the audit from reading "unused" as "wrong." Per-card `model_override` was flagged as plumbed-but-unwired in 0.14 — "not a feature to just turn on."
- **What:** **`checkpoints.enabled: true` recorded as inert** in the MCP-only model (filed #824 to drop it + the false "reversibility" comment from all 5 profiles). **Why:** Hermes checkpoints trigger only on native `write_file`/`patch`/`terminal`, none of which any Memoria profile ships; real reversibility comes from the `policy_hook` before/after-hash audit pair, not shadow-git (`hermes-014-utilization-audit.md` B5). *(Filed, not landed this release.)*

### Gate / provenance / trust / audit

- **What:** **`process` added to `DENY_DIRECT_TOOLS`** in the policy gate (`src/.memoria/mcp/policy_hook.py`, #825). **Why:** `process` is a real tool in Hermes's `terminal` toolset (`toolsets.py:142-145`) that the denylist missed, so the gate returned `{}` (allow) for it — the exact config-drift the denylist's own "fail closed" comment exists to prevent. The audit ranked this the highest-leverage real gap because ADRs 03/21/46 all ride this one mechanism (`adr-implementation-gap-audit.md` cross-cutting #1).
- **What:** **New `hermes_contract_doctor.py`** — reads the toolset registry from the *installed* Hermes and exits non-zero when any `file`/`terminal`/`code_execution` tool is not in `DENY_DIRECT_TOOLS` (importing the denylist as single source of truth, not re-listing it); also reports (as a non-failing warning) the ~36 egress/side-effect tools the gate allows by default (`src/.memoria/mcp/hermes_contract_doctor.py`, #825; wired into runbook S4). **Why:** to make denylist drift *catchable instead of silent* — "the executable form of AGENTS.md 'build on the installed version, upgrade by verifying'" and the generalization of the ADR-106 cost doctor. A completeness sweep test (`test_policy_gate_completeness.py`) proves the list is enforced, not merely declared, with a `process` regression guard.
- **What:** **Gate audit-completion failures now emit a diagnostics event** (`record_event(component="mcp.policy_hook", code="audit_completion_failed")`) in addition to stderr (`policy_hook.py` `evaluate_post`). **Why:** the audit pair (before/after hash) is the real reversibility/provenance record (ADR-104); a silent audit-completion failure previously left only a stderr line — routing it to the content-light diagnostic plane makes it operator-visible without breaking the agent loop.
- **What:** **Plugin fail-closed contract made explicit** — the gate plugin's broad `except` now carries `# noqa: BLE001 -- policy gate fails closed; any error blocks the write` and the post-hook `# noqa: BLE001 -- never block the agent on audit-completion failures` (`plugins/memoria-policy-gate/__init__.py`, #804/#825). **Why:** the two exception paths have *opposite* correct behaviors (pre-hook must block on error, post-hook must not) and the new AGENTS.md rule requires every `# noqa` to state its rationale on-line.
- **What:** **New working principle "Enforcement is a mechanism, not a label"** in AGENTS.md, with named done-right exemplars (ADR-55/74/80/105) and done-wrong-corrected list (ADR-28/23/60/04/46/41) (`AGENTS.md`, #820). **Why:** the through-line of the entire audit — a guarantee kept being credited to a *description* (a config key, a taxonomy, "the architecture") when a *mechanism* elsewhere actually held it. The canonical example is `disabled_toolsets`: it only hides a tool from the model; `registry.dispatch` still runs any registered tool by name, so the policy plugin's hard-deny is the boundary (`hermes-014-utilization-audit.md` B1; `adr-enforcement-audit.md`).
- **What:** **Enforcement-attribution corrections applied in-place to six ADRs** — 28 (`disabled_toolsets` is UX; plugin hard-deny + default-deny is enforcement), 46 ("only through MCP" bound by `DENY_DIRECT_TOOLS`+default-deny, not the layer contract), 04 (lifecycle write rules enforced by lane globs, not folder naming), 23 (substrate split is a routing table; isolation = per-profile dirs + denylist + card channel), 41 (card column-advance is process discipline; the code stop is at the write in `decision.py`), 60 (the named cross-vault enforcer is *not yet implemented* — a deferred design, not a live guarantee) (`docs/adr/{04,23,28,41,46,60}-*.md`, #820). **Why:** each credited a boundary to a non-mechanism; verified on-box against installed v0.14.0 (`registry.dispatch` has no enablement check; `vault_root()` returns a single root). ADR-60's correction explicitly warns it "must not be read as a live guarantee" and names what a real cross-vault boundary would require (foreign-vault scope + a deny-write test).

### Retrieval / derived store

- **What:** **Pre-file similarity shadow telemetry** — QuickAdd source/claim creation now runs a report-only `qmd` neighbour check (top-3 within `notes/claims/`, `notes/sources/`), writes a `pre-file-similarity.jsonl` row, and renders a `[!similarity]` callout; **never blocks, auto-merges, or applies a calibrated threshold** (`src/system/scripts/quickadd-similarity.js`, #774; ADR-38 note). **Why:** alpha.8 shipped the shadow primitive as a passive filing-side signal; calibration/enforcement stays deferred to #562 — the ADR-38 prose was updated to say "shadow primitive, not enforcement" so it isn't read as a live gate.
- **What:** **ADR-10 supersession gap recorded as a live correctness bug** (filed #826) — the "exclude superseded from `query`/`write` by default" half is enforced only in `link-claim.js` ranking + the FAMA linter, with **no retrieval-layer filter** in the find/draft skills (`docs/adr/10`, `adr-implementation-gap-audit.md` ADR-10 PARTIAL). **Why:** a query can surface, and a draft can cite, an already-retired claim — the exact FAMA failure ADR-10 exists to prevent, live in alpha.9's own domain. The fix is deterministic and cheap (honor the human-set `superseded_by` pointer), so it belongs in the alpha.9 recommendations cut, not the deferred contradiction-precision engine.
- **What:** **Retrieval recommendations re-anchored to `qmd`** (the existing local hybrid BM25 + vector + cross-encoder-rerank index) instead of an abstract "BM25+dense" (`design-update-recommendations.md` §0c/§4.1; #818). **Why:** a grounding error symmetric to the qwen one — the recs proposed building a retriever that already ships. The substantive shift: since qmd *already* cross-encoder-reranks, the near-term action is to **evaluate that reranker on reasoning-relevance** (Wei 2026: MS-MARCO cross-encoders hurt there), not "add a reranker"; and what is retired is **cosine's role as *arbiter*** (the pre-file gate), never cosine as retriever — measured 0/8 on negation traps where NLI scored 8/8 (§0a).
- **What:** **Contradiction detection re-tiered down to a cheap soft-flag for alpha.9**; the NLI + decomposed-variable-match gate + MaxSAT apparatus **deferred** (`design-update-recommendations.md` §0e/§3.1). **Why:** a new N=1 error-cost model — a missed/late contradiction is *self-correcting and low-cost* for a single PI, so a precision engine is over-built and its propose-only queue (~28% precision, Mitchell 2022) would dominate the scarce supervision budget. On-box measurement showed NLI *confidently mints false contradictions* (0.94-1.00) on same-topic/different-variable pairs, unrecoverable by a confidence threshold; the decomposed gate helped (recall 4/6, FP 6/7→1/7) but still leaks negation/argument-swap. So the engine is gated on in-domain proof that the miss costs something (#813, #815, #816).

### Security

- **What:** **Egress allow-by-default recorded as a live P0** (filed as interim hard-deny #832 + structural default-deny #822) — the gate hard-denies file/terminal/code-exec but allow-by-defaults `web_*`/`browser_*`/`send_message`/`delegate_task` (returns `{}`), so for egress the sandbox rests entirely on `disabled_toolsets` schema-hiding (`hermes-014-utilization-audit.md` B2; `hermes-clean-slate-design.md` §2a). **Why:** on the injection path this is unmitigated — the threat model is now stated: a **poisoned source/paper** whose text reaches the LLM on every ingest call (data > code, Greshake/Debenedetti) and induces a tool call; concrete because ingest processes untrusted papers. The critique round forced two admissions onto the page: **merged ≠ deployed** (the deployed `~/Memoria-test` gate is a pre-#825 build with no `process`, so even the landed fix isn't running — #832 includes a redeploy + deploy-freshness check), and the second independent **fail-open** (the gate fails open if the plugin fails to register/import — filed #823). *(Findings + interim fix filed this checkpoint; the structural default-deny is the follow-through.)*
- **What:** **MCP-only sandbox reframed "necessary, not sufficient"** (`design-update-recommendations.md` §2/§0b.5). **Why:** least-privilege doesn't defend against poisoned *data*, which is the dominant threat — an architectural finding, not benchmarkable, that reframes the whole safety cluster (ADR-27/32/46/28) as a *write* boundary that says nothing about egress under injection.

### Removed / retired (and why)

- **Retired: cosine similarity as the contradiction/supersession *arbiter*** (kept as `qmd`'s retriever). **Why:** measured negation-blind (0/8) — a similarity score cannot tell "same" from "opposite" (`design-update-recommendations.md` §7, `[provisional, n≈6]` + BRIGHT corroboration).
- **Dropped: the QLoRA fine-tuning budget.** **Why:** the live engine is an API (not fine-tuneable) and the frozen test floor already clears schema compliance (6/6) — `[architecture]`, doubly moot (#814, #818).
- **Deferred/killed for alpha.9: the high-precision contradiction engine** (NLI + decomposed gate + MaxSAT), **blanket self-consistency**, the **greenfield retriever/clustering stack**, and **auto-applied contradiction/supersession links**. **Why:** the N=1 error-cost model (missed contradiction is low-cost/self-correcting), the API-per-token-bill fact, the already-built `qmd`/`cluster_mcp`, and the propose-only-is-the-most-expensive-default budget arithmetic, respectively (`design-update-recommendations.md` §1 "Explicitly NOT doing", §7).
- **Removed: `memoria_runtime/` compat tree, `_parse_ts`/`_safe_name` aliases, hand-rolled `is_relative_to`.** **Why:** ponytail-audit clean-wins after the `memoria/` package spine landed — no behavior change (#788, #800, #804).
- **Retired framing: qwen-as-production-engine, "cheap local model" / "local-by-choice" as production bets, "keep the bits local."** **Why:** the qwen-is-test-fixture correction — these described a system that doesn't exist; the live engine is an API, so vault text leaves the box on every call (#818). *(Whether high-volume ingest should move to a local model is left open as §0d/§8.1, not decided.)*
