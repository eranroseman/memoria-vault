## alpha.11 — The clean-slate reset: one knowledge graph, trusted-writer + checked read-barrier, capability Concepts

**Theme.** alpha.11 is the single largest redesign in Memoria's history — a full re-derivation "from requirements, intellectual lineage, and best practice, *not* from the current implementation" (`_notes/0.1.0-alpha.11-design.md` §Method). It tears out the profile/fleet runtime, the lifecycle+maturity+review-gate write model, the durable-Inbox-card workflow, and the 27-type schema, and replaces them with: a three-bundle OKF workspace (`catalog/` + `knowledge/` + `capabilities/`), a **four-type knowledge model** (`source`→`digest`→`note`+`hub`), a **trusted-writer + staging/promotion + `check_status` read-barrier** integrity spine, **operations as policy Concepts** (with a generated `ai-catalog.json`), **cascade rollback** over a PROV-DM journal, and a **direct-API/Hermes LLM runner split**. What drove it: the prior milestone chain hit a wall where "the only outcome is proceed to the next" (§15), and the two theses — **direction not authorship** and **traceability not approval** (§2) — made the old per-write human gate both wrong and unfalsifiable. The reset is deliberately shipped onto a disposable test Bundle, gated on a seeded-error verdict (§14/§16).

Note on method: alpha.11 landed as one squashed implementation commit (`1b24453b`, PR #1027) preceded by a large docs/rehome runway (PRs #930–#1026). The design doc `_notes/0.1.0-alpha.11-design.md` (identical copy in `scratch/releases/0.1.0-alpha.11/`) is the authoritative statement of intent; ADRs are corroboration.

### Vault / concepts / storage

**What:** The shipped vault moved from `src/` to `vault-template/`, and the installable Python package moved to `src/memoria_vault/`, with offline cassettes under `tests/fixtures/` (commit `a966cb54`, "Rehome vault template and Python package"). — **Why:** to stop conflating "the shipped vault source" with "the Python package source" under one `src/` root; the rehome commit body states it moved "the installable package to `src/memoria_vault`, … the shipped vault source to `vault-template`" and updated installers/CI/pre-commit/docs accordingly. Clean separation was a precondition for the clean-slate skeleton to be validated independently.

**What:** Top-level workspace layout changed from six flat categories `[catalog, notes, projects, inbox, spaces, system]` to **three OKF-compatible bundle roots + workspace-level state**: `catalog/`, `knowledge/`, `capabilities/`, plus workspace-owned `journal/`, `.memoria/`, `steering.md`, `index.md`, `references.bib` (`vault-template/.memoria/schemas/folders.yaml` `categories`/`bundle_roots`; old `src/.memoria/schemas/folders.yaml`). — **Why:** §6.1/§6.2 — "A Memoria install is a **workspace** … containing three OKF-compatible bundle roots"; the parent workspace is deliberately **not** an OKF bundle so cross-bundle state (`steering.md`, journal, worker/index) "do not have to pretend to be Concepts." The rejected alternative was keeping flat topic/lifecycle folders (old ADR-04, now retired); OKF was adopted because it "independently specifies the §6 substrate … a fourth re-derivation of the storage layer" (Provenance).

**What:** The catalog was restructured from per-kind folders (`catalog/papers`, `people`, `organizations`, `venues`, `datasets`, `repositories`) to **`catalog/sources/<source_id>/{source.md, content.md, raw/}` + `catalog/entities/<id>.md`** (`folders.yaml` `homes`; `vault-template/catalog/`). — **Why:** §5 "The catalog stays separate — objective, deterministic," and §6.1 "the Concept is the *doc*; some carry referenced **attachments** — a source = `source.md` (the Concept) + `content.md` + `raw/` (attachments, *not* Concepts)." Datasets/repositories collapse into `source.item_type` rather than separate folders ("No 'Work' Concept — version grouping is a `version_of` Link"). person/org/venue merge into one `entities/` home because they are one objective-identity family.

**What:** Retired the `inbox/` category and all inbox-resident durable Concepts; replaced with **generated attention projections** backed by `.memoria/queue/{pending,running,done,failed}`, `.memoria/staging/{catalog,knowledge,capabilities}`, and `.memoria/quarantine/` (new dirs in `folders.yaml` skeleton; old skeleton had `inbox`, `spaces`, `system/board`, `system/worklists`). — **Why:** §8.2 "**Attention is generated, not authored** … markdown, Bases, Canvas, or plugin 'cards' are projections and never canonical Concepts"; §17 "Old Inbox card types do not remain durable Concepts." The alternative (durable Inbox cards as the canonical workflow model) is explicitly shed in §17 because workflow state belongs in "queue (live) + journal (durable) + projections (visibility)," never in card Concepts.

**What:** Introduced a **workspace-level append-only `journal/<machine>.jsonl`** as the durable record, with content-hashed **PROV-DM** events (`derived`/`run`/`model_call`/`check-fired`/`resolved`) — `vault-template/journal/.gitkeep`, schema §7. — **Why:** §6 "Record (authored, not regenerable) … + the **journal** (append-only, content-hashed, per-machine; **W3C PROV-DM**)"; §4.1 makes the journal the tamper-evident derivation graph that substitutes for authorship. "Partition, don't chain" (per-machine journal) is chosen over a chained/DB log to keep multi-device convergence at the git text layer (§6, §4.1).

**What:** Formalized the **record vs projection** split: tracked generated projections are `index.md` / `ai-catalog.json` / `references.bib`; disposable projections (`.memoria/index/qmd`, embeddings, DAG, open-flags, queue) are gitignored (`folders.yaml` skeleton; `vault-template/references.bib`, `catalog/index.md`, `knowledge/index.md`, `capabilities/index.md`). — **Why:** §6 "a small **record**; everything else regenerates." A dedicated projection generator + drift check was mandated ("a hand-edited projection does not satisfy acceptance," §6.1), which is why `regenerate-tracked-projections` (with `required_checks: [projection-drift]`) ships as an operation.

**What:** `references.bib` moved to the workspace root (from `src/.memoria/memoria.bib`), and `catalog.base` moved under `catalog/` (`vault-template/references.bib`, `vault-template/catalog/catalog.base`). — **Why:** §13 "**`references.bib` / CSL-JSON are generated tracked projections** … Memoria does not hand-maintain a bibliography," generated from catalog sources by `regenerate-references-bib`.

### Knowledge model — the four-type collapse

**What:** The type vocabulary collapsed from **25 types to 12** (19 removed + 6 kept → 6 added + 6 kept; count from the type-schema glob). Removed: `alert, candidate, claim, code-note, dataset, eval-task, flag, fleeting, gap, maintenance, paper, pattern, queue, space, thesis, work-prompt, worker-card, worklist-item`. Added: `digest, note, operation, mcp, skill, workflow`. Kept (reshaped): `hub, organization, person, project, source, venue` (`vault-template/.memoria/schemas/types/*.yaml` vs `src/.memoria/schemas/types/*.yaml`). — **Why:** §5/§7 define exactly **three Concept families** (catalog / knowledge-graph / capabilities) and the knowledge graph as **three node-kinds by authoring role**. §Vocabulary note: "`annotation`/`claim`/`fleeting`/`source-digest`/ZK-`index` are not separate types." The many operational types (alert/flag/gap/queue/worker-card/worklist-item) were retired because "Queue, flags, gaps, work cards, dashboards, and indexes are projections or worker state, not durable Concepts" (§7.1 table).

**What:** `paper` + `source` merged into one **`source`** type at `catalog/sources/<source_id>/source.md`. The old `paper` (`citekey`, `doi`, `authors`, `year`, `venue`, `ingest_status{tier0|enriched|complete|needs-human}`) and old `source` (an entity-pointer note with `evidence_level` CEBM grades, `entity` wikilink) became: `source_id` (immutable path), `item_type{article|book|webpage|dataset|repository|software|audio|video|note|report}`, `identifiers{}` map, `csl_json`, `raw_text_sha256`/`normalized_text_sha256`, `metadata_status{verified|partial|unverified|not-indexed}`, `citekey` (demoted to optional alias) (`types/source.yaml`). — **Why:** §6.1 "the catalog path is an **immutable `source_id`**, not the citekey — citekeys are metadata-derived and get corrected, so they are a **generated display/export alias**, never the path (… fixture-verified across a citekey correction)." CEBM `evidence_level` and the entity-pointer indirection were dropped because catalog is now "identity + snapshot + metadata + graph, nothing more" (§13); creators/venue "are **not** standalone fields — they live in `csl_json` + the `authored_by`/`published_in` Links" (§7.1).

**What:** `claim` + `fleeting` + note-`source` merged into one **`note`** type at `knowledge/notes/<id>.md` — an atomic Concept with an *optional* **anchor** (`quote`, `text_sha256`, `source_id`, `annotation_ref`) and an *optional* **claim block** (`claim_text`, `qualifier`, `tense`, `temporal_scope_*`, `status{candidate|accepted|rejected|superseded|needs_review}`, `evidence_set`, `extraction_confidence`) (`types/note.yaml`). The old `claim` type's `maturity{seedling|budding|evergreen}` and mandatory `sources: list` provenance guardrail are gone; the old `fleeting` type's `origin{human|agent|chat}` is gone. — **Why:** §5 "**one note type** … optional anchor = 'annotation', optional assertion = 'claim'." **`no maturity`** because "trust = the checks" (§5) — maturity was a human-set gate proxy, now replaced by `check_status`. A `#todo` tag replaces the `fleeting` type ("A reminder is a `#todo` tag; **no `maturity`**"). `annotation_ref` is a **W3C Web-Annotation selector subset** so spans "resolve regardless of parser" (§7.1). `extraction_confidence` is "internal telemetry — **never surfaced**" (§7.1) — a direct response to Jacobs 2021 (don't show an agent verdict).

**What:** New **`digest`** type (`knowledge/digests/<source_id>.md`) — machine-authored per-source synthesis, 1:1 with a catalog source, carrying `massw{context,key_idea,method,outcome,projected_impact}`, `confidence`, `contested`, `contradictions[]` (`types/digest.yaml`). — **Why:** §5 "`digest` (per-source, **machine**) — 'what this source says,' in corpus context, compiled from the capture conversation; the reading unit, and what hubs draw from." This is the net-new Karpathy "wiki" role — "the bookkeeping humans abandon" (§5), "born `unchecked` + read-barrier'd because they're machine-made over untrusted sources, not because they're disposable" (§6). `massw` fields "feed gap analysis" (§7.1/§9).

**What:** `hub` reshaped from a review-gated (`gated: true`) topic node keyed on a single `topic:` string to a `check_status`-governed **machine-synthesized + human-curated** node with `members[]`, optional `project`, `contested`, `confidence` (`types/hub.yaml`). — **Why:** §5 "the merged Karpathy concept-page × ZK MOC: the machine keeps the topic synthesis current, the human owns the structure + argument." The gated-zone model is retired in favor of ownership+suggestion ("never rewriting the curation"), and "a project's hub *is* its draft outline and the **entry point** (no separate ZK *index* type)" — which is why the old `notes/indexes` folder and any index type are gone.

**What:** `project` reshaped: the old model had separate `project`/`thesis`/`code-note` types under `projects/`; now a single **`project`** type at `knowledge/projects/<id>/project.md` with an *optional* `thesis` field and a `scope` map (`types/project.yaml`; `folders.yaml`). — **Why:** §3 "one project type; the thesis is an optional field that switches on the argument-health gap lens (§9)"; §5 "**project = Link-defined subgraph, not a bundle** (`in_project` membership)." `code-note` and coding/autonomy artifacts are explicitly out of scope (§7.1 table, §15). This is corroborated by ADR-119 (schema-driven document creation) which made `scope_topics` creation-optional and the thesis field conditional on `output_mode` (commits `8e63ff1a`, `87a363e8`).

**What:** Typed Links standardized to the predicate set `{supports|challenges|extends|supersedes|in_project}` + Citations; `version_of` restricted to catalog source↔source only (§7.1 "Typed Links"; old ADR-08 "typed-relations-frontmatter" retired). — **Why:** §5 "**Notes cite sources … not machine-synthesis nodes**, so the machine can recompound freely without breaking citations"; §7.1 "**`version_of` is a *catalog* source↔source Link, not a ZK predicate**." High-risk directional Links (contradiction, supersession) stay human-set ("automatic = candidates").

### Gate / provenance / integrity spine

**What:** The write model changed from **per-write human approval into review-gated folders** (old `folders.yaml` `gated_prefixes: [notes/claims/, notes/hubs/]`, enforced by the policy MCP, old ADR-03) to a **trusted-writer + staging → promotion + `check_status` read-barrier**. `gated_prefixes` is removed entirely; every type now carries `check_status: [unchecked, checked, quarantined]` with `initial_check_status: unchecked` (`types/*.yaml`; `folders.yaml` `machine_staging_roots`, `quarantine_root`). — **Why:** §2 Thesis 2 "**traceability, not approval**" — approving every machine write "rubber-stamps under load and checks the wrong object (integrity is **structural**)"; and per-atom approval is "empirically dubious … a confident wrong output drags the reviewer *below* baseline (Jacobs 2021)." §4.4 makes consumption (not just writing) the barrier: machine write-requests land in `.memoria/staging/`, the worker runs on-write checks, and "the trusted writer **promotes** a file into the target bundle only with `check_status: checked`." This retires ADR-03's structural-review-gate as the operative model (though the ADR file survives with `nav_exclude`).

**What:** `lifecycle` (`proposed|provisional|current|retracted|archived`) as the universal state field was removed from knowledge Concepts and replaced by `check_status`; `lifecycle` survives only on `source` (`current|retracted|archived`) as a factual bibliographic state (`types/source.yaml` vs old `types/*.yaml` all carrying `initial_lifecycle`). — **Why:** §4.1 "The authoritative field is frontmatter **`check_status`**. Allowed values: `unchecked` … `checked` … `quarantined`. **Only the trusted worker/check runner transitions this field**." lifecycle-as-workflow-state conflated "is this true/current" with "has this been checked"; the reset separates factual source lifecycle (retraction) from consumption eligibility.

**What:** Codified **two write paths** as operation Concepts: the **agent path** (`capture-source`, `compile-source-digest`, `propose-note-candidates` → staging → promote, `actor: operation`) and the **PI path** (`observe-pi-edits` file-watch backfill + `mark-checked`, `actor: pi`) (`vault-template/capabilities/operations/{observe-pi-edits,mark-checked}.md`; §4.1). — **Why:** §4.1 "There are **two write paths** … editing machine-derived content *keeps* the upstream so cascade rollback still reaches it." `observe-pi-edits` "Backfill journal events for PI edits detected in git status" and `mark-checked` "Record a worker-owned `check_status: checked` transition for an observed PI edit" — the PI is trusted and edits Obsidian directly, but the worker still owns machine-consumer eligibility.

**What:** Added **cascade rollback** as a first-class operation with `rollback` unit = output Concept, quarantining machine descendants and flagging PI-directed ones (`operations/cascade-rollback.md`, `risk_class: high`, `required_checks: [trace-integrity]`, writes `.memoria/quarantine/`). — **Why:** §4.2 "Invalidate the offending **output Concept** … and everything **transitively derived from it** … machine-derived downstream auto-reverts; **human-directed downstream halts auto-revert but still propagates a flag** … **The commit is the atomic *write* unit; the *rollback* unit is the output Concept.**"

**What:** Added the **trace-integrity scan** (`operations/trace-integrity-scan.md`) that quarantines untraced/foreign commits (Obsidian Git, manual git) into `.memoria/quarantine/`. — **Why:** §4.1 "Local git cannot *prevent* another committer … the worker is the **supported** write path, **not an enforceable sole committer**. The enforced boundary is **trace integrity on scan** … a commit lacking a coupled `derived` event is **quarantined and routed through a backfill/import command**." §8.1 explicitly names Obsidian-Git auto-commit as a rejected foreign committer.

**What:** Added the structural detection suite as **integrity-* operation Concepts**: `integrity-claim-quote-check`, `integrity-contradiction-check`, `integrity-evidence-check`, `integrity-link-target-check`, `integrity-prompt-injection-check`, `integrity-quote-anchor-check`, `integrity-provenance-checkpoint`, plus `check-source-metadata` and `check-falsifiability` (`vault-template/capabilities/operations/`). — **Why:** §10 "The integrity mechanism the human gate used to be. Zero-LLM where possible; flag, never silent-mutate; on write and on schedule." §10 splits "bibliographic hygiene" from knowledge integrity via a separate `check_source_metadata`. `integrity-provenance-checkpoint` implements the §12 "provenance-triggered human checkpoint on injection-prone content — derived from a freshly-ingested, uncorroborated external source" — the single most load-bearing residual-risk mitigation ("**flag checked notes or digests that depend on checked sources whose metadata is still partial, unverified, or not indexed**").

**What:** Calibration is now shipped as `calibration.yaml` with `production_enabled: false` and `null` thresholds across `clustering`, `hybrid_scores.candidate_rank`, and pinned model-version slots (`vault-template/.memoria/schemas/calibration.yaml`). — **Why:** §10 "**shadow-first, error-budgeted calibration discipline**: no score affects routing until grounded within an error budget; ungrounded ⇒ report-only; drift ⇒ revert to shadow." The file's own comment: "Null thresholds mean shadow/report-only."

### Profiles / runtime — operations replace profiles

**What:** The **five-profile / seven-specialist / fleet** runtime model was retired; policy now binds to **operations, not profile identity**. Operations are Concepts with policy in frontmatter: `operation_id`, `allowed_tools`, `allowed_paths`, `allowed_network`, `runner`, `model`, `prompt_version`, `io_schema`, `risk_class`, `required_checks` (`vault-template/.memoria/schemas/types/operation.yaml`; 40 `capabilities/operations/*.md`). Old ADR-02 "seven-specialist-profiles", ADR-37 "retriever-scout-profile", ADR-42 "profile-compilation" all deleted. — **Why:** §11 "Policy binds to **operations**, not profiles/fleet identity. An operation policy is the runtime boundary"; §17 "A role/profile name is never the enforcement boundary; enforcement comes from the operation's tool, path, network, check, and audit rules … The old five-profile/fleet shape is not carried forward."

**What:** **Separation of duties** encoded per operation — generators (`compile-source-digest`, `propose-note-candidates`, `analyze-claims`, `red-team-argument`) are never their own checkers (`integrity-*`, `check-*`); generators use `runner: direct_api`, checkers use `runner: local` with `integrity_checker`/`evaluator` tools (compare frontmatter across `capabilities/operations/`). — **Why:** §11 "**Separation of duties:** the operation that *generates* is never the one that *checks*." The loop is "**not** generate→approve→repeat: an output is **born `unchecked`**, an independent checker (a different op/model) flips it `checked` *or* raises a flag routed `act`/`ask`/`drop`."

**What:** New **LLM runner split** — every operation declares `runner: direct_api` (default, generators) or `runner: local` (deterministic checks/writers) or (per policy) `hermes` (`operation.yaml` `runner` field; `capture-source` `runner: local`, `compile-source-digest` `runner: direct_api`). — **Why:** §8.2/§13 "the default runner is a direct OpenAI-compatible API call owned by the worker … Hermes is a **library adapter**, not the framework." §11 "**Direct API is the baseline** because it leaves Memoria in charge of request shape, schema validation, replay evidence, and tool-call logging." Commit `2ff5a49d` ("Clarify alpha.11 llm runner split") is the docs runway; the model is corroborated by the retirement of ADR-22/ADR-27 (build-on-Hermes-runtime / Hermes-native-config-and-gate-enforcement was deleted) — Hermes drops from control-plane to specialized adapter. The shipped ai-catalog pins `model: deterministic-fixture` because "**local `qwen2.5:7b` = test fixture only**" (§8 Integrations) — the alpha.9 "qwen is test fixture not runtime" correction carried forward.

**What:** `steering.md` (renamed from `research-focus.md`) is the authored **program** — the `program.md`-equivalent (`vault-template/steering.md`; body text identical except the substrate is now named "steering … the program-wide steering substrate"). — **Why:** §11 "**Direction is authored as the program, not the content.** The human's authored surface is `steering.md` + the project framing + operation policy + the wiki `SCHEMA.md` … the `program.md`-equivalent (Karpathy `autoresearch`)." The rename makes the Karpathy "edit `program.md`, never touch the code" framing literal (§0).

**What:** **Autonomy ceiling** made explicit in policy — no autonomous keep/discard loop for synthesis; code-lane autonomy deferred (old ADR-34 "code-artifact-autopilot" deleted; §11). — **Why:** §11 "An autonomous keep/discard loop is admissible **only** where … a scalar, monotonic metric … reversible … independent, comparable experiments [hold]. They **fail for synthesis** … so the knowledge lane is **human-directed**, and the **integrity spine is the substitute for the missing metric**" (Karpathy `autoresearch`).

### Pipeline / ingest / enrichment

**What:** The whole **ingest spine — capture → conversation → digest+hub compounding → notes** — was materialized as operation Concepts: `capture-source` (+ adapters `capture-pdf-source`, `capture-url-source`, `capture-bibtex-source`, `capture-zotero-source`), `record-copi-interview`, `compile-source-digest`, `propose-note-candidates`, `curate-note-candidate`, `curate-note-link`, `extract-claim-stubs` (`vault-template/capabilities/operations/`; §9). — **Why:** §3/§9 "The §3 ingest spine … *is* the mechanism." The **conversation is the hinge**: `record-copi-interview` ("Record a PI interview answer for later digest synthesis") implements §9 "**Conversation (the interview — the hinge)** … it is *not* an optional side chat." `compile-source-digest` implements the **compounding step** ("compile the per-source digest, then compound hub updates … curated hub changes stay as suggestions") — §5's Co-PI differentiator, "**core — the Co-PI differentiator, not deferrable**."

**What:** Capture adapters are a **small fixed set** (`capture-pdf/url/bibtex/zotero-source`), not a translator engine; Zotero import is via the read-only **Zotero Local API** (`capture-zotero-source.md`; §9/§13). — **Why:** §9 "Capture adapters are a small fixed set … **not a translator engine** … lean on the Zotero connector + **Zotero Local API** only for item/source import"; §13 "Zotero annotations are not an alpha.11 import path; Memoria-owned PDF capture stores canonical quote/span/hash selectors itself."

**What:** New QuickAdd surface `capture-note.js` (writes an **unchecked** note under `knowledge/notes/`), plus `open-inbox.js`, `quickadd-utils.js`, `restore-memoria-shell.js`, and a heavily rewritten `resolve-inbox-card.js` (106-line change) (`vault-template/system/scripts/`). — **Why:** the PI-path note-authoring surface must land content as `unchecked` and let "The worker/check loop own promotion" (`capture-note.js` header comment) — enforcing §4.4's read-barrier even for human-authored notes. This is the plugin-boundary rule from ADR-121: the surface enqueues/writes-unchecked, never promotes.

**What:** **Gap analysis** shipped as `analyze-gaps` (source/note topic-mismatch classification, `runner: local`, reads checked Concepts only) plus `analyze-project-argument`, `render-project-argument-canvas`, `surface-tensions`, `compare-and-contrast` (`vault-template/capabilities/operations/`). — **Why:** §9 "**Gap analysis (over *both* graphs and their correspondence)** … read the **mismatch** — the primary signal" (thin/thin, dense/dense, sources-dense/notes-thin, notes-dense/sources-thin). §3 "**Gap analysis is first-class** — the thread that makes Memoria *directed*." `analyze-gaps` reads "checked current Concepts" only — enforcing the read barrier on the gap signal itself.

**What:** All pipeline operations write to `.memoria/staging/knowledge/` (`output_target` in `compile-source-digest`, `propose-note-candidates`) and are `initial_check_status: unchecked`. — **Why:** §4.4 the universal staging path is the enforcement mechanism, "not just a flag."

### Retrieval / derived store

**What:** The disposable index is now **qmd**, rebuilt **from checked-only Concepts** via `rebuild-checked-qmd-source` (`.memoria/index/qmd`; `folders.yaml` skeleton; §6). — **Why:** §6 "the **query index** — an *interface* (BM25 + vector + hybrid) … **`qmd` is the backend** (Karpathy names it; disposable-bundle BM25 index/search verified pre-alpha.11), SQLite FTS5+sqlite-vec the zero-dependency baseline/fallback." Crucially the rebuild is **checked-only** ("Refresh the checked-only qmd input tree") — §4.4 "retrieval/qmd projections … must filter to `check_status: checked`."

**What:** Added `answer-query` and `summarize-for-recall` operations implementing the **Ask** contract (`vault-template/capabilities/operations/`; §9). — **Why:** §3 "The one distinct read surface is **Ask**: a retrieval-backed, **cited** query over wiki + notes (with temporal/supersession filters)"; §9 "**The Ask/Query answer contract is sources-plus-unknowns** … a confident answer with no stated edges is a defect." This supersedes the earlier Explore/Ask/Transform triad (§3, §Surface-orchestration ADR cluster).

**What:** Retrieval currency (hide superseded/non-current by default) preserved as a design invariant, exercised by the `stale-as-current` seeded-error case (`vault-template/system/eval/alpha11-seeded-errors.json`; §9/§17). — **Why:** §9 "never present a superseded claim as current"; §17 "retrieval hides superseded/non-current content by default; historical lookup must be explicit."

### Discovery / capabilities registry

**What:** New **`capabilities/` OKF bundle** (`operations/`, `skills/`, `mcp/`, `workflows/`) with a generated `ai-catalog.json` projection and `capabilities.base` (`vault-template/capabilities/`). — **Why:** §11 "Capabilities are Concepts; discovery is ai-catalog … Operations, skills, MCP configs, and workflows live in the `capabilities/` bundle as **OKF Concepts** (one per capability, policy in frontmatter). A **generated tracked `ai-catalog.json` projection** indexes them." §13 "**ai-catalog = the capability registry** (OKF is the knowledge registry)."

**What:** `ai-catalog.json` carries a **trust manifest** per entry (`trust: {sha256, source}`) (`vault-template/capabilities/ai-catalog.json`). — **Why:** §12 "**Capability import is supply chain, not just data** … Vet it before the worker runs it via **ai-catalog's trust manifest** … an unsigned/unattested imported capability is **quarantined, not run**." §11 "its **trust manifest is exercised only on third-party import**, and **exposing Memoria's operations as MCP tools externally is deferred**." The full **ARD federated discovery service is rejected** — "it solves a cross-org, web-scale problem the single-user local design doesn't have."

**What:** Projection regeneration operations added: `regenerate-ai-catalog`, `regenerate-references-bib`, `regenerate-indexes`, `regenerate-tracked-projections` (the umbrella, with `required_checks: [projection-drift]`) (`vault-template/capabilities/operations/`). — **Why:** §6.1 "**Generator/check expectation:** alpha.11 must add one worker-owned projection generator and a drift check that regenerates these files into a temp tree and diffs them against the tracked copies; a hand-edited projection does not satisfy acceptance."

### UI / navigation / dashboards

**What:** New **space projections** shipped: `spaces/knowledge.md`, `spaces/project.md`, `spaces/inbox.md`, `spaces/library.md` (plus reworked `maintenance.md`), each embedding a `.base` view (`![[knowledge.base#Digests]]` etc.) and a **read-barrier callout** ("`check_status: checked` is the read barrier. ### Removed / retired (summary, with why)

- **Per-write approval + review-gated folders** (`gated_prefixes`) → retired for the `check_status` read-barrier: per-atom approval "checks the wrong object" and rubber-stamps under load (§2, Jacobs 2021).
- **`maturity` (seedling/budding/evergreen) + universal `lifecycle`** → retired: "trust = the checks" (§5); consumption eligibility is `check_status`, not a human-set maturity.
- **Durable Inbox card Concepts** (candidate/gap/flag/alert/work-prompt/worker-card/worklist-item) → retired: attention is a **generated projection** over journal/check/queue state (§8.2, §17).
- **Profiles / fleet / five-lane runtime** → retired: policy binds to operations, not identity (§11, §17).
- **Hermes as control-plane** (session DB, memory, Kanban, cron, goals as source of truth) → demoted to a specialized library adapter (§8.2, §11, §13; ADR-22/27 deleted).
- **Separate `paper`/`dataset`/`repository`/`code-note`/`thesis` types** → collapsed into `source.item_type` and the single `project` type (§5, §7.1).
- **CEBM `evidence_level`, `entity`-pointer source notes, mandatory `claim.sources`** → replaced by `csl_json` + typed Links + `evidence_set` warrant (§7.1).
- **Embedded `creation:` form blocks inside type schemas** → moved to QuickAdd/Modal-Form scripts + generated forms (schema-driven creation, ADR-119); schemas now declare only shape/enums.
- **The bundled `mediterranean-diet` sample skeleton** (old `folders.yaml` `.memoria/samples/…`) → removed: alpha.11 seeds "only test projects and captured sources created through the alpha.11 pipeline" (§20).
- **Full ARD federated discovery, RO-Crate/BagIt, external MCP exposure, in-toto/SLSA, draft/verify/export, code-lane autonomy** → explicitly **deferred** (not retired): boundary/scope decisions in §11/§13/§15/§16, kept as named future work so the alpha stays "the basic knowledge cycle, end to end, with the plugin."
