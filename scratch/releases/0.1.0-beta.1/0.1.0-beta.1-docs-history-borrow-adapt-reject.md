## Headline

- **Roughly one-third of the raw pool survives as distinct items**: ~215 raw findings collapse to ~70 after dedup. Almost nothing was discarded as worthless — the redundancy was convergence, not noise: the top ideas were mined independently by 4–8 readers each, which is itself a signal about where the current design is thinnest.
- **The single biggest gap: alpha.15 ships disposition telemetry with no presentation or interpretation contract.** The deleted corpus supplies both halves fully worked out: honesty-card suggestion format (argument-for/against, never a verdict), batch worklists for high-cardinality decisions, two-sided accept-rate bands with accept-unedited tracking, review-queue back-pressure, and a precision-floor/volume-cap rule for suggestion surfaces. This cluster is the highest-value borrow and conflicts with nothing current.
- **beta.1's `superseded:true` flag and Crossref allowlist have no consumers.** Default-exclude superseded from Ask retrieval (counted in the coverage line), flag drafts citing superseded works, and run a retraction sweep via the already-allowlisted api.crossref.org. The deleted record shows the missing retrieval filter was a live correctness bug for months last time.
- **beta.1's operational layer is unspecified today but fully designed in deleted docs**: JobRunner lifecycle (idempotency keys, retry ceilings, heartbeat-on-success), Guard fail-closed liveness with negative deny-tests, lifecycle-layered upgrade reconcile, hostile-plugin defense, and citekey pin-at-import with correction-as-revertible-run.
- **The alpha.16 writing module is largely pre-designed**: argument graph distinct from knowledge map (outline derives from the argument, else you get a survey), a deterministic gap taxonomy, saturation with a refutation floor and researcher stamp, counter-outlines with a CHOSEN record, and a draft claim-trace loop — all deterministic mechanisms compatible with meaning-only frontmatter and the no-oracle rule.
- **Strongest re-rejections, each already killed 2–3 times**: confidence-routed gate bypass, second-model consensus checking, nightly autonomous discovery, maturity/lifecycle ladders and machine-written frontmatter, and agent-executed code loops. The reject records were themselves deleted, which is why they keep resurfacing — they need durable homes.
- **Process: alpha.11's evidence discipline is the proven practice set** (pre-implementation assumption ledger, seeded-error integrity harness with named metrics, tmp-disposition table on deletion) and should be reinstated before alpha.15 implementation starts. Every numeric threshold anywhere in this pool is an untested placeholder — borrow shapes and knobs, never values.

## Patterns in what was deleted

**Deletion was almost never a verdict.** The overwhelming majority of these 520 pages died in packaging events — slug renames (#300), the docs2 promotion, the docs/design retirement (#406), tmp pruning at checkpoint close — not on merits. The dangerous consequence: architecture-independent policies (suggestion UX contracts, epistemics rules, test practices) died as collateral of architecture retirements they merely rode on. The honesty card died with the inbox, back-pressure died with the kanban board, the loudness ladder died with the dashboards — yet none of them depended on those hosts. When an architecture is retired, its surface-level policies need explicit re-homing, or this archaeology has to be re-run.

**The project repeatedly builds the enforcement half and loses the consumption half.** The superseded flag shipped without a retrieval filter and was a live correctness bug for months (alpha.9's audit caught drafts citing retired claims); deny logs existed without spike detection; disposition telemetry is currently shipping without interpretation bands or an emission-point contract. A mechanism without a named consumer decays into decoration. The fix is procedural: every flag/log/score in a design names its consumers in the same document.

**A stable set of zombie ideas gets re-invented and re-killed on identical grounds.** Maturity/lifecycle ladders were designed, hedged, and deleted across at least three doc generations; nightly discovery, second-model consensus, Telegram channels, frontmatter-as-machine-scratch, and adoption triggers each died twice or more. The rejection rationale was re-derived from scratch each time because the reject record lived in the doc tree that got deleted. Under the ADR-only model, rejects that keep tempting deserve durable ADR-status records — the Reject table above is effectively the backlog for that.

**Frontmatter is the perpetual dumping ground, and the pressure is structural.** Every generation materialized status, scores, proposals, or aspect fields into frontmatter for one reason: Obsidian's native surfaces (Bases) read only frontmatter. That constraint has not changed, so the "just write a derived scalar so the view can filter" proposal will re-arrive during alpha.15 implementation, from view code, framed as pragmatic. Meaning-only frontmatter survives only if the sanctioned alternative (derived state in the DB, one generated index artifact with computed_at, read-only surfaces) is convenient enough to reach for first.

**Every era minted confident numbers from zero data — but the knobs recur even when the values are fiction.** 0.8 dedup similarity, 15–20 hub thresholds, WIP 5, 90-day retention, 0.92 batch-approve confidence: none was ever validated, yet the same knobs reappear independently across generations, which is decent evidence the dimensions are real. The current shadow-first doctrine is the correct residue, but the deleted corpus supplies its two missing complements: supervision-budget arithmetic (a single researcher's annual labels cannot fund many calibratable thresholds — count before adding a score) and display floors (never render a band over sparse data).

**Enforcement theater is the recurring audit finding.** Fail-open registration, merged-but-not-deployed gate fixes, ~20% of accepted ADR claims not holding against artifacts, a security regression invisible from the repo because only the deployment contradicted the source. The countermeasures that worked — liveness assertions, negative deny-tests, the THEATER taxonomy, absence pins — were themselves deleted with checkpoint scratch. Alpha.11's tmp-disposition table (record where every deleted file's durable content went) is the single deletion done right in the whole history, and the existence of this report is the cost of not doing it everywhere else.
