# Beta.1 design alternatives summary

Date: 2026-07-03. Alternatives memory for the beta.1 charter. Three
clean-slate designs competed; adoption-first Design A won and became the
charter skeleton. This file keeps the verdicts, adopted grafts, and attack
summary only; detailed alternative bodies stay out of scratch because their
useful decisions are already folded into the charter.

## Judge verdicts

- **Design A — adoption-first, winner.** Best product legibility and strongest
  focus on the dominant risk: abandonment. It made Ask the everyday payoff
  moment, kept capture/annotation friction measurable, and put evaluation in
  service of real use rather than the other way around.
- **Design B — integrity-first, losing source of grafts.** Best ideas were the
  absence drill, migration-backed keep-test stability, immediate-stop treatment
  for human-text loss, and cost honesty. It over-weighted trust plumbing before
  proving that the daily loop was worth using.
- **Design C — evaluation-first, losing source of grafts.** Best ideas were the
  fixed review calendar, three-verdict exit, channel discipline, and calibrated
  threshold ledger. It risked making the measurement apparatus the product.

## Adopted grafts

1. Replace the seven-day abandonment trigger with a fourteen-day trigger; keep
   seven days as a warning marker and require one capped resumption attempt
   before declaring abandonment.
2. Treat file-format stability as a migration/keep-test outcome: every format
   change must ship with a lossless, exercised migration rather than freezing
   guessed beta formats.
3. Add the absence drill: after two weeks of zero use, the researcher must be
   able to resume normal capture/compose work without a catch-up queue, repair
   task, or nag backlog.
4. Make silent loss of human-authored text an immediate-stop incident, not a
   week-14 exit-review item.
5. Use a three-verdict exit: proceed, targeted redesign, or kill. Two
   consecutive targeted-redesign verdicts on the same flow escalate to kill.
6. Keep release-channel discipline: unreleased builds touch only the sandbox
   vault; only a tagged beta build touches the real corpus after a pre-upgrade
   snapshot.
7. Make Ask and gap reports use the widest local searchable denominator
   available, with coverage stated plainly. Digests enrich retrieval; they are
   not the only evidence substrate.
8. Cap cost prompts: routine operations use fixed ceiling prices and the
   monthly meter; explicit confirmation is reserved for large operations or
   cap-crossing work.
9. Keep a weekly sampled citation/claim audit and a written trust-incident
   taxonomy so "zero fabricated citations" means more than "none noticed."
10. Track total attention cost, including review, verification, retries, and
    failure handling, rather than shifting product friction out of the primary
    verb metrics.
11. Label all thresholds as pre-registered guesses until beta.1 data calibrates
    them.
12. Cut organization-theater operations from beta.1. Keep safeguards that
    protect irreplaceable data; defer support SLAs, debug-bundle polish, and
    continuous egress machinery until there is a real support surface.

## Attack summary

| Risk | Adopted response |
| --- | --- |
| Ask can look empty when the corpus is undigested. | Search raw/full-text local material first, enrich with digests, and print the searched/undigested/no-fulltext denominator. |
| Machine or repair code could destroy human words. | Immediate-stop rule, pre-operation snapshots, diff review for human-text files, and restore/revert drills. |
| Null-workflow reversion is invisible. | Keep passive Memoria logs plus a tiny explicit fallback counter; do not pretend external tools are fully instrumented. |
| Developer-as-subject contamination can make tinkering look like research. | Separate researcher-hat events from maintainer work, record release markers, and keep fixed review dates. |
| Prompt injection and vault egress can leak unpublished work. | Keep markdown egress linting, Rule-of-Two checks on request plans, and stdio/local-only MCP assumptions for beta.1. |
| Metrics compare unlike work or omit review cost. | Publish a verb-mapping table and a weekly total-attention budget before day 1. |
| Format promises can freeze guessed beta schemas. | Promise lossless, tested migration and keep-test survival, not byte-for-byte immobility. |
| Reuse-rate criteria can punish healthy broad capture. | Scope reuse to beta-period/project-relevant works; treat whole-corpus reuse as descriptive. |
| Cost previews can become per-item approval dialogs. | Fixed price ceilings for routine calls, confirmations only above threshold, visible monthly spend and caps. |
| Measurement burden can cause the abandonment it measures. | Baseline only the few null-workflow facts needed for comparison; product telemetry stays passive. |
