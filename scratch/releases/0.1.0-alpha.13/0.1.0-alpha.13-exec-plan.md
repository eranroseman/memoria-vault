# Memoria — 0.1.0-alpha.13 exec-plan

Pairs with `0.1.0-alpha.13-design.md` (this dir). Builds the **multi-source catalog enrichment subsystem** on the alpha.12 engine. **Assumes alpha.12 is in place** — the engine spine + write lifecycle (alpha.12 WP1), the SQLite catalog + `references.bib` (alpha.12 WP6), and the operations/runner (alpha.12 WP5). Sandbox-only (`Memoria-test`); no migration target; scratch work → `scratch` branch, committed directly (no PR).

## Goal
Every catalog add is **enriched**: resolve identifiers → type-branch (DOI / ISBN / URL) → fan out to the providers → merge with **per-field provenance** → check (incl. **retraction**) → commit through the alpha.12 lifecycle, with a journal `api_call` event per provider call. **MVP = DOI items via Crossref + OpenAlex + Unpaywall;** everything else adds by need.

## Settled (design §8 — don't re-litigate)
- Enrichment is a **worker operation**, not an agent tool (MCP-only sandbox; declared HTTP allowlist).
- **DOI is the master key; type-branch at resolution** (DOI / ISBN / URL-primary-source).
- **Field-authority merge + per-field provenance;** high-authority conflict → flag + `metadata_status`, never silent.
- **Identity via external IDs** (ORCID/ROR/ISSN/OpenAlex/S2), not strings; human merges preserved on re-enrich.
- **Retracted/contested → flagged, never silently `checked`.**
- Raw payloads cached as content-addressed blobs; enrichment **idempotent + re-runnable**.
- **Required providers block ingest; optional enrichers async-backfill.**
- **Six adapters + one reconciler;** add an abstraction only when a 7th demands it.
- **No `catalog/` markdown folder** — catalog is SQLite-SSOT; the **digest is the source's graph presence**; `source_id` link model (not wikilinks); PI catalog via a plugin over the DB; `references.bib` is the portable takeout.

## Deferred / async
- Full **citation graph** + **embeddings** → async backfill, not blocking ingest.
- **Tier-2/3 providers** (CORE, bioRxiv/medRxiv, NASA ADS/INSPIRE, OpenCitations, OpenAIRE, Wikidata; commercial) — by need only.
- **Primary-source/web archiving** depth (Wayback) — on demand.

## Guardrails (standing)
- **MCP-only sandbox** — enrichment is worker-side HTTP on a declared allowlist; agents get no network/code-exec.
- **Provider etiquette/compliance is non-negotiable** — `mailto`/email/keys + per-provider rate limits + terms (polite pools, Unpaywall email, gentle arXiv).
- **Integrity** — retracted/conflicted → flagged, never silently `checked`.
- **PI-direct-access** — catalog is now DB-only, so the plugin catalog surface (WP9) is required, not optional.
- **Own branch/worktree;** scratch work committed directly to `scratch` (no PR).

## Phase 0 — locks (design §9 gates + §10 decisions)
Turn the §9 pre-implementation gates into runnable contract tests (resolution, merge determinism, conflict, retraction, provenance, idempotency, etiquette, identity). Lock the §10 decisions:
- **D1 Title-match confidence threshold** — auto-accept a resolved DOI vs flag for human.
- **D2 Full-text** — store extracted text (CORE/OA PDF) or just link?
- **D3 Re-enrichment cadence** — when to refresh volatile fields (cited_by_count, retraction).
- **D4 Required-vs-optional provider set** per branch.
- **Accept:** every §9 gate has a passing contract test; D1–D4 resolved or explicitly deferred.

## Phase 1 — resolution + fetch spine
**WP1 — Identifier resolution + type-branch**
- Goal: from any input (DOI / arXiv / PMID / ISBN / title+authors / URL) → canonical key set + branch.
- Build: DOI master + crosswalks (NCBI idconv, arXiv↔DOI); title-match with confidence threshold + flag; the branch router (DOI-scholarly / ISBN-book / URL-primary-source).
- Accept: each branch yields the right key set; **no-confident-key → flagged, not guessed.**
- Depends: alpha.12 catalog (WP6).

**WP2 — Provider adapters + fetch governor**
- Goal: a thin adapter per provider → common `ProviderRecord`; governed concurrent fetch + cache.
- Build: the six core adapters (Crossref / OpenAlex / S2 / PubMed / Unpaywall / arXiv); **per-provider token-bucket** governor; timeout + bounded retry; **content-addressed payload cache** (→ `raw/`); a journal `api_call` event per call.
- Accept: each adapter normalizes + caches + journals; rate limits honored; a provider outage degrades gracefully (flag-for-retry, never fail the item).
- Depends: WP1.

## Phase 2 — merge + write
**WP3 — Reconciler (field-authority merge + provenance)**
- Goal: merge `ProviderRecord`s → canonical `csl_json` + `field_provenance`; conflicts → flag.
- Build: the field-authority table; per-field provenance; conflict detection → `check-fired` + `metadata_status`; deterministic merge.
- Accept: identical payloads → identical record + provenance; high-authority conflict → flag + `partial/unverified`, never silent.
- Depends: WP2.

**WP4 — Identity reconciliation**
- Goal: link authors/venues/institutions by external IDs; preserve human merges.
- Build: `external_ids` on sources/entities; ORCID/ROR/ISSN/OpenAlex/S2 linking; entity dedup that preserves curated merge decisions.
- Accept: `authored_by` edges carry external IDs; **re-enrich preserves prior merge decisions** (alpha.12 author-reconcile gate).
- Depends: WP3.

**WP5 — Commit + data model (no catalog markdown)**
- Goal: write enrichment through the alpha.12 lifecycle; the digest is the source's graph presence.
- Build: `enrichment_runs` / `provider_payloads` / `field_provenance` / `external_ids` tables; `sources` / `entities` / `catalog_links` writes; `references.bib` materialize + drift check; the `source_id` link model (no `[[wikilink]]`); **no `catalog/` markdown written**.
- Accept: enriched source committed; `references.bib` drift-clean; drop SQLite → re-enrich/rebuild; the digest is the only file-side source node.
- Depends: WP3, WP4, alpha.12 lifecycle (WP1/WP5).

## Phase 3 — integrity + Tier-1
**WP6 — Retraction / integrity check**
- Goal: retracted/contested → flagged, never `checked`.
- Build: Crossref Retraction Watch + PubMed pub-types + PubPeer adapters; the retraction check; the `source.lifecycle: retracted` hook.
- Accept: a retracted DOI → flagged, non-consumable-`checked`; later retraction (on re-enrich) re-flags + cascades.
- Depends: WP2, WP3.

**WP7 — Tier-1 additions (by need)**
- Goal: ORCID/ROR resolvers; books (Open Library / Google Books, the ISBN branch); DataCite/Zenodo (data/software).
- Build: the adapters; wire into the branch router + reconciler + field-authority table.
- Accept: an ISBN item enriches via the book branch; a dataset DOI via DataCite; author/org resolve to canonical records.
- Depends: WP1, WP2, WP4.

## Phase 4 — async + PI surface
**WP8 — Async backfill**
- Goal: non-blocking enrichers (embeddings, citation graph, topics).
- Build: a follow-up operation; bounded; the source is usable as soon as required providers land.
- Accept: required-provider success unblocks ingest; backfill updates the record later without re-blocking.
- Depends: WP5.

**WP9 — PI catalog surface (plugin)** *(required — PI-direct-access)*
- Goal: PI views/edits the DB-only catalog from Obsidian (replacing the removed `catalog/` folder).
- Build: a plugin catalog view over the DB; source/entity edits routed through an engine operation (never direct DB writes); the digest as the source's Obsidian node.
- Accept: the PI can view/edit a source from Obsidian; edits go through the worker + checks; **every catalog feature reachable from Obsidian directly**.
- Depends: WP5.

## Cross-cutting
- **Config:** `mailto`/email + per-provider keys + rate limits + required/optional set + OA-only-PDF + cache TTL.
- **Eval:** an enrichment accuracy/coverage baseline on a sample of known-good sources (field-authority correctness, dedup precision).
- **ADRs:** the enrichment subsystem + provider field-authority + the catalog-folder-removal (refines alpha.12 §3).

## Critical path
`Phase 0 → WP1 → WP2 → WP3 → {WP4, WP6} → WP5 → {WP7, WP8, WP9}`

## Top risks
- **Provider terms / rate limits** — compliance is non-negotiable; the governor + `mailto`/keys are load-bearing, not optional.
- **Title-match false positives** (no-DOI items) — confidence threshold + human flag; never auto-attach a guessed DOI.
- **Provider disagreement** — field-authority + provenance + flag; never silently pick a winner on a critical field.
- **Bulk enrichment of ~800 sources** — rate-limit-bound, not CPU-bound; concurrent fetch + cache + batch; respect daily caps.
- **PI-direct-access regression** — catalog is DB-only now, so WP9 must ship or the PI loses catalog access entirely.
