# Memoria — 0.1.0-alpha.13 design (catalog enrichment)

Date: 2026-06-30
Status: working design for `0.1.0-alpha.13`. Adds the **multi-source catalog enrichment subsystem** on top of the alpha.12 engine (SQLite-SSOT catalog, the worker/operation/journal boundary, the §3–§4 ER model). **Builds on, does not supersede, alpha.12.** Sandbox-only (`Memoria-test`); no migration target.
Method: derived from the alpha.12 catalog ER model + the requirement "every catalog item is enriched from multiple scholarly APIs (OpenAlex, Semantic Scholar, PubMed, Unpaywall, Crossref, arXiv)."

---

## 0. What alpha.13 adds
- An **`enrich-source` operation** invoked on every catalog add: resolve identifiers → fan out to scholarly providers → merge into one canonical record with **per-field provenance** → validate/check (incl. **retraction**) → commit through the alpha.12 lifecycle.
- New tables on the §3 ER model: `external_ids`, `enrichment_runs` / `provider_payloads`, `field_provenance`; provider data populates author/venue/institution/topic `entities` + citation/OA edges in `catalog_links`.
- **Type-branched ingest:** DOI-keyed scholarly · ISBN-keyed book · URL/primary-source — each with its own provider set, all merging into the same canonical `sources` model.

---

## 1. Where it lives (alpha.12 fit)
- `enrich-source` is a **worker operation** in the ingest path; it runs after `capture-source`, feeding the source's DB commit.
- **Sandbox respected:** the worker (trusted code) makes HTTP to a **declared network allowlist**; agents never get network or code-execution. Enrichment is a worker capability, not an agent tool.
- Writes through the alpha.12 lifecycle `requested → stage → validate → check → DB commit → references.bib → project`. **Each provider call is a journal `api_call` event** (provider, endpoint, request, `response_hash`, status, latency), so every field is traceable to who supplied it.

---

## 2. Provider roles (who is authoritative for what)

| Provider | Key | Authoritative for | Etiquette / auth |
|---|---|---|---|
| **Crossref** | DOI | Registered bibliographic metadata, references, funders, license, container/ISSN | polite pool + `mailto`; no key; ~50/s |
| **OpenAlex** | DOI / OpenAlex ID | Disambiguated authors (ORCID), venues (ISSN), institutions (ROR), **topics**, **citation graph** (`referenced_works`), OA locations, `cited_by_count` | `mailto`; ~10/s, 100k/day |
| **Semantic Scholar** | DOI / arXiv / PMID | Abstract, **TLDR**, **SPECTER embedding**, citation **intent/influential** flags, fields-of-study, S2 author IDs | API key (1/s unauth) |
| **PubMed** (E-utils) | PMID | **MeSH** terms, biomedical abstract, pub types, grants, affiliations, PMCID | NCBI key for 10/s; idconv for PMID↔DOI |
| **Unpaywall** | DOI | **Best legal OA PDF** location, OA status/version/license | `email` param; ~100k/day |
| **arXiv** | arXiv ID | Preprint metadata, categories, versions, preprint↔published DOI link | gentle (~1 req/3s) |

Routing is conditional: PubMed only when a PMID resolves (or a biomed venue), arXiv only for preprints — never call a provider an item cannot key into.

### Tier-1 additions (fill real gaps; add by need)
- **Crossref Retraction Watch data + PubPeer** — *integrity-critical, on-thesis.* Retracted/contested status feeds `check_status` + a flag. (Partly reachable via Crossref `update-to`.)
- **ORCID + ROR** — authoritative resolvers for author / institution identity (OpenAlex/S2 give the IDs; these resolve them).
- **Open Library / Google Books (ISBN)** — books/monographs, which none of the six DOI-centric providers handle (essential for a humanities/historian library).
- **DataCite (+ Zenodo)** — datasets, software, many preprints (the non-Crossref-DOI half), pairing with the CFF/CodeMeta/DataCite citation standards.

### Tier-2 (situational / domain-conditional)
CORE (aggregated OA **full text**, not just a link) · bioRxiv/medRxiv (biomed preprints) · NASA ADS / INSPIRE-HEP (astro/physics) · OpenCitations / OpenAIRE / Crossref Event Data (citation/funding breadth) · Wikidata (entity linking / QIDs).

### Tier-3 (skip: cost/license, open equivalents exist)
Scopus, Web of Science, Dimensions, Altmetric, PlumX.

## 3. The pipeline
**Stage A — Resolve identifiers (type-branch).** From whatever the item has (DOI / arXiv / PMID / ISBN / title+authors / PDF / URL), build the canonical key set. **DOI is the master key**; backfill via cross-walk (NCBI idconv for PMID↔DOI↔PMCID, arXiv↔published-DOI, Crossref/OpenAlex title search → candidate DOI with a confidence threshold). No confident key → **flag for human confirmation, do not guess.** Branch:
- **DOI-keyed scholarly** → the six core + Tier-1 (retraction/identity).
- **ISBN-keyed book** → Open Library / Google Books.
- **URL / primary source** (a historian's web/archival materials, no identifier) → web metadata extraction (defuddle / cite-from-URL) + archiving (Wayback `availability`).

**Stage B — Fan-out fetch (concurrent, rate-governed).** Each provider has a thin **adapter** that fetches by key, caches the **raw payload as a content-addressed blob** (in `raw/`, hash in the DB), and normalizes it into a common `ProviderRecord` (CSL-ish fields + external IDs + relations + `raw_hash`). A shared **token-bucket governor** enforces per-provider rate limits; per-provider timeout + bounded retry; **graceful degradation** — the item proceeds with whoever answered.

**Stage C — Merge + reconcile (the heart).** A shared reconciler merges the `ProviderRecord`s by a **field-authority table** into the canonical `sources.csl_json`, recording **per-field provenance**:

| Field | Authority order |
|---|---|
| Title, authors(names), dates, container, license, references | Crossref → OpenAlex → PubMed → S2 |
| Author identity (ORCID/IDs), venue ISSN, institution ROR, topics, citation edges | OpenAlex → S2 |
| Abstract | PubMed → Crossref → S2 → OpenAlex; TLDR from S2 |
| MeSH / biomedical terms | PubMed |
| OA PDF location | Unpaywall → OpenAlex/S2 OA locations |
| Embedding | S2 (SPECTER) |
| Preprint version/category | arXiv |

Conflicts between high-authority providers on a critical field (title mismatch, divergent DOI) → a `check-fired` flag and `metadata_status = partial/unverified`.

**Stage D — Validate + check.** Schema (Pydantic adapter), identifier integrity, cross-provider consistency, and a **retraction/erratum check** (Crossref Retraction Watch + PubMed pub-types + PubPeer). Derive `metadata_status` from provider agreement/coverage and `check_status` from the checks.

**Stage E — Commit + project.** One worker transaction writes `sources`, the populated `entities`/`catalog_links`, `external_ids`, and `field_provenance`; materializes `references.bib`; updates the index. **Async backfill** (embeddings, full citation graph, topics) runs as a follow-up non-blocking operation so the source is usable as soon as the *required* providers land.

## 4. Identity reconciliation (solves the alpha.12 `sources ↔ authors` gap)
Authors are linked by **ORCID / OpenAlex Author ID / S2 author ID** (not name strings), venues by **ISSN-L**, institutions by **ROR**. So `catalog_links` `authored_by` edges carry external IDs → **global, stable disambiguation** instead of fragile local string matching. `csl_json` keeps the literal names; the entity graph carries the resolved identities. On re-enrichment, prior human merge/disambiguation decisions are preserved (FK-by-identity match), per the alpha.12 author-reconcile gate.

---

## 5. Data-model additions (on the alpha.12 §3 ER model)
- **`external_ids`** (on sources/entities): sources — `doi, openalex_id, s2_paper_id, pmid, pmcid, arxiv_id, isbn, datacite_doi, mag_id`; authors — `orcid, openalex_author_id, s2_author_id`; venues — `issn_l, openalex_source_id`; institutions — `ror`.
- **`enrichment_runs`** / **`provider_payloads`**: per (source, provider) — `raw_hash` (→ blob), `fetched_at`, `status`, `provider_version`, `rate_limit_info`. Re-runnable and reproducible.
- **`field_provenance`**: per canonical field → winning provider + alternatives (audit + conflict surfacing).
- Naturally populated: author/venue/institution/topic `entities`; `cites` edges in `catalog_links` (among *your* sources only); `best_oa_location` on the source.

---

## 6. Integrity & provenance
- **Provenance:** each provider call → an `api_call` journal event; the cached payload makes re-merge deterministic.
- **Sandbox:** the worker does the HTTP against a declared allowlist; agents never get network/code-exec. Enrichment is a worker capability, not an agent tool.
- **Retraction is never silently swallowed:** a retracted/contested source is **flagged and cannot become consumable-`checked`** without explicit human resolution.
- **Idempotency:** keyed by (identifier, provider, provider-data-version); re-enrich updates only changed fields as a new run, preserving history.

---

## 7. Resilience & etiquette (do not cut)
- Concurrent fan-out + **per-provider token-bucket** rate limiting; per-provider timeout + bounded retry/backoff.
- **Content-addressed cache** of raw payloads → cheap, reproducible re-runs; respect cache TTL for volatile fields (`cited_by_count`, retraction status).
- **Graceful degradation:** required providers (DOI items: Crossref + OpenAlex + Unpaywall) block ingest; optional enrichers async-backfill; a provider outage flags-for-retry, never fails the item.
- **One config** for `mailto`/email + per-provider API keys + rate limits + required-vs-optional set + OA-only-PDF toggle.

---

## 8. Decisions (this iteration)
- **Enrichment is a worker operation, not an agent tool** (MCP-only sandbox; declared network allowlist).
- **DOI is the master key; type-branch at resolution** (DOI / ISBN / URL-primary-source).
- **Field-authority merge with per-field provenance;** high-authority conflicts → flag + `metadata_status`, never silent.
- **Author/venue/institution identity via external IDs** (ORCID/ROR/ISSN/OpenAlex/S2), not strings.
- **Retracted/contested source → flagged, never silently `checked`.**
- **Provider raw payloads cached as content-addressed blobs;** enrichment idempotent + re-runnable.
- **Required providers block ingest; optional enrichers async-backfill.**
- **Six concrete adapters + one reconciler;** add a provider abstraction only when a 7th demands it.

---

## 9. Pre-implementation gates
- **Resolution:** DOI / ISBN / URL each produce the right key set; no-confident-key → flagged, not guessed.
- **Merge determinism:** identical provider payloads → identical canonical record + provenance.
- **Conflict:** high-authority disagreement → flag + `partial/unverified`, never silent.
- **Retraction:** a retracted DOI → flagged, never consumable-`checked`.
- **Provenance:** every field traces to a provider + a cached payload; every call journaled.
- **Idempotency:** re-enrich updates only changed fields, preserves history + prior merge decisions.
- **Etiquette:** rate-limit governor honored; `mailto`/keys present; degradation on provider failure/outage.
- **Identity:** ORCID/ROR/ISSN link entities; human merges preserved across re-enrich (ties to the alpha.12 author-reconcile gate).

## 10. Open questions
- **Title-match confidence threshold** — when to auto-accept a resolved DOI vs flag for human.
- **Citation-graph depth/cost** — how aggressively to pull `referenced_works`/citations (async + bounded).
- **Full-text acquisition** — store extracted text (CORE/OA PDF) or just the link? (ties to the ingest/digest text needs.)
- **Primary-source/web enrichment depth** — archive every URL via Wayback, or only on demand?
- **Re-enrichment cadence** — providers update (cited_by_count, retraction status); when to refresh.
