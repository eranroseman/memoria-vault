## Status

Draft sandbox release plan for the alpha.15 engine + integrity hardening cut. It
is a **delta on alpha.14**: Memoria inherits the alpha.14 standalone CLI + engine
whole and extends — does not rewrite — its lifecycle, read barrier, and rollback
model. Alpha.15 pins the clean-slate stack (`uv`, Typer, stdlib `sqlite3` no-ORM,
`.sql`/`user_version`) and lands the integrity mechanisms that make the engine
defensibly correct (quarantine-and-verify with a consumption-hash guard,
cross-store ACID, git/journal division + commit contract, storage recovery
classes, NLI Tier-1 proposer), plus the minimum product loop (enrichment merge,
gap analysis, Markdown export, recover) and the CLI Co-PI agent contract.

## Scope

Alpha.15 delivers, in `Memoria-test`, the **safety/integrity core plus the
minimum end-to-end loop** (design §S6):

- Stack cutover: `uv tool install memoria` (`pip install -e .` fallback), Typer
  CLI, stdlib `sqlite3` with WAL and FK, `.sql` + `PRAGMA user_version`
  migrations, no ORM.
- Universal quarantine-and-verify invariant + consumption-hash demote-only read
  guard; `workspace scan` as sole verifier/promoter.
- Cross-store ACID: 2PC lifecycle, write-ahead fsync ordering, workspace-scoped
  lock, crash-at-each-boundary recovery.
- Git/journal division + workspace commit contract: git owns file history +
  `.memoria/journal-head`; journal stores git refs; one stage-only-materialized-
  paths commit per operation; rollback = git restore + DB inverse.
- Storage recovery classes: authority-vs-recovery split, fail-closed-to-
  `unchecked` on DB loss, blob-replay records, human-override log
  `.memoria/overrides.jsonl`.
- Enrichment merge by field authority with deterministic replay; native gap
  analysis with an ordinal severity scale; runner policy (two OpenAI-compatible
  providers, per-op test/production model pin, `--mode`); NLI Tier-1 with a HANS
  acceptance gate; raw deterministic edge extraction → unchecked candidates;
  engine read-API + verdict-tagging + CLI Co-PI; the §S5 integrity-principle sweep.

The minimum loop runs end to end: `init` → `work capture` → `work enrich` →
`work digest` → `note propose/accept` → `project gaps` → `memoria ask` /
`project ask` → `project export` (Markdown) → `workspace recover`, with no
Obsidian/Zotero/Hermes/MCP/profile dependency.

Deferred (from §S6, not in this cut): paper/outcome framing; `eval
select-models`; LLM-judge Tier-2; edge-extraction act-promotion/tuning; per-Co-PI
read scoping + the MCP transport (optional read-only pull-in only if cheap);
HTTP+OAuth remote; UX surfaces (control-surface UI, Obsidian plugin,
`agent-client`, TUI/web); beta.2 real-time change detection.

## Known Limitations

- Limitation: sandbox-only; no installed-workspace migration and no
  backward-compatibility mode. Impact: alpha.15 starts from fresh disposable
  workspaces only. Workaround: test only in `~/Memoria-test`. Tracking: alpha.15
  parent.
- Limitation: CLI Co-PI is all-vault (no exfiltration scoping until the MCP
  transport). Impact: the accepted single-user-own-vault trust boundary; the MCP
  server, per-Co-PI read scoping, and UX surfaces are deferred. Workaround:
  operate via `memoria … --json` and a plain editor; don't-amplify ships in 15.
  Tracking: adapter follow-up.
- Limitation: paper/outcome framing deferred. Impact: `project export` ships in
  plain Markdown as the loop's export step; paper-readiness framing moves to the
  next slice. Workaround: none needed for the minimum loop. Tracking: next slice.
- Limitation: re-enrichment cadence and export format beyond Markdown +
  `references.bib` remain open (§15). Impact: do not block the first checked work,
  digest, ask, project export, or recovery. Tracking: alpha.15 open questions.
- Limitation: real-time change detection deferred to beta.2. Impact: direct edits
  are caught at scan time or fail-closed at consumption; the consumption guard
  makes quarantine immediate-on-consumption. Tracking: beta.2.

## Candidate Checks

1. Required CI is green on `main`.
2. `scripts/verify pr` passes after code changes.
3. `scripts/verify package` passes after shipped workspace/manifest/projection
   changes.
4. `scripts/verify runtime` or `scripts/verify rc` passes, or Runtime is skipped
   with a concrete environment reason.
5. `uv tool install .` then `memoria init`/`memoria doctor` succeeds against a
   fresh disposable workspace in `~/Memoria-test`; `pip install -e .` yields the
   same working CLI; WAL SQLite from `.sql`; no ORM/migration framework.
6. Recovery evidence: crash injected at each lifecycle boundary recovers to
   complete-or-non-consumable; DB dropped → `workspace rebuild` starts fully
   `unchecked` (frontmatter status ignored); records recover by blob replay then
   override-log apply.
7. Commit evidence: exactly one commit per operation staging only materialized
   paths + `.memoria/journal-head` + `.memoria/overrides.jsonl`; a foreign dirty
   tracked file is not swept in; no `git add -A`.
8. Product/manual evidence covers DOI success, required-provider failure,
   conflict, retraction, degraded acceptance (`metadata_status = unverified`, no
   `checked`), qmd doctor/rebuild, gap findings with stable severities, note
   curation, generic/project ask, Markdown project export, two-provider `--mode`
   dispatch, NLI Tier-1 + HANS gate, raw edge extraction (unchecked candidates),
   citation survival, and the seeded-error verdict bars.
9. The `rg`/architecture gates confirm: no baseline surface labels `checked` as
   approved/verified/trusted; no adapter reads raw SQLite/files; the CLI holds no
   logic outside `engine/api`.
10. Documentation integrity: shipped behavior covered; the ADR-worthy mechanisms
    routed to ADRs; deferred items marked.
11. `scratch/releases/0.1.0-alpha.15/` scratch is deleted from the `scratch`
    branch after durable content moves to ADRs, docs, release notes, or issues.
