# ExecPlan - alpha.15 engine + integrity hardening (delta on alpha.14)

Living plan. Self-contained enough for a fresh agent to continue without chat
history. Tactical progress here; durable decisions to ADRs, product state to
issues.

Design authority: `scratch/releases/0.1.0-alpha.15/0.1.0-alpha.15-design.md`.
When plan and design disagree, the design wins; fix the plan. The in-15 vs defer
cut line is design §S6 — it is the scope authority for this plan.

## 0. Metadata

- Task: implement the alpha.15 delta — the safety/integrity engine core plus the
  minimum end-to-end loop, on the alpha.14 standalone CLI + engine baseline. No
  backward-compatibility or migration path.
- Plan branch and worktree: `scratch` at `~/mv/scratch`.
- Implementation base: `origin/main` + the alpha.14 CLI/engine cut (see
  `0.1.0-alpha.14-exec-plan.md`). Each PR gets its own worktree under
  `~/mv/alpha15-<slug>` and branch `feat/alpha15-<slug>` off `origin/main`.
- Release/checkpoint: `0.1.0-alpha.15`.
- Owner: agent-assisted implementation for `eranroseman/memoria-vault`.
- Relationship to alpha.14: alpha.15 **inherits alpha.14 whole** and extends —
  does not rewrite — the lifecycle, read barrier, and rollback model. It adds
  the stack cutover and the integrity mechanisms listed below. Where alpha.14 has
  not yet landed, its slices are prerequisites; this plan covers only the
  alpha.15 deltas.

## 1. Purpose

Alpha.15 hardens the alpha.14 engine into a defensibly-correct one and pins the
clean-slate stack. Over alpha.14 it lands:

- **Stack**: `uv` distribution, Typer CLI, stdlib `sqlite3` with **no ORM**, WAL,
  `.sql` + `PRAGMA user_version` migrations (design §S1/§S2/§11).
- **Universal quarantine-and-verify invariant** with a consumption-hash
  demote-only read guard; `workspace scan` is the sole verifier/promoter (§2/§9).
- **Cross-store ACID contract**: 2PC lifecycle, write-ahead fsync ordering,
  workspace-scoped lock, idempotent demote-on-read, crash-at-each-boundary
  recovery (§6).
- **Git/journal division + commit contract**: git owns file history + the
  `.memoria/journal-head` anchor; the journal stores git refs; one
  stage-only-materialized-paths commit per operation; file rollback = git restore
  + DB inverse (§6/§11).
- **Storage recovery classes**: authority-vs-recovery split, fail-closed-to-
  `unchecked` on DB loss, blob-replay for provider-derived records, and the
  human-override recovery log `.memoria/overrides.jsonl` (§3).
- **Enrichment merge** by static field authority with deterministic replay (§8).
- **Native gap analysis** with an ordinal severity scale (§10).
- **Runner policy**: two OpenAI-compatible providers (`local`, `gateway`),
  per-operation test/production model pin, `--mode` (§4/§7).
- **Contradiction Tier-1**: NLI + abstain + HANS acceptance gate (§9).
- **Deterministic edge extraction (raw)**: explicit-typed links → unchecked
  candidate edges (§4/§9).
- **Engine read-API + verdict-tagging + CLI Co-PI**: `engine/api` single module,
  read commands emit `--json` carrying `check_status`, untrusted text returned as
  data (§2/§5, §1/§2).
- **§S5 integrity principle**: no "checked = approved"; frontmatter `check_status`
  is a generated projection ignored on read.

Observable outcome: a fresh `uv tool install`ed `memoria` that runs the minimum
loop (`init` → `work capture` → `work enrich` → `work digest` → `note
propose/accept` → `project gaps` → `memoria ask` → `project export` (Markdown) →
`workspace recover`), survives a crash injected at each lifecycle boundary, fails
closed to `unchecked` on DB loss, and passes the seeded-error bars — with the
stack, ACID, and quarantine guarantees above.

## 2. Context — delta on alpha.14

- alpha.15 is a delta; §S6 is the scope authority. Everything not in the §S6
  in-15 set is deferred (see §6 of this plan).
- The alpha.14 CLI spine, SQLite engine, request envelope, worker lifecycle,
  read barrier, enrichment substrate, qmd wiring, and capability manifests are
  the base. alpha.15 sharpens them; it does not rebuild them.
- Two things alpha.15 changes that touch existing alpha.14 code and need care:
  the **frontmatter `check_status` supersession** (it becomes a generated
  projection ignored on read — the alpha.14 field contract is overridden), and
  the **commit path** (must become stage-only-materialized-paths, never
  `git add -A`).

## 3. Implementation slices

Small PRs off `origin/main`. Each slice names its design section and its
load-bearing gates (full gate list in design §14 and the §S3 per-section gate
blocks). Write failing tests first.

- **PR-A Stack cutover** (§S1/§S2/§11): `uv` packaging (`uv tool install`;
  `pip install -e .` still works), Typer CLI skeleton, stdlib `sqlite3` with FK
  ON + WAL, `.sql` DDL + `PRAGMA user_version`, no ORM/Alembic dependency.
  Gates: `uv tool install .` yields a working `memoria`; fresh DB reports
  `user_version` + `journal_mode = wal`; dependency-set assertion has no
  ORM/migration framework; schema from checked-in `.sql`, re-init is a no-op.
- **PR-B Storage model + recovery** (§3): schema split (concept mirror vs verdict
  state), authority/recovery matrix, fail-closed-to-`unchecked` on DB loss,
  human-override log `.memoria/overrides.jsonl`, frontmatter `check_status`
  supersession. Gates: rebuild-from-files starts fully `unchecked` and ignores
  frontmatter status; mirror rebuildable, verdict state not; records recover by
  blob replay **then** override-log apply; no human datum in accepted-loss.
- **PR-C Quarantine-and-verify + consumption guard** (§2/§9): the invariant,
  `workspace scan` as sole verifier/promoter, read-barrier demote-only guard
  (mtime+size dirty pre-check → SHA), promotion-only-via-scan. Gates: an edited
  checked file fails closed on next read (demote + enqueue scan); no consumption
  path sets `checked`; every changed origin non-consumable until act/ask/drop.
- **PR-D Cross-store ACID** (§6): 2PC lifecycle, write-ahead fsync ordering
  (payload fsynced before DB commit; DB commit durable before materialize),
  workspace-scoped lock held for the whole operation, idempotent serialized
  demote-on-read. Gates: crash injected at each boundary recovers to
  complete-or-non-consumable; missing-payload-after-commit fails loudly;
  concurrent writers serialize.
- **PR-E Git/journal + commit contract** (§6/§9/§11): git owns file history +
  `.memoria/journal-head` anchor; journal stores git refs not content; one
  commit per operation staging **only** materialized paths + journal-head +
  `.memoria/overrides.jsonl`; file rollback = git restore + DB inverse; `init`
  writes `.gitignore` + repo. Gates: 1:1 commit↔operation; a foreign dirty file
  is not swept in; journal stores refs; init tracked/gitignored set correct.
- **PR-F Enrichment merge** (§8): static field-authority table, provider adapters
  as plain functions, content-addressed payload blobs, deterministic remerge,
  degraded acceptance sets `metadata_status = unverified` and bypasses no content
  check, retraction blocks checked. Gates: identical payloads → identical records
  + `references.bib` hash; adapter HTTP through the shared allowlist; conflict →
  attention; blob-replay reconstructs records.
- **PR-G Native gap analysis** (§10): checked-only inputs, the gap kinds, the
  fixed `{0,1,2}` ordinal severity scale (`score 0..8`, bands ≥4/≥2/else),
  `--json` findings, discovery → unchecked candidates. Gates: reads only checked
  + explicit candidate rows; stable kinds/severities on the fixture; discovery
  never promotes.
- **PR-H Runner policy** (§4/§7): `providers.yaml` two connections
  (`local`, `gateway`), per-operation `runner.test`/`runner.production`,
  `--mode`, resolved model recorded per run; manifest is the hard ceiling. Gates:
  both branches required to load; `--mode` selects the branch; a prompt cannot
  run on an undeclared model; a model change re-triggers the seeded-error verdict.
- **PR-I Contradiction Tier-1** (§9): NLI comparator (pinned, local-capable),
  tri-class SUPPORTED/REFUTED/NOTENOUGHINFO + abstain threshold + FP budget,
  canonical-ID dedup, **HANS acceptance gate** before it may gate. Gates: NLI
  passes HANS (overlap-but-opposite) or does not gate; no auto-accept; beats the
  cosine/BM25 baseline on the fixture.
- **PR-J Deterministic edge extraction (raw)** (§4/§9): parse explicit-typed
  links on materialization → unchecked candidate edges (attention); no
  act-promotion, no argument-class auto-mint. Gates: deterministic (no LLM);
  bare wikilink never yields `supports`/`contradicts`/`extends`; extracted edges
  land unchecked.
- **PR-K Engine read-API + CLI Co-PI** (§2/§5, §1/§2): `engine/api` single module
  (read_* verdict-tagged, write_* → queued unchecked request); CLI is a pure
  wrapper holding no logic of its own; read commands emit stable `--json` with
  `check_status`; untrusted work text returned data-tagged; read-completeness
  checklist tracked. Gates: no adapter reads raw SQLite/files; every read item
  carries its verdict; CLI holds no logic outside `engine/api`; a CLI Co-PI write
  lands `unchecked` and passes checks.
- **PR-L §S5 principle + terminology sweep** (§S5/§3): restate the principle;
  `rg` gate that no baseline surface labels `checked` as approved/verified/
  trusted; frontmatter status generated-and-ignored; the human-checkpoint
  calibration fixture (checkpoint must beat its no-checkpoint baseline).

Ordering note: PR-A → PR-B/PR-C/PR-D/PR-E are the integrity spine and come first;
PR-F/PR-G/PR-H/PR-I/PR-J/PR-K/PR-L layer on. PR-K's `engine/api` factoring must
land as a real single module so the deferred MCP transport is later a skin
(design §S6 factoring discipline).

## 5. Testing & verification

- Failing-first per slice, mapped to the design §14 gate list and the §S3
  per-section gate blocks (quarantine, ACID, git/journal, storage recovery,
  runner policy, read-API, contradiction, edge extraction).
- The **seeded-error verdict** (§9) runs against the configured **production**
  models; passing on test models does not license non-sandbox use.
- Runtime-local gate in `~/Memoria-test` on replay + crash fixtures (no external
  providers or timing dependence).

## 6. Deferred (from §S6) — not in this plan

- Paper/outcome framing (`paper_plan`/`outcome_frame`/`frame-paper`/`export
  --ready-only`) — product expansion beyond the loop, next slice.
- `eval select-models` (hand-pin models in 15) and LLM-judge Tier-2.
- Edge-extraction act-promotion/allowlist tuning.
- Per-Co-PI read scoping + MCP-granted tool restriction (with the MCP transport;
  alpha.15 CLI Co-PI is all-vault).
- MCP server transport (optional read-only pull-in only if cheap), HTTP+OAuth
  remote, UX surfaces (control-surface UI, Obsidian plugin, `agent-client`,
  TUI/web).
- beta.2 real-time change detection (bindingly deferred; the consumption guard
  covers beta.1).

## 7. Known limitations / open questions

- Sandbox-only; fresh disposable workspaces in `~/Memoria-test`; no migration.
- Re-enrichment cadence for volatile provider fields and minimal export format
  beyond Markdown + `references.bib` remain open (§15).
- CLI Co-PI is all-vault (no exfiltration scoping until MCP); the accepted
  single-user-own-vault trust boundary, with don't-amplify shipping in 15.

## 8. Risks

- **Commit path**: `git add -A` anywhere reintroduces the quarantine-bypass
  (finding #3). PR-E must enforce stage-only-materialized-paths with a gate.
- **fsync ordering**: durability/atomicity depend on payload-before-commit and
  commit-before-materialize; PR-D must pin and test it.
- **`engine/api` factoring**: if the CLI grows logic outside `engine/api`, the
  deferred MCP transport becomes a refactor. PR-K's purity gate prevents it.
- **NLI without HANS**: an NLI comparator that has not passed HANS is no better
  than cosine on the cases that justify it; PR-I gates on it.
