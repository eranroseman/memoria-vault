## Scope Boundary

Alpha.15 is an engine and integrity checkpoint, not an Obsidian plugin release.
It keeps the standalone CLI/engine, read API, local HTTP transport, scoped MCP
transport, quarantine-and-verify path, recovery model, four-type Concept model,
and machine-judgment layers in scope.

Out of scope for alpha.15: Obsidian plugin implementation, Obsidian plugin
manifests/packages, command-palette wiring, adapter-specific UI, rich UI
frameworks, per-host wrapper packages, remote/OAuth, multi-user infrastructure,
and edge-extraction act-promotion/tuning without real usage data. No alpha.15
slice adds, updates, packages, or tests an Obsidian plugin, plugin manifest,
command-palette command, Inspector panel, or Obsidian-specific adapter. The
read/view/action contracts may be shaped and tested in alpha.15 only through the
engine, CLI, local HTTP, and scoped MCP surfaces; any Obsidian rendering of
those contracts needs a future adapter ADR when scheduled. This is a negative
scope gate: alpha.15 cannot close with Obsidian plugin implementation counted as
release scope.
