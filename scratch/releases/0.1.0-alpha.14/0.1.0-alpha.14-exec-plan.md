## 1. Purpose

The alpha.14 design makes Memoria a standalone CLI + engine. The alpha.12 SQLite
working-state boundary (ADR-122) and alpha.13 DOI enrichment (ADR-123) already
landed on `main`, so this is not greenfield. The pivot alpha.14 lands is:

- `memoria` CLI is the product surface (the console script now exists; parity
  and runtime gates are still incomplete).
- SQLite is the OpenAlex-style graph authority; markdown Work, Author, OpenAlex
  Source, and Institution records are removed (capture currently dual-writes
  both).
- Full-text acquisition, first-order citation/topic graph import, embeddings,
  richer retrieval, gap analysis, and discovery candidates are baseline
  functionality because checked digests and useful project gaps depend on them.
- qmd is the required local search backend for checked-only keyword and semantic
  retrieval; SQLite remains the graph/ops authority.
- The runner is pydantic-ai over an OpenAI-compatible endpoint; Hermes and
  `direct_api`-as-a-product-runner are dropped.
- Obsidian, Zotero, MCP-as-required, installed profiles, and lane/fleet are
  removed from the baseline - not deferred.
- Operation manifests are file-per-capability; optional same-stem asset folders
  are allowed only for large prompts, schemas, or examples.

Observable outcome: a fresh checkout where `pip install -e .` yields a working
`memoria` command that runs the thin end-to-end cycle (`init` -> `work capture`
-> `work enrich` -> `work update` -> `work interview` -> `work digest` ->
`note propose/accept/link` -> `memoria ask` / `project ask` -> `project export`
-> `workspace recover`) with no Obsidian, Zotero, Hermes, MCP, or profile
environment, and `rg` proves those paths are gone from runtime and installer.

## 5. Validation And Acceptance

Alpha.14 is complete only when all of these hold (design sections 5, 9, 11, 14):

- `memoria --help` and every command import with no Obsidian/Zotero/Hermes/MCP/
  profile env; CLI, scan, and scheduled ingress all route through request rows
  with no dedicated ingress-type field.
- Every mutating command writes a durable request row before work; `status` and
  `doctor` are read-only (doctor repairs only with an explicit flag).
- DOI capture creates an unchecked SQLite Work and no markdown Work, Author,
  OpenAlex Source, or Institution records,
  and does not update `references.bib`.
- Required-provider enrichment materializes `references.bib` before completion;
  required-provider failure leaves the Work unchecked and creates attention.
- Full-text acquisition is required for checked digest: abstract-only or
  metadata-only fixtures block digest and create attention; acquired text is
  content-addressed, anchored, and replayable.
- URL page/PDF capture and user-supplied PDF/text fixtures produce checked
  digests only after anchored full text exists.
- BibTeX/CSL DOI and ISBN/book imports create unchecked SQLite Work rows with no
  Zotero and no legacy Work markdown; book digest remains blocked without full
  text or user-supplied equivalent.
- First-order citation/reference/topic graph import creates graph rows and
  unchecked discovery candidates without checking discovered Works.
- `workspace rebuild --search --embeddings` rebuilds the checked-only qmd input
  tree and qmd keyword/semantic index over acquired text, digests, notes, hubs,
  compact citations, and graph neighborhoods; retrieval/gap tests prove
  qmd+graph beats BM25-only.
- qmd is required: `doctor --check qmd` proves Node >=22, absolute qmd executable
  resolution, workspace-local qmd config/index state, collection
  `memoria-checked` with root `.memoria/index/qmd/checked/` and mask `**/*.md`,
  and no ambiguous bare-`PATH` binary.
- A provider endpoint enabled in config is covered by the operation allowlist; an
  adapter HTTP call bypassing the shared allowlist fails closed.
- High-authority DOI/title conflict and retraction/contested evidence both block
  checked promotion.
- A keep-set work reference with no compact `citations[]` fails validation.
- Crash after DB commit but before materialization recovers from the durable
  payload; lookup/digest/references/attention/rollback/recovery all work by
  `work_id` with no legacy markdown Work path.
- `work update` archives/restores Works, applies human classification, refreshes
  active retrieval/views, and resolves linked attention.
- `work interview` records guided answers and `work digest` consumes them without
  profile/Hermes memory.
- `note accept`/`note reject`/`note link` preserve explicit curation and typed
  link journal evidence.
- Attention items preserve finding/proposal/verification/work-prompt shapes;
  worklists preserve batch review and dedupe behavior; queue back-pressure and
  `request retry|cancel|amend` are visible in `status --json`.
- File-per-capability manifests load; same-stem asset folders are optional for
  oversized prompts, schemas, or examples; directory-only manifests do not load.
- Universal Concept frontmatter requires `id`, `standing`, and map-shaped `links`;
  typed facets, steering, vocabulary, asset Concepts, and `knowledge/` +
  `capabilities/` validate as OKF Knowledge Bundles.
- Every operation manifest declares `method_class`; hybrid/generative manifests
  without baseline, runner, allowlist, output-intent, and check policy fail.
- `work import --format bibtex`/`work import --format csl` seed unchecked
  work rows with no Zotero and no work markdown; `work import --format
  zotero-export` fails if it needs a live Zotero.
- `workspace rebuild` emits `knowledge/index.md`, `knowledge/_views/`,
  `capabilities/index.md`, and `capabilities/_generated/capability-index.json`.
- `workspace check`, scheduled jobs with `--schedule-id`, `journal list/show`,
  `doctor bundle`, and `doctor self-test` cover the current linter, session
  summary, diagnostics, and scheduled-maintenance workflows.
- `memoria ask` answers over checked workspace content without a project id;
  `project ask` adds project context.
- `project export --format markdown` renders a checked composition and resolves
  citations through `references.bib`; Pandoc formats fail clearly when Pandoc is
  absent.
- The `rg` negative gate passes: no runtime/installer/current-doc path requires
  Hermes, Obsidian, Zotero, MCP, profiles, or legacy markdown Work paths.
- The parity fixture exercises the five palette actions, the adapter-only startup
  macro, and the manifest-derived operation set through CLI/engine.
- The historical-command audit stays a negative fixture (removed sample/project/
  mapping/delegation/lane/draft/verify/workspace commands are not reintroduced).
- The installed runtime-local gate and seeded-error verdict pass before any
  non-sandbox use.
- A keep-set edit imports as a PI edit (rollback-reachable inputs); an edit to a
  generated/worker/`references.bib`/capability path is quarantined, not consumed.
- A `note` validates its anchor and Toulmin claim block; the quote-anchor and
  model-free warrant checks fail an unresolvable quote or a claim whose cited span
  lacks its quote.
- Contradiction, supersession, and dedup use NLI (tri-class + abstain), pass an
  adversarial word-overlap fixture, and never decide equivalence on cosine alone.
- Extraction runs closed-domain at temperature 0 with verbatim grounding and
  self-consistency (disagreement sets the uncertainty flag).
- `digest`/`hub` carry `confidence`/`contested`/`contradictions`, and lint
  surfaces `confidence: low`/`contested`/single-work claims.
- Retrieval hides superseded/retracted content by default; a substantial Ask
  answer files back as a Concept.
- DOI resolution abstains to attention unless >=2 of title/author/year agree;
  digest fails closed without acquired full text or user-supplied equivalent.
- The argument-health operations run via `operation run` and update the project
  argument graph.
- No autonomy loop is built: no co-trained reviewer, tournament, tree-search over
  synthesis, confidence-routed bypass, or preference-into-weights.

## 6. Idempotence And Recovery

- All testing uses disposable workspaces (`/tmp/memoria-a14`, `~/Memoria-test/...`);
  never a personal runtime workspace. `Memoria-test` is the sandbox; `memoria-private`
  is hands-off.
- Provider tests fake HTTP or use checked-in replay fixtures. No live provider
  call is required for any merge gate.
- SQLite schema changes are additive within the checkpoint; there is no
  production-workspace migration target. There is no backward-compatibility branch.
- If a PR branch tangles, stash with `git stash push -u -m alpha14-wip`, recreate
  the worktree from `origin/main`, and reapply only the relevant paths. Verify
  `git rev-parse origin/main` first - concurrent sessions move this branch.
- Crash between DB commit and materialization is resumed by `memoria workspace recover`
  via `state.recover_pending_materializations` and the blob payload.

## 10. Interfaces And Dependencies

- Keep the CLI on stdlib `argparse`, matching the existing worker entrypoint
  style. No new CLI framework.
- Reuse the existing SQLite queue/journal in `state.py`; do not add an event
  framework or workflow engine.

  dispatch; it does not gate Work capture, graph persistence, or enrichment.

  config lives at `<workspace>/.memoria/config/providers.yaml`; the runtime never
  reads provider policy from a repository checkout or template directory.

  without the matching Markdown file must not load.

  `runtime/` - shed MCP/Hermes/profile transport, preserve all deterministic
  logic. `runtime/` becomes the single operation home; nothing runs from
  `.memoria/operations/` after PR 6.

  `capability-index.json`) must name their SQLite/Concept owner and be covered by a
  drift check.

  survive the Hermes/MCP/profile removal - re-read ADR-32/46/28/27 before cutting
  the sandbox plumbing.
