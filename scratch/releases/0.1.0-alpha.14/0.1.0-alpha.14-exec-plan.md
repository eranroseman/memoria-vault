## 1. Purpose

The alpha.14 design makes Memoria a standalone CLI + engine. The alpha.12 SQLite
working-state boundary (ADR-122) and alpha.13 DOI enrichment (ADR-123) already
landed on `main`, so this is not greenfield. The pivot alpha.14 lands is:

- `memoria` CLI is the product surface (today there is no console script).
- SQLite is the catalog source of truth; markdown catalog and entity Concepts
  are removed (capture currently dual-writes both).
- The runner is pydantic-ai over an OpenAI-compatible endpoint; Hermes and
  `direct_api`-as-a-product-runner are dropped.
- Obsidian, Zotero, MCP-as-required, installed profiles, and lane/fleet are
  removed from the baseline - not deferred.
- Operation manifests are directory-per-capability; flat markdown stops loading.

Observable outcome: a fresh checkout where `pip install -e .` yields a working
`memoria` command that runs the thin end-to-end cycle (init -> capture-source ->
enrich-source -> compile-digest -> ask -> recover) with no Obsidian, Zotero,
Hermes, MCP, or profile environment, and `rg` proves those paths are gone from
runtime and installer.

## 5. Validation And Acceptance

Alpha.14 is complete only when all of these hold (design sections 5, 9, 11, 14):

- `memoria --help` and every command import with no Obsidian/Zotero/Hermes/MCP/
  profile env; CLI, scan, and scheduled ingress all route through request rows
  with no dedicated ingress-type field.
- Every mutating command writes a durable request row before work; `status` and
  `doctor` are read-only (doctor repairs only with an explicit flag).
- DOI capture creates an unchecked SQLite source and no source/entity markdown,
  and does not update `references.bib`.
- Required-provider enrichment materializes `references.bib` before completion;
  required-provider failure leaves the source unchecked and creates attention.
- A provider endpoint enabled in config is covered by the operation allowlist; an
  adapter HTTP call bypassing the shared allowlist fails closed.
- High-authority DOI/title conflict and retraction/contested evidence both block
  checked promotion.
- A keep-set source reference with no compact `citations[]` fails validation.
- Crash after DB commit but before materialization recovers from the durable
  payload; lookup/digest/references/attention/rollback/recovery all work by
  `source_id` with no `catalog/sources/*/source.md`.
- Directory-per-capability manifests load; flat operation markdown does not.
- Universal Concept frontmatter requires `id`, `standing`, and map-shaped `links`;
  `knowledge/` and `capabilities/` validate as OKF Knowledge Bundles.
- `import-bibtex`/`import-csl` seed unchecked source rows with no Zotero and no
  source markdown; `capture-zotero-source` fails if it needs a live Zotero.
- `regenerate-projections` emits `knowledge/index.md`, `knowledge/_views/`,
  `capabilities/index.md`, and `capabilities/_generated/ai-catalog.json`.
- The `rg` negative gate passes: no runtime/installer/current-doc path requires
  Hermes, Obsidian, Zotero, MCP, profiles, or `source.md`.
- The parity fixture exercises the five palette actions, the adapter-only startup
  macro, and the manifest-derived operation set through CLI/engine.
- The historical-command audit stays a negative fixture (removed sample/project/
  mapping/delegation/lane/draft/verify/workspace commands are not reintroduced).
- The installed runtime-local gate and seeded-error verdict pass before any
  non-sandbox use.
- A keep-set edit imports as a PI edit (rollback-reachable inputs); an edit to a
  generated/worker/`references.bib`/capability path is quarantined, not consumed.
- A `note` validates its anchor and Toulmin claim block; the quote-anchor and
  model-free warrant checks fail an unresolvable quote or a claim whose cited span
  lacks its quote.
- Contradiction, supersession, and dedup use NLI (tri-class + abstain), pass an
  adversarial word-overlap fixture, and never decide equivalence on cosine alone.
- Extraction runs closed-domain at temperature 0 with verbatim grounding and
  self-consistency (disagreement sets the uncertainty flag).
- `digest`/`hub` carry `confidence`/`contested`/`contradictions`, and lint
  surfaces `confidence: low`/`contested`/single-source claims.
- Retrieval hides superseded/retracted content by default; a substantial Ask
  answer files back as a Concept.
- DOI resolution abstains to attention unless >=2 of title/author/year agree; the
  digest stamps its source-text tier and confidence.
- The argument-health operations run via `operation run` and update the project
  argument graph.
- No autonomy loop is built: no co-trained reviewer, tournament, tree-search over
  synthesis, confidence-routed bypass, or preference-into-weights.

## 6. Idempotence And Recovery

- All testing uses disposable vaults (`/tmp/memoria-a14`, `~/Memoria-test/...`);
  never the real runtime vault. `Memoria-test` is the sandbox; `memoria-private`
  is hands-off.
- Provider tests fake HTTP or use checked-in replay fixtures. No live provider
  call is required for any merge gate.
- SQLite schema changes are additive within the checkpoint; there is no
  production-vault migration target. There is no backward-compatibility branch.
- If a PR branch tangles, stash with `git stash push -u -m alpha14-wip`, recreate
  the worktree from `origin/main`, and reapply only the relevant paths. Verify
  `git rev-parse origin/main` first - concurrent sessions move this branch.
- Crash between DB commit and materialization is resumed by `memoria recover`
  via `state.recover_pending_materializations` and the blob payload.

- `main` carries two disjoint operation subsystems: the alpha.12 SQLite engine
  (`src/memoria_vault/runtime/`, 26 ops) and a legacy `.memoria/operations/` tree
  (26 deterministic modules: ingest, eval, linter, retraction, resolve-merge,
  reconcile, project, lib). `runtime/` does not import the latter. Alpha.14 ports
  all of it into the engine (drop no logic, PR 6), then removes the vacated
  originals (PR 7). `ingest`/`eval`/`lint` are real subsystems, not aliases.

## 10. Interfaces And Dependencies

- Keep the CLI on stdlib `argparse`, matching the existing worker entrypoint
  style. No new CLI framework.
- Reuse the existing SQLite queue/journal in `state.py`; do not add an event
  framework or workflow engine.
- pydantic-ai is runner scope only. It replaces the `direct_api`/`hermes`
  dispatch; it does not gate capture/catalog/enrichment.
- The existing template provider config is conversion input. Runtime provider
  config lives at `<workspace>/.memoria/config/providers.yaml`; the runtime never
  reads provider policy from a source checkout or template directory.
- Operation manifests move to `operations/<name>/capability.md`; the flat path is
  migration input only and must stop loading.
- The `.memoria/operations/` subsystem (26 modules) is ported wholesale into
  `runtime/` - shed MCP/Hermes/profile transport, preserve all deterministic
  logic. `runtime/` becomes the single operation home; nothing runs from
  `.memoria/operations/` after PR 6.
- Generated projections (`references.bib`, indexes, `knowledge/_views/`,
  `ai-catalog.json`) must name their SQLite/Concept owner and be covered by a
  drift check.
- Security controls in design section 9 and the MCP-only sandbox guarantee must
  survive the Hermes/MCP/profile removal - re-read ADR-32/46/28/27 before cutting
  the sandbox plumbing.
