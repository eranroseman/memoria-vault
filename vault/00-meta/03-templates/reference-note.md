# `reference-note` template

Stable reference page for a concept, method, or venue. Lives in `30-synthesis/02-reference/`. Agent drafts; human finalizes. Compiled from matured claim notes.

## Frontmatter

```yaml
---
title: ""
type: reference-note
category: concept
lifecycle: proposed        # proposed (draft) | current (canonical, after human review and link consolidation)
moc: []
projects: []
aliases: []
schema_version: 1
created:
updated:
sources: []
---
```

## Body

```md
# Definition
Concise definition or description.

# Why it matters
- Why this concept is useful.
- Where it appears in the corpus.

# Related terms
- Similar concept.
- Contrasting concept.
- Parent or child concept.

# Notes
- Stable details worth remembering.
- Boundaries and exceptions.
```

## Notes

A reference note is promoted from a stable claim note (or compiled from several). If a reference note changes substantially every week, it isn't a reference note — it's a budding claim note. Move it back.
